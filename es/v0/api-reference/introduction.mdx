<div id="base-url">
  ## URL base
</div>

Todas las solicitudes usan la siguiente URL base:

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## Autenticación
</div>

Para la autenticación, es obligatorio incluir un encabezado Authorization. El encabezado debe contener `Bearer fc_123456789`, donde `fc_123456789` representa tu clave de API.

```bash
Authorization: Bearer fc_123456789
```

​

<div id="response-codes">
  ## Códigos de respuesta
</div>

Firecrawl emplea códigos de estado HTTP convencionales para indicar el resultado de tus solicitudes.

Normalmente, los códigos de estado HTTP 2xx indican éxito, los códigos 4xx representan errores relacionados con el usuario y los 5xx señalan problemas de infraestructura.

Status | Descripción
--- | ---
200 | La solicitud se realizó correctamente.
400 | Verifica que los parámetros sean correctos.
401 | No se proporcionó la clave de API.
402 | Pago requerido.
404 | No se pudo encontrar el recurso solicitado.
429 | Se ha superado el límite de solicitudes.
5xx | Indica un error de servidor en Firecrawl.

Consulta la sección «Códigos de error» para una explicación detallada de todos los posibles errores de la API.

​

<div id="rate-limit">
  ## Límite de tasa
</div>

La API de Firecrawl tiene un límite de tasa para garantizar la estabilidad y la confiabilidad del servicio. Este límite se aplica a todos los puntos de conexión y se basa en la cantidad de solicitudes realizadas dentro de un periodo de tiempo específico.

Si excedes el límite de tasa, recibirás un código de respuesta 429.

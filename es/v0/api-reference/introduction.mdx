<div id="base-url">
  ## URL base
</div>

Todas las solicitudes usan la siguiente URL base:

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## Autenticación
</div>

Para autenticarse, debes incluir un encabezado Authorization. El encabezado debe contener `Bearer fc_123456789`, donde `fc_123456789` es tu clave de API.

```bash
Authorization: Bearer fc_123456789
```

​

<div id="response-codes">
  ## Códigos de respuesta
</div>

Firecrawl utiliza códigos de estado HTTP convencionales para indicar el resultado de tus solicitudes.

Por lo general, los códigos de estado HTTP 2xx indican éxito, los códigos 4xx representan errores relacionados con el usuario y los códigos 5xx señalan problemas de infraestructura.

Status | Descripción
--- | ---
200 | Solicitud exitosa.
400 | Verifica que los parámetros sean correctos.
401 | No se proporcionó la clave de API.
402 | Pago requerido.
404 | No se pudo localizar el recurso solicitado.
429 | Se superó el límite de tasa.
5xx | Indica un error del servidor de Firecrawl.

Consulta la sección Códigos de error para una explicación detallada de todos los posibles errores de la API.

​

<div id="rate-limit">
  ## Límite de tasa
</div>

La API de Firecrawl tiene un límite de tasa para garantizar la estabilidad y la confiabilidad del servicio. Este límite se aplica a todos los puntos de conexión y se basa en la cantidad de solicitudes realizadas dentro de un periodo de tiempo específico.

Si excedes el límite de tasa, recibirás un código de respuesta 429.

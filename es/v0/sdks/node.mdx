<div id="installation">
  ## Instalación
</div>

Para instalar el SDK de Firecrawl para Node, puedes usar npm:

```bash
npm install @mendable/firecrawl-js@0.0.36
```

<div id="usage">
  ## Uso
</div>

1. Obtén una clave de API en [firecrawl.dev](https://firecrawl.dev)
2. Configura la clave de API como una variable de entorno llamada `FIRECRAWL_API_KEY` o pásala como parámetro a la clase `FirecrawlApp`.

Aquí tienes un ejemplo de cómo usar el SDK con gestión de errores:

```js
import FirecrawlApp from '@mendable/firecrawl-js';

// Inicializa FirecrawlApp con tu clave de API
const app = new FirecrawlApp({ apiKey: "YOUR_API_KEY" });

// Extrae una única URL
const url = 'https://docs.firecrawl.dev';
const scrapedData = await app.scrapeUrl(url);

// Rastrea un sitio web
const crawlUrl = 'https://docs.firecrawl.dev';
const params = {
  crawlerOptions: {
    excludes: ['blog/'],
    includes: [], // déjalo vacío para incluir todas las páginas
    limit: 1000,
  },
  pageOptions: {
      onlyMainContent: true
  }
};

const crawlResult = await app.crawlUrl(crawlUrl, params);
```

<div id="scraping-a-url">
  ### Extracción de una URL
</div>

Para extraer una única URL con manejo de errores, usa el método `scrapeUrl`. Recibe la URL como parámetro y devuelve los datos extraídos como un diccionario.

```js
const url = 'https://example.com';
const scrapedData = await app.scrapeUrl(url);
```

<div id="crawling-a-website">
  ### Rastreo de un sitio web
</div>

Para rastrear un sitio web con manejo de errores, utiliza el método `crawlUrl`. Recibe la URL inicial y parámetros opcionales como argumentos. El parámetro `params` te permite especificar opciones adicionales para la tarea de rastreo, como el número máximo de páginas a rastrear, los dominios permitidos y el formato de salida.

```js
const crawlUrl = 'https://example.com';

const params = {
  crawlerOptions: {
    excludes: ['blog/'],
    includes: [], // dejar en blanco para incluir todas las páginas
    limit: 1000,
  },
  pageOptions: {
    onlyMainContent: true
  }
};

const waitUntilDone = true;
const pollInterval = 5;

const crawlResult = await app.crawlUrl(
  crawlUrl,
  params,
  waitUntilDone,
  pollInterval
);
```

<div id="checking-crawl-status">
  ### Comprobar el estado del rastreo
</div>

Para verificar el estado de una tarea de rastreo con manejo de errores, usa el método `checkCrawlStatus`. Recibe el ID de la tarea como parámetro y devuelve el estado actual del rastreo.

```js
const status = await app.checkCrawlStatus(jobId);
```

<div id="extracting-structured-data-from-a-url">
  ### Extracción de datos estructurados desde una URL
</div>

Con la extracción mediante LLM, puedes extraer fácilmente datos estructurados de cualquier URL. Compatibilizamos esquemas de Zod para facilitarte el proceso. Aquí te mostramos cómo usarlo:

```js
import FirecrawlApp from "@mendable/firecrawl-js";
import { z } from "zod";

const app = new FirecrawlApp({
  apiKey: "fc-YOUR_API_KEY",
});

// Define un esquema para extraer el contenido
const schema = z.object({
  top: z
    .array(
      z.object({
        title: z.string(),
        points: z.number(),
        by: z.string(),
        commentsURL: z.string(),
      })
    )
    .length(5)
    .describe("Las 5 principales noticias en Hacker News"),
});

const scrapeResult = await app.scrapeUrl("https://firecrawl.dev", {
  extractorOptions: { extractionSchema: schema },
});

console.log(scrapeResult.data["llm_extraction"]);
```

<div id="search-for-a-query">
  ### Buscar una consulta
</div>

Con el método `search`, puedes realizar una búsqueda en un motor y obtener los principales resultados junto con el contenido de la página de cada uno. El método recibe la consulta como parámetro y devuelve los resultados de la búsqueda.

```js
const query = '¿Qué es Mendable?';
const searchResults = await app.search(query, {
  pageOptions: {
    fetchPageContent: true // Obtener el contenido de la página para cada resultado de la búsqueda
  }
});
```

<div id="error-handling">
  ## Manejo de errores
</div>

El SDK gestiona los errores que devuelve la API de Firecrawl y arroja las excepciones correspondientes. Si se produce un error durante una solicitud, se generará una excepción con un mensaje descriptivo. Los ejemplos anteriores muestran cómo manejar estos errores con bloques `try/catch`.

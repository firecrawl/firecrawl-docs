Los webhooks te permiten recibir notificaciones en tiempo real sobre tus operaciones de Firecrawl conforme progresan, en lugar de consultar periódicamente el estado.


<div id="supported-operations">
  ## Operaciones admitidas
</div>

| Operación       | Eventos                             |
| --------------- | ----------------------------------- |
| Crawl           | `started`, `page`, `completed`      |
| Batch Scrape    | `started`, `page`, `completed`      |
| Extract         | `started`, `completed`, `failed`    |

<div id="configuration">
  ## Configuración
</div>

Añade un objeto `webhook` a tu solicitud:

<WebhookConfigBasic />

| Campo      | Tipo   | Requerido | Descripción                                   |
| ---------- | ------ | --------- | --------------------------------------------- |
| `url`      | string | Sí        | La URL de tu endpoint (HTTPS)                |
| `headers`  | object | No        | Encabezados personalizados para incluir en las solicitudes del webhook |
| `metadata` | object | No        | Datos personalizados incluidos en las cargas del webhook      |
| `events`   | array  | No        | Tipos de eventos a recibir (por defecto: todos los eventos) |

<div id="timeouts-retries">
  ## Tiempos de espera y reintentos
</div>

Tu endpoint debe responder con un estado `2xx` en un plazo de **10 segundos**.

Si la entrega falla (se excede el tiempo de espera, el estado no es 2xx o hay un error de red), Firecrawl reintenta automáticamente:

| Reintento | Retraso después del fallo |
| --------- | ------------------------- |
| 1.º       | 1 minuto                  |
| 2.º       | 5 minutos                 |
| 3.º       | 15 minutos                |

Después de 3 reintentos fallidos, el webhook se marca como fallido y no se realizan más intentos.

<div id="overview">
  ## Resumen
</div>

Usa la API `fetch` de Deno Deploy para llamar a Firecrawl.

<div id="prerequisites">
  ## Requisitos previos
</div>

* Una clave de API de Firecrawl
* Un proyecto de Deno Deploy

<div id="install">
  ## Instalación
</div>

No se requieren paquetes adicionales; Deno incluye fetch.

<div id="configure-environment">
  ## Configurar el entorno
</div>

Configura la clave de API en el entorno del proyecto de Deno Deploy.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## Añade una ruta del servidor
</div>

main.ts

```ts
Deno.serve(async (req) => {
  const { url } = await req.json();
  const response = await fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${Deno.env.get('FIRECRAWL_API_KEY')}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ url, formats: ['markdown'] }),
  });

  return new Response(await response.text(), {
    headers: { 'Content-Type': 'application/json' },
  });
});
```

<div id="call-it-from-your-app">
  ## Invócalo desde tu aplicación
</div>

```js
const response = await fetch('https://your-app.deno.dev', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

<div id="notes">
  ## Notas
</div>

* Deno Deploy solo permite solicitudes salientes con `fetch`; mantén todas las llamadas a Firecrawl del lado del servidor.

<div id="overview">
  ## Resumen
</div>

Usa el SDK de Firecrawl para Node en una ruta del servidor Express.

<div id="prerequisites">
  ## Requisitos previos
</div>

* Una clave de API de Firecrawl
* Una aplicación de Express
* Node.js 18+

<div id="install">
  ## Instalación
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="configure-environment">
  ## Configurar el entorno
</div>

Guarda la clave de API en `.env` o en las variables de entorno del proceso.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## Agrega una ruta en el servidor
</div>

server.js

```js
import express from 'express';
import Firecrawl from '@mendable/firecrawl-js';

const app = express();
app.use(express.json());

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

app.post('/api/firecrawl', async (req, res) => {
  const { url } = req.body;
  const doc = await client.scrape(url, { formats: ['markdown'] });
  res.json({ markdown: doc.markdown });
});

app.listen(3000, () => console.log('Server listening on :3000'));
```

<div id="call-it-from-your-app">
  ## Invócalo desde tu aplicación
</div>

```js
const response = await fetch('http://localhost:3000/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

<div id="notes">
  ## Notas
</div>

* Mantén las llamadas a Firecrawl en el servidor; nunca expongas claves de API en los bundles del cliente.

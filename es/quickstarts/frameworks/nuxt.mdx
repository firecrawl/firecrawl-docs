<div id="overview">
  ## Resumen
</div>

Usa una ruta de API del servidor de Nuxt para que tu clave de API de Firecrawl permanezca privada.

<div id="prerequisites">
  ## Requisitos previos
</div>

* Una clave de API de Firecrawl
* Una aplicación de Nuxt 3
* Node.js 18+

<div id="install">
  ## Instalación
</div>

No se requieren paquetes adicionales; las rutas del servidor de Nuxt pueden usar fetch directamente.

<div id="configure-environment">
  ## Configurar el entorno
</div>

Añade la clave de API a `.env` y expónla mediante la configuración en tiempo de ejecución.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    firecrawlApiKey: process.env.FIRECRAWL_API_KEY,
  },
});
```

<div id="add-a-server-route">
  ## Añade una ruta de servidor
</div>

server/api/firecrawl.post.ts

```ts
export default defineEventHandler(async (event) => {
  const { url } = await readBody(event);
  const config = useRuntimeConfig();

  return await $fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${config.firecrawlApiKey}`,
      'Content-Type': 'application/json',
    },
    body: { url, formats: ['markdown'] },
  });
});
```

<div id="call-it-from-your-app">
  ## Invócalo desde tu aplicación
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

<div id="notes">
  ## Notas
</div>

* Mantén la clave de API de Firecrawl en `runtimeConfig` y evita exponerla en el cliente.

<div id="overview">
  ## Resumen
</div>

Mantén tu clave de API de Firecrawl en el servidor realizando la llamada desde un Route Handler de Next.js.

<div id="prerequisites">
  ## Requisitos previos
</div>

* Una clave de API de Firecrawl
* Aplicación de Next.js (App Router)
* Node.js 18+

<div id="install">
  ## Instalación
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="configure-environment">
  ## Configura el entorno
</div>

Añade la clave de API a `.env.local` para que solo esté disponible en el servidor.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## Añade una ruta del servidor
</div>

app/api/firecrawl/route.ts

```ts
import Firecrawl from '@mendable/firecrawl-js';

export async function POST(request: Request) {
  const { url } = await request.json();
  const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

  const doc = await client.scrape(url, { formats: ['markdown'] });
  return Response.json({ markdown: doc.markdown });
}
```

<div id="call-it-from-your-app">
  ## Invócalo desde tu aplicación
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

<div id="notes">
  ## Notas
</div>

* Llama a Firecrawl solo desde rutas del servidor para evitar exponer la clave de tu API.
* Si usas el Pages Router, crea `pages/api/firecrawl.ts` en su lugar.

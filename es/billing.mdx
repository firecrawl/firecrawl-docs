<div id="overview">
  ## Descripción general
</div>

Firecrawl utiliza un **sistema de facturación basado en créditos**. Cada llamada a la API que haces consume créditos, y la cantidad de créditos consumidos depende del endpoint y de las opciones que utilices. Recibes una asignación mensual de créditos según tu plan y puedes comprar créditos adicionales mediante paquetes de recarga automática si necesitas más.

Para ver los precios de los planes vigentes, visita la [página de precios de Firecrawl](https://www.firecrawl.dev/pricing).

<Note>
  Todas las facturas de Firecrawl se emiten en **dólares estadounidenses (USD)**, independientemente de tu dirección de facturación o método de pago.
</Note>

<div id="credits">
  ## Créditos
</div>

Los créditos son la unidad de consumo en Firecrawl. Cada plan incluye una cuota mensual de créditos que se reinicia al comienzo de cada ciclo de facturación. Los distintos endpoints de la API consumen diferentes cantidades de créditos.

<div id="credit-costs-per-endpoint">
  ### Costo de créditos por endpoint
</div>

| Endpoint    | Credit Cost                      | Notes                                                                                                                                                                                                                                                                                                                                                            |
| ----------- | -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Scrape**  | 1 crédito / página               | Convierte una única URL en markdown limpio, HTML o datos estructurados. Se aplican créditos adicionales al usar opciones de extracción (ver más abajo).                                                                                                                                                                                                          |
| **Crawl**   | 1 crédito / página               | Extrae un sitio web completo siguiendo enlaces desde una URL inicial. Los mismos costos por página de las opciones de extracción se aplican a cada página rastreada.                                                                                                                                                                                             |
| **Map**     | 1 crédito / llamada              | Descubre todas las URL de un sitio web sin extraer su contenido.                                                                                                                                                                                                                                                                                                 |
| **Search**  | 2 créditos / 10 resultados       | Busca en la web y, opcionalmente, extrae los resultados. Redondeado hacia arriba por cada 10 resultados (p. ej., 11 resultados = 4 créditos). Se aplican costos adicionales de extracción por página a cada resultado que se extrae. La búsqueda ZDR Enterprise cuesta 10 créditos / 10 resultados (ver [ZDR Search](/es/features/search#zero-data-retention-zdr)). |
| **Browser** | 2 créditos / minuto de navegador | Sesión interactiva de navegador en sandbox, facturada por minuto.                                                                                                                                                                                                                                                                                                |
| **Agent**   | Dinámico                         | Agente autónomo de investigación web. 5 ejecuciones diarias gratuitas; precios basados en uso a partir de ese límite.                                                                                                                                                                                                                                            |

<div id="additional-credit-costs-for-scrape-options">
  ### Costos de crédito adicionales para las opciones de extracción
</div>

Ciertas opciones de extracción añaden créditos sobre el costo base por página:

| Option | Additional Cost | Description |
|--------|----------------|-------------|
| PDF parsing | +1 crédito / página PDF | Extrae contenido de documentos PDF |
| JSON format (LLM extraction) | +4 créditos / página | Usa un LLM para extraer datos JSON estructurados de la página |
| Enhanced Mode | +4 créditos / página | Extracción mejorada para páginas a las que es difícil acceder |
| Zero Data Retention (ZDR) | +1 crédito / página | Garantiza que no se conserve ningún dato más allá de la solicitud (consulta [Scrape ZDR](/es/features/scrape#zero-data-retention-zdr)) |

Estos modificadores se acumulan. Por ejemplo, extraer una página con JSON format y Enhanced Mode cuesta **1 + 4 + 4 = 9 créditos** por página. Estos mismos modificadores se aplican a los endpoints Crawl y Search, ya que usan scrape internamente para cada página.

<div id="when-credits-are-charged">
  ### Cuándo se cobran créditos
</div>

Los créditos se cobran siempre que la infraestructura de Firecrawl procesa una solicitud, incluso si el sitio de destino devuelve un código de estado de error HTTP como 403 Forbidden o 404 Not Found. Esto se debe a que la infraestructura de scraping (renderizado en el navegador, proxy, etc.) se utiliza por completo independientemente de la respuesta del sitio de destino. Puedes revisar el campo `metadata.statusCode` en la respuesta de la API para detectar estos casos y evitar reintentar URLs que se bloquean de forma constante.

Para trabajos de **batch scrape** y **crawl**, los créditos se cobran de forma asíncrona a medida que cada página termina de procesarse, no cuando se envía el trabajo. Esto significa que puede haber un retraso entre el envío de un trabajo y el momento en que se refleja el costo total en créditos en tu cuenta. Si un lote contiene muchas URLs o las páginas se ponen en cola durante períodos de alto tráfico, los créditos pueden seguir registrándose minutos u horas después del envío. Hacer *polling* o comprobar el estado del batch no consume créditos.

<Note>
  **Verificación previa de créditos para crawl:** Antes de que comience un trabajo de crawl, Firecrawl verifica que tu saldo de créditos disponible pueda cubrir por completo el parámetro `limit` que solicitaste. Si tu saldo es inferior a `limit`, la solicitud devuelve un 402 incluso si el crawl hubiera descubierto menos páginas. El valor predeterminado de `limit` es **10,000**, por lo que, si lo omites, debes tener 10,000 créditos disponibles por adelantado. Para evitarlo, pasa un `limit` explícito que coincida con la cantidad de páginas que realmente quieres recorrer (por ejemplo, `limit: 100`).
</Note>

<div id="tracking-your-usage">
  ### Seguimiento del uso
</div>

Puedes monitorear tu consumo de créditos de dos maneras:

* **Dashboard**: Consulta tu consumo actual e histórico en [firecrawl.dev/app](https://www.firecrawl.dev/app)
* **API**: Usa los endpoints [Credit Usage](/es/api-reference/endpoint/credit-usage) y [Credit Usage Historical](/es/api-reference/endpoint/credit-usage-historical) para comprobar tu consumo de forma programática

<Note>
  Estamos trabajando activamente en mejoras para que el uso de créditos sea más fácil de entender. Mantente al tanto de las novedades.
</Note>

<div id="plans">
  ## Planes
</div>

Firecrawl ofrece planes mensuales de suscripción. No existe una opción de pago por uso puro, pero la recarga automática (descrita más abajo) permite escalar de forma flexible por encima de tu plan base.

<div id="available-plans">
  ### Planes disponibles
</div>

| Plan         | Créditos mensuales | Navegadores concurrentes |
| ------------ | ------------------ | -----------------------: |
| **Free**     | 1,000              |                        2 |
| **Hobby**    | 5,000              |                        5 |
| **Standard** | 100,000            |                       50 |
| **Growth**   | 500,000            |                      100 |
| **escala**   | 1,000,000          |                      150 |

<Note>
  Para necesidades más allá del plan escala, Firecrawl ofrece planes **Enterprise** con créditos personalizados, soporte dedicado, SLAs, descuentos por volumen, sin retención de datos y SSO. Visita la [página Enterprise](https://www.firecrawl.dev/enterprise) para más detalles.
</Note>

Todos los planes de pago están disponibles con facturación **mensual** o **anual**. La facturación anual ofrece un descuento frente a pagar mes a mes. Para ver los precios actuales de cada plan, visita la [página de precios](https://www.firecrawl.dev/pricing).

<div id="choosing-a-credit-tier-with-the-pricing-slider">
  ### Elegir un nivel de créditos con el control deslizante de precios
</div>

La [página de precios](https://www.firecrawl.dev/pricing) usa un único control deslizante que te permite recorrer todas las opciones de crédito, desde el plan Free hasta escala y Custom. Además de los niveles principales de cada plan, cada plan de pago ofrece niveles intermedios de crédito para que puedas ajustar tu suscripción más a tu uso real sin tener que pasar al siguiente plan:

| Plan         | Niveles de crédito disponibles |
| ------------ | ------------------------------ |
| **Hobby**    | 5,000 · 6,500 · 8,000          |
| **Standard** | 100,000 · 130,000 · 160,000    |
| **Growth**   | 500,000 · 650,000              |
| **Escala**   | 1,000,000                      |

Los navegadores concurrentes, los límites de tasa y el nivel de soporte los determina el plan, no el nivel específico de créditos: elegir 6,500 créditos en Hobby te da la misma concurrencia que elegir 5,000. Usa el control deslizante para ajustar la cantidad de créditos y luego suscríbete a ese nivel.

<div id="concurrent-browsers">
  ### Navegadores concurrentes
</div>

Los navegadores concurrentes representan cuántas páginas web puede procesar Firecrawl simultáneamente para ti. Tu plan determina este límite. Si lo superas, las tareas adicionales esperarán en una cola hasta que se libere un cupo. Consulta [Límites de tasa](/es/rate-limits) para obtener todos los detalles sobre concurrencia y límites de tasa de la API.

<div id="handling-usage-spikes">
  ## Cómo gestionar los picos de uso
</div>

Firecrawl ofrece dos mecanismos opcionales para que las solicitudes sigan fluyendo cuando te quedas sin créditos a mitad del ciclo: **Auto-Recharge** y **Smart Upgrade**. Ambos se configuran desde tu [configuración de facturación](https://www.firecrawl.dev/app/settings?tab=billing) y son **mutuamente excluyentes**: tu organización usa uno u otro, no ambos.

|                   | Auto-Recharge                                                    | Smart Upgrade                                                     |
| ----------------- | ---------------------------------------------------------------- | ----------------------------------------------------------------- |
| Qué hace          | Compra un paquete de créditos puntual y lo añade a tu saldo      | Te pasa al siguiente nivel de créditos en la escala de precios    |
| Cuándo se activa  | El saldo baja hasta tu umbral (predeterminado: 0)                | El saldo llega a 0                                                |
| Cargo             | Precio fijo del paquete                                          | Diferencia prorrateada entre tu nivel actual y el siguiente       |
| Créditos añadidos | El paquete se conserva durante 1 año y no se restablece cada mes | Asignación completa del nuevo nivel, que sustituye al anterior    |
| Ideal para        | Picos ocasionales además de un plan estable                      | Crecimiento sostenido: subir de nivel a medida que aumenta el uso |

<div id="auto-recharge">
  ### Recarga automática
</div>

Cuando tus créditos restantes bajen de un umbral, Firecrawl compra automáticamente un paquete de créditos adicional y lo añade a tu saldo. De forma predeterminada, el umbral es de **0 créditos**, pero puedes personalizarlo en tu [configuración de facturación](https://www.firecrawl.dev/app/settings?tab=billing) para activar las recargas antes — por ejemplo, establecer un umbral más alto garantiza que siempre tengas un colchón de créditos disponible, lo cual es útil si ejecutas rastreos grandes. Ten en cuenta que esta comprobación ocurre cuando haces una solicitud a la API — tu saldo no se recargará hasta que tu siguiente solicitud active la recarga.

* Los paquetes de recarga automática están disponibles en todos los planes de pago
* El tamaño y el precio de los paquetes varían según el plan (visible en la [página de precios](https://www.firecrawl.dev/pricing))
* Puedes activar o desactivar la recarga automática en cualquier momento
* La recarga automática está limitada a **4 paquetes al mes**
* Los créditos de los paquetes de recarga automática **no se restablecen mensualmente** — se conservan entre ciclos de facturación y vencen **1 año** después de la compra, a diferencia de los créditos mensuales de tu plan, que se restablecen en cada período.

La recarga automática es ideal para manejar picos ocasionales de uso. Si ves que superas constantemente la asignación de tu plan, actualizar a un plan superior — o habilitar Smart Upgrade — te dará un mejor precio por crédito.

<div id="smart-upgrade">
  ### Smart Upgrade
</div>

Smart Upgrade es un modo de facturación opcional que te pasa automáticamente al siguiente nivel de créditos cada vez que tu saldo llega a cero, para que tu uso no se interrumpa. En lugar de comprar un paquete de créditos puntual, se te migra al siguiente nivel de suscripción del selector (p. ej., Hobby 5k → Hobby 6.5k → Hobby 8k → Standard 100k → … → Escala 1M).

Cómo funciona:

* **Activación:** Smart Upgrade se activa cuando tu saldo de créditos llega a 0. Si todavía te queda algún crédito, incluidos los créditos de recarga automática sin usar, no se produce ningún cambio de nivel.
* **Cargo:** Se te factura un importe prorrateado equivalente a la diferencia entre tu nivel actual y el siguiente. En los planes **mensuales**, es la diferencia de precio mensual; en los planes **anuales**, es `(meses restantes del ciclo ÷ 12) × diferencia de precio anual`. El prorrateo se calcula con meses naturales completos desde el inicio de tu período de facturación actual.
* **Créditos otorgados:** Recibes la asignación completa de créditos del nuevo nivel, menos la asignación de tu nivel anterior (así que solo pagas la diferencia).
* **Notificaciones:** Cada vez que Smart Upgrade se activa, recibes un correo electrónico con la confirmación del cambio de nivel, el importe cobrado y una vista previa del siguiente escalón. Puedes deshabilitar Smart Upgrade en la [configuración de facturación](https://www.firecrawl.dev/app/settings?tab=billing) en cualquier momento para evitar cambios de nivel posteriores.
* **Parte superior de la escala:** Una vez que llegas al nivel Escala (1.000.000 de créditos/mes), Smart Upgrade ya no tiene más escalones. Para aumentar los créditos por encima de Escala, [contacta con ventas](https://www.firecrawl.dev/enterprise) para consultar un plan Enterprise personalizado.

Smart Upgrade se habilita en el momento en que te suscribes o cambias de plan (verás el interruptor en la confirmación de compra) y luego puede activarse o desactivarse desde la configuración de facturación. Está disponible para organizaciones de un solo equipo con planes estándar; las organizaciones con contratos Enterprise personalizados siguen usando su modelo de facturación negociado.

Smart Upgrade es la mejor opción si tu uso crece de forma constante y prefieres subir automáticamente por la escala de planes en lugar de recargar con paquetes de créditos. Usa Auto-Recharge si tu plan base es el adecuado y solo necesitas recargas puntuales de vez en cuando.

<div id="coupons">
  ## Cupones
</div>

Firecrawl admite dos tipos de cupones:

* Los **cupones de suscripción** aplican un descuento a la suscripción de tu plan (p. ej., un porcentaje de descuento en el precio mensual o anual). Estos **solo** se pueden aplicar durante el proceso de pago de Stripe, cuando te suscribes por primera vez a un plan de pago o cambias de plan. No puedes aplicar un cupón de suscripción una vez completado el pago.
* Los **cupones de créditos** añaden créditos extra a tu cuenta. Estos se pueden canjear desde la sección de **Facturación** de tu panel de control en [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing). Busca el campo de cupón en la página de facturación para aplicar tu código. Los créditos extra de los cupones de créditos son independientes de la asignación mensual de tu plan y se conservan aunque cambies a un plan superior o inferior.

Si tienes un código de cupón y no estás seguro de qué tipo es, intenta aplicarlo primero en la sección de facturación de tu panel de control. Si es un cupón de suscripción, tendrás que usarlo en la página de pago de Stripe.

<div id="billing-cycle">
  ## Ciclo de facturación
</div>

* **Planes mensuales**: Los créditos se restablecen en tu fecha de renovación mensual
* **Planes anuales**: Se te factura anualmente, pero los créditos igual se restablecen cada mes en tu fecha de renovación mensual virtual
* **Los créditos del plan no utilizados no se acumulan de forma predeterminada** — tu asignación mensual se restablece cada mes. **Los planes anuales Escala permiten acumular los créditos del plan no utilizados durante 1 mes**, y **los planes anuales Enterprise permiten acumularlos durante 2 meses**. Los créditos de los paquetes de recarga automática no están vinculados a tu ciclo de facturación — se conservan y vencen **1 año** después de la fecha de compra.

<div id="upgrading-and-downgrading">
  ## Actualizar o bajar de plan
</div>

* **Las actualizaciones de plan** surten efecto de inmediato. Hoy se te cobra el precio completo del nuevo plan (sin prorrateo) y tu ciclo de facturación se restablece: tu próxima renovación será un mes o un año después de la fecha de actualización. Los créditos no usados de tu plan anterior se transfieren, y tu nueva asignación de créditos y tus límites de concurrencia se aplican al instante.
* **Los cambios a un plan inferior** se programan para entrar en vigor en tu próxima fecha de renovación. Mantienes los créditos y límites de tu plan actual hasta entonces. El tiempo no usado de tu plan actual no se acredita ni se reembolsa. Puedes deshacer un cambio programado a un plan inferior desde tu [configuración de facturación](https://www.firecrawl.dev/app/settings?tab=billing) en cualquier momento antes de la fecha de entrada en vigor.

<div id="switching-between-monthly-and-yearly-billing">
  ### Cambio entre facturación mensual y anual
</div>

* **Mensual → Anual** en el mismo nivel de créditos o en uno superior se considera una mejora inmediata.
* **Anual → Mensual** se considera una mejora inmediata solo si cambias a un nivel de créditos estrictamente superior.

Los paquetes de recarga automática que ya hayas comprado no se ven afectados por los cambios de plan: conservan su caducidad original de 1 año, independientemente de las mejoras o de los cambios a un plan inferior.

<div id="what-happens-when-you-run-out-of-credits">
  ## Qué sucede cuando te quedas sin créditos
</div>

Si agotas tu asignación de créditos y no tienes activados Auto-Recharge ni Smart Upgrade, las solicitudes a la API que consumen créditos devolverán un error **HTTP 402 (Pago requerido)**.

* Con **Auto-Recharge** activado, el uso continúa mientras los paquetes de recarga se compran automáticamente; pero si se alcanza el límite mensual de paquetes (4) o falla una recarga, tu saldo puede quedar en negativo hasta el siguiente ciclo de facturación.
* Con **Smart Upgrade** activado, tu suscripción pasa automáticamente al siguiente nivel de créditos en el control deslizante y recibes de inmediato la asignación de créditos del nuevo nivel. Smart Upgrade se detiene cuando alcanzas el nivel Escala.

Para reanudar el uso después de una interrupción total, puedes:

1. Activar [Auto-Recharge](#auto-recharge) o [Smart Upgrade](#smart-upgrade)
2. Cambiar a un plan superior
3. Esperar a que tus créditos se restablezcan en el siguiente ciclo de facturación

<div id="free-plan">
  ## Free Plan
</div>

El plan gratuito ofrece **1,000 créditos por mes** sin necesidad de tarjeta de crédito. Los créditos se restablecen cada mes en tu fecha de renovación mensual: una vez que se agoten, puedes esperar al siguiente reinicio o actualizar a un plan de pago para obtener una asignación mayor. El plan gratuito también tiene límites de tasa y concurrencia más bajos en comparación con los planes de pago (consulta los [Límites de tasa](/es/rate-limits)).

<AccordionGroup>
  <Accordion title="¿Los créditos no utilizados se trasladan al mes siguiente?">
    Los **créditos del plan** no se acumulan de forma predeterminada — tu asignación mensual se restablece cada mes. **Los planes anuales escala trasladan los créditos del plan no utilizados durante 1 mes**, y **los planes Enterprise anuales los trasladan durante 2 meses**. Sin embargo, los créditos comprados mediante **paquetes de recarga automática** no están vinculados a tu ciclo de facturación y sí se conservan en cualquier plan. Los créditos de recarga automática vencen **1 año** después de la fecha de compra.
  </Accordion>

  <Accordion title="¿Cuál es la diferencia entre Auto-Recharge y Smart Upgrade?">
    **Auto-Recharge** compra un paquete de créditos puntual cuando tu saldo alcanza un umbral, manteniéndote en tu plan actual. **Smart Upgrade** te mueve al siguiente nivel de créditos en la escala de precios (p. ej., Hobby 5k → 6.5k → 8k → Standard 100k → …) cuando tu saldo llega a 0, cobrándote la diferencia prorrateada. Son mutuamente excluyentes — tu organización usa uno u otro. Usa Auto-Recharge para picos ocasionales, y Smart Upgrade si tu uso crece de forma constante y prefieres ir subiendo de nivel automáticamente.
  </Accordion>

  <Accordion title="¿Cómo se calcula el cargo de Smart Upgrade?">

En los planes **anuales**, es `(meses restantes en tu ciclo de facturación ÷ 12) × diferencia de precio anual`, usando meses calendario completos desde el inicio de tu período actual. También recibes la diferencia de créditos entre los dos niveles, por lo que solo pagas por lo que obtienes.

Estos niveles intermedios comparten la misma concurrencia y el mismo nivel de soporte que el nivel principal de su plan; solo cambian la asignación de créditos y el precio.
  </Accordion>

  <Accordion title="¿Cómo sé cuántos créditos me quedan?">

  </Accordion>

  <Accordion title="¿Cuál es la diferencia entre créditos y tokens?">
    Los créditos son la unidad de facturación para las llamadas a la API de Firecrawl. Los tokens se refieren a los tokens de LLM utilizados internamente por endpoints como Extract y Agent. Cada crédito equivale a 15 tokens. Para la mayoría de los usuarios, solo necesitas pensar en términos de créditos.
  </Accordion>

Las próximas facturas incluirán automáticamente la información actualizada. Si necesitas que se vuelva a generar una factura anterior con los nuevos datos, ponte en contacto con [help@firecrawl.dev](mailto:help@firecrawl.dev).

</div>

Todas las solicitudes usan la siguiente URL base:

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## Autenticación
</div>

Para autenticarse, es obligatorio incluir el encabezado Authorization. Este encabezado debe contener `Bearer fc-123456789`, donde `fc-123456789` corresponde a tu clave de API.

```bash
Authorization: Bearer fc-123456789
```

​

<div id="response-codes">
  ## Códigos de respuesta
</div>

Firecrawl utiliza códigos de estado HTTP convencionales para indicar el resultado de tus solicitudes.

Por lo general, los códigos 2xx indican éxito, los 4xx representan errores del cliente y los 5xx señalan problemas de infraestructura.

Status | Description
--- | ---
200 | Solicitud exitosa.
400 | Verifica que los parámetros sean correctos.
401 | No se proporcionó la clave de API.
402 | Pago requerido.
404 | No se encontró el recurso solicitado.
429 | Se superó el límite de solicitudes.
5xx | Indica un error del servidor de Firecrawl.

Consulta la sección Códigos de error para una explicación detallada de todos los posibles errores de la API.

<div id="firecrawl-error-codes-5xx">
  ### Códigos de error de Firecrawl (5xx)
</div>

Cuando ocurre un error 5xx, Firecrawl proporciona códigos de error más específicos para aclarar qué salió mal.​

Error Code | Status | Descripción\
--- | --- | ---\
`SCRAPE_ALL_ENGINES_FAILED` | 500 | Fallaron todos los motores de scraping.\
`SCRAPE_SSL_ERROR` | 500 | El certificado SSL de la página no es válido. Puedes usar `skipTlsVerification:true` para omitir esta verificación.\
`SCRAPE_SITE_ERROR` | 500 | Error irrecuperable del sitio.\
`SCRAPE_DNS_RESOLUTION_ERROR` | 500 | Falló la resolución de DNS.\
`SCRAPE_ACTION_ERROR` | 500 | Error al ejecutar una acción en la página.\
`SCRAPE_PDF_PREFETCH_FAILED` | 500 | Error al realizar la precarga del PDF.\
`SCRAPE_PDF_INSUFFICIENT_TIME_ERROR` | 500 | No hubo suficiente tiempo para procesar el PDF.\
`SCRAPE_PDF_ANTIBOT_ERROR` | 500 | PDF bloqueado por mecanismos antibot.\
`SCRAPE_ZDR_VIOLATION_ERROR` | 500 | Conflicto de Zero Data Retention: ocurre cuando `zeroDataRetention:true`, pero otra opción (p. ej., `screenshot`) requiere almacenamiento temporal.\
`SCRAPE_UNSUPPORTED_FILE_ERROR` | 500 | Tipo de archivo no compatible.\
`UNKNOWN_ERROR` | 500 | Error genérico o inesperado.

<div id="rate-limit">
  ## Límites de tasa
</div>

La API de Firecrawl tiene límites de tasa para garantizar la estabilidad y la fiabilidad del servicio. Estos límites se aplican a todos los puntos de conexión y se basan en la cantidad de solicitudes realizadas dentro de un periodo de tiempo específico.

Si superas el límite, recibirás un código de respuesta 429.

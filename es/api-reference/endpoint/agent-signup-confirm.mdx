---
title: "Confirmar registro del agente"
description: "Verifica y vincula una clave de API aprovisionada por un agente a la cuenta del patrocinador."
---

Confirma un registro pendiente de agente. Si el patrocinador tiene una cuenta existente de Firecrawl, la clave se fusiona con ella. Si no, la cuenta en entorno aislado se promociona a una cuenta completa bajo el correo electrónico del patrocinador. No se requiere autenticación: el token de verificación actúa como prueba.

<div id="headers">
  ## Cabeceras
</div>

| Cabecera       | Valor              |
| -------------- | ------------------ |
| `Content-Type` | `application/json` |

<div id="request-body">
  ## Cuerpo de la solicitud
</div>

| Parámetro | Tipo | Obligatorio | Descripción |
|-----------|------|-------------|-------------|
| `agent_signup_token` | string | **Sí** | El token de verificación del enlace de confirmación del correo electrónico. |

<div id="response">
  ## Respuesta
</div>

| Campo     | Tipo    | Descripción                                        |
| --------- | ------- | -------------------------------------------------- |
| `success` | boolean | Indica si la confirmación se realizó correctamente |
| `message` | string  | Mensaje del resultado legible para humanos         |

<div id="example-request">
  ### Ejemplo de solicitud
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/agent-signup/confirm" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_signup_token": "a1b2c3d4..."
  }'
```

<div id="example-response">
  ### Respuesta de ejemplo
</div>

```json
{
  "success": true,
  "message": "Clave del agente confirmada y vinculada a tu cuenta."
}
```

<div id="what-happens-on-confirm">
  ## Qué sucede al confirmar
</div>

- **Cuenta existente**: La clave de API del agente se transfiere al equipo del patrocinador. La cuenta en entorno aislado se desactiva.
- **Sin cuenta**: El correo electrónico de la cuenta en entorno aislado se actualiza con el correo electrónico del patrocinador, y pasa a ser su cuenta real.
- En ambos casos, se elimina el límite de 50 créditos y la clave hereda el plan completo del patrocinador.

<div id="error-responses">
  ## Respuestas de error
</div>

| Estado | Error | Cuándo |
|--------|-------|--------|
| `400` | Solicitud no válida | Falta `agent_signup_token` |
| `403` | El plazo de verificación ha vencido | La ventana de confirmación de 5 días ha expirado |
| `403` | Se ha bloqueado este registro del agente | El patrocinador ya bloqueó esta clave |
| `404` | Token de verificación no válido o caducado | Token no encontrado o ya utilizado |

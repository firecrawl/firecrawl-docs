<div id="browser-actions">
  ## Acciones del navegador
</div>

Puedes realizar interacciones en el navegador antes de hacer scraping usando el parámetro `actions`. Esto te permite:

* Esperar a que se carguen elementos
* Hacer clic en botones o enlaces
* Rellenar campos de formularios
* Pulsar teclas
* Desplazar páginas (scroll)
* Tomar capturas de pantalla
* Ejecutar JavaScript personalizado
* Generar archivos PDF

Las acciones admitidas incluyen `wait`, `click`, `write`, `press`, `scroll`, `screenshot`, `scrape`, `executeJavascript` y `pdf`. Para ver la documentación detallada con ejemplos, consulta la [Guía avanzada de scraping](https://docs.firecrawl.dev/advanced-scraping-guide#actions-actions).

<div id="persistent-profiles">
  ## Perfiles persistentes
</div>

Pasa el parámetro `profile` para conservar el estado del navegador (cookies, localStorage, sesiones) entre sesiones de scrape y de [interact](/es/features/interact). Las operaciones de scrape con el mismo nombre de perfil comparten el estado del navegador, lo que resulta útil para mantener la sesión iniciada o conservar las preferencias.

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/dashboard",
    "profile": {
      "name": "my-session",
      "saveChanges": true
    }
  }'
```

| Parámetro             | Tipo    | Obligatorio | Predeterminado | Descripción                                                                                                                                                                   |
| --------------------- | ------- | ----------- | -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `profile.name`        | string  | Sí          | —              | Un nombre para el perfil (1-128 caracteres). Las operaciones de scraping con el mismo nombre comparten el estado del navegador.                                               |
| `profile.saveChanges` | boolean | No          | `true`         | Cuando es `true`, el estado del navegador se guarda de nuevo en el perfil al finalizar la sesión de Interact. Establece `false` para cargar datos existentes sin escribirlos. |

<Note>
  Solo una sesión puede guardar en un perfil a la vez. Si otra sesión ya lo está guardando, recibirás un error `409`. Aun así, puedes abrir el mismo perfil con `saveChanges: false` o volver a intentarlo más tarde.
</Note>

Para más detalles, consulta [Interact — Perfiles persistentes](/es/features/interact#persistent-profiles).

---
title: "Registro de agente"
description: "Crea una API key de Firecrawl en un entorno aislado de pruebas en nombre de un patrocinador humano."
---

Aprovisiona una nueva API key de inmediato, en un entorno aislado con 50 créditos. Se envía un correo electrónico de verificación al patrocinador. No se requiere autenticación.

Para obtener una visión general completa del flujo, consulta la [guía de registro de agente](/es/features/agent-signup).

<div id="headers">
  ## Encabezados
</div>

| Encabezado     | Valor              |
| -------------- | ------------------ |
| `Content-Type` | `application/json` |

<div id="request-body">
  ## Cuerpo de la solicitud
</div>

| Parámetro      | Tipo    | Obligatorio | Descripción                                                                                                       |
| -------------- | ------- | ----------- | ----------------------------------------------------------------------------------------------------------------- |
| `email`        | string  | **Sí**      | Dirección de correo electrónico del patrocinador humano. Debe ser una dirección de correo electrónico válida.     |
| `agent_name`   | string  | **Sí**      | Nombre del agente que crea la clave (de 1 a 100 caracteres). Se muestra en el correo electrónico de verificación. |
| `accept_terms` | boolean | **Sí**      | Debe ser `true`. Confirma la aceptación de los [Términos del servicio](https://firecrawl.dev/terms-of-service).   |

<div id="response">
  ## Respuesta
</div>

| Field | Type | Description |
|-------|------|-------------|
| `success` | boolean | Si la clave se creó |
| `api_key` | string | La API key aprovisionada (empieza con `fc-`) |
| `sponsor_status` | string | Siempre `"pending"` al crearse |
| `credit_limit` | number | Máximo de crédito disponible antes de la verificación (50) |
| `credits_remaining` | number | Crédito restante (50 al crearse) |
| `verification_deadline_at` | string | Marca temporal ISO 8601. El patrocinador debe confirmar antes de esta fecha límite (5 días). |
| `tos_url` | string | URL de los Términos del Servicio aceptados |

<div id="example-request">
  ### Ejemplo de solicitud
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/agent-signup" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@company.com",
    "agent_name": "Research Assistant",
    "accept_terms": true
  }'
```

<div id="example-response">
  ### Ejemplo de respuesta
</div>

```json
{
  "success": true,
  "api_key": "fc-abc123...",
  "sponsor_status": "pending",
  "credit_limit": 50,
  "credits_remaining": 50,
  "verification_deadline_at": "2026-03-23T12:00:00.000Z",
  "tos_url": "https://firecrawl.dev/terms-of-service"
}
```

<div id="error-responses">
  ## Respuestas de error
</div>

| Status | Error | When |
|--------|-------|------|
| `400` | Cuerpo de la solicitud no válido | Faltan campos o hay campos no válidos |
| `403` | Este correo electrónico ha bloqueado los registros de agente | El patrocinador bloqueó previamente los registros de agente |
| `403` | La verificación de un registro anterior de agente ha expirado | Un registro anterior expiró; el patrocinador debe iniciar sesión |
| `409` | Ya se ha enviado una confirmación pendiente de registro de agente | Ya hay una clave pendiente para este correo electrónico |

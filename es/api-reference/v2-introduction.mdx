<div id="base-url">
  ## URL base
</div>

Todas las solicitudes utilizan la siguiente URL base:

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## Autenticación
</div>

Para autenticarse, debes incluir un encabezado Authorization. Este encabezado debe contener `Bearer fc-123456789`, donde `fc-123456789` es tu clave de API.

```bash
Authorization: Bearer fc-123456789
```

​

<div id="response-codes">
  ## Códigos de respuesta
</div>

Firecrawl utiliza códigos de estado HTTP convencionales para indicar el resultado de tus solicitudes.

Por lo general, los códigos 2xx indican éxito, los 4xx representan errores del cliente y los 5xx señalan problemas de infraestructura.

Status | Description
--- | ---
200 | Solicitud realizada correctamente.
400 | Verifica que los parámetros sean correctos.
401 | No se proporcionó la clave de API.
402 | Pago requerido.
404 | No se encontró el recurso solicitado.
429 | Se superó el límite de tasa.
5xx | Indica un error del servidor de Firecrawl.

Consulta la sección de Códigos de error para una explicación detallada de todos los posibles errores de la API.

​

<div id="rate-limit">
  ## Límites de tasa
</div>

La API de Firecrawl tiene límites de tasa para garantizar la estabilidad y confiabilidad del servicio. Estos límites se aplican a todos los puntos de conexión y dependen de la cantidad de solicitudes realizadas dentro de un periodo de tiempo específico.

Si superas el límite, recibirás un código de respuesta 429.

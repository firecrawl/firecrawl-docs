El formato `menu` extrae un menú de restaurante estructurado **de forma determinista**: el mismo tipo de salida estructurada que el [formato `json`](/es/features/scrape#extract-structured-data), pero sin una llamada a un LLM ni un esquema definido por ti, y diseñado específicamente para páginas de menús de restaurantes. Si has estado extrayendo campos de menús con un esquema `json`, usa `formats: ["menu"]` en su lugar: es más rápido y más barato, aunque se limita a menús.

Devuelve un objeto `menu` con el establecimiento, la moneda y una lista de secciones, donde cada sección incluye ítems, y cada ítem incluye descripción, imágenes, precio, disponibilidad, etiquetas dietéticas, calorías y grupos de opciones; útil para la ingesta de menús, la supervisión de precios o herramientas de entrega y pedidos de comida.

<div id="menu-object-structure">
  ## Estructura del objeto de menú
</div>

El objeto `menu` contiene las siguientes propiedades:

* `isMenu`: Indica si la página se identificó como un menú
* `confidence`: Una puntuación de confianza entre `0` y `1` para la extracción del menú
* `merchant`: El establecimiento al que pertenece el menú (opcional):
  * `name`: El nombre del establecimiento
  * `type`: El tipo de establecimiento, p. ej. `"restaurant"` (opcional)
* `currency`: El código de moneda del menú, solo se informa cuando aparece en la página (opcional)
* `sections`: array de secciones del menú. Cada sección tiene:
  * `id`: El identificador de la sección (opcional)
  * `name`: El nombre de la sección, p. ej. `"Appetizers"`
  * `description`: La descripción de la sección (opcional)
  * `items`: array de ítems del menú. Cada ítem tiene:
    * `id`: El identificador del ítem (opcional)
    * `name`: El nombre del ítem
    * `description`: La descripción del ítem (opcional)
    * `images`: array de imágenes del ítem, cada una con una `url` y texto `alt` opcional (opcional)
    * `price`: el objeto de precio (opcional):
      * `amount`: El valor numérico del precio
      * `currency`: El código de moneda, solo se informa cuando aparece en la página (opcional)
      * `formatted`: El precio tal como se muestra en la página (opcional)
    * `availability`: información de disponibilidad (opcional):
      * `inStock`: Indica si el ítem está disponible
      * `text`: El texto de disponibilidad original de la página (opcional)
    * `dietary`: array de etiquetas dietéticas, p. ej. `["vegetarian"]` (opcional)
    * `calories`: La cantidad de calorías del ítem (opcional)
    * `optionGroups`: Array de grupos de opciones/modificadores para el ítem (opcional)
    * `identifiers`: identificadores específicos del establecimiento para el ítem (opcional):
      * `merchantItemId`: El ID propio del establecimiento para el ítem (opcional)
    * `url`: La URL canónica del ítem (opcional)
    * `sourceUrl`: La URL de la que se extrajo el ítem (opcional)
* `sourceUrl`: La URL de la que se extrajo el menú (opcional)

<div id="how-menu-extraction-works">
  ## Cómo funciona la extracción de menús
</div>

El formato `menu` extrae el menú mediante una cadena determinista, que se prueba en este orden: **adaptadores de plataforma (p. ej., Toast) &gt; JSON-LD de schema.org &gt; microdatos / estado JSON incrustado &gt; un parser de texto visible/markdown**. Devuelve el resultado más completo que produzca la cadena y solo recurre a una última pasada con un LLM cuando la cadena determinista no encuentra nada o arroja un resultado de baja calidad. Los nombres de los ítems y los precios siempre se basan en la página: la extracción nunca inventa ítems. La divisa solo se informa cuando la página la proporciona.

<Note>
  La extracción de menús funciona con fail-closed: las páginas que no son menús no devuelven ningún menú. En una página sin un menú extraíble, la respuesta omite el objeto `menu` y añade `warning`.
</Note>

<Note>
  **Autoalojamiento:** el formato `menu` se apoya en un servicio dedicado de extracción de menús. En Firecrawl Cloud funciona de inmediato. Si lo autoalojas, configura `MENU_EXTRACTION_SERVICE_URL` para que apunte a ese servicio; cuando no está configurado, solicitar el formato `menu` devuelve una advertencia de &quot;formato `menu` is not available&quot; y no incluye el campo `menu` (el scraping sigue completándose correctamente).
</Note>

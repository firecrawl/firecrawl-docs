Firecrawl ofrece diferentes tipos de proxy para ayudarte a extraer datos de sitios web con distintos niveles de protección antibot. Puedes especificar el tipo de proxy con el parámetro `proxy`.

> De forma predeterminada, Firecrawl dirige todas las solicitudes a través de proxies para garantizar la fiabilidad y el acceso, incluso si no especificas un tipo de proxy o una ubicación.

<div id="how-to-specify-proxy-location">
  ## Cómo especificar la ubicación del proxy
</div>

Puedes solicitar una ubicación de proxy específica estableciendo el parámetro `location.country` en tu solicitud. Por ejemplo, para usar un proxy de Brasil, configura `location.country` en `BR`.

Para ver todos los detalles, consulta la [referencia de la API de `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location).

<div id="proxy-types">
  ## Tipos de proxy
</div>

Firecrawl admite tres tipos de proxies:

* **basic**: Proxies para extraer datos (scraping) de sitios con soluciones antibots nulas o básicas. Son rápidos y suelen funcionar.
* **stealth**: Proxies sigilosos para extraer datos de sitios con soluciones antibots avanzadas o que bloquean proxies comunes. Más lentos, pero más fiables en ciertos sitios. [Más información sobre Stealth Mode →](/es/features/stealth-mode)
* **auto**: Firecrawl reintentará automáticamente con proxies stealth si falla el proxy basic. Si el reintento con stealth tiene éxito, se cobrarán 5 créditos por la extracción. Si el primer intento con basic tiene éxito, solo se cobrará el costo regular.

***

> **Nota:** Para obtener información detallada sobre el uso de proxies stealth, incluidos los costos en créditos y las estrategias de reintento, consulta la [documentación de Stealth Mode](/es/features/stealth-mode).

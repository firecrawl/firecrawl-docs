<div id="firecrawl-elixir-agent-quickstart">
  # Guía de inicio rápido del agente de Firecrawl para Elixir
</div>

Este archivo es la guía de inicio rápido de referencia para agentes externos y se genera a partir del código fuente del SDK y de la especificación OpenAPI v2.

<div id="install">
  ## Instalación
</div>

Añade la dependencia en `mix.exs`:

```elixir
def deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

A continuación, ejecuta:

```bash
mix deps.get
```

<div id="authenticate">
  ## Autenticación
</div>

```elixir
# config/runtime.exs o config/config.exs
config :firecrawl, api_key: "fc-your-api-key"
```

O pásalo en cada llamada:

```elixir
Firecrawl.scrape_and_extract_from_url([url: "https://example.com"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: usar cuando partes de una consulta y necesitas descubrir resultados.
* `scrape`: usar cuando ya tienes una URL y quieres el contenido de la página.
* `interact`: usar cuando la página requiere clics, formularios o acciones del navegador posteriores al scraping.

<div id="search">
  ## Búsqueda
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `search` para encontrar URLs relevantes a partir de una consulta y, opcionalmente, hacer scraping de cada resultado. Puedes limitar las consultas a un solo sitio con el operador estándar `site:` (por ejemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de hacer scraping de una URL específica.

### Método preferido del SDK

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  limit: 5
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "firecrawl pricing",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  enterprise: ["anon"],
  scrape_options: [
    formats: [
      "markdown",
      [type: "json", prompt: "Extract pricing tiers and limits."]
    ],
    only_main_content: true,
    proxy: :enhanced
  ],
  timeout: 60000
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Parámetros
</div>

* `query`: Cadena de consulta de búsqueda. Siempre obligatoria.
* `sources`: Fuentes en las que buscar. Valores admitidos: `"web"`, `"news"`, `"images"` u objetos con formato `[%{type: "web" | "news" | "images"}]`. Úsalo para controlar qué arrays de resultados se devuelven.
* `categories`: Filtros de resultados. Valores admitidos: `"github"`, `"research"`, `"pdf"` u objetos con formato `[%{type: "github" | "research" | "pdf"}]`. Úsalo para restringir los resultados por categoría.
* `limit`: Número máximo de resultados que se devolverán. Úsalo para limitar el tamaño de la respuesta.
* `tbs`: Cadena de filtro de búsqueda basada en el tiempo. Úsalo para limitar por rango temporal (p. ej., `qdr:d`).
* `location`: Cadena de ubicación para resultados localizados. Úsalo para consultas específicas por ubicación geográfica.
* `country`: Código de país ISO para resultados de búsqueda segmentados geográficamente. Úsalo junto con `location`.
* `timeout`: Tiempo de espera de la solicitud en milisegundos. Úsalo para limitar el tiempo de búsqueda.
* `ignore_invalid_urls`: Excluye las URL no válidas de los resultados. Úsalo al encadenarlo con otros endpoints.
* `enterprise`: Opciones de búsqueda enterprise. Valores admitidos: `"anon"` (ZDR anonimizado) o `"zdr"` (ZDR de extremo a extremo).
* `scrape_options`: Opciones de scraping aplicadas a cada resultado (las mismas claves que `scrape`). Úsalo cuando quieras obtener contenido junto con los resultados.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `scrape` cuando ya conozcas la URL y necesites el contenido en formatos específicos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`Firecrawl.scrape_and_extract_from_url(params, opts)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com",
  formats: ["markdown"]
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/docs",
  formats: [
    "markdown",
    [type: "json", prompt: "Extract headings and slugs."],
    [type: "changeTracking", modes: ["git-diff"], tag: "docs"],
    [type: "screenshot", full_page: true, viewport: [width: 1280, height: 720]]
  ],
  actions: [
    [type: "click", selector: "button#accept"],
    [type: "scroll", direction: "down"]
  ],
  parsers: [[type: "pdf", mode: "auto", max_pages: 5]],
  location: [country: "US", languages: ["en-US"]],
  proxy: :enhanced,
  profile: [name: "docs-session", save_changes: true],
  max_age: 3_600_000,
  min_age: 0,
  store_in_cache: true,
  zero_data_retention: false
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Parámetros
</div>

* `url`: URL para hacer scraping. Siempre obligatoria.
* `formats`: Formatos de salida que se incluirán. Cadenas de formato compatibles: `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`. Objetos de formato: `json` `[type: "json", prompt?, schema?]`, `changeTracking` `[type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?]`, `screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`.
* `headers`: Encabezados HTTP adicionales. Úsalos para autenticación, cookies o user-agent.
* `include_tags`: Incluye solo contenido de estas etiquetas HTML. Úsalo para acotar la extracción.
* `exclude_tags`: Excluye contenido de estas etiquetas HTML. Úsalo para eliminar ruido.
* `only_main_content`: Extrae solo el contenido principal. Úsalo para omitir la navegación y el pie de página.
* `timeout`: Tiempo de espera del scraping en milisegundos. Úsalo para limitar el tiempo de ejecución.
* `wait_for`: Retraso antes del scraping en milisegundos. Úsalo para páginas renderizadas con JS.
* `mobile`: Emula un dispositivo móvil. Úsalo para diseños móviles.
* `parsers`: Lista o configuración de analizadores. Configuración de PDF compatible: `[type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: number]` o entradas de cadena (por ejemplo, `"pdf"`).
* `actions`: Acciones del navegador que se ejecutarán antes del scraping. Acciones compatibles: `wait` `[type: "wait", milliseconds? | selector?]`, `screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`, `click` `[type: "click", selector, all?]`, `write` `[type: "write", text]`, `press` `[type: "press", key]`, `scroll` `[type: "scroll", direction: "up" | "down", selector?]`, `scrape` `[type: "scrape"]`, `executeJavascript` `[type: "executeJavascript", script]`, `pdf` `[type: "pdf", format?, landscape?, scale?]` con formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuración de ubicación `[country, languages]`. Úsala para páginas específicas de una región.
* `skip_tls_verification`: Omite la verificación del certificado TLS. Úsalo para certificados autofirmados.
* `remove_base64_images`: Elimina las imágenes en base64 de la salida markdown. Úsalo para reducir el tamaño de la carga útil.
* `block_ads`: Habilita el bloqueo de anuncios y cookies. Úsalo para reducir el ruido.
* `proxy`: Modo de proxy `:basic`, `:enhanced`, `:auto`.
* `max_age`: Acepta contenido en caché con una antigüedad inferior a esta (ms). Úsalo para favorecer aciertos de caché.
* `min_age`: Devuelve solo contenido en caché con al menos esta antigüedad (ms). Úsalo para lecturas exclusivas desde caché.
* `store_in_cache`: Almacena el resultado en la caché/índice. Úsalo cuando quieras reutilizarlo.
* `profile`: Perfil persistente del navegador `[name, save_changes]`. Úsalo para conservar el estado de la sesión.
* `zero_data_retention`: Habilita la retención de datos cero para el scraping. Úsalo cuando ZDR esté habilitado para tu equipo.

<div id="interact">
  ## Interactúa
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `interact` cuando la página requiera hacer clic, completar formularios u otras acciones del navegador después del scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.url())"
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.title())",
  language: :node,
  timeout: 60,
  origin: "agent-docs"
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Parámetros
</div>

* `job_id`: ID del trabajo de scraping (de `metadata.scrapeId`). Siempre es obligatorio.
* `code`: Código que se ejecutará en el sandbox del navegador. Obligatorio.
* `language`: Lenguaje de ejecución `:python`, `:node` o `:bash` (predeterminado: `:node`). Úsalo para que coincida con tu código.
* `timeout`: Tiempo de espera de ejecución en segundos. Úsalo para limitar ejecuciones largas.
* `origin`: Etiqueta de origen opcional. Úsala para atribuir la solicitud.

<div id="notes">
  ## Notas
</div>

* Los parámetros se pasan como listas de palabras clave en snake&#95;case validadas por `NimbleOptions`.
* Las funciones se generan automáticamente a partir de la especificación OpenAPI; sus nombres siguen los IDs de operación de OpenAPI.
* Cada función tiene una variante con bang que lanza una excepción en caso de error (por ejemplo, `search_and_scrape!`).
* El SDK de Elixir expone únicamente `interact` mediante código (sin el parámetro `prompt`).
* Todas las funciones aceptan las opciones `api_key` y `base_url` para sobrescrituras por solicitud.

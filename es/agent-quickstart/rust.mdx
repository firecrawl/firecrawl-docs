<div id="firecrawl-rust-agent-quickstart">
  # Guía de inicio rápido del agente de Rust de Firecrawl
</div>

Este archivo es la guía de inicio rápido de referencia para agentes externos y se genera a partir del código fuente del SDK y de la especificación OpenAPI v2.

<div id="install">
  ## Instalación
</div>

```bash
cargo add firecrawl
```

<div id="authenticate">
  ## Autenticación
</div>

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: usar cuando empiezas con una consulta y necesitas descubrir resultados.
* `scrape`: usar cuando ya tienes una URL y quieres el contenido de la página.
* `interact`: usar cuando la página requiere clics, formularios o acciones del navegador posteriores al scraping.

<div id="search">
  ## Búsqueda
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `search` para descubrir URL relevantes a partir de una consulta y, opcionalmente, hacer scraping de cada resultado. Puedes limitar las consultas a un solo sitio con el operador estándar `site:` (por ejemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de hacer scraping de una URL concreta.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.search(query, options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
let results = client.search("site:docs.firecrawl.dev crawl webhooks", None).await?;

if let Some(web) = results.data.web {
    println!("{:?}", web.first());
}
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```rust
use firecrawl::v2::{Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format};

let client = Client::new("fc-your-api-key")?;
let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:w".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Json]),
        only_main_content: Some(true),
        ..Default::default()
    }),
    integration: Some("agent-quickstart".to_string()),
};

let results = client.search("firecrawl pricing", options).await?;
```

<div id="parameters">
  ### Parámetros
</div>

* `query`: Cadena de consulta de búsqueda. Siempre obligatoria.
* `limit`: Número máximo de resultados que se devolverán. Úsalo para limitar el tamaño de la respuesta.
* `sources`: Fuentes donde buscar. Valores admitidos: `SearchSource::Web`, `SearchSource::News`, `SearchSource::Images`.
* `categories`: Filtros de resultados. Valores admitidos: `SearchCategory::Github`, `SearchCategory::Research`, `SearchCategory::Pdf`.
* `tbs`: Cadena de filtro de búsqueda basada en tiempo. Úsala para limitar por rango temporal (p. ej., `qdr:d`).
* `location`: Cadena de ubicación para resultados localizados. Úsala para consultas geográficas específicas.
* `ignore_invalid_urls`: Excluye las URL no válidas de los resultados. Úsalo al canalizar la salida a otros endpoints.
* `timeout`: Tiempo de espera de la solicitud en milisegundos. Úsalo para limitar el tiempo de búsqueda.
* `scrape_options`: Opciones de scraping aplicadas a cada resultado (los mismos campos que `scrape`). Úsalas cuando quieras incluir contenido en los resultados.
* `integration`: Identificador de integración. Úsalo para seguimiento y analíticas.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `scrape` si ya conoces la URL y quieres el contenido en formatos específicos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```rust
use firecrawl::v2::{Client, ScrapeOptions, Format};

let client = Client::new("fc-your-api-key")?;
let doc = client
    .scrape(
        "https://example.com",
        ScrapeOptions {
            formats: Some(vec![Format::Markdown]),
            ..Default::default()
        },
    )
    .await?;

println!("{:?}", doc.markdown);
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```rust
use firecrawl::v2::{
    Action, ChangeTrackingMode, ChangeTrackingOptions, Client, Format, JsonOptions, LocationConfig,
    ParserConfig, ProxyType, ScrapeOptions, ScreenshotOptions, ScrollDirection, Viewport,
};

let client = Client::new("fc-your-api-key")?;
let options = ScrapeOptions {
    formats: Some(vec![Format::Markdown, Format::Json, Format::ChangeTracking, Format::Screenshot]),
    json_options: Some(JsonOptions {
        prompt: Some("Extract headings and slugs.".to_string()),
        ..Default::default()
    }),
    change_tracking_options: Some(ChangeTrackingOptions {
        modes: Some(vec![ChangeTrackingMode::GitDiff]),
        tag: Some("docs".to_string()),
        ..Default::default()
    }),
    screenshot_options: Some(ScreenshotOptions {
        full_page: Some(true),
        viewport: Some(Viewport { width: 1280, height: 720 }),
        ..Default::default()
    }),
    actions: Some(vec![
        Action::Wait {
            milliseconds: Some(1000),
            selector: None,
        },
        Action::Click {
            selector: "button#accept".to_string(),
        },
        Action::Scroll {
            direction: ScrollDirection::Down,
            selector: None,
        },
    ]),
    parsers: Some(vec![ParserConfig::Pdf {
        parser_type: "pdf".to_string(),
        max_pages: Some(5),
    }]),
    location: Some(LocationConfig {
        country: Some("US".to_string()),
        languages: Some(vec!["en-US".to_string()]),
    }),
    proxy: Some(ProxyType::Stealth),
    ..Default::default()
};

let doc = client.scrape("https://example.com/docs", options).await?;
```

<div id="parameters">
  ### Parámetros
</div>

* `url`: URL para hacer scraping. Siempre obligatoria.
* `formats`: Formatos de salida que se incluirán. Valores compatibles: `Format::Markdown`, `Format::Html`, `Format::RawHtml`, `Format::Links`, `Format::Images`, `Format::Screenshot`, `Format::Summary`, `Format::ChangeTracking`, `Format::Json`, `Format::Attributes`, `Format::Branding`, `Format::Audio`.
* `headers`: Encabezados HTTP adicionales. Usar para autenticación, cookies o user-agent.
* `include_tags`: Incluye solo contenido de estas etiquetas HTML. Usar para acotar la extracción.
* `exclude_tags`: Excluye contenido de estas etiquetas HTML. Usar para eliminar ruido.
* `only_main_content`: Extrae solo el contenido principal. Usar para omitir la navegación y el pie de página.
* `timeout`: Tiempo de espera del scraping en milisegundos. Usar para limitar el tiempo de ejecución.
* `wait_for`: Espera antes del scraping en milisegundos. Usar para páginas renderizadas con JS.
* `mobile`: Emula un dispositivo móvil. Usar para diseños móviles.
* `parsers`: Lista o configuraciones de analizadores. Valores compatibles: `ParserConfig::Simple(String)` o `ParserConfig::Pdf { parser_type: "pdf", max_pages: Option<u32> }`.
* `actions`: Acciones del navegador que se ejecutarán antes del scraping. Acciones compatibles: `Wait { milliseconds?, selector? }`, `Screenshot { full_page?, quality?, viewport? }`, `Click { selector }`, `Write { text }`, `Press { key }`, `Scroll { direction: Up | Down, selector? }`, `Scrape`, `ExecuteJavascript { script }`, `Pdf { format?, landscape?, scale? }` con formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuración de ubicación `{ country, languages }`. Usar para páginas específicas por geolocalización.
* `skip_tls_verification`: Omite la verificación del certificado TLS. Usar para certificados autofirmados.
* `remove_base64_images`: Elimina imágenes base64 de la salida en markdown. Usar para reducir el tamaño de la carga útil.
* `block_ads`: Habilita el bloqueo de anuncios y cookies. Usar para reducir el ruido.
* `proxy`: Modo proxy `ProxyType::Basic`, `ProxyType::Stealth`, `ProxyType::Enhanced`, `ProxyType::Auto`.
* `max_age`: Acepta contenido en caché con una antigüedad inferior a esta (ms). Usar para favorecer aciertos de caché.
* `min_age`: Devuelve solo contenido en caché con al menos esta antigüedad (ms). Usar para lecturas solo desde caché.
* `store_in_cache`: Almacena el resultado en la caché/índice. Usar cuando quieras reutilizarlo.
* `profile`: Perfil persistente del navegador `ProfileConfig { name, save_changes }`. Usar para conservar el estado de la sesión.
* `integration`: Identificador de integración. Usar para seguimiento/analíticas.
* `json_options`: Opciones de extracción JSON para el formato `Format::Json` (`schema`, `system_prompt`, `prompt`).
* `screenshot_options`: Opciones de captura de pantalla para el formato `Format::Screenshot` (`full_page`, `quality`, `viewport`).
* `change_tracking_options`: Opciones de seguimiento de cambios para el formato `Format::ChangeTracking` (`modes`, `schema`, `prompt`, `tag`). Modos: `ChangeTrackingMode::GitDiff`, `ChangeTrackingMode::Json`.
* `attribute_selectors`: Selectores de atributos para el formato `Format::Attributes` (`selector`, `attribute`).

<div id="interact">
  ## Interactuar
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `interact` cuando la página requiera hacer clic, rellenar formularios u otras acciones en el navegador después de un scraping.

### Método preferido del SDK

`client.interact(job_id, options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```rust
use firecrawl::v2::{Client, ScrapeExecuteOptions};

let client = Client::new("fc-your-api-key")?;
let run = client
    .interact(
        "scrape-job-id",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.url())".to_string()),
            ..Default::default()
        },
    )
    .await?;

println!("{:?}", run.stdout);
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```rust
use firecrawl::v2::{Client, ScrapeExecuteLanguage, ScrapeExecuteOptions};

let client = Client::new("fc-your-api-key")?;
let run = client
    .interact(
        "scrape-job-id",
        ScrapeExecuteOptions {
            prompt: Some("Open the pricing page and list the plan names.".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            origin: Some("agent-docs".to_string()),
            ..Default::default()
        },
    )
    .await?;
```

<div id="parameters">
  ### Parámetros
</div>

* `job_id`: ID del trabajo de scraping (de `document.metadata.scrape_id`). Siempre es obligatorio.
* `code`: Código para ejecutar en el sandbox del navegador. Úsalo para interacciones mediante scripts.
* `prompt`: Instrucción en lenguaje natural para el agente del navegador. Úsalo cuando prefieras acciones guiadas por el agente.
* `language`: Lenguaje de ejecución `ScrapeExecuteLanguage::Python`, `ScrapeExecuteLanguage::Node` o `ScrapeExecuteLanguage::Bash` (el valor predeterminado es `Node`).
* `timeout`: Tiempo de espera de ejecución en segundos. Úsalo para limitar ejecuciones prolongadas.
* `origin`: Etiqueta de origen opcional. Úsala para atribuir la solicitud.

<div id="notes">
  ## Notas
</div>

* Los parámetros usan `snake_case` en Rust; JSON se serializa en `camelCase`.
* Método de conveniencia: `search_and_scrape` realiza una búsqueda con scraping activado.
* Alias obsoletos: `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (usa `interact` y `stop_interaction`).

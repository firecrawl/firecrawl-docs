<div id="firecrawl-javascripttypescript-agent-quickstart">
  # Inicio rápido del agente de Firecrawl para JavaScript/TypeScript
</div>

Este archivo es la guía de inicio rápido de referencia para agentes externos y se genera a partir del código fuente del SDK y de la especificación OpenAPI v2.

<div id="install">
  ## Instalación
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="authenticate">
  ## Autenticación
</div>

```ts
import { Firecrawl } from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // opcional para instancias autogestionadas
});
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: úsalo cuando partas de una consulta y necesites descubrir resultados.
* `scrape`: úsalo cuando ya tengas una URL y quieras el contenido de la página.
* `interact`: úsalo cuando la página requiera clics, formularios o acciones del browser después del scraping.

<div id="search">
  ## Búsqueda
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `search` para descubrir URL relevantes a partir de una consulta y, opcionalmente, hacer scraping de cada resultado. Puedes limitar las consultas a un solo sitio con el operador estándar `site:` (por ejemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de hacer scraping de una URL específica.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.search(query, options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const results = await client.search(
  "site:docs.firecrawl.dev crawl webhooks",
  { limit: 5 }
);

console.log(results.web?.[0]);
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```ts
const results = await client.search("firecrawl pricing", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  scrapeOptions: {
    formats: [
      "markdown",
      { type: "json", prompt: "Extract pricing tiers and limits." }
    ],
    onlyMainContent: true,
    actions: [
      { type: "wait", milliseconds: 1000 },
      { type: "scroll", direction: "down" }
    ],
    proxy: "stealth"
  },
  integration: "agent-quickstart",
  origin: "agent-docs"
});
```

<div id="parameters">
  ### Parámetros
</div>

* `query`: Cadena de búsqueda. Siempre obligatoria.
* `sources`: Fuentes donde buscar. Valores admitidos: `"web"`, `"news"`, `"images"` o `{ type: "web" | "news" | "images" }`. Úsalo para controlar qué arrays de resultados se devuelven.
* `categories`: Filtros de resultados. Valores admitidos: `"github"`, `"research"`, `"pdf"` o `{ type: "github" | "research" | "pdf" }`. Úsalo para limitar los resultados por categoría.
* `limit`: Número máximo de resultados que se devolverán. Úsalo para limitar el tamaño de la respuesta.
* `tbs`: Cadena de filtro de búsqueda basada en el tiempo. Úsalo para limitar por rango temporal (p. ej., `qdr:d`).
* `location`: Cadena de ubicación para resultados localizados. Úsalo para consultas específicas por ubicación geográfica.
* `ignoreInvalidURLs`: Excluye las URL no válidas de los resultados. Úsalo al encadenarlo con otros endpoints.
* `timeout`: Tiempo de espera de la solicitud en milisegundos. Úsalo para limitar el tiempo de búsqueda.
* `scrapeOptions`: Opciones de scraping aplicadas a cada resultado (los mismos campos que `scrape`). Úsalo cuando quieras incluir contenido junto con los resultados.
* `integration`: Identificador de integración. Úsalo para seguimiento y analítica.
* `origin`: Etiqueta de origen opcional. Úsalo para atribuir solicitudes.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `scrape` cuando ya conoces la URL y quieres obtener contenido en formatos específicos.

### Método del SDK recomendado

`client.scrape(url, options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const doc = await client.scrape("https://example.com", {
  formats: ["markdown"]
});

console.log(doc.markdown);
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```ts
const doc = await client.scrape("https://example.com/docs", {
  formats: [
    "markdown",
    { type: "json", prompt: "Extract headings and their slugs." },
    { type: "changeTracking", modes: ["git-diff"], tag: "docs" },
    { type: "screenshot", fullPage: true, viewport: { width: 1280, height: 720 } },
    {
      type: "attributes",
      selectors: [{ selector: "a", attribute: "href" }]
    },
    { type: "query", prompt: "What is the pricing model?" }
  ],
  onlyMainContent: true,
  waitFor: 1500,
  actions: [
    { type: "click", selector: "button#accept" },
    { type: "scroll", direction: "down" }
  ],
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  location: { country: "US", languages: ["en-US"] },
  proxy: "stealth",
  profile: { name: "docs-session", saveChanges: true },
  maxAge: 3600000
});
```

<div id="parameters">
  ### Parámetros
</div>

* `url`: URL sobre la que hacer scraping. Siempre obligatoria.
* `formats`: Formatos de salida que se incluirán. Cadenas de formato compatibles: `markdown`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `json`, `attributes`, `branding`, `audio`. Objetos de formato: `json` `{ type: "json", prompt?, schema? }` (requiere `prompt` o `schema`), `changeTracking` `{ type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag? }`, `screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`, `attributes` `{ type: "attributes", selectors: [{ selector, attribute }] }`, `query` `{ type: "query", prompt }`, además de `{ type: "<format>" }` para cualquier formato de cadena.
* `headers`: Encabezados HTTP adicionales. Úsalo para autenticación, cookies o `user-agent`.
* `includeTags`: Incluye solo contenido de estas etiquetas HTML. Úsalo para acotar la extracción.
* `excludeTags`: Excluye contenido de estas etiquetas HTML. Úsalo para eliminar ruido.
* `onlyMainContent`: Extrae solo el contenido principal. Úsalo para omitir la navegación y el pie de página.
* `timeout`: Tiempo de espera del scraping en milisegundos. Úsalo para limitar el tiempo de ejecución.
* `waitFor`: Retraso antes del scraping en milisegundos. Úsalo para páginas renderizadas con JS.
* `mobile`: Emula un dispositivo móvil. Úsalo para diseños móviles.
* `parsers`: Lista o configuración de analizadores. Configuración de PDF compatible: `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }` o entradas de cadena (por ejemplo, `"pdf"`).
* `actions`: Acciones del navegador que se ejecutan antes del scraping. Acciones compatibles: `wait` `{ type: "wait", milliseconds? | selector? }`, `screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`, `click` `{ type: "click", selector }`, `write` `{ type: "write", text }`, `press` `{ type: "press", key }`, `scroll` `{ type: "scroll", direction: "up" | "down", selector? }`, `scrape` `{ type: "scrape" }`, `executeJavascript` `{ type: "executeJavascript", script }`, `pdf` `{ type: "pdf", format?, landscape?, scale? }` con formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuración de ubicación `{ country, languages }`. Úsalo para páginas específicas por ubicación geográfica.
* `skipTlsVerification`: Omite la verificación del certificado TLS. Úsalo para certificados autofirmados.
* `removeBase64Images`: Elimina las imágenes base64 de la salida markdown. Úsalo para reducir el tamaño de la carga útil.
* `fastMode`: Habilita el modo rápido. Úsalo cuando la velocidad sea lo más importante.
* `useMock`: Usa un fixture simulado con nombre (interno/testing).
* `blockAds`: Habilita el bloqueo de anuncios y cookies. Úsalo para reducir ruido.
* `proxy`: Modo proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"` o una cadena de proxy personalizada. Úsalo para sitios con protección antibots.
* `maxAge`: Acepta contenido en caché con una antigüedad inferior a este valor (ms). Úsalo para favorecer aciertos de caché.
* `minAge`: Devuelve solo contenido en caché con al menos esta antigüedad (ms). Úsalo para lecturas solo de caché.
* `storeInCache`: Almacena el resultado en caché/índice. Úsalo cuando quieras reutilizarlo.
* `profile`: Perfil persistente del navegador `{ name, saveChanges }`. Úsalo para conservar el estado de la sesión.
* `integration`: Identificador de integración. Úsalo para seguimiento y analítica.
* `origin`: Etiqueta de origen opcional. Úsalo para atribuir solicitudes.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `interact` cuando la página requiera hacer clic, completar formularios u otras acciones en el navegador después del scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.interact(jobId, args)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const run = await client.interact("scrape-job-id", {
  code: "console.log(await page.url())"
});

console.log(run.stdout);
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```ts
const run = await client.interact("scrape-job-id", {
  prompt: "Open the pricing page and list the plan names.",
  language: "node",
  timeout: 60,
  origin: "agent-docs"
});

console.log(run.output);
```

<div id="parameters">
  ### Parámetros
</div>

* `jobId`: ID del trabajo de scraping (de `document.metadata.scrapeId`). Siempre es obligatorio.
* `code`: Código que se ejecuta en el sandbox del navegador. Úsalo para interacciones mediante scripts.
* `prompt`: Instrucción en lenguaje natural para el agente del navegador. Úsalo cuando prefieras acciones guiadas por el agente.
* `language`: Lenguaje de ejecución `"python"`, `"node"` o `"bash"` (predeterminado: `"node"`). Úsalo para que coincida con tu código.
* `timeout`: Tiempo de espera de ejecución en segundos. Úsalo para limitar ejecuciones largas.
* `origin`: Etiqueta de origen opcional. Úsala para atribuir la solicitud.

<div id="notes">
  ## Notas
</div>

* Los parámetros usan camelCase en JS/TS.
* Alias obsoletos: `scrapeExecute`, `stopInteractiveBrowser`, `deleteScrapeBrowser` (usa `interact` y `stopInteraction`).
* `Firecrawl` es el cliente unificado de v2; v1 sigue disponible en `.v1` para compatibilidad con versiones anteriores.

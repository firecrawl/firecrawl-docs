<div id="firecrawl-python-agent-quickstart">
  # Firecrawl Guía de inicio rápido del agente de Python
</div>

Este archivo es la guía de inicio rápido de referencia para agentes externos y se genera a partir del código fuente del SDK y de la especificación OpenAPI v2.

<div id="install">
  ## Instalación
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## Autenticación
</div>

```python
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])
```

<div id="when-to-use-what">
  ## Cuándo usar cada uno
</div>

* `search`: úsalo cuando empiezas con una consulta y necesitas descubrir resultados.
* `scrape`: úsalo cuando ya tienes una URL y quieres extraer el contenido de la página.
* `interact`: úsalo cuando la página requiere clics, formularios o acciones del navegador después del scraping.

<div id="search">
  ## Búsqueda
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `search` para descubrir URL relevantes a partir de una consulta y, de forma opcional, hacer scraping de cada resultado. Puedes limitar las consultas a un solo sitio con el operador estándar `site:` (por ejemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de hacer scraping de una URL específica.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.search(query, **options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```python
results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    limit=5,
)

print(results.web[0] if results.web else None)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```python
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "firecrawl pricing",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:w",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            {"type": "json", "prompt": "Extract pricing tiers and limits."},
        ],
        only_main_content=True,
        proxy="stealth",
    ),
    integration="agent-quickstart",
)
```

<div id="parameters">
  ### Parámetros
</div>

* `query`: Cadena de consulta de búsqueda. Siempre obligatorio.
* `sources`: Fuentes donde buscar. Valores admitidos: `"web"`, `"news"`, `"images"` u objetos `{"type": ...}`. Úsalo para controlar qué arrays de resultados se devuelven.
* `categories`: Filtros de resultados. Valores admitidos: `"github"`, `"research"`, `"pdf"` u objetos `{"type": ...}`. Úsalo para limitar los resultados por categoría.
* `limit`: Número máximo de resultados que se devolverán. Predeterminado: `5`.
* `tbs`: Cadena de filtro de búsqueda basada en tiempo. Úsala para limitar por rango temporal (p. ej., `qdr:d`).
* `location`: Cadena de ubicación para resultados localizados. Úsala para consultas específicas por ubicación geográfica.
* `ignore_invalid_urls`: Excluye las URL no válidas de los resultados. Úsalo al canalizar la salida a otros endpoints.
* `timeout`: Tiempo de espera de la solicitud en milisegundos. Predeterminado: `300000`.
* `scrape_options`: Opciones de scraping aplicadas a cada resultado (los mismos campos que `scrape`). Úsalo cuando quieras incluir contenido junto con los resultados.
* `integration`: Identificador de integración. Úsalo para seguimiento y analítica.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `scrape` cuando ya conozcas la URL y quieras obtener contenido en formatos específicos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.scrape(url, **options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```python
doc = client.scrape(
    "https://example.com",
    formats=["markdown"],
)

print(doc.markdown)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```python
doc = client.scrape(
    "https://example.com/docs",
    formats=[
        "markdown",
        {"type": "json", "prompt": "Extract headings and slugs."},
        {"type": "change_tracking", "modes": ["git-diff"], "tag": "docs"},
        {"type": "screenshot", "full_page": True, "viewport": {"width": 1280, "height": 720}},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
        {"type": "query", "prompt": "What is the pricing model?"},
    ],
    only_main_content=True,
    wait_for=1500,
    actions=[
        {"type": "click", "selector": "button#accept"},
        {"type": "scroll", "direction": "down"},
    ],
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    location={"country": "US", "languages": ["en-US"]},
    proxy="stealth",
    profile={"name": "docs-session", "save_changes": True},
    max_age=3_600_000,
)
```

<div id="parameters">
  ### Parámetros
</div>

* `url`: URL para hacer scraping. Siempre es obligatorio.
* `formats`: Formatos de salida que se incluirán. Acepta un objeto `ScrapeFormats` (campos `markdown`, `html`, `raw_html`, `summary`, `links`, `images`, `screenshot`, `change_tracking`, `json`) o una lista de cadenas u objetos de formato. Las cadenas de formato compatibles incluyen `markdown`, `html`, `rawHtml`, `raw_html`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `change_tracking`, `json`, `attributes`, `branding`, `query`, `audio`. Objetos de formato: `json` `{type: "json", prompt?, schema?}` (requiere `prompt` o `schema`), `changeTracking` `{type: "changeTracking" | "change_tracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `attributes` `{type: "attributes", selectors: [{selector, attribute}]}`, `query` `{type: "query", prompt}`.
* `headers`: Encabezados HTTP adicionales. Úsalo para autenticación, cookies o user-agent.
* `include_tags`: Incluye solo contenido de estas etiquetas HTML. Úsalo para acotar la extracción.
* `exclude_tags`: Excluye contenido de estas etiquetas HTML. Úsalo para eliminar ruido.
* `only_main_content`: Extrae solo el contenido principal. Úsalo para omitir la navegación y el pie de página.
* `timeout`: Tiempo de espera del scraping en milisegundos. Úsalo para limitar el tiempo de ejecución.
* `wait_for`: Retraso antes del scraping en milisegundos. Úsalo para páginas renderizadas con JS.
* `mobile`: Emula un dispositivo móvil. Úsalo para diseños móviles.
* `parsers`: Lista de analizadores o configuraciones. Configuración de PDF compatible: `{type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: int}` o entradas de cadena (por ejemplo, `"pdf"`).
* `actions`: Acciones del navegador que se ejecutan antes del scraping. Acciones compatibles: `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `click` `{type: "click", selector}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}` con formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuración de ubicación `{country, languages}`. Úsalo para páginas específicas por geolocalización.
* `skip_tls_verification`: Omite la verificación del certificado TLS. Úsalo para certificados autofirmados.
* `remove_base64_images`: Elimina imágenes en base64 de la salida markdown. Úsalo para reducir el tamaño de la carga útil.
* `fast_mode`: Habilita el modo rápido. Úsalo cuando la velocidad sea lo más importante.
* `use_mock`: Usa un mock fixture con nombre (interno/pruebas).
* `block_ads`: Habilita el bloqueo de anuncios y cookies. Úsalo para reducir el ruido.
* `proxy`: Modo de proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`.
* `max_age`: Acepta contenido en caché con una antigüedad inferior a esta (ms). Úsalo para favorecer los aciertos de caché.
* `store_in_cache`: Almacena el resultado en la caché o el índice. Úsalo cuando quieras reutilizarlo.
* `profile`: Perfil persistente del navegador `{name, save_changes}` (también acepta `saveChanges`). Úsalo para conservar el estado de la sesión.
* `integration`: Identificador de integración. Úsalo para seguimiento y analítica.

<div id="interact">
  ## Interactuar
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `interact` cuando la página requiera clics, completar formularios u otras acciones en el navegador tras un scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.interact(job_id, code=None, *, prompt=None, **options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```python
run = client.interact(
    "scrape-job-id",
    code="print(await page.url())",
)

print(run.stdout)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```python
run = client.interact(
    "scrape-job-id",
    prompt="Open the pricing page and list the plan names.",
    language="python",
    timeout=60,
    origin="agent-docs",
)

print(run.output)
```

<div id="parameters">
  ### Parámetros
</div>

* `job_id`: ID del trabajo de scraping (de `document.metadata_typed.scrape_id`). Siempre es obligatorio.
* `code`: Código que se ejecuta en el sandbox del navegador. Úsalo para interacciones mediante scripts.
* `prompt`: Instrucción en lenguaje natural para el agente del navegador. Úsalo cuando prefieras acciones dirigidas por el agente.
* `language`: Lenguaje de ejecución `"python"`, `"node"` o `"bash"` (predeterminado: `"node"`). Úsalo para ajustarlo a tu código.
* `timeout`: Tiempo de espera de ejecución en segundos. Úsalo para limitar ejecuciones largas.
* `origin`: Etiqueta de origen opcional. Úsala para atribuir solicitudes.

<div id="notes">
  ## Notas
</div>

* Los parámetros usan snake&#95;case en Python; el SDK normaliza las cargas útiles al camelCase de la API.
* Alias obsoletos: `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (usa `interact` y `stop_interaction`).
* `Firecrawl` es el cliente unificado de v2; v1 sigue disponible en `.v1` para compatibilidad heredada.

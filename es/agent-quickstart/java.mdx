<div id="firecrawl-java-agent-quickstart">
  # Guía de inicio rápido del agente de Java de Firecrawl
</div>

Este archivo es la guía de inicio rápido de referencia para agentes externos y se genera a partir del código fuente del SDK y de la especificación OpenAPI v2.

<div id="install">
  ## Instalar
</div>

```gradle
dependencies {
  implementation("com.firecrawl:firecrawl-java:1.2.0")
}
```

<div id="authenticate">
  ## Autenticación
</div>

```java
import com.firecrawl.client.FirecrawlClient;

FirecrawlClient client = FirecrawlClient.builder()
    .apiKey(System.getenv("FIRECRAWL_API_KEY"))
    .build();
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: úsalo cuando partes de una consulta y necesitas descubrir resultados.
* `scrape`: úsalo cuando ya tienes una URL y quieres extraer el contenido de la página.
* `interact`: úsalo cuando la página requiere clics, formularios o acciones del navegador después del scraping.

<div id="search">
  ## Búsqueda
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `search` para encontrar URL relevantes a partir de una consulta y, opcionalmente, hacer scraping de cada resultado. Puedes restringir la consulta a un solo sitio con el operador estándar `site:` (por ejemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de hacer scraping de una URL específica.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.search(query, options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```java
import com.firecrawl.models.SearchData;

SearchData data = client.search("site:docs.firecrawl.dev crawl webhooks");
System.out.println(data.getWeb());
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```java
import com.firecrawl.models.SearchData;
import com.firecrawl.models.SearchOptions;
import com.firecrawl.models.ScrapeOptions;

SearchData data = client.search(
    "firecrawl pricing",
    SearchOptions.builder()
        .sources(java.util.List.of("web", "news"))
        .categories(java.util.List.of("research"))
        .limit(10)
        .tbs("qdr:w")
        .location("San Francisco,California,United States")
        .ignoreInvalidURLs(true)
        .scrapeOptions(
            ScrapeOptions.builder()
                .formats(java.util.List.of(
                    "markdown",
                    java.util.Map.of("type", "json", "prompt", "Extract pricing tiers and limits.")
                ))
                .onlyMainContent(true)
                .proxy("stealth")
                .build()
        )
        .integration("agent-quickstart")
        .build()
);
```

<div id="parameters">
  ### Parámetros
</div>

* `query`: Cadena de consulta de búsqueda. Siempre obligatorio.
* `sources`: Fuentes donde buscar. Valores compatibles: `"web"`, `"news"`, `"images"` o mapeos `{type: "web" | "news" | "images"}`. Úsalo para controlar qué arrays de resultados se devuelven.
* `categories`: Filtros de resultados. Valores compatibles: `"github"`, `"research"`, `"pdf"` o mapeos `{type: "github" | "research" | "pdf"}`. Úsalo para acotar los resultados por categoría.
* `limit`: Número máximo de resultados que se devolverán. Úsalo para limitar el tamaño de la respuesta.
* `tbs`: Cadena de filtro de búsqueda basada en tiempo. Úsalo para limitar por rango temporal (p. ej., `qdr:d`).
* `location`: Cadena de ubicación para resultados localizados. Úsalo para consultas geográficas específicas.
* `ignoreInvalidURLs`: Excluye las URL no válidas de los resultados. Úsalo al encadenar con otros endpoints.
* `timeout`: Tiempo de espera de la solicitud en milisegundos. Úsalo para limitar el tiempo de búsqueda.
* `scrapeOptions`: Opciones de scraping aplicadas a cada resultado (los mismos campos que `scrape`). Úsalo cuando quieras obtener contenido junto con los resultados.
* `integration`: Identificador de integración. Úsalo para seguimiento y analítica.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `scrape` cuando ya conozcas la URL y quieras obtener contenido en formatos específicos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com",
    ScrapeOptions.builder()
        .formats(java.util.List.of("markdown"))
        .build()
);

System.out.println(doc.getMarkdown());
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.LocationConfig;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com/docs",
    ScrapeOptions.builder()
        .formats(java.util.List.of(
            "markdown",
            java.util.Map.of("type", "json", "prompt", "Extract headings and slugs."),
            java.util.Map.of("type", "changeTracking", "modes", java.util.List.of("git-diff"))
        ))
        .onlyMainContent(true)
        .waitFor(1500)
        .actions(java.util.List.of(
            java.util.Map.of("type", "click", "selector", "button#accept"),
            java.util.Map.of("type", "scroll", "direction", "down")
        ))
        .parsers(java.util.List.of(java.util.Map.of("type", "pdf", "mode", "auto", "maxPages", 5)))
        .location(LocationConfig.builder()
            .country("US")
            .languages(java.util.List.of("en-US"))
            .build())
        .proxy("stealth")
        .maxAge(3_600_000L)
        .build()
);
```

<div id="parameters">
  ### Parámetros
</div>

* `url`: URL para hacer scraping. Siempre es obligatoria.
* `formats`: Formatos de salida que se incluirán. Cadenas de formato compatibles: `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`. Objetos de formato: `json` `{type: "json", prompt?, schema?}`, `changeTracking` `{type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`.
* `headers`: Encabezados HTTP adicionales. Úsalos para autenticación, cookies o user-agent.
* `includeTags`: Incluye solo contenido de estas etiquetas HTML. Úsalo para acotar la extracción.
* `excludeTags`: Excluye el contenido de estas etiquetas HTML. Úsalo para eliminar ruido.
* `onlyMainContent`: Extrae solo el contenido principal. Úsalo para omitir la navegación y el pie de página.
* `timeout`: Tiempo de espera del scraping en milisegundos. Úsalo para limitar el tiempo de ejecución.
* `waitFor`: Retraso antes del scraping en milisegundos. Úsalo para páginas renderizadas con JS.
* `mobile`: Emula un dispositivo móvil. Úsalo para diseños móviles.
* `parsers`: Lista o configuraciones de analizadores. Configuración de PDF compatible: `{type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number}` o entradas de texto (p. ej., `"pdf"`).
* `actions`: Acciones del navegador que se ejecutan antes del scraping. Acciones compatibles: `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`, `click` `{type: "click", selector, all?}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}` con formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuración de ubicación (`country`, `languages`). Úsalo para páginas específicas por región.
* `skipTlsVerification`: Omite la verificación del certificado TLS. Úsalo para certificados autofirmados.
* `removeBase64Images`: Elimina las imágenes base64 de la salida en markdown. Úsalo para reducir el tamaño de la carga útil.
* `blockAds`: Activa el bloqueo de anuncios y cookies. Úsalo para reducir ruido.
* `proxy`: Modo proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"` o una URL de proxy personalizada.
* `maxAge`: Acepta contenido en caché con una antigüedad inferior a esta (ms). Úsalo para favorecer aciertos de caché.
* `storeInCache`: Almacena el resultado en la caché/índice. Úsalo cuando quieras reutilizarlo.
* `integration`: Identificador de integración. Úsalo para seguimiento y analítica.

<div id="interact">
  ## Interactuar
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa `interact` cuando la página requiera hacer clic, completar formularios u otras acciones del navegador después del scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.interact(jobId, code, language, timeout, origin)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());"
);

System.out.println(exec.getStdout());
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());",
    "node",
    60,
    "agent-docs"
);
```

<div id="parameters">
  ### Parámetros
</div>

* `jobId`: ID del trabajo de scraping (de `document.metadata.scrapeId`). Siempre es obligatorio.
* `code`: Código que se ejecutará en el sandbox del navegador. Úsalo para interacciones por script.
* `language`: Lenguaje de ejecución `"python"`, `"node"` o `"bash"` (predeterminado: `"node"`). Úsalo para que coincida con tu código.
* `timeout`: Tiempo de espera de ejecución en segundos. Úsalo para limitar ejecuciones prolongadas.
* `origin`: Etiqueta de origen opcional. Úsala para atribuir la solicitud.

<div id="notes">
  ## Notas
</div>

* Los parámetros usan camelCase en Java.
* `ScrapeOptions` y `SearchOptions` usan el patrón builder.
* Alias obsoletos: `scrapeExecute` y `deleteScrapeBrowser` (usa `interact` y `stopInteractiveBrowser`).
* El Java SDK solo expone `interact` basado en código (sin el parámetro `prompt`).

<div id="scrape-endpoint">
  ## extremo /scrape
</div>

El extremo `/scrape` se ha actualizado y rediseñado para mejorar la confiabilidad y la facilidad de uso. La estructura del nuevo cuerpo de la solicitud a `/scrape` es la siguiente:

```json
{
  "url": "<string>",
  "formats": ["markdown", "html", "rawHtml", "links", "screenshot", "json"],
  "includeTags": ["<string>"],
  "excludeTags": ["<string>"],
  "headers": { "<key>": "<value>" },
  "waitFor": 123,
  "timeout": 123
}
```

<div id="formats">
  ### Formatos
</div>

Ahora puedes elegir en qué formatos quieres el resultado. Puedes especificar varios formatos de salida. Los formatos compatibles son:

* Markdown (markdown)
* HTML (html)
* HTML en bruto (rawHtml) (sin modificaciones)
* Captura de pantalla (screenshot o screenshot@fullPage)
* Enlaces (links)
* JSON (json)
  De forma predeterminada, el resultado incluirá solo el formato markdown.

<div id="details-on-the-new-request-body">
  ### Detalles sobre el nuevo cuerpo de la solicitud
</div>

La siguiente tabla describe los cambios en los parámetros del cuerpo de la solicitud para el extremo `/scrape` en V1.

| Parámetro | Cambio | Descripción |
| --------- | ------ | ----------- |
| `onlyIncludeTags` | Movido y renombrado | Se movió al nivel raíz y se renombró a `includeTags`. |
| `removeTags` | Movido y renombrado | Se movió al nivel raíz y se renombró a `excludeTags`. |
| `onlyMainContent`| Movido | Se movió al nivel raíz. `true` por defecto. |
| `waitFor`| Movido | Se movió al nivel raíz. |
| `headers`| Movido | Se movió al nivel raíz. |
| `parsePDF`| Movido | Se movió al nivel raíz. |
| `extractorOptions`| Sin cambios ||
| `timeout`| Sin cambios ||
| `pageOptions` | Eliminado | Ya no se necesita el parámetro `pageOptions`. Las opciones de scrape se movieron al nivel raíz. |
| `replaceAllPathsWithAbsolutePaths` | Eliminado | `replaceAllPathsWithAbsolutePaths` ya no es necesario. Ahora todas las rutas son absolutas por defecto. |
| `includeHtml`| Eliminado | Agrega `"html"` a `formatos` en su lugar. |
| `includeRawHtml`| Eliminado | Agrega `"rawHtml"` a `formatos` en su lugar. |
| `screenshot`| Eliminado | Agrega `"screenshot"` a `formatos` en su lugar. |
| `fullPageScreenshot`| Eliminado | Agrega `"screenshot@fullPage"` a `formatos` en su lugar. |
| `extractorOptions` | Eliminado | Usa el formato `"json"` con el objeto `jsonOptions`. |

El nuevo formato `json` se describe en la sección [llm-extract](/es/features/extract).

<div id="crawl-endpoint">
  ## extremo /crawl
</div>

También actualizamos el extremo `/crawl` en `v1`. Consulta a continuación el cuerpo de la solicitud mejorado:

```json
{
  "url": "<string>",
  "excludePaths": ["<string>"],
  "includePaths": ["<string>"],
  "maxDepth": 2,
  "ignoreSitemap": true,
  "limit": 10,
  "allowBackwardLinks": true,
  "allowExternalLinks": true,
  "scrapeOptions": {
    // mismas opciones que en /scrape
    "formats": ["markdown", "html", "rawHtml", "screenshot", "links"],
    "headers": { "<key>": "<value>" },
    "includeTags": ["<string>"],
    "excludeTags": ["<string>"],
    "onlyMainContent": true,
    "waitFor": 123
  }
}
```

<div id="details-on-the-new-request-body">
  ### Detalles del nuevo cuerpo de la solicitud
</div>

La tabla siguiente detalla los cambios en los parámetros del cuerpo de la solicitud para el extremo `/crawl` en V1.

| Parámetro | Cambio | Descripción |
| --------- | ------ | ----------- |
| `pageOptions` | Renombrado | Cambiado a `scrapeOptions`. |
| `includes` | Movido y renombrado | Trasladado al nivel raíz. Cambiado a `includePaths`. |
| `excludes` | Movido y renombrado | Trasladado al nivel raíz. Cambiado a `excludePaths`. |
| `allowBackwardCrawling` | Movido y renombrado | Trasladado al nivel raíz. Cambiado a `allowBackwardLinks`. |
| `allowExternalLinks` | Movido | Trasladado al nivel raíz. |
| `maxDepth` | Movido | Trasladado al nivel raíz. |
| `ignoreSitemap` | Movido | Trasladado al nivel raíz. |
| `limit` | Movido | Trasladado al nivel raíz. |
| `crawlerOptions` | Eliminado | No se requiere el parámetro `crawlerOptions`. Las opciones de crawl se trasladaron al nivel raíz. |
| `timeout` | Eliminado | Usa `timeout` dentro de `scrapeOptions`. |

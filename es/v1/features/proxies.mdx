Firecrawl ofrece diferentes tipos de proxy para ayudarte a extraer datos de sitios web con distintos niveles de protección antibots. El tipo de proxy se puede especificar mediante el parámetro `proxy`.

> De forma predeterminada, Firecrawl enruta todas las solicitudes a través de proxies para garantizar la fiabilidad y el acceso, incluso si no especificas un tipo de proxy o una ubicación.

<div id="location-based-proxy-selection">
  ## Selección de proxy por ubicación
</div>

Firecrawl selecciona automáticamente el mejor proxy según tu ubicación especificada o detectada. Esto ayuda a optimizar el rendimiento y la fiabilidad del scraping. Sin embargo, no todas las ubicaciones están admitidas actualmente. Las siguientes ubicaciones están disponibles:

| Country Code | Country Name         | Stealth Mode Support |
|--------------|----------------------|----------------------|
| AE           | United Arab Emirates | No                   |
| AU           | Australia            | No                   |
| BR           | Brazil               | Yes                  |
| CN           | China                | No                   |
| DE           | Germany              | Yes                  |
| FR           | France               | Yes                  |
| GB           | United Kingdom       | No                   |
| JP           | Japan                | No                   |
| QA           | Qatar                | No                   |
| TR           | Turkey               | No                   |
| US           | United States        | Yes                  |
| VN           | Vietnam              | No                   |

<Warning>La lista de ubicaciones de proxy admitidas se actualizó por última vez el 15 de mayo de 2025. La disponibilidad puede cambiar con el tiempo.</Warning>

Si necesitas proxies en una ubicación que no aparece arriba, por favor [contáctanos](mailto:help@firecrawl.com) y cuéntanos tus requisitos.

Si no especificas un proxy o una ubicación, Firecrawl seleccionará automáticamente la mejor opción según el sitio de destino y tu solicitud.

<div id="how-to-specify-proxy-location">
  ## Cómo especificar la ubicación del proxy
</div>

Puedes solicitar una ubicación de proxy específica configurando el parámetro `location.country` en tu solicitud. Por ejemplo, para usar un proxy de Brasil, establece `location.country` en `BR`.

Para obtener todos los detalles, consulta la [referencia de la API de `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location-country).

<div id="proxy-types">
  ## Tipos de proxy
</div>

Firecrawl admite tres tipos de proxy:

* **basic**: Proxies para extraer datos de sitios con soluciones anti-bot nulas o básicas. Rápidos y suelen funcionar.
* **stealth**: Proxies sigilosos para extraer datos de sitios con soluciones anti-bot avanzadas o que bloquean proxies comunes. Más lentos, pero más fiables en ciertos sitios. [Más información sobre el modo Stealth →](/es/features/stealth-mode)
* **auto**: Firecrawl volverá a intentar automáticamente con proxies stealth si el proxy basic falla. Si el reintento con stealth tiene éxito, se cobrarán 5 créditos por la extracción. Si el primer intento con basic tiene éxito, solo se cobrará el costo regular.

***

> **Nota:** Para información detallada sobre el uso de proxies stealth, incluidos los costos en créditos y las estrategias de reintento, consulta la [documentación del modo Stealth](/es/features/stealth-mode).

El seguimiento de cambios es aún más potente cuando se combina con el rastreo. Mientras que el seguimiento en páginas individuales muestra cambios de contenido, usarlo con el rastreo permite monitorear toda la estructura del sitio: detectar páginas nuevas, eliminadas y las que han quedado ocultas.

<div id="basic-usage">
  ## Uso básico
</div>

Para habilitar el seguimientoDeCambios durante un rastreo, inclúyelo en la matriz `formats` de tus `scrapeOptions`:

```typescript
// JavaScript/TypeScript
const app = new FirecrawlApp({ apiKey: 'tu-api-key' });
const result = await app.crawl('https://example.com', {
  scrapeOptions: {
    formats: ['markdown', 'seguimientoDeCambios']
  }
});
```

```python
# Python
app = FirecrawlApp(api_key='tu-api-key')
result = app.crawl('https://firecrawl.dev', {
    'scrapeOptions': {
        'formats': ['markdown', 'seguimientoDeCambios']
    }
})
```

<div id="understanding-change-status">
  ## Comprender el estado de cambios
</div>

Al usar el seguimientoDeCambios con crawl, el campo `changeStatus` resulta especialmente útil:

* `new`: Una página que no existía en tu rastreo anterior
* `same`: Una página que existe y no ha cambiado desde tu último rastreo
* `changed`: Una página que existe pero ha sido modificada desde tu último rastreo
* `removed`: Una página que existía en tu rastreo anterior pero ya no se encuentra

<div id="page-visibility">
  ## Visibilidad de la página
</div>

El campo `visibility` te ayuda a entender cómo se descubren las páginas:

* `visible`: La página se puede encontrar a través de enlaces o del sitemap
* `hidden`: La página sigue existiendo, pero ya no está enlazada ni aparece en el sitemap

Esto es especialmente útil para:

* Detectar contenido huérfano
* Encontrar páginas eliminadas accidentalmente de la navegación
* Supervisar cambios en la estructura del sitio
* Identificar contenido que debería volver a enlazarse o eliminarse

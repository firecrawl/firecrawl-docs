<div id="old-version">
  # Versión antigua
</div>

Con el nuevo formato `changeTracking`, puedes monitorear los cambios en un sitio web.

El formato `changeTracking` devuelve los siguientes campos en la respuesta:

* `previousScrapeAt`: La marca de tiempo del scrape anterior con el que se compara la página actual. (`null` si no hubo scrape previo)
* `changeStatus`: El resultado de la comparación entre las dos versiones de la página
  * `new`: Esta página no existía o no se había descubierto antes. (generalmente tiene `previousScrapeAt` en `null`)
  * `same`: El contenido de esta página no ha cambiado desde el último scrape.
  * `changed`: El contenido de esta página ha cambiado desde el último scrape.
  * `removed`: Esta página fue eliminada desde el último scrape.
* `visibility`: La visibilidad de la página o URL actual.
  * `visible`: Esta página es visible, lo que significa que su URL se descubrió por una ruta orgánica (a través de enlaces en otras páginas visibles o del sitemap).
  * `hidden`: Esta página no es visible, lo que significa que su URL se recuperó de memoria a partir de rastreos anteriores de la misma URL.

Algunos datos útiles:

* Los scrapes siempre se comparan mediante su respuesta en formato markdown.
  * El formato `markdown` también debe especificarse al usar el formato `changeTracking`. Además, se pueden incluir otros formatos.
  * El algoritmo de comparación es resistente a cambios en espacios en blanco y en el orden del contenido. Actualmente se ignoran las URLs de origen de iframes para mitigar captchas y antibots con URLs aleatorias.
* Los scrapes previos con los que se compara actualmente solo se emparejan por la URL de origen, el ID del equipo y el formato `markdown`, y no por otras opciones de scrape o crawl.
  * Para una comparación efectiva, la URL de entrada debe ser exactamente la misma que la solicitud anterior para el mismo contenido. Esta es una limitación que estamos trabajando arduamente para superar.
  * Rastrear las mismas URLs con diferentes `includePaths`/`excludePaths` tendrá inconsistencias al usar `changeTracking`.
  * Hacer scrape de las mismas URLs con diferentes `includeTags`/`excludeTags`/`onlyMainContent` tendrá inconsistencias al usar `changeTracking`.
  * Las páginas también se compararán contra scrapes previos que solo tengan el formato `markdown` sin el formato `changeTracking`.
  * Las comparaciones están acotadas a tu equipo. Si haces scrape de una URL por primera vez con tu clave de API, su `changeStatus` siempre será `new`, incluso si otros usuarios de Firecrawl la han scrapeado antes.
* Mientras esté en Beta, se recomienda monitorear el campo `warning` del documento resultante y manejar el caso en que el objeto `changeTracking` pueda faltar en la respuesta. Estaremos monitoreando el sistema para asegurar que esto ocurra lo menos posible.
  * Esto puede suceder cuando la consulta a nuestra base de datos para encontrar el scrape previo con el que comparar supera el tiempo de espera. Estaremos monitoreando esta situación y publicaremos mejoras y optimizaciones antes del lanzamiento oficial de la función.
* Actualmente no devolvemos un diff del contenido de la página que cambió.
  * No sabemos cuál sería el formato óptimo para esto. Si quieres tener el diff, probablemente ya sabes en qué formato lo quieres. Por favor, contáctanos en [mogery@firecrawl.com](mailto:mogery@firecrawl.com), ¡y lo añadiremos!

<div id="examples">
  ## Ejemplos
</div>

<div id="scrape">
  ### Scrape
</div>

```json
{
    "url": "https://firecrawl.dev",
    "formats": ["markdown", "seguimientoDeCambios"]
}
```

```json
{
  "success": true,
  "data": {
    "markdown": "...",
    "metadata": {...},
    "changeTracking": {
      "previousScrapeAt": "2025-03-30T15:07:17.543071+00:00",
      "changeStatus": "same",
      "visibility": "visible"
    }
  }
}
```

<div id="crawl">
  ### Rastreo
</div>

```json
{
    "url": "https://firecrawl.dev",
    "scrapeOptions": {
        "formats": ["markdown", "seguimientoDeCambios"]
    }
}
```

<div id="introducing-extract-beta">
  ## Presentación de /extract (Beta)
</div>

<Warning>
  [Usa la nueva documentación del Open Beta de /extract haciendo clic aquí 🔥](/es/features/extract)
</Warning>

Extrae datos estructurados de una o varias URL o de sitios web completos usando modelos de lenguaje (LLMs). Nuestro nuevo punto de conexión `/extract` te permite:

* Extraer datos estructurados de sitios web completos de una sola vez
* Conectar o crear aplicaciones de enriquecimiento de datos que necesitan datos estructurados de sitios web
* Desarrollar aplicaciones de IA que necesitan datos limpios de múltiples sitios web

<div id="considerations">
  ## Consideraciones
</div>

El punto de conexión `/extract` ofrece una extracción de datos flexible con esquemas personalizables. Los resultados pueden mejorarse ajustando el prompt. Actualmente está en beta y agradecemos tus comentarios.

<div id="extract-endpoint">
  ### punto de conexión /extract
</div>

Se usa para extraer datos estructurados de sitios web completos.

Al especificar URLs, puedes añadir `/*` a la URL para extraer información de toda la ruta del sitio web en lugar de solo una página.

Por ejemplo, `https://firecrawl.dev/*` intentará extraer datos de todas las páginas del dominio firecrawl.dev. El uso de `/*` aún está en pruebas; si tienes algún problema, escríbenos a help@firecrawl.com.

<div id="pending-response">
  #### Respuesta pendiente
</div>

Los trabajos de extracción pueden tener uno de los siguientes estados:

* `completed`: El trabajo de extracción se completó correctamente.
* `processing`: El trabajo de extracción aún está en curso.
* `failed`: El trabajo de extracción presentó un error y no se completó.
* `cancelled`: El trabajo de extracción fue cancelado por el usuario.

<div id="improving-results-with-web-search-external-links">
  ### Mejora de resultados con búsqueda web y enlaces externos
</div>

Si quieres mejorar los resultados de la extracción, puedes pasar el parámetro `enableWebSearch` al punto de conexión. Esto permitirá intentar encontrar los datos en enlaces externos, fuera del alcance de las URL proporcionadas.

<div id="billing">
  ### Facturación
</div>

Mientras `/extract` esté en beta, cobramos 5 créditos por cada URL extraída que se utilice para formar la respuesta final. Esto es para prevenir abusos. Esto cambiará en el futuro.

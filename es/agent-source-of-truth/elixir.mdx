<div id="firecrawl-elixir-source-of-truth">
  # Firecrawl Elixir fuente canónica
</div>

Este archivo es la fuente pública canónica para los agentes. Se genera a partir del SDK de Firecrawl Elixir (con estructura OpenAPI) y de la especificación OpenAPI v2.

<div id="install">
  ## Instalación
</div>

```elixir
# mix.exs
defp deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

<div id="authenticate">
  ## Autenticación
</div>

Configura tu clave de API en `config/runtime.exs`:

```elixir
config :firecrawl, api_key: System.fetch_env!("FIRECRAWL_API_KEY")
```

O pasa `api_key` en cada solicitud:

```elixir
{:ok, resp} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-...")
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: parte de una consulta para descubrir URL relevantes, incluidas las restricciones `site:`.
* `scrape`: obtiene y extrae contenido de una URL conocida.
* `interact`: ejecuta acciones del navegador o código cuando una página requiere clics, formularios o pasos posteriores al scraping.

<div id="search">
  ## Búsqueda
</div>

### Por qué usarlo

Usa search cuando necesites encontrar resultados a partir de una consulta y quieras que Firecrawl devuelva URL (y, opcionalmente, haga scraping de cada resultado). Puedes limitar los resultados a un dominio con el operador estándar `site:`, por ejemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000,
    actions: [
      %{type: "click", selector: "button#accept"},
      %{type: "wait", milliseconds: 750},
      %{type: "scrape"}
    ]
  ]
)
```

<div id="parameters">
  ### Parámetros
</div>

| Parámetro             | Tipo / Opciones                                               | Descripción                                                                                  |
| --------------------- | ------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| `query`               | `String`                                                      | Consulta de búsqueda. Usa `site:domain` para limitar los resultados a un sitio.              |
| `sources`             | `List` (`"web"`, `"news"`, `"images"` o mapas con `type`)     | Fuentes donde buscar.                                                                        |
| `categories`          | `List` (`"github"`, `"research"`, `"pdf"` o mapas con `type`) | Categorías para filtrar los resultados.                                                      |
| `limit`               | `integer`                                                     | Número máximo de resultados que se devolverán.                                               |
| `tbs`                 | `String`                                                      | Filtro de búsqueda por tiempo (por ejemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).                |
| `location`            | `String`                                                      | Cadena de ubicación para los resultados de búsqueda.                                         |
| `country`             | `String`                                                      | Código de país ISO para segmentar geográficamente los resultados de búsqueda.                |
| `enterprise`          | `List[String]` (`"anon"`, `"zdr"`)                            | Opciones ZDR empresariales cuando están habilitadas para tu equipo.                          |
| `ignore_invalid_urls` | `boolean`                                                     | Excluye las URL no válidas que otros endpoints de Firecrawl no pueden procesar con scraping. |
| `timeout`             | `integer`                                                     | Timeout en milisegundos.                                                                     |
| `scrape_options`      | `keyword`                                                     | Opciones de scraping aplicadas a cada resultado (los mismos campos que `scrape`).            |

<div id="scrape">
  ## Scraping
</div>

### Por qué usarlo

Usa `scrape` si ya tienes una URL y quieres que Firecrawl devuelva contenido estructurado y formatos opcionales.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`Firecrawl.scrape_and_extract_from_url(params, opts)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  include_tags: ["main", "article"],
  exclude_tags: ["nav", "footer"],
  only_main_content: true,
  wait_for: 1000,
  mobile: false,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "write", text: "firecrawl"},
    %{type: "press", key: "Enter"},
    %{type: "wait", milliseconds: 1500},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  skip_tls_verification: true,
  remove_base64_images: true,
  block_ads: true,
  proxy: "auto",
  max_age: 86400000,
  store_in_cache: true,
  profile: [name: "docs-session", saveChanges: true]
)
```

<div id="parameters">
  ### Parámetros
</div>

| Parámetro               | Tipo / Opciones    | Descripción                                                                                                                                               |                   |                |
| ----------------------- | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------- | -------------- |
| `url`                   | `String`           | URL que se va a scrapear.                                                                                                                                 |                   |                |
| `formats`               | `List`             | Formatos de salida que se deben incluir. Cadenas u objetos como `%{type: "json", prompt, schema}` o `%{type: "screenshot", fullPage, quality, viewport}`. |                   |                |
| `only_main_content`     | `boolean`          | Conserva solo el contenido principal.                                                                                                                     |                   |                |
| `include_tags`          | `List[String]`     | Etiquetas que se deben incluir.                                                                                                                           |                   |                |
| `exclude_tags`          | `List[String]`     | Etiquetas que se deben excluir.                                                                                                                           |                   |                |
| `max_age`               | `integer`          | Da preferencia a los datos en caché hasta esta antigüedad.                                                                                                |                   |                |
| `min_age`               | `integer`          | Solo devuelve datos en caché si alcanzan la antigüedad mínima.                                                                                            |                   |                |
| `headers`               | `map`              | Encabezados personalizados.                                                                                                                               |                   |                |
| `wait_for`              | `integer`          | Tiempo de espera en milisegundos antes del scraping.                                                                                                      |                   |                |
| `mobile`                | `boolean`          | Emula un dispositivo móvil.                                                                                                                               |                   |                |
| `skip_tls_verification` | `boolean`          | Omite la verificación del certificado TLS.                                                                                                                |                   |                |
| `timeout`               | `integer`          | Tiempo de espera en milisegundos.                                                                                                                         |                   |                |
| `parsers`               | `List`             | Configuración del parser (se admiten opciones de PDF).                                                                                                    |                   |                |
| `actions`               | `List`             | Acciones del navegador que se ejecutan antes del scraping.                                                                                                |                   |                |
| `location`              | `keyword`          | Configuración de ubicación para la solicitud.                                                                                                             |                   |                |
| `remove_base64_images`  | `boolean`          | Elimina las imágenes en base64 de la salida en markdown.                                                                                                  |                   |                |
| `block_ads`             | `boolean`          | Activa el bloqueo de anuncios y ventanas emergentes de cookies.                                                                                           |                   |                |
| `proxy`                 | `&quot;basic&quot; | &quot;enhanced&quot;                                                                                                                                      | &quot;auto&quot;` | Modo de proxy. |
| `store_in_cache`        | `boolean`          | Almacena los resultados en caché.                                                                                                                         |                   |                |
| `profile`               | `keyword`          | Perfil persistente del navegador, por ejemplo `[name: "profile", saveChanges: true]`.                                                                     |                   |                |
| `zero_data_retention`   | `boolean`          | Activa la retención de datos cero cuando tu cuenta lo permita.                                                                                            |                   |                |

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usar interact
</div>

Usa interact cuando una página requiera acciones del navegador o la ejecución de código después de que se inicie el scraping (clics, formularios o pasos mediante scripts).

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```elixir
{:ok, scrape_resp} = Firecrawl.scrape_and_extract_from_url(url: "https://example.com")
job_id = get_in(scrape_resp.body, ["data", "metadata", "scrapeId"])

{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  job_id,
  code: "console.log(document.title)",
  language: :node
)
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```elixir
{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  "<scrape_job_id>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="parameters">
  ### Parámetros
</div>

| Parámetro  | Tipo / Opciones | Descripción                                                            |        |                                                                        |
| ---------- | --------------- | ---------------------------------------------------------------------- | ------ | ---------------------------------------------------------------------- |
| `job_id`   | `String`        | ID del trabajo de scraping de `data.metadata.scrapeId`.                |        |                                                                        |
| `code`     | `String`        | Código que se ejecutará en la sesión del navegador. Obligatorio.       |        |                                                                        |
| `language` | `:python        | :node                                                                  | :bash` | Lenguaje de ejecución. El valor predeterminado es `:node` si se omite. |
| `timeout`  | `integer`       | Tiempo de espera de la ejecución, en segundos.                         |        |                                                                        |
| `origin`   | `String`        | Solo para uso interno. Reservado para la instrumentación de Firecrawl. |        |                                                                        |

<div id="notes">
  ## Notas
</div>

* Las funciones del cliente de Elixir tienen el formato de OpenAPI y aceptan parámetros de palabra clave en snake&#95;case que se convierten en campos JSON en camelCase.

Fuente canónica de curl de Firecrawl para agentes. Generada a partir del código fuente del SDK y de la especificación OpenAPI v2."

<div id="authenticate">
  ## Autenticación
</div>

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="when-to-use-what">
  ## Cuándo usar cada uno
</div>

* `search`: úsalo cuando empieces con una consulta y necesites descubrir resultados.
* `scrape`: úsalo cuando ya tengas una URL y quieras el contenido de la página.
* `interact`: úsalo cuando la página requiera clics, formularios o acciones del navegador posteriores al scraping.

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa search para descubrir páginas relevantes a partir de una consulta y luego elegir URLs para hacer scraping o interactuar con ellas. Puedes restringir los resultados a un sitio con `site:`, por ejemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`POST /search`

<div id="simple-example">
  ### Ejemplo simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev webhook retries"
  }'
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev crawl webhooks",
    "sources": ["web", "news"],
    "categories": ["research"],
    "limit": 10,
    "tbs": "qdr:m",
    "location": "San Francisco,California,United States",
    "country": "US",
    "ignoreInvalidURLs": true,
    "timeout": 60000,
    "enterprise": ["anon"],
    "scrapeOptions": {
      "formats": [
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract title and key endpoints."}
      ],
      "onlyMainContent": true,
      "includeTags": ["main", "article"],
      "excludeTags": ["nav", "footer"],
      "waitFor": 1000
    }
  }'
```

<div id="parameters">
  ### Parámetros
</div>

* `query`
  * Tipo: string
  * Usar cuando: necesites una consulta de búsqueda.
  * Notas: usa `site:example.com` para limitar los resultados a un dominio.

* `sources`
  * Tipo: array de nombres de fuentes u objetos de fuente tipados
  * Usar cuando: quieras controlar en qué fuentes se busca.
  * Valores confirmados:
    * `"web"`: resultados del índice web
    * `"news"`: resultados de noticias
    * `"images"`: resultados de imágenes
    * `{ "type": "web" | "news" | "images" }`: formato de objeto de fuente tipado

* `categories`
  * Tipo: array de nombres de categorías u objetos de categoría tipados
  * Usar cuando: quieras filtrar los resultados por categoría.
  * Valores confirmados:
    * `"github"`: resultados centrados en GitHub
    * `"research"`: resultados de investigación y académicos
    * `"pdf"`: resultados centrados en PDF
    * `{ "type": "github" | "research" | "pdf" }`: formato de objeto de categoría tipado

* `limit`
  * Tipo: number
  * Usar cuando: quieras limitar la cantidad de resultados.

* `tbs`
  * Tipo: string
  * Usar cuando: necesites un filtro temporal (por ejemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `location`
  * Tipo: string
  * Usar cuando: quieras resultados localizados.

* `country`
  * Tipo: string
  * Usar cuando: quieras segmentación ISO 3166-1 alpha-2 (por ejemplo, `"US"`).

* `ignoreInvalidURLs`
  * Tipo: boolean
  * Usar cuando: quieras descartar URL que otros endpoints no puedan extraer.

* `timeout`
  * Tipo: number
  * Usar cuando: necesites un tiempo de espera de la solicitud en milisegundos.

* `enterprise`
  * Tipo: array de strings
  * Usar cuando: necesites controles de búsqueda empresariales.
  * Valores confirmados:
    * `"zdr"`: retención de datos cero de extremo a extremo
    * `"anon"`: retención de datos cero anonimizada

* `scrapeOptions`
  * Tipo: object
  * Usar cuando: quieras extraer cada resultado de búsqueda (consulta los parámetros de scraping para ver los campos).

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa scrape cuando ya tengas una URL y quieras contenido estructurado en uno o varios formatos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`POST /scrape`

<div id="simple-example">
  ### Ejemplo simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.firecrawl.dev",
    "formats": ["markdown"]
  }'
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": [
      "markdown",
      "links",
      {"type": "json", "prompt": "Extract plan names and prices."},
      {"type": "screenshot", "fullPage": true, "quality": 80, "viewport": {"width": 1280, "height": 720}},
      {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    "headers": {"User-Agent": "FirecrawlDocsBot/1.0"},
    "onlyMainContent": true,
    "waitFor": 1000,
    "parsers": [{"type": "pdf", "mode": "auto", "maxPages": 5}],
    "actions": [
      {"type": "click", "selector": "#accept"},
      {"type": "wait", "milliseconds": 750},
      {"type": "scrape"}
    ],
    "location": {"country": "US", "languages": ["en-US"]},
    "removeBase64Images": true,
    "blockAds": true,
    "proxy": "auto",
    "maxAge": 86400000,
    "minAge": 1,
    "storeInCache": true,
    "profile": {"name": "docs-session", "saveChanges": true},
    "zeroDataRetention": false
  }'
```

<div id="parameters">
  ### Parámetros
</div>

* `url`
  * Tipo: string
  * Usar cuando: quieres hacer scraping de una página específica.

* `formats`
  * Tipo: array de cadenas de formato u objetos de formato
  * Usar cuando: quieres varios formatos de salida.
  * Cadenas de formato confirmadas:
    * `"markdown"`: contenido en markdown
    * `"html"`: HTML limpio
    * `"rawHtml"`: HTML sin procesar
    * `"links"`: enlaces de la página
    * `"images"`: URL de imágenes
    * `"screenshot"`: salida de captura de pantalla
    * `"summary"`: salida de resumen
    * `"changeTracking"`: salida de seguimiento de cambios
    * `"json"`: extracción JSON
    * `"branding"`: salida del perfil de marca
    * `"audio"`: extracción de audio
  * Campos del objeto de formato:
    * `type`: una de las cadenas de formato anteriores
    * `prompt`, `schema`: opciones de extracción JSON para `type: "json"`
    * `modes`, `schema`, `prompt`, `tag`: opciones de seguimiento de cambios para `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: opciones de captura de pantalla para `type: "screenshot"`

* `headers`
  * Tipo: object
  * Usar cuando: necesitas encabezados de solicitud personalizados.

* `includeTags`
  * Tipo: array de strings
  * Usar cuando: quieres incluir solo etiquetas HTML específicas.

* `excludeTags`
  * Tipo: array de strings
  * Usar cuando: quieres excluir etiquetas HTML específicas.

* `onlyMainContent`
  * Tipo: boolean
  * Usar cuando: quieres eliminar la navegación, el pie de página y otros elementos repetitivos.

* `timeout`
  * Tipo: number
  * Usar cuando: necesitas un tiempo de espera en milisegundos.

* `waitFor`
  * Tipo: number
  * Usar cuando: necesitas esperar a que la página se renderice (milisegundos).

* `mobile`
  * Tipo: boolean
  * Usar cuando: quieres una viewport móvil.

* `parsers`
  * Tipo: array de cadenas de parser u objetos
  * Usar cuando: necesitas controles de análisis de archivos.
  * Valores confirmados:
    * `"pdf"`
    * `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "maxPages": number }`

* `actions`
  * Tipo: array de objetos de acción
  * Usar cuando: necesitas acciones ligeras previas al scraping.
  * Tipos de acción confirmados:
    * `wait`: `milliseconds` o `selector` obligatorios
    * `screenshot`: `fullPage`, `quality`, `viewport` opcionales
    * `click`: `selector` obligatorio, `all` opcional
    * `write`: `text` obligatorio (haz clic primero para enfocar el campo de entrada)
    * `press`: `key` obligatoria
    * `scroll`: `direction` (`up` o `down`) obligatoria, `selector` opcional
    * `scrape`: sin campos adicionales
    * `executeJavascript`: `script` obligatorio
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` opcionales

* `location`
  * Tipo: objeto con `country` y `languages`
  * Usar cuando: necesitas scraping según la ubicación o el idioma.

* `skipTlsVerification`
  * Tipo: boolean
  * Usar cuando: necesitas omitir la verificación de TLS.

* `removeBase64Images`
  * Tipo: boolean
  * Usar cuando: quieres eliminar imágenes en base64 de la salida markdown.

* `blockAds`
  * Tipo: boolean
  * Usar cuando: quieres bloquear anuncios y ventanas emergentes de cookies.

* `proxy`
  * Tipo: string
  * Usar cuando: necesitas controlar el proxy.
  * Valores confirmados: `"basic"`, `"enhanced"`, `"auto"`

* `maxAge`
  * Tipo: number
  * Usar cuando: quieres datos en caché con una antigüedad máxima determinada (milisegundos).

* `minAge`
  * Tipo: number
  * Usar cuando: quieres datos en caché solo si tienen al menos esta antigüedad (milisegundos).

* `storeInCache`
  * Tipo: boolean
  * Usar cuando: quieres que Firecrawl almacene el resultado en caché.

* `profile`
  * Tipo: objeto con `name` y `saveChanges` opcional
  * Usar cuando: quieres un perfil de browser persistente compartido entre scrapes e interacciones.

* `zeroDataRetention`
  * Tipo: boolean
  * Usar cuando: quieres retención de datos cero para este scraping.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa Interact cuando una página requiera acciones del navegador o la ejecución de código una vez iniciado el scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`POST /scrape/{jobId}/interact`

<div id="simple-example">
  ### Ejemplo simple
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "console.log(await page.title());",
    "language": "node",
    "timeout": 60
  }'
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Click the pricing tab and summarize the plans.",
    "language": "node",
    "timeout": 60
  }'
```

<div id="parameters">
  ### Parámetros
</div>

* `jobId`
  * Tipo: string
  * Úsalo cuando: tengas el ID de un trabajo de scraping.

* `code`
  * Tipo: string
  * Úsalo cuando: quieras ejecutar código en la sesión del navegador.

* `prompt`
  * Tipo: string
  * Úsalo cuando: quieras que el agente del navegador siga una instrucción en lenguaje natural.

* `language`
  * Tipo: string
  * Úsalo cuando: necesites un entorno de ejecución específico.
  * Valores confirmados: `"python"`, `"node"`, `"bash"`

* `timeout`
  * Tipo: number
  * Úsalo cuando: necesites un tiempo de espera de ejecución en segundos.

<div id="notes">
  ## Notas
</div>

* Usa `interact` para flujos de trabajo de navegador de varios pasos; las `actions` de scraping son más adecuadas para pasos previos al scraping pequeños.

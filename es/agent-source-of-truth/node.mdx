Fuente canónica de Firecrawl en Node.js para agentes. Generada a partir del código fuente del SDK y de la especificación OpenAPI v2.

<div id="install">
  ## Instalación
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="authenticate">
  ## Autenticación
</div>

```ts
import Firecrawl from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // opcional para instancias autogestionadas
});
```

<div id="when-to-use-what">
  ## Cuándo usar cada opción
</div>

* `search`: úsalo cuando empieces con una consulta y necesites descubrir resultados.
* `scrape`: úsalo cuando ya tengas una URL y quieras el contenido de la página.
* `interact`: úsalo cuando la página requiera clics, formularios o acciones en el navegador después del scraping.

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa search para descubrir páginas relevantes a partir de una consulta y luego elegir las URLs que quieras usar para hacer scraping o interactuar con ellas. Puedes restringir los resultados a un sitio con `site:`, por ejemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.search(query, options?)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev webhook retries");
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev crawl webhooks", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  timeout: 60000,
  scrapeOptions: {
    formats: [
      "markdown",
      "links",
      { type: "json", prompt: "Extract title and key endpoints." }
    ],
    onlyMainContent: true,
    includeTags: ["main", "article"],
    excludeTags: ["nav", "footer"],
    waitFor: 1000
  }
});
```

<div id="parameters">
  ### Parámetros
</div>

* `query`
  * Tipo: string
  * Úsalo cuando: necesites una consulta de búsqueda.
  * Notas: usa `site:example.com` para limitar los resultados a un dominio.

* `options.sources`
  * Tipo: array de nombres de fuentes u objetos de fuente con tipo
  * Úsalo cuando: quieras controlar en qué fuentes se busca.
  * Valores confirmados:
    * `"web"`: resultados del índice web
    * `"news"`: resultados de noticias
    * `"images"`: resultados de imágenes
    * `{ type: "web" | "news" | "images" }`: formato de objeto de fuente con tipo

* `options.categories`
  * Tipo: array de nombres de categorías u objetos de categoría con tipo
  * Úsalo cuando: quieras filtrar los resultados por categoría.
  * Valores confirmados:
    * `"github"`: resultados centrados en GitHub
    * `"research"`: resultados de investigación y académicos
    * `"pdf"`: resultados centrados en PDF
    * `{ type: "github" | "research" | "pdf" }`: formato de objeto de categoría con tipo

* `options.limit`
  * Tipo: number
  * Úsalo cuando: quieras limitar los resultados.

* `options.tbs`
  * Tipo: string
  * Úsalo cuando: necesites un filtro temporal (por ejemplo `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).

* `options.location`
  * Tipo: string
  * Úsalo cuando: quieras resultados localizados.

* `options.ignoreInvalidURLs`
  * Tipo: boolean
  * Úsalo cuando: quieras omitir las URL que otros endpoints no pueden extraer.

* `options.timeout`
  * Tipo: number
  * Úsalo cuando: necesites un tiempo de espera de la solicitud en milisegundos.

* `options.scrapeOptions`
  * Tipo: ScrapeOptions
  * Úsalo cuando: quieras extraer cada resultado de búsqueda (consulta los parámetros de scraping para ver los campos).

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa scraping cuando ya tengas una URL y quieras contenido estructurado en uno o más formatos.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.scrape(url, options?)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const doc = await client.scrape("https://docs.firecrawl.dev", {
  formats: ["markdown"]
});
```

### Ejemplo avanzado

```ts
const doc = await client.scrape("https://example.com/pricing", {
  formats: [
    "markdown",
    "links",
    { type: "json", prompt: "Extract plan names and prices." },
    { type: "screenshot", fullPage: true, quality: 80, viewport: { width: 1280, height: 720 } },
    { type: "changeTracking", modes: ["git-diff"], tag: "pricing" },
    { type: "attributes", selectors: [{ selector: "a", attribute: "href" }] }
  ],
  headers: {
    "User-Agent": "FirecrawlDocsBot/1.0"
  },
  onlyMainContent: true,
  waitFor: 1000,
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  actions: [
    { type: "click", selector: "#accept" },
    { type: "wait", milliseconds: 750 },
    { type: "scrape" }
  ],
  location: { country: "US", languages: ["en-US"] },
  removeBase64Images: true,
  fastMode: true,
  blockAds: true,
  proxy: "auto",
  maxAge: 86400000,
  minAge: 1,
  storeInCache: true,
  profile: { name: "docs-session", saveChanges: true }
});
```

<div id="parameters">
  ### Parámetros
</div>

* `url`
  * Tipo: string
  * Úsalo cuando: quieras hacer scraping de una página específica.

* `options.formats`
  * Tipo: array de cadenas de formato u objetos de formato
  * Usar cuando: necesitas varios formatos de salida.
  * Cadenas de formato admitidas:
    * `"markdown"`: contenido en markdown
    * `"html"`: HTML limpio
    * `"rawHtml"`: HTML sin procesar
    * `"links"`: enlaces de la página
    * `"images"`: URL de imágenes
    * `"screenshot"`: salida de captura de pantalla
    * `"summary"`: salida de resumen
    * `"changeTracking"`: salida de seguimiento de cambios
    * `"json"`: extracción JSON
    * `"attributes"`: extracción de atributos
    * `"branding"`: salida del perfil de marca
    * `"audio"`: extracción de audio
    * `"query"`: salida de preguntas y respuestas
  * Campos del objeto de formato:
    * `type`: una de las cadenas de formato anteriores
    * `prompt`: obligatorio para `type: "query"`; obligatorio para `type: "json"` salvo que se proporcione `schema`
    * `schema`: esquema JSON (o esquema Zod) para `type: "json"` o para el modo JSON de seguimiento de cambios
    * `modes`: array de `"git-diff"` o `"json"` para `type: "changeTracking"`
    * `tag`: etiqueta de seguimiento de cambios para `type: "changeTracking"`
    * `fullPage`, `quality`, `viewport`: opciones de captura de pantalla para `type: "screenshot"`
    * `selectors`: array de `{ selector, attribute }` para `type: "attributes"`

* `options.headers`
  * Tipo: registro de cadenas a cadenas
  * Úsalo cuando: necesites cabeceras de solicitud personalizadas.

* `options.includeTags`
  * Tipo: array de cadenas
  * Úsalo si: quieres incluir solo etiquetas HTML específicas.

* `options.excludeTags`
  * Tipo: array de cadenas
  * Úsalo cuando quieras excluir etiquetas HTML específicas.

* `options.onlyMainContent`
  * Tipo: boolean
  * Úsalo cuando: quieras eliminar los menús de navegación, el pie de página y otros elementos repetitivos.

* `options.timeout`
  * Tipo: número
  * Úsalo cuando: necesites un timeout en milisegundos.

* `options.waitFor`
  * Tipo: número
  * Úsalo cuando: necesites esperar a que la página termine de renderizarse (milisegundos).

* `options.mobile`
  * Tipo: boolean
  * Úsalo cuando: quieras un viewport móvil.

* `options.parsers`
  * Tipo: array de nombres de analizadores o de objetos de analizador
  * Usar cuando: necesites controlar el análisis de archivos (por ejemplo, de PDF).
  * Valores admitidos:
    * `"pdf"`: habilita el análisis de PDF
    * `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }`: opciones del analizador de PDF

* `options.actions`
  * Tipo: array de objetos de acción
  * Úsalo cuando: necesites acciones ligeras previas al scraping.
  * Tipos de acción admitidos:
    * `wait`: se requiere `milliseconds` o `selector`
    * `screenshot`: `fullPage`, `quality`, `viewport` son opcionales
    * `click`: se requiere `selector`
    * `write`: se requiere `text` (haz clic primero para enfocar el campo de entrada)
    * `press`: se requiere `key`
    * `scroll`: se requiere `direction` (`up` o `down`), `selector` es opcional
    * `scrape`: sin campos adicionales
    * `executeJavascript`: se requiere `script`
    * `pdf`: `format` (A0, A1, A2, A3, A4, A5, A6, Letter, Legal, Tabloid, Ledger), `landscape`, `scale` son opcionales

* `options.location`
  * Tipo: objeto con `country` y `languages`
  * Úsalo cuando: necesites scraping según la ubicación o el idioma.

* `options.skipTlsVerification`
  * Tipo: boolean
  * Úsalo cuando necesites omitir la verificación de TLS.

* `options.removeBase64Images`
  * Tipo: boolean
  * Úsalo cuando: quieras excluir las imágenes en base64 de la salida Markdown.

* `options.fastMode`
  * Tipo: booleano
  * Úsalo cuando: quieras hacer scraping más rápido con menor fidelidad.

* `options.blockAds`
  * Tipo: boolean
  * Se usa cuando: quieres bloquear anuncios y ventanas emergentes de cookies.

* `options.proxy`
  * Tipo: string
  * Úsalo cuando: necesites controlar el proxy.
  * Valores admitidos:
    * `"basic"`
    * `"stealth"`
    * `"enhanced"`
    * `"auto"`
    * cadena de URL de proxy personalizada

* `options.maxAge`
  * Tipo: number
  * Úsalo cuando: quieras datos en caché de hasta una antigüedad máxima (en milisegundos).

* `options.minAge`
  * Type: number
  * Úsalo cuando: quieras datos en caché solo si tienen al menos esta antigüedad (milisegundos).

* `options.storeInCache`
  * Tipo: boolean
  * Úsalo cuando: quieras que Firecrawl guarde el resultado en caché.

* `options.profile`
  * Tipo: objeto con `name` y `saveChanges` opcional
  * Úsalo cuando: quieras un perfil de navegador persistente compartido entre operaciones de scraping e interacciones.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Por qué usarlo
</div>

Usa interact cuando una página requiere acciones del navegador o la ejecución de código después de que se inicia un scraping.

<div id="preferred-sdk-method">
  ### Método recomendado del SDK
</div>

`client.interact(jobId, args)`

<div id="simple-example">
  ### Ejemplo simple
</div>

```ts
const doc = await client.scrape("https://example.com", { formats: ["markdown"] });
const jobId = doc.metadata?.scrapeId;
if (!jobId) throw new Error("Falta scrapeId en la respuesta del scraping");

const result = await client.interact(jobId, {
  prompt: "Haz clic en la pestaña de precios y resume los planes."
});
```

<div id="complex-example">
  ### Ejemplo complejo
</div>

```ts
const result = await client.interact("<scrapeJobId>", {
  code: "console.log(await page.title());",
  language: "node",
  timeout: 60
});
```

<div id="parameters">
  ### Parámetros
</div>

* `jobId`
  * Tipo: string
  * Úsalo cuando: tengas un ID de trabajo de scraping de `document.metadata.scrapeId`.

* `args.code`
  * Tipo: string
  * Úsalo cuando: quieras ejecutar código en la sesión del navegador.

* `args.prompt`
  * Tipo: string
  * Úsalo cuando: quieras que el agente del navegador siga una instrucción en lenguaje natural.

* `args.language`
  * Tipo: string
  * Úsalo cuando: necesites un entorno de ejecución específico.
  * Valores confirmados: `"python"`, `"node"`, `"bash"`

* `args.timeout`
  * Tipo: number
  * Úsalo cuando: necesites un tiempo de espera de ejecución en segundos.

<div id="notes">
  ## Notas
</div>

* Alias obsoletos: `scrapeExecute`, `stopInteractiveBrowser` y `deleteScrapeBrowser` corresponden a `interact` y `stopInteraction`.
* La exportación predeterminada de `Firecrawl` es el cliente de v2; la versión heredada de v1 permanece en `client.v1`.
* Los esquemas de Zod que se pasan a `formats` (para `json` o `changeTracking`) el SDK los convierte a JSON Schema.

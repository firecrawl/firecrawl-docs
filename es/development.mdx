<Info>
  **Requisito previo** Debes tener instalado Node.js (versión 18.10.0 o
  superior).
</Info>

Paso 1. Instala Mintlify en tu sistema operativo:

<CodeGroup>
  ```bash npm
  npm i -g mintlify
  ```

  ```bash yarn
  yarn global add mintlify
  ```
</CodeGroup>

Paso 2. Ve al directorio donde están las docs (donde puedes encontrar `mint.json`) y ejecuta el siguiente comando:

```bash
mintlify dev
```

El sitio web de la documentación ya está disponible en `http://localhost:3000`.

<div id="custom-ports">
  ### Puertos personalizados
</div>

Mintlify usa el puerto 3000 de forma predeterminada. Puedes usar la opción `--port` para personalizar el puerto en el que se ejecuta Mintlify. Por ejemplo, usa este comando para ejecutarlo en el puerto 3333:

```bash
mintlify dev --port 3333
```

Verás un error como este si intentas ejecutar Mintlify en un puerto que ya está ocupado:

```md
Error: listen EADDRINUSE: la dirección ya está en uso :::3000
```

<div id="mintlify-versions">
  ## Versiones de Mintlify
</div>

Cada CLI está vinculado a una versión específica de Mintlify. Actualiza la CLI si tu sitio web local se ve diferente al de producción.

<CodeGroup>
  ```bash npm
  npm i -g mintlify@latest
  ```

  ```bash yarn
  yarn global upgrade mintlify
  ```
</CodeGroup>

<div id="troubleshooting">
  ## Solución de problemas
</div>

Así puedes resolver algunos problemas comunes al trabajar con la CLI.

<AccordionGroup>
  <Accordion title="Mintlify no carga">
    Actualiza a Node v18. Ejecuta `mintlify install` e inténtalo de nuevo.
  </Accordion>

  <Accordion title="No existe el archivo o directorio en Windows">
    Ve al directorio `C:/Users/Username/.mintlify/` y elimina la carpeta `mint`.
    Luego abre Git Bash en esa ubicación y ejecuta `git clone
    https://github.com/mintlify/mint.git`.

    Repite el paso 3.
  </Accordion>

  <Accordion title="Se produce un error desconocido">
    Ve a la raíz de tu dispositivo y elimina la carpeta ~/.mintlify.
    Luego ejecuta `mintlify dev` otra vez.
  </Accordion>
</AccordionGroup>

Firecrawl Research は、科学・工学分野の研究エージェント向けに特化して設計されたインデックスです。論文の検索、論文メタデータの確認、関連する本文箇所の閲覧、構造的拡張による関連論文の発見、研究関連の GitHub リポジトリの検索に対応した、研究専用のツールセットを提供します。

* トピック、手法、ベンチマーク、著者、またはカテゴリで論文を探す
* 正規の論文メタデータとソース ID を確認する
* 特定の質問に答える、1 本の論文内の本文箇所を読む
* 有力なシード論文から、関連論文、引用論文、または参考文献へと広げる
* 実装メモ、バグ、設計に関する議論を見つけるために、GitHub の履歴や README を検索する

<Note>
  エージェントに Research Index へのアクセスを付与するには、[CLI](/ja/sdks/cli) または [MCP](/ja/mcp-server) を、[**専用の research skill**](https://github.com/firecrawl/skills/blob/main/skills/firecrawl-research-index/SKILL.md) と組み合わせて使うことを強くおすすめします。インストールには次のコマンドを使用します。

  ```bash
  npx skills add firecrawl/skills@firecrawl-research-index
  ```
</Note>

<div id="search-papers">
  ## 論文を検索
</div>

自然言語のクエリで論文のアブストラクトを検索します。レスポンスでは、正規の `paperId`、優先される `primaryId`、ソース ID、タイトル、アブストラクト、スコア、任意のランキングシグナルを含む、順位付けされた論文が返されます。

<CodeGroup>
  ```bash cURL
  # 利用開始に API key は不要です。より高いレート制限が必要な場合は -H "Authorization: Bearer $FIRECRAWL_API_KEY" を追加してください:
  curl -s "https://api.firecrawl.dev/v2/search/research/papers?query=diffusion%20image%20synthesis&k=20"
  ```

  ```bash CLI
  firecrawl research search-papers "diffusion image synthesis" --limit 20
  ```

  ```python Python
  from firecrawl import Firecrawl

  firecrawl = Firecrawl(
    # 利用開始に API key は不要です。より高いレート制限が必要な場合は追加してください:
    # api_key="fc-YOUR-API-KEY",
  )

  result = firecrawl.v2.search_papers("diffusion image synthesis", k=20)
  print(result)
  ```

  ```js Node
  import { Firecrawl } from "firecrawl";

  const firecrawl = new Firecrawl({
    // 利用開始に API key は不要です。より高いレート制限が必要な場合は追加してください:
    // apiKey: "fc-YOUR-API-KEY",
  });

  const result = await firecrawl.research.searchPapers("diffusion image synthesis", {
    k: 20,
  });
  console.log(result);
  ```
</CodeGroup>

任意のフィルタ:

* `authors`: 著者名の部分文字列で絞り込みます。すべてのフィルタ条件に一致する必要があります
* `categories`: `cs.LG` などの論文カテゴリで絞り込みます
* `from`: 作成日/更新日の下限 (この日付を含む) 。`YYYY-MM-DD`
* `to`: 作成日/更新日の上限 (この日付を含む) 。`YYYY-MM-DD`

<div id="read-paper-passages">
  ## 論文の該当箇所を読む
</div>

同じ論文パスに `query` を追加すると、質問に対する本文中の関連度が高い本文箇所を取得できます。これは、候補の論文を含める前に、その論文に手法、データセット、制約、または結果が実際に記載されているかを確認するのに役立ちます。

<CodeGroup>
  ```bash cURL
  # 利用開始に API key は不要です。より高いレート制限を利用するには -H "Authorization: Bearer $FIRECRAWL_API_KEY" を追加してください:
  curl -s "https://api.firecrawl.dev/v2/search/research/papers/arxiv:1706.03762?query=what%20is%20the%20attention%20mechanism&k=4"
  ```

  ```bash CLI
  firecrawl research read-paper arxiv:1706.03762 --question "What is the attention mechanism?" --limit 4
  ```

  ```python Python
  from firecrawl import Firecrawl

  firecrawl = Firecrawl(
    # 利用開始に API key は不要です。より高いレート制限を利用するには追加してください:
    # api_key="fc-YOUR-API-KEY",
  )

  passages = firecrawl.v2.read_paper(
      "arxiv:1706.03762",
      "What is the attention mechanism?",
      k=4,
  )
  print(passages)
  ```

  ```js Node
  import { Firecrawl } from "firecrawl";

  const firecrawl = new Firecrawl({
    // 利用開始に API key は不要です。より高いレート制限を利用するには追加してください:
    // apiKey: "fc-YOUR-API-KEY",
  });

  const passages = await firecrawl.research.getPaper("arxiv:1706.03762", {
    query: "What is the attention mechanism?",
    k: 4,
  });
  console.log(passages);
  ```
</CodeGroup>

<div id="find-related-papers">
  ## 関連する論文を見つける
</div>

1本以上のシード論文を起点にセマンティック拡張し、自然言語の`intent`に基づいて候補を順位付けします。

<CodeGroup>
  ```bash cURL
  # 開始するのに API key は不要です。より高いレート制限が必要な場合は、-H "Authorization: Bearer $FIRECRAWL_API_KEY" を追加してください:
  curl -s "https://api.firecrawl.dev/v2/search/research/papers/arxiv:1706.03762/similar?intent=efficient%20transformers&mode=similar&k=20"
  ```

  ```bash CLI
  firecrawl research related-papers arxiv:1706.03762 --intent "efficient transformers" --limit 20
  ```

  ```python Python
  from firecrawl import Firecrawl

  firecrawl = Firecrawl(
    # 開始するのに API key は不要です。より高いレート制限が必要な場合は追加してください:
    # api_key="fc-YOUR-API-KEY",
  )

  papers = firecrawl.v2.related_papers(
      "arxiv:1706.03762",
      "efficient transformers",
      mode="similar",
      k=20,
  )
  print(papers)
  ```

  ```js Node
  import { Firecrawl } from "firecrawl";

  const firecrawl = new Firecrawl({
    // 開始するのに API key は不要です。より高いレート制限が必要な場合は追加してください:
    // apiKey: "fc-YOUR-API-KEY",
  });

  const papers = await firecrawl.research.similarPapers("arxiv:1706.03762", {
    intent: "efficient transformers",
    mode: "similar",
    k: 20,
  });
  console.log(papers);
  ```
</CodeGroup>

モード:

* `similar`: 共引用および書誌結合による近傍
* `citers`: シードを引用している論文
* `references`: シードが引用している論文

<div id="search-github-history">
  ## GitHub の履歴を検索
</div>

GitHub の issue、pull request、discussion、リポジトリの README を検索して、実装の詳細や過去のエンジニアリング事例を調べられます。

<CodeGroup>
  ```bash cURL
  # 開始するのに API キーは不要です。より高い rate limits を利用するには -H "Authorization: Bearer $FIRECRAWL_API_KEY" を追加してください:
  curl -s "https://api.firecrawl.dev/v2/search/research/github?query=flash%20attention%20implementation%20notes&k=10"
  ```

  ```bash CLI
  firecrawl research search-github "flash attention implementation notes" --limit 10
  ```

  ```python Python
  from firecrawl import Firecrawl

  firecrawl = Firecrawl(
    # 開始するのに API キーは不要です。より高い rate limits を利用するには追加してください:
    # api_key="fc-YOUR-API-KEY",
  )

  results = firecrawl.v2.search_github("flash attention implementation notes", k=10)
  print(results)
  ```

  ```js Node
  import { Firecrawl } from "firecrawl";

  const firecrawl = new Firecrawl({
    // 開始するのに API キーは不要です。より高い rate limits を利用するには追加してください:
    // apiKey: "fc-YOUR-API-KEY",
  });

  const results = await firecrawl.research.searchGithub(
    "flash attention implementation notes",
    { k: 10 },
  );
  console.log(results);
  ```
</CodeGroup>

GitHub の検索結果には、リポジトリ、URL、利用可能な場合は issue/PR のメタデータ、スニペット、利用可能な場合は一致した markdown content が含まれます。

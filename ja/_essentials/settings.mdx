---
title: "グローバル設定"
description: "Mintlify では、mint.json ファイルによりドキュメントの見た目と使い勝手を細かく制御できます"
icon: 'gear'
---

すべての Mintlify サイトには、基本的な設定を含む `mint.json` ファイルが必要です。以下の[プロパティ](#properties)をご覧ください。

<div id="properties">
  ## プロパティ
</div>

<ResponseField name="name" type="string" required>
  プロジェクト名。グローバルタイトルに使用されます。

  例: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  各グループに属するすべてのページを含むグループ配列

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      グループ名。

      例: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      ページとして使用する Markdown ファイルへの相対パス。

      例: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  ロゴ画像へのパス、または「light」「dark」モードのロゴ画像パスを持つオブジェクト

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      ライトモード用ロゴへのパス
    </ResponseField>

    <ResponseField name="dark" type="string">
      ダークモード用ロゴへのパス
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      ロゴをクリックしたときのリンク先
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  ファビコン画像へのパス
</ResponseField>

<ResponseField name="colors" type="Colors">
  グローバルテーマの 16 進カラーコード

  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      プライマリカラー。主にハイライトされたコンテンツ、セクションヘッダー、アクセントに使用され、ライトモードで適用
    </ResponseField>

    <ResponseField name="light" type="string">
      ダークモード用のプライマリカラー。主にハイライトされたコンテンツ、セクションヘッダー、アクセントに使用され、ダークモードで適用
    </ResponseField>

    <ResponseField name="dark" type="string">
      重要なボタン用のプライマリカラー
    </ResponseField>

    <ResponseField name="background" type="object">
      ライト・ダーク両モードの背景色

      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          ライトモードの背景の 16 進カラーコード
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          ダークモードの背景の 16 進カラーコード
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  トップバーに含めるリンクの `name` と `url` の配列

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      ボタン名。

      例: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      ボタンをクリックした際の URL。例: `https://mintlify.com/contact`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      link はボタンを表示します。github は指定した URL のリポジトリ情報（GitHub スター数を含む）を表示します。
    </ResponseField>

    <ResponseField name="url" type="string">
      `link` の場合: ボタンのリンク先。

      `github` の場合: GitHub 情報を取得するリポジトリへのリンク。
    </ResponseField>

    <ResponseField name="name" type="string">
      ボタン内のテキスト。`type` が `link` の場合のみ必須。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  バージョン名の配列。ナビゲーションバーのドロップダウンで
  ドキュメントの異なるバージョンを表示する場合にのみ使用します。
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  アンカーの配列。`icon`、`color`、`url` を含みます。

  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
      アンカーに使用する [Font Awesome](https://fontawesome.com/search?s=brands%2Cduotone) のアイコン。

      例: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      アンカーのラベル名。

      例: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      アンカーに含めるページを判別するための URL の先頭部分。一般的には、ページを配置するフォルダー名です。
    </ResponseField>

    <ResponseField name="color" type="string">
      アンカーアイコンの背景色（hex）。`from` と `to` のプロパティを持つオブジェクトを渡すとグラデーションも指定でき、それぞれに hex 色を設定します。
    </ResponseField>

    <ResponseField name="version" type="string">
      適切なドキュメントバージョンが選択されるまでアンカーを非表示にしたい場合に使用します。
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      その中のドキュメントへの直接リンクがあるまでアンカーを非表示にしたい場合は、`true` を渡します。
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      次のいずれか: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot;, &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  最上位アンカーのデフォルト設定を上書きします。

  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      最上位アンカーの名前。
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome のアイコン。
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      次のいずれか: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot;, &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  ナビゲーションタブの配列。

  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      タブのラベル名。
    </ResponseField>

    <ResponseField name="url" type="string">
      タブに含めるページを判別するための URL の先頭部分。一般的には、ページを配置するフォルダー名です。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  API 設定の構成です。API ページの詳細は [API Components](/ja/api-playground/demo) を参照してください。

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      すべての API エンドポイントの基底 URL。`baseUrl` が配列の場合、ユーザーが切り替え可能な複数の基底 URL オプションを有効にします。
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          すべての API エンドポイントで使用する認証方式。
        </ResponseField>

        <ResponseField name="name" type="string">
          API Playground で使用する認証パラメータ名。

          method が `basic` の場合、形式は `[usernameName]:[passwordName]` としてください。
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          認証入力フィールドの接頭辞となる既定値。

          例: `inputPrefix` に `AuthKey` を指定すると、認証フィールドの既定入力値は `AuthKey` になります。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      API Playground の設定

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          Playground を表示するか、非表示にするか、またはユーザー操作なしでエンドポイントのみを表示する（`simple`）かを指定します。

          詳細は [playground guides](/ja/api-playground/demo) を参照してください。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      このフラグを有効にすると、OpenAPI ページのキー順序が OpenAPI ファイルで定義されたキー順序と一致します。

      <Warning>この挙動はまもなく既定で有効になり、その時点でこのフィールドは非推奨となります。</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  OpenAPI ファイルを指す URL または相対パスの文字列、またはそれらの文字列配列。

  例:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  ソーシャルメディアアカウントのオブジェクト。key:property の組で、ソーシャルプラットフォームとアカウントの URL を表します。

  例:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      次のいずれかの値: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      例: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      ソーシャルプラットフォームの URL。

      例: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  フィードバックボタンを有効化するための設定

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      ユーザーがプルリクエストで編集提案できるボタンを有効にします。
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      ドキュメントに関する Issue を起票できるボタンを有効にします。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  ダークモードのトグルをカスタマイズします。

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      新規ユーザーに常にライトまたはダークモードを表示するかを設定します。未設定の場合は、ユーザーのOSと同じモードが既定になります。
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      ダーク/ライトモードのトグルを非表示にするには true に設定します。`isHidden` と `default` を組み合わせることで、ドキュメントをライトまたはダークのいずれかに固定できます。例:

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  すべてのページの背面に表示される背景画像です。例として
  [Infisical](https://infisical.com/docs) と [FRPC](https://frpc.io) を参照してください。
</ResponseField>

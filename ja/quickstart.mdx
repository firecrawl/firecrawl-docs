<div id="setup-your-development">
  ## 開発環境のセットアップ
</div>

ローカルでドキュメントを更新し、公開にデプロイする方法を学びます。

<div id="edit-and-preview">
  ### 編集とプレビュー
</div>

<AccordionGroup>
  <Accordion icon="github" title="ドキュメントをローカルにクローンする">
    オンボーディングの際に、あなたの GitHub 上にドキュメントコンテンツを含むリポジトリを作成しました。リポジトリは
    [ダッシュボード](https://dashboard.mintlify.com) から確認できます。ローカルにクローンするには、ターミナルで
    [こちらの手順](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)
    に従ってください。
  </Accordion>

  <Accordion icon="rectangle-terminal" title="変更をプレビューする">
    プレビューすると、変更が意図どおりに表示されているか確認できます。ローカルで変更をレンダリングするためのコマンドラインインターフェースを用意しています。1. 次のコマンドで
    [Mintlify CLI](https://www.npmjs.com/package/mintlify) をインストールし、ローカルでドキュメントの変更をプレビューします: `npm i -g mintlify`
    2. ドキュメントのルート（`mint.json` がある場所）で次のコマンドを実行します: `mintlify dev`
  </Accordion>
</AccordionGroup>

<div id="deploy-your-changes">
  ### 変更をデプロイする
</div>

<AccordionGroup>
  <Accordion icon="message-bot" title="GitHub アプリをインストールする">
    当社の GitHub アプリは、ドキュメントサイトへの変更を自動的にデプロイします。手動でデプロイを管理する必要はありません。インストール用のリンクは
    [ダッシュボード](https://dashboard.mintlify.com)にあります。ボットのインストールが完了すると、リポジトリのコミットハッシュの横にチェックマークが表示されます。
  </Accordion>

  <Accordion icon="rocket" title="変更をプッシュする">
    ドキュメントサイトを更新するには、
    [変更をコミットして Git にプッシュ](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push)
    してください。プッシュ後に GitHub アプリでデプロイが確認できない場合は、
    [ダッシュボード](https://dashboard.mintlify.com)から手動でドキュメントを更新できます。
  </Accordion>
</AccordionGroup>

<div id="update-your-docs">
  ## ドキュメントを更新
</div>

MDX構文とReactコンポーネントで、ファイル内に直接コンテンツを追加できます。用意されたコンポーネントを使うことも、自作することも可能です。

<CardGroup>
  <Card title="ドキュメントのスタイル設定" icon="paintbrush" href="/ja/settings/global">
    ブランディングをカスタマイズして、ドキュメントに個性を加えましょう。
  </Card>

  <Card title="APIエンドポイントの追加" icon="square-code" href="/ja/api-playground/configuration">
    OpenAPI仕様を取り込み、APIのインタラクションを有効化します。
  </Card>

  <Card title="アナリティクスの統合" icon="chart-mixed" href="/ja/analytics/supported-integrations">
    ドキュメント上のユーザー行動からインサイトを得ましょう。
  </Card>

  <Card title="カスタムドメインでホスト" icon="browser" href="/ja/settings/custom-domain/subdomain">
    自身のウェブサイトのサブドメインでドキュメントを公開できます。
  </Card>
</CardGroup>

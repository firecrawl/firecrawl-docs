<div id="overview">
  ## 概要
</div>

Firecrawl は **クレジットベースの課金システム**を採用しています。行う API 呼び出しごとにクレジットが消費され、消費されるクレジット数は利用するエンドポイントやオプションによって異なります。プランごとに毎月のクレジット割り当てが決まっており、必要に応じて自動チャージパックで追加クレジットを購入できます。

現在のプラン料金については、[Firecrawl の料金ページ](https://www.firecrawl.dev/pricing)を参照してください。

<Note>
  Firecrawl の請求書はすべて、請求先住所や支払い方法にかかわらず、**米ドル (USD) **で請求されます。
</Note>

<div id="credits">
  ## クレジット
</div>

クレジットは、Firecrawl における利用を測る単位です。各プランには毎月のクレジット割り当てが含まれており、請求サイクルの開始時にリセットされます。API の各エンドポイントごとに、消費するクレジット数は異なります。

<div id="credit-costs-per-endpoint">
  ### エンドポイントごとのクレジットコスト
</div>

| Endpoint    | Credit Cost       | Notes                                                                                                                                                                                                                               |
| ----------- | ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **スクレイピング**  | 1 クレジット / ページ     | 1 つの URL をクリーンな Markdown、HTML、または構造化データに変換します。スクレイピングのオプションを使用する場合は追加クレジットが発生します (下記参照) 。                                                                                                                                           |
| **Crawl**   | 1 クレジット / ページ     | 開始 URL からリンクをたどって、Web サイト全体をスクレイピングします。クロールされた各ページには、同じページ単位のスクレイピングのオプションのコストが適用されます。                                                                                                                                              |
| **Map**     | 1 クレジット / コール     | コンテンツをスクレイピングせずに、Web サイト上のすべての URL を検出します。                                                                                                                                                                                          |
| **Search**  | 2 クレジット / 10 件の結果 | Web を検索し、必要に応じて結果をスクレイピングします。10 件ごとに切り上げて計算されます (例: 11 件の結果 = 4 クレジット) 。スクレイピングされた各結果には、追加のページ単位のスクレイピングのコストが適用されます。Enterprise ZDR Search のコストは 10 クレジット / 10 件の結果です ([ZDR Search](/ja/features/search#zero-data-retention-zdr) を参照) 。 |
| **Browser** | 2 クレジット / ブラウザ分   | 分単位で課金されるインタラクティブなブラウザサンドボックスセッションです。                                                                                                                                                                                               |
| **Agent**   | Dynamic           | 自律型の Web リサーチエージェントです。1 日あたり 5 回まで無料で実行でき、それ以降は利用量に応じた課金となります。                                                                                                                                                                      |

<div id="additional-credit-costs-for-scrape-options">
  ### スクレイピングのオプションによる追加クレジットコスト
</div>

一部のスクレイピングのオプションは、ページごとの基本コストに加えてクレジットが加算されます:

| Option | Additional Cost | Description |
|--------|----------------|-------------|
| PDF parsing | +1 credit / PDF page | PDF ドキュメントからコンテンツを抽出します |
| JSON format (LLM extraction) | +4 credits / page | LLM を使用してページから構造化された JSON データを抽出します |
| Enhanced Mode | +4 credits / page | アクセスが難しいページ向けにスクレイピングを強化します |
| Zero Data Retention (ZDR) | +1 credit / page | リクエストを超えてデータが保持されないことを保証します ([Scrape ZDR](/ja/features/scrape#zero-data-retention-zdr) を参照)  |

これらの修飾子は重複して適用されます。たとえば、JSON format と Enhanced Mode の両方を使用して 1 ページをスクレイピングすると、そのページのコストは **1 + 4 + 4 = 9 クレジット** になります。これらと同じ修飾子は、各ページに対して内部的にスクレイピングを使用するため、クロールおよび Search エンドポイントにも適用されます。

<div id="when-credits-are-charged">
  ### クレジットが消費されるタイミング
</div>

クレジットは、対象サイトが 403 Forbidden や 404 Not Found などの HTTP エラーステータスコードを返した場合でも、Firecrawl のインフラがリクエストを処理するたびに消費されます。これは、対象サイトのレスポンスに関係なく、スクレイピング用インフラ (ブラウザレンダリング、proxy など) がフルに使用されるためです。こうしたケースは API レスポンスの `metadata.statusCode` フィールドで検出できるため、継続的にブロックされる URL への再試行を避けられます。

**batch scrape** や **crawl** ジョブの場合、クレジットはジョブの送信時ではなく、各ページの処理が完了するたびに非同期で課金されます。つまり、ジョブを送信してから、アカウント上に最終的なクレジット消費が反映されるまでにタイムラグが生じる場合があります。バッチに多くの URL が含まれている場合や、高トラフィック時にページがキューに入れられる場合、送信後も数分から数時間にわたってクレジットが順次反映され続けることがあります。バッチのステータスをポーリングまたは確認しても、クレジットは消費されません。

<Note>
  **クロール開始前のクレジット確認:** クロールジョブを開始する前に、Firecrawl は残りのクレジット残高で、要求した `limit` パラメータの全量をカバーできるかを確認します。残高が `limit` を下回っている場合、実際のクロールで見つかるページ数がそれより少なくても、リクエストは 402 を返します。デフォルトの `limit` は **10,000** のため、これを省略すると、事前に 10,000 クレジットを利用可能な状態にしておく必要があります。これを避けるには、実際にクロールする予定のページ数に合わせて `limit` を明示的に指定してください (例: `limit: 100`)。
</Note>

<div id="tracking-your-usage">
  ### 利用状況の確認
</div>

クレジットの利用状況は、次の2つの方法で確認できます。

* **ダッシュボード**: 現在および過去の利用状況を [firecrawl.dev/app](https://www.firecrawl.dev/app) で確認できます
* **API**: [Credit Usage](/ja/api-reference/endpoint/credit-usage) および [Credit Usage Historical](/ja/api-reference/endpoint/credit-usage-historical) エンドポイントを使用して、プログラムから利用状況を確認できます

<Note>
  クレジット利用状況をより理解しやすくするための改善に継続的に取り組んでいます。今後のアップデートをお待ちください。
</Note>

<div id="plans">
  ## プラン
</div>

Firecrawl はサブスクリプション制の月額プランを提供しています。純粋な従量課金プランはありませんが、後述の自動チャージ機能により、ベースプランに対して柔軟にスケールさせることができます。

<div id="available-plans">
  ### 利用可能なプラン
</div>

| プラン          | 月間クレジット   | 同時ブラウザ数 |
| ------------ | --------- | ------: |
| **Free**     | 1,000     |       2 |
| **Hobby**    | 5,000     |       5 |
| **Standard** | 100,000   |      50 |
| **Growth**   | 500,000   |     100 |
| **Scale**    | 1,000,000 |     150 |

<Note>
  Scale を超える利用ニーズには、Firecrawl は **Enterprise** プランを提供しています。これには、カスタムクレジット、専任サポート、SLA、一括割引、データ非保持 (ゼロデータ保持) 、SSO が含まれます。詳細は [Enterprise ページ](https://www.firecrawl.dev/enterprise) を参照してください。
</Note>

すべての有料プランは **月払い** または **年払い** に対応しています。年払いは月払いと比べて割引があります。各プランの最新の料金については、[料金ページ](https://www.firecrawl.dev/pricing) を参照してください。

<div id="choosing-a-credit-tier-with-the-pricing-slider">
  ### 料金スライダーでクレジットティアを選ぶ
</div>

[料金ページ](https://www.firecrawl.dev/pricing)では、フリープランから Scale、Custom まで、すべてのクレジットオプションを1つのスライダーで確認できます。各プランの代表的なサイズに加え、有料プランには中間のクレジットティアも用意されているため、1つ上のプランに移行せずに、実際の使用量により近い形でサブスクリプションを調整できます。

| Plan         | Available credit tiers      |
| ------------ | --------------------------- |
| **Hobby**    | 5,000 · 6,500 · 8,000       |
| **Standard** | 100,000 · 130,000 · 160,000 |
| **Growth**   | 500,000 · 650,000           |
| **Scale**    | 1,000,000                   |

同時ブラウザ数、レート制限、サポートレベルは、個別のクレジットティアではなくプランによって決まります。つまり、Hobby で 6,500 クレジットを選んでも、5,000 を選んだ場合と同じ同時実行数になります。スライダーでクレジット数を調整し、そのティアで契約してください。

<div id="concurrent-browsers">
  ### 同時ブラウザ数
</div>

同時ブラウザ数は、Firecrawl が同時に処理できる Web ページ数を表します。この上限はご利用プランによって決まります。上限を超えた場合、追加のジョブは空きが出るまでキューで待機します。同時実行数と API のレート制限の詳細については、[Rate Limits](/ja/rate-limits) を参照してください。

<div id="handling-usage-spikes">
  ## 使用量の急増への対処
</div>

Firecrawlでは、請求サイクルの途中でクレジットを使い切ってもリクエストを継続できるように、オプトイン可能な仕組みを2つ提供しています。**自動チャージ**と**Smart Upgrade**です。どちらも[課金設定](https://www.firecrawl.dev/app/settings?tab=billing)から設定でき、**併用はできません**。つまり、組織ではどちらか一方のみを使用し、両方を同時に使うことはできません。

|            | 自動チャージ                     | Smart Upgrade                     |
| ---------- | -------------------------- | ------------------------------- |
| 役割         | 単発のクレジットパックを購入して残高に追加する    | 料金プラン上の次のクレジットティアに移行する          |
| 発動条件       | 残高がしきい値 (デフォルトは0) まで下がる    | 残高が0になる                         |
| 請求         | 固定のパック料金                   | 現在のティアと次のティアの差額を日割りで請求          |
| 追加されるクレジット | パックは1年間有効で、毎月リセットされない      | 以前のティアに代わって、新しいティアの割り当てが全量付与される |
| 最適な用途      | 安定したプランに加えて、突発的な使用量増加がある場合 | 使用量の増加に合わせて段階的に上位ティアへ移行する場合     |

<div id="auto-recharge">
  ### 自動チャージ
</div>

残りクレジットがしきい値を下回ると、Firecrawl が自動的に追加のクレジットパックを購入し、残高に加算します。デフォルトではしきい値は **0 クレジット** ですが、[課金設定](https://www.firecrawl.dev/app/settings?tab=billing) でカスタマイズして、より早い段階でチャージが実行されるようにできます — たとえば、しきい値を高めに設定すると、常に一定のクレジットの余裕を確保できるため、大規模なクロールを実行する場合に便利です。このチェックは API リクエストを行ったときに実行される点に注意してください — 次のリクエストでチャージがトリガーされるまで、残高は補充されません。

* 自動チャージ用のパックは、すべての有料プランで利用できます
* パックのサイズと価格はプランごとに異なります ([料金ページ](https://www.firecrawl.dev/pricing) で確認できます)
* 自動チャージはいつでもオン／オフを切り替えられます
* 自動チャージは **月あたり4パックまで** に制限されています
* 自動チャージ用パックのクレジットは **毎月リセットされません** — 月次プランのクレジットが各期間ごとにリセットされるのとは異なり、請求サイクルをまたいで保持され、購入から **1年後** に失効します。

自動チャージは、利用量が一時的にスパイクするケースへの対応に最適です。継続的にプランの割り当てを超えてしまう場合は、より上位のプランにアップグレードするか、Smart Upgrade を有効にすると、クレジットあたりのコスト面で有利になります。

<div id="smart-upgrade">
  ### Smart Upgrade
</div>

Smart Upgrade は、残高がゼロになるたびに自動で次のクレジットティアへ移行するオプトイン型の課金モードです。これにより、利用が中断されません。単発のクレジットパックを購入する代わりに、スライダー上の次のサブスクリプションティアへ移行されます (例: Hobby 5k → Hobby 6.5k → Hobby 8k → Standard 100k → … → Scale 1M) 。

仕組み:

* **Trigger:** Smart Upgrade は、残りのクレジット残高が 0 になると発動します。未使用の自動チャージ分を含め、クレジットが少しでも残っている場合はアップグレードされません。
* **Charge:** 現在のティアと次のティアの差額に相当する金額が日割りで請求されます。**monthly** プランでは月額料金の差額、**yearly** プランでは `(請求サイクルの残り月数 ÷ 12) × 年額料金の差額` です。日割り計算には、現在の課金期間の開始時点からの暦上の満月数が使われます。
* **Credits granted:** 新しいティアのクレジット割り当てから、以前のティアの割り当て分を差し引いた分のクレジットが付与されます (つまり、支払うのは差分のみです) 。
* **Notifications:** Smart Upgrade が発動するたびに、アップグレード内容、請求額、次に移行するティアの案内を含むメールが送信されます。以降のアップグレードを停止するには、いつでも [課金設定](https://www.firecrawl.dev/app/settings?tab=billing) で Smart Upgrade を無効にできます。
* **Top of the ladder:** Scale ティア (1,000,000 credits/month) に達すると、Smart Upgrade でそれ以上移行できる先はありません。Scale を超えてクレジットを増やす場合は、カスタム Enterprise プランについて [営業にお問い合わせください](https://www.firecrawl.dev/enterprise)。

Smart Upgrade は、サブスクライブ時またはプラン変更時に有効になります (チェックアウト確認画面にトグルが表示されます) 。その後は課金設定から切り替えることもできます。標準プランの単一チーム組織で利用でき、カスタム Enterprise 契約の組織では引き続き個別に合意した課金モデルが適用されます。

Smart Upgrade は、使用量が継続的に増えていて、クレジットパックで都度補充するよりも自動で上位プランへ移行したい場合に最適です。ベースラインのプランが適切で、たまに単発の追加補充だけ必要な場合は、Auto-Recharge を使用してください。

<div id="coupons">
  ## クーポン
</div>

Firecrawl は、2種類のクーポンに対応しています。

* **サブスクリプションクーポン**は、プランのサブスクリプション料金に割引を適用します (例: 月額または年額料金の一定割合の割引) 。これらは、有料プランに初めて登録する際、またはプラン変更時の Stripe のチェックアウトフロー中に**のみ**適用できます。チェックアウト完了後にサブスクリプションクーポンを適用することはできません。
* **クレジットクーポン**は、アカウントにボーナスクレジットを追加します。これらは、ダッシュボードの**課金**セクションにある [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing) から引き換えできます。コードを適用するには、課金ページのクーポン入力欄を探してください。クレジットクーポンによるボーナスクレジットは、プランの月次割り当てとは別であり、プランをアップグレードまたはダウングレードしても保持されます。

クーポンコードをお持ちで、どちらの種類かわからない場合は、まずダッシュボードの課金セクションで適用してみてください。サブスクリプションクーポンの場合は、代わりに Stripe のチェックアウトページで使用する必要があります。

<div id="billing-cycle">
  ## 請求サイクル
</div>

* **月額プラン**: クレジットは毎月の更新日にリセットされます
* **年額プラン**: 請求は年に一度ですが、クレジットはシステム上の毎月の更新日にリセットされます
* **未使用のプランのクレジットはデフォルトでは繰り越されません** — 毎月の割り当ては毎月リセットされます。**年額のScaleプランでは未使用のプランのクレジットを1か月繰り越せます**、また**年額のEnterpriseプランでは2か月繰り越せます**。自動チャージパックのクレジットは請求サイクルに紐づいていません — 購入日から**1年**後に失効するまで保持されます。

<div id="upgrading-and-downgrading">
  ## アップグレードとダウングレード
</div>

* **アップグレード** は即時に反映されます。本日、新しいプランの料金全額が請求され (日割り計算はありません) 、請求サイクルはリセットされます。次回の更新日は、アップグレード日から1か月後または1年後です。以前のプランで未使用のクレジットは繰り越され、新しいクレジットの割り当てと同時実行数の上限がすぐに適用されます。
* **ダウングレード** は次回の更新日に反映されるようにスケジュールされます。それまでは現在のプランのクレジットと上限を維持します。現在のプランの未使用期間については、クレジットの付与や返金はありません。適用日の前であれば、いつでも[課金設定](https://www.firecrawl.dev/app/settings?tab=billing)からスケジュール済みのダウングレードを取り消せます。

<div id="switching-between-monthly-and-yearly-billing">
  ### 月次課金と年次課金の切り替え
</div>

* **月次 → 年次**で、同じまたは上位のクレジットティアに切り替える場合は、即時アップグレードとして扱われます。
* **年次 → 月次**は、現在より上位のクレジットティアに切り替える場合にのみ、即時アップグレードとして扱われます。

すでに購入済みの自動チャージパックは、プラン変更の影響を受けません。アップグレードやダウングレードにかかわらず、購入時点の1年間の有効期限がそのまま適用されます。

<div id="what-happens-when-you-run-out-of-credits">
  ## クレジットを使い切った場合
</div>

クレジットの割り当てを使い切り、かつ自動チャージまたはSmart Upgradeを有効にしていない場合、クレジットを消費するAPIリクエストは **HTTP 402 (Payment Required)** エラーを返します。

* **自動チャージ** を有効にしている場合、自動的にリチャージパックが購入されるため、利用を継続できます。ただし、月間パック上限 (4) に達した場合やリチャージに失敗した場合は、次の課金サイクルまで残高がマイナスになることがあります。
* **Smart Upgrade** を有効にしている場合、サブスクリプションはスライダー上の次のクレジットティアに自動的に移行し、新しい階層のクレジット割り当てが即座に付与されます。Smart UpgradeはScale階層に到達すると停止します。

利用が完全に停止した後で再開するには、次のいずれかを行います。

1. [自動チャージ](#auto-recharge) または [Smart Upgrade](#smart-upgrade) を有効にする
2. より上位のプランにアップグレードする
3. 次の課金サイクルでクレジットがリセットされるまで待つ

<div id="free-plan">
  ## 無料プラン
</div>

無料プランでは、**毎月 1,000 クレジット**が付与され、クレジットカードは不要です。クレジットは毎月の更新日にリセットされるため、使い切った場合は次回のリセットを待つか、より多くの割り当てを利用するために有料プランへアップグレードできます。無料プランは、有料プランと比べてレート制限および同時実行数が低くなっています ([レート制限](/ja/rate-limits) を参照) 。

  ## FAQ
</div>

<AccordionGroup>
  <Accordion title="未使用のクレジットは翌月に繰り越されますか？">
    **プランクレジット** はデフォルトでは繰り越されません — 毎月の割り当ては毎月リセットされます。**年間 Scale プランでは未使用のプランクレジットが 1 か月繰り越され**、**年間 Enterprise プランでは 2 か月繰り越されます**。ただし、**自動チャージパック** で購入したクレジットは、どのプランでも請求サイクルに紐づいていないため、翌月以降に繰り越されます。自動チャージクレジットの有効期限は購入日から **1 年** です。
  </Accordion>

  <Accordion title="自動チャージ と Smart Upgrade の違いは何ですか？">
    **自動チャージ** は、残高がしきい値に達したときに単発のクレジットパックを購入し、現在のプランを維持します。**Smart Upgrade** は、残高が 0 になると料金体系上の次のクレジットティア (例: Hobby 5k → 6.5k → 8k → Standard 100k → …) に移行し、日割り計算された差額が請求されます。両者は併用できず、組織ではどちらか一方のみを使用します。一時的な利用増には 自動チャージ を、利用量が着実に増えていて自動的に上位ティアへ移行したい場合は Smart Upgrade を使用してください。
  </Accordion>

  <Accordion title="Smart Upgrade の料金はどのように計算されますか？">
    Smart Upgrade では、現在のティアと次のティアの差額を日割りで請求します。**月額**プランでは、これは単純に月額料金の差額です。**年額**プランでは、現在の期間の開始時点からの暦上の完全な月数を基に、`(請求サイクルの残り月数 ÷ 12) × 年額料金の差額` で計算されます。さらに、2 つのティア間のクレジット差分も受け取れるため、増える分に対してのみ支払うことになります。
  </Accordion>

  <Accordion title="2 つのプランの中間のクレジット量を選べますか？">
    はい。[料金スライダー](https://www.firecrawl.dev/pricing) では、各プラン内の中間クレジットティアを選択できます — Hobby には 5k/6.5k/8k、Standard には 100k/130k/160k、Growth には 500k/650k があります。これらの中間ティアは、各プランの代表ティアと同じ同時実行数およびサポートレベルで、異なるのはクレジットの割り当てと価格のみです。
  </Accordion>

  <Accordion title="残りのクレジット数はどこで確認できますか？">
    [firecrawl.dev/app](https://www.firecrawl.dev/app) のダッシュボードを確認するか、プログラムからは [Credit Usage API エンドポイント](/ja/api-reference/endpoint/credit-usage) を呼び出してください。
  </Accordion>

  <Accordion title="クレジットとトークンの違いは何ですか？">
    クレジットは Firecrawl API 呼び出しの課金単位です。トークンは、Extract や Agent などのエンドポイントで内部的に使用される LLM のトークンを指します。1 クレジットは 15 トークンに相当します。ほとんどのユーザーは、クレジットだけを意識していれば問題ありません。
  </Accordion>

  <Accordion title="クーポンコードはどこで適用しますか？">
    クーポンの種類によって異なります。**クレジットクーポン** はダッシュボードの 課金 セクションで適用します。**サブスクリプションクーポン** (プラン料金の割引) は、プランの契約時または変更時に Stripe のチェックアウトページでのみ適用できます。
  </Accordion>

  <Accordion title="より高い同時実行数やカスタムプランが必要です。どこに連絡すればよいですか？">
    [help@firecrawl.dev](mailto:help@firecrawl.dev) までお問い合わせいただくか、[Enterprise ページ](https://www.firecrawl.dev/enterprise)でカスタムプランの詳細をご確認ください。
  </Accordion>

  <Accordion title="請求書はどの通貨で発行されますか？">
    Firecrawl のすべての請求書は、請求先住所や支払い方法にかかわらず、**米ドル (USD)** で請求されます。
  </Accordion>

  <Accordion title="請求書に VAT 番号、会社名、または請求先住所を追加するにはどうすればよいですか？">
    [課金設定](https://www.firecrawl.dev/app/settings?tab=billing) に移動し、**Manage Subscriptions** をクリックして、Stripe ポータルで請求先住所、会社名、VAT 番号を更新してください。今後の請求書には、更新後の情報が自動的に反映されます。新しい情報を反映した過去の請求書の再発行が必要な場合は、[help@firecrawl.dev](mailto:help@firecrawl.dev) までご連絡ください。
  </Accordion>

<div id="overview">
  ## 概要
</div>

Firecrawl は **クレジットベースの課金システム**を採用しています。行う API 呼び出しごとにクレジットが消費され、消費されるクレジット数は利用するエンドポイントやオプションによって異なります。プランごとに毎月のクレジット枠が決まっており、必要に応じて自動チャージパックで追加クレジットを購入できます。

現在のプラン料金については、[Firecrawl の料金ページ](https://www.firecrawl.dev/pricing)を参照してください。

<div id="credits">
  ## クレジット
</div>

クレジットは、Firecrawl における利用を測る単位です。各プランには毎月のクレジット割り当てが含まれており、請求サイクルの開始時にリセットされます。API の各エンドポイントごとに、消費するクレジット数は異なります。

<div id="credit-costs-per-endpoint">
  ### エンドポイントごとのクレジットコスト
</div>

| Endpoint | クレジットコスト | 補足 |
|----------|------------|-------|
| **Scrape** | 1 クレジット / ページ | 1 つの URL をクリーンな Markdown、HTML、または構造化データに変換します。Scrape オプションを使用する場合は追加クレジットが発生します (下記参照) 。 |
| **Crawl** | 1 クレジット / ページ | 開始 URL からリンクをたどって、サイト全体をスクレイピングします。クロールされた各ページには、同じページ単位の Scrape オプションのクレジットコストが適用されます。 |
| **Map** | 1 クレジット / コール | コンテンツをスクレイピングせずに、サイト上のすべての URL を検出します。 |
| **Search** | 2 クレジット / 10 件の結果 | Web を検索し、必要に応じて結果をスクレイピングします。スクレイピングされた各結果には、追加のページ単位の Scrape クレジットコストが発生します。 |
| **Browser** | 2 クレジット / ブラウザ利用分 | 分単位で課金されるインタラクティブなブラウザサンドボックスセッションです。 |
| **Agent** | 変動制 | 自律的な Web リサーチエージェント。1 日あたり 5 回まで無料で実行でき、それ以降は利用量に応じた課金となります。 |

<div id="additional-credit-costs-for-scrape-options">
  ### スクレイプオプションによる追加クレジットコスト
</div>

一部のスクレイプオプションは、ページごとの基本コストに加えてクレジットが加算されます:

| Option | Additional Cost | Description |
|--------|----------------|-------------|
| PDF parsing | +1 credit / PDF page | PDF ドキュメントからコンテンツを抽出します |
| JSON format (LLM extraction) | +4 credits / page | LLM を使用してページから構造化された JSON データを抽出します |
| Enhanced Mode | +4 credits / page | アクセスが難しいページ向けにスクレイピングを強化します |

これらのオプションは併用でき、追加コストは合算されます。たとえば、JSON format と Enhanced Mode の両方を有効にして 1 ページをスクレイプすると、そのページには **1 + 4 + 4 = 9 クレジット**が必要です。これらと同じ追加コストは、各ページに対して内部的にスクレイプ処理を行うため、Crawl エンドポイントと Search エンドポイントにも適用されます。

<div id="when-credits-are-charged">
  ### クレジットが消費されるタイミング
</div>

**batch scrape** や **crawl** ジョブの場合、クレジットはジョブの送信時ではなく、各ページの処理が完了するたびに非同期で課金されます。つまり、ジョブを送信してから、アカウント上に最終的なクレジット消費が反映されるまでにタイムラグが生じる場合があります。バッチに多くの URL が含まれている場合や、高トラフィック時にページがキューに入れられる場合、送信後も数分から数時間にわたってクレジットが順次反映され続けることがあります。バッチのステータスをポーリングまたは確認しても、クレジットは消費されません。

<div id="tracking-your-usage">
  ### 利用状況の確認
</div>

クレジットの利用状況は、次の2つの方法で確認できます。

* **ダッシュボード**: 現在および過去の利用状況を [firecrawl.dev/app](https://www.firecrawl.dev/app) で確認できます
* **API**: [Credit Usage](/ja/api-reference/endpoint/credit-usage) および [Credit Usage Historical](/ja/api-reference/endpoint/credit-usage-historical) エンドポイントを使用して、プログラムから利用状況を確認できます

<Note>
  クレジット利用状況をより理解しやすくするための改善に継続的に取り組んでいます。今後のアップデートをお待ちください。
</Note>

<div id="plans">
  ## プラン
</div>

Firecrawl はサブスクリプション制の月額プランを提供しています。純粋な従量課金プランはありませんが、後述の自動チャージ機能により、ベースプランに対して柔軟にスケールさせることができます。

<div id="available-plans">
  ### 利用可能なプラン
</div>

| プラン | 月間クレジット | 同時実行ブラウザ数 |
|------|----------------|--------------------:|
| **Free** | 500 (1回限り)  | 2 |
| **Hobby** | 3,000 | 5 |
| **Standard** | 100,000 | 50 |
| **Growth** | 500,000 | 100 |

<Note>
  Growth プランを超える大規模な利用ニーズには、Firecrawl は **Scale** および **Enterprise** プランを提供しています。これらは 100 万クレジット以上、150 台以上の同時実行ブラウザ、専任サポート、SLA、一括割引、データ非保持 (ゼロデータ保持) 、SSO などに対応しています。詳細は [Enterprise ページ](https://www.firecrawl.dev/enterprise) を参照してください。
</Note>

すべての有料プランは **月払い** または **年払い** に対応しています。年払いは月払いと比べて割引があります。各プランの最新の料金については、[料金ページ](https://www.firecrawl.dev/pricing) を参照してください。

<div id="concurrent-browsers">
  ### 同時ブラウザ数
</div>

同時ブラウザ数は、Firecrawl が同時に処理できる Web ページ数を表します。この上限はご利用プランによって決まります。上限を超えた場合、追加のジョブは空きが出るまでキューで待機します。同時実行数と API のレート制限の詳細については、[Rate Limits](/ja/rate-limits) を参照してください。

<div id="auto-recharge">
  ## 自動チャージ
</div>

ときどきプランに含まれるクレジット以上に利用する必要がある場合は、ダッシュボードから **自動チャージ** を有効にできます。残りクレジットが設定したしきい値を下回ると、Firecrawl が自動的に追加のクレジットパックを購入し、残高に加算します。

* 自動チャージ用のパックは、すべての有料プランで利用できます
* パックのサイズと価格はプランごとに異なります ([料金ページ](https://www.firecrawl.dev/pricing) で確認できます)
* 自動チャージのしきい値を設定でき、自動チャージはいつでもオン／オフを切り替えられます
* 自動チャージは **月あたり4パックまで** に制限されています

自動チャージは、利用量が一時的にスパイクするケースへの対応に最適です。継続的にプランの割り当てを超えてしまう場合は、より上位のプランにアップグレードした方が、クレジットあたりのコスト面で有利になります。

<div id="coupons">
  ## クーポン
</div>

Firecrawl は、2種類のクーポンに対応しています。

* **サブスクリプションクーポン**は、プランのサブスクリプション料金に割引を適用します (例: 月額または年額料金の一定割合の割引) 。これらは、有料プランに初めて登録する際、またはプラン変更時の Stripe のチェックアウトフロー中に**のみ**適用できます。チェックアウト完了後にサブスクリプションクーポンを適用することはできません。
* **クレジットクーポン**は、アカウントにボーナスクレジットを追加します。これらは、ダッシュボードの**課金**セクションにある [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing) から引き換えできます。コードを適用するには、課金ページのクーポン入力欄を探してください。クレジットクーポンによるボーナスクレジットは、プランの月次割り当てとは別であり、プランをアップグレードまたはダウングレードしても保持されます。

クーポンコードをお持ちで、どちらの種類かわからない場合は、まずダッシュボードの課金セクションで適用してみてください。サブスクリプションクーポンの場合は、代わりに Stripe のチェックアウトページで使用する必要があります。

<div id="billing-cycle">
  ## 請求サイクル
</div>

* **月額プラン**: クレジットは毎月の更新日にリセットされます
* **年額プラン**: 請求は年に一度ですが、クレジットはシステム上の毎月の更新日にリセットされます
* **未使用のクレジットは繰り越されません** — 各請求期間の開始時に、割り当てはプランで定められたクレジット数にリセットされます

<div id="upgrading-and-downgrading">
  ## アップグレードとダウングレード
</div>

* **アップグレード** は即時に反映されます。現在の請求期間の残り日数に応じた差額が請求され、クレジットの割り当てと上限はすぐに更新されます。
* **ダウングレード** は次回の更新日に反映されるようにスケジュールされます。それまでは現在のプランのクレジットと上限を維持します。

<div id="what-happens-when-you-run-out-of-credits">
  ## クレジットを使い切った場合
</div>

クレジット残高を使い切り、かつ自動チャージを有効にしていない場合、クレジットを消費する API リクエストは **HTTP 402 (Payment Required)** エラーを返します。自動チャージを有効にしている場合は、自動的にリチャージパックが購入されるあいだは利用を継続できますが、月間パック上限に達した場合やリチャージに失敗した場合は、次の請求サイクルまたは手動でチャージするまで残高がマイナスになることがあります。強制停止後に利用を再開するには、次のいずれかを行ってください。

1. 自動チャージを有効にしてクレジットを自動購入する
2. 上位プランにアップグレードする
3. 次の請求サイクルでクレジットがリセットされるのを待つ

<div id="free-plan">
  ## 無料プラン
</div>

無料プランでは、**一度きりで 500 クレジットが付与**され、クレジットカードは不要です。これらのクレジットは更新されないため、使い切った後も Firecrawl を継続利用するには有料プランへのアップグレードが必要です。無料プランは、有料プランと比べてレート制限および同時実行数が低くなっています ([レート制限](/ja/rate-limits) を参照) 。

<div id="faqs">
  ## FAQ
</div>

<AccordionGroup>
  <Accordion title="未使用のクレジットは翌月に繰り越されますか？">
    いいえ。クレジットの割り当ては各請求期間の開始時にリセットされます。未使用のクレジットは繰り越されません。
  </Accordion>

  <Accordion title="残りのクレジット数はどこで確認できますか？">
    [firecrawl.dev/app](https://www.firecrawl.dev/app) のダッシュボードを確認するか、プログラムからは [Credit Usage API エンドポイント](/ja/api-reference/endpoint/credit-usage) を呼び出してください。
  </Accordion>

  <Accordion title="クレジットとトークンの違いは何ですか？">
    クレジットは Firecrawl API 呼び出しの課金単位です。トークンは、Extract や Agent などのエンドポイントで内部的に使用される LLM のトークンを指します。1 クレジットは 15 トークンに相当します。ほとんどのユーザーは、クレジットだけを意識していれば問題ありません。
  </Accordion>

  <Accordion title="クーポンコードはどこで適用しますか？">
    クーポンの種類によって異なります。**クレジットクーポン** はダッシュボードの Billing セクションで適用します。**サブスクリプションクーポン** (プラン料金の割引) は、プランの契約時または変更時に Stripe のチェックアウトページでのみ適用できます。
  </Accordion>

  <Accordion title="より高い同時実行数やカスタムプランが必要です。どこに連絡すればよいですか？">
    [help@firecrawl.dev](mailto:help@firecrawl.dev) までお問い合わせいただくか、[Enterprise ページ](https://www.firecrawl.dev/enterprise)でカスタムプランの詳細をご確認ください。
  </Accordion>
</AccordionGroup>

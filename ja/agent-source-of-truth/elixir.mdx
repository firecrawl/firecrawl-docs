<div id="firecrawl-elixir-source-of-truth">
  # Firecrawl Elixir の信頼できる唯一の情報源
</div>

このファイルは、エージェント向けの公開された正規の信頼できる唯一の情報源です。Firecrawl Elixir SDK (OpenAPI形式) と v2 OpenAPI 仕様から生成されています。

<div id="install">
  ## インストール
</div>

```elixir
# mix.exs
defp deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

<div id="authenticate">
  ## 認証
</div>

`config/runtime.exs` でAPIキーを設定します：

```elixir
config :firecrawl, api_key: System.fetch_env!("FIRECRAWL_API_KEY")
```

または、各リクエストで `api_key` を渡します:

```elixir
{:ok, resp} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-...")
```

<div id="when-to-use-what">
  ## 何を使い分けるべきか
</div>

* `search`: クエリから始めて、`site:` 制約を含む関連 URL を見つけます。
* `scrape`: 既知の URL からコンテンツを取得・抽出します。
* `interact`: ページでクリックやフォーム入力、またはスクレイピング後の処理が必要な場合に、ブラウザーのアクションやコードを実行します。

<div id="search">
  ## 検索
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

クエリに基づいて候補を見つけ、Firecrawl に URL を返させたい場合 (必要に応じて各結果をスクレイピングすることも可能) は、search を使用します。たとえば、標準の `site:` 演算子を使って、結果を特定のドメインに限定できます。例: `site:docs.firecrawl.dev crawl webhooks`

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### シンプルな例
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### 複雑な例
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000,
    actions: [
      %{type: "click", selector: "button#accept"},
      %{type: "wait", milliseconds: 750},
      %{type: "scrape"}
    ]
  ]
)
```

<div id="parameters">
  ### パラメータ
</div>

| パラメータ                 | 型 / オプション                                                    | 説明                                                   |
| --------------------- | ------------------------------------------------------------ | ---------------------------------------------------- |
| `query`               | `String`                                                     | 検索クエリ。`site:domain` を使うと、結果を特定のサイトに限定できます。           |
| `sources`             | `List` (`"web"`, `"news"`, `"images"` または `type` を持つマップ)     | 検索対象のソース。                                            |
| `categories`          | `List` (`"github"`, `"research"`, `"pdf"` または `type` を持つマップ) | 結果を絞り込むカテゴリ。                                         |
| `limit`               | `integer`                                                    | 返される結果の最大件数。                                         |
| `tbs`                 | `String`                                                     | 時間ベースの検索フィルター (例: `qdr:d`, `qdr:w`, `sbd:1,qdr:m`) 。 |
| `location`            | `String`                                                     | 検索結果用のロケーション文字列。                                     |
| `country`             | `String`                                                     | 検索結果の地域ターゲティングに使用する ISO 国コード。                        |
| `enterprise`          | `List[String]` (`"anon"`, `"zdr"`)                           | チームで有効になっている場合の Enterprise ZDR オプション。                |
| `ignore_invalid_urls` | `boolean`                                                    | 他の Firecrawl エンドポイントでスクレイピングできない無効な URL を除外します。      |
| `timeout`             | `integer`                                                    | タイムアウト (ミリ秒) 。                                       |
| `scrape_options`      | `keyword`                                                    | 各結果に適用されるスクレイピングのオプション (`scrape` と同じフィールド) 。         |

<div id="scrape">
  ## スクレイピング
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

URL がすでに分かっており、Firecrawl から構造化されたコンテンツと任意のフォーマットを取得したい場合は、スクレイピングを使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.scrape_and_extract_from_url(params, opts)`

### 基本的な例

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

### 応用例

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  include_tags: ["main", "article"],
  exclude_tags: ["nav", "footer"],
  only_main_content: true,
  wait_for: 1000,
  mobile: false,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "write", text: "firecrawl"},
    %{type: "press", key: "Enter"},
    %{type: "wait", milliseconds: 1500},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  skip_tls_verification: true,
  remove_base64_images: true,
  block_ads: true,
  proxy: "auto",
  max_age: 86400000,
  store_in_cache: true,
  profile: [name: "docs-session", saveChanges: true]
)
```

<div id="parameters">
  ### パラメータ
</div>

| パラメータ                   | 型 / オプション          | 説明                                                                                                                    |                   |          |
| ----------------------- | ------------------ | --------------------------------------------------------------------------------------------------------------------- | ----------------- | -------- |
| `url`                   | `String`           | スクレイピング対象の URL。                                                                                                       |                   |          |
| `formats`               | `List`             | 含める出力フォーマット。`%{type: "json", prompt, schema}` や `%{type: "screenshot", fullPage, quality, viewport}` などの文字列またはオブジェクト。 |                   |          |
| `only_main_content`     | `boolean`          | メインコンテンツのみを保持します。                                                                                                     |                   |          |
| `include_tags`          | `List[String]`     | 含めるタグ。                                                                                                                |                   |          |
| `exclude_tags`          | `List[String]`     | 除外するタグ。                                                                                                               |                   |          |
| `max_age`               | `integer`          | この期間以内の古さのキャッシュデータを優先します。                                                                                             |                   |          |
| `min_age`               | `integer`          | 最小経過時間を満たすキャッシュデータのみを返します。                                                                                            |                   |          |
| `headers`               | `map`              | カスタムヘッダー。                                                                                                             |                   |          |
| `wait_for`              | `integer`          | スクレイピング前の待機時間 (ミリ秒) 。                                                                                                 |                   |          |
| `mobile`                | `boolean`          | モバイルデバイスをエミュレートします。                                                                                                   |                   |          |
| `skip_tls_verification` | `boolean`          | TLS 証明書の確認をスキップします。                                                                                                   |                   |          |
| `timeout`               | `integer`          | タイムアウト時間 (ミリ秒) 。                                                                                                      |                   |          |
| `parsers`               | `List`             | パーサーの設定 (PDF オプション対応) 。                                                                                               |                   |          |
| `actions`               | `List`             | スクレイピング前に実行するブラウザアクション。                                                                                               |                   |          |
| `location`              | `keyword`          | リクエストのロケーション設定。                                                                                                       |                   |          |
| `remove_base64_images`  | `boolean`          | markdown 出力から base64 画像を削除します。                                                                                        |                   |          |
| `block_ads`             | `boolean`          | 広告と Cookie ポップアップのブロックを有効にします。                                                                                        |                   |          |
| `proxy`                 | `&quot;basic&quot; | &quot;enhanced&quot;                                                                                                  | &quot;auto&quot;` | プロキシモード。 |
| `store_in_cache`        | `boolean`          | 結果をキャッシュに保存します。                                                                                                       |                   |          |
| `profile`               | `keyword`          | 永続的なブラウザプロファイル。例: `[name: "profile", saveChanges: true]`。                                                             |                   |          |
| `zero_data_retention`   | `boolean`          | アカウントで許可されている場合にゼロデータ保持を有効にします。                                                                                       |                   |          |

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

ページで、スクレイピング開始後にブラウザのアクションやコードの実行 (クリック、フォーム入力、スクリプトによる手順など) が必要な場合は、Interact を使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

<div id="simple-example">
  ### シンプルな例
</div>

```elixir
{:ok, scrape_resp} = Firecrawl.scrape_and_extract_from_url(url: "https://example.com")
job_id = get_in(scrape_resp.body, ["data", "metadata", "scrapeId"])

{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  job_id,
  code: "console.log(document.title)",
  language: :node
)
```

<div id="complex-example">
  ### 複雑な例
</div>

```elixir
{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  "<scrape_job_id>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="parameters">
  ### パラメータ
</div>

| パラメータ      | 型 / オプション | 説明                                       |        |                             |
| ---------- | --------- | ---------------------------------------- | ------ | --------------------------- |
| `job_id`   | `String`  | `data.metadata.scrapeId` のスクレイピングジョブ ID。 |        |                             |
| `code`     | `String`  | ブラウザセッションで実行するコード。必須。                    |        |                             |
| `language` | `:python  | :node                                    | :bash` | 実行言語。省略時のデフォルトは `:node` です。 |
| `timeout`  | `integer` | 実行タイムアウト (秒) 。                           |        |                             |
| `origin`   | `String`  | 内部専用。Firecrawl の計測用に予約されています。            |        |                             |

<div id="notes">
  ## 注記

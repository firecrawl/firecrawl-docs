  ## 7日目 – インテグレーション・デー
</div>

![Firecrawl のインテグレーション](/images/launch-week/lw3-d7-1.webp)

<div id="overview">
  ### 概要
</div>

Launch Week III の最終日に、普段お使いのツールやプラットフォームと Firecrawl をより簡単に連携できる、新たな統合とアップデートを公開します。

オートメーションプラットフォームから AI パイプラインまで、Firecrawl は現在 20 以上のサービスと連携し、ウェブデータからワークフロー実行までの到達時間をさらに短縮します。

<div id="key-integration-updates">
  ### 主要なインテグレーション更新
</div>

- **Discord Bot:** サーバー内でスクレイプを実行し、構造化結果を受け取れます。 [詳細はこちら](https://github.com/mendableai/firecrawl-discord-bot)
- **Make Integration:** Firecrawlのスクレイピングと抽出を活用したワークフローをビジュアルに構築できます。 [詳細はこちら](https://www.make.com/en/integrations/firecrawl)
- **n8n Integration:** Firecrawlをカスタムオートメーションフローに接続できます。 [詳細はこちら](https://n8n.io/)
- **Langflow:** FirecrawlのエージェントをLangflowのパイプラインにシームレスに組み込めます。 [詳細はこちら](https://docs.langflow.org/)
- **LlamaIndex:** FirecrawlをLlamaIndexと組み合わせて、データの強化と高精度な取得を行えます。 [詳細はこちら](https://www.llamaindex.ai/)
- **Dify:** FirecrawlをDifyと統合して、AIワークフローを自動化します。 [詳細はこちら](https://dify.ai/)

今後もインテグレーションを順次追加予定です。ご希望のインテグレーションがあれば、ぜひお知らせください。

<div id="day-6-firecrawl-mcp">
  ## Day 6 – Firecrawl MCP
</div>

![Firecrawl MCP!](/images/launch-week/lw3-d6-1.webp)

<div id="overview">
  ### 概要
</div>

本日、LLM 接続のスクレイピングワークフロー向け Model Context Protocol（MCP）実装である Firecrawl MCP サーバーに大規模アップグレードをリリースします。

このリリースでは MCP に FIRE-1（エージェント）対応を追加し、ログインやボタンなどのインタラクションの壁の先にあるデータを、/scrape エンドポイントと /extract エンドポイント経由で取得できるようにします。 

また、ローカル利用向けに SSE（サーバー送信イベント）にも対応し、セットアップとリアルタイム連携がこれまで以上に簡単になります。

これらのアップデートにより、インテリジェントなエージェントが重作業を担い、ウェブデータを LLM パイプラインへストリーミングしやすくなります。

### 主な機能

* FIRE-1（エージェント）対応: Model Context Protocol（MCP）の /scrape および /extract エンドポイント経由でウェブアクションのエージェントを利用できます。
* インタラクション認識: 複雑なウェブ上のバリアを自動で突破・ナビゲートします。
* SSE（サーバー送信イベント）対応: ローカル環境でも当社のホスト型 MCP サーバー経由でも、リアルタイムで出力をストリーミングします。
* プラグアンドプレイ: 既存のスタックに最小限のセットアップで統合できます。

<div id="day-4-llmstxtnew">
  ## Day 4 – LLMstxt.new
</div>

![LLMs.txt メイン](/images/launch-week/lw3-d4-1.webp)

<div id="overview">
  ### 概要
</div>

本日、[http://llmstxt.new](http://llmstxt.new) を発表します — あらゆるウェブサイトを、LLM向けのクリーンで統合されたテキストファイルに最速で変換する方法です。

任意のURLの先頭に `llmstxt.new/` を付けるだけで、AIの学習と推論に最適化されたプレーンな `.txt` ファイルが返ってきます。不要なボイラープレートもノイズもなく、有用なコンテンツだけ。

Firecrawl の上に構築されたこのツールは、実世界のウェブコンテンツを LLM パイプラインで活用できる形に、手間なく整備できます。

![LLMs.txt Generator Interface](/images/launch-week/lw3-d4-2.webp)

### 主な機能  

- **すぐに使える**: 任意のURLの前に `llmstxt.new/` を付けるだけ。  
- **2種類の出力**: `llms.txt` は簡潔なサマリー用、`llms-full.txt` は全文用。  
- **APIフレンドリー**: `http://llmstxt.new/{YOUR_URL}` で呼び出すか、Firecrawl の API キーで全文出力を取得。  
- **LLM対応フォーマット**: 学習と推論の両コンテキスト向けに設計。  

<div id="day-3-extract-v2">
  ## Day 3 – /extract v2
</div>

![/extract v2 メイン](/images/launch-week/lw3-d3-l.webp)

<div id="overview">
  ### 概要
</div>

本日、FIRE-1（エージェント）によって強化された抽出システムの大規模アップグレードである **/extract v2** をリリースしました。

**ページネーション、マルチステップのフロー、動的なインタラクション**をフルサポートし、/extract v2 は1月に提供した機能を大きく上回ります。さらに、内蔵の検索レイヤーで目的のコンテンツを見つけることで、**URL なし**でもデータを抽出できるようになりました。

内部を一から再設計し、モデルの改良、優れたアーキテクチャ、そして社内ベンチマーク全体での大幅なパフォーマンス向上を実現しています。

![Extract v2 の動作例](/images/launch-week/lw3-d3-2.webp)

<div id="key-features">
  ### 主要機能
</div>

- **FIRE-1（エージェント）統合**: ページのアクション、ナビゲーション、インタラクションを自動処理します。  
- **マルチページ抽出**: ページネーションやステップベースのフロー全体からデータを抽出します。  
- **URL不要**: 組み込み検索で、直接リンクがなくてもコンテンツを抽出できます。  
- **精度向上**: 新しいモデルとアーキテクチャにより、より高精度かつ高速な結果を提供します。

  ## Day 2 - FIRE-1 エージェント

![FIRE-1 エージェントのビジュアライゼーション](/images/launch-week/lw3d2-1.webp)

<div id="overview">

**FIRE-1** をご紹介します。Firecrawl 初の AI エージェントで、ウェブスクレイピングを次の段階へ引き上げます。高度なナビゲーションとインタラクションにより、FIRE-1 は従来のスクレイピング手法を大きく超えます。

ページネーションの処理から、ボタンやリンクなどの動的要素との対話まで、FIRE-1 は強力でコンテキスト認識型のスクレイピング／抽出ワークフローを実現します。

- **スマートナビゲーション**: ページネーションや複数ステップのコンテンツを自動で横断します。
- **ダイナミックインタラクション**: ボタンのクリック、リンクの追跡、JavaScriptでレンダリングされたコンポーネントとの対話を自動化します。

<div id="day-1-change-tracking">
  ## Day 1 - 変更トラッキング
</div>

![変更トラッキング（メイン）](/images/launch-week/lw3d1.webp)

<div id="overview">
  ### 概要
</div>

変更トラッキングは、ウェブコンテンツの変化を経時的に監視・検出できる強力な機能です。JavaScript と Python の両方の SDK で利用できます。
![変更トラッキング](/images/launch-week/lw3d12.webp)

### 主要な機能

- **変更の検出**: 前回のスクレイプ以降にウェブページが変更されたかを判定します。
- **特定の変更の表示**: スクレイプ間で何が変わったかの詳細にアクセスできます。
- **構造化データ**: 変更内容の構造化データを受け取れます。
- **可視性の制御**: 自サイト上で変更の可視性を管理します。

`変更トラッキング` フォーマットを使用すると、サイトの変更を効果的に監視でき、前回スクレイプのタイムスタンプ、2つのページバージョンの比較結果、現在のページ／URLの可視性に関する包括的な情報を受け取れます。

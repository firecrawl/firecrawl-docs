<div id="base-url">
  ## ベースURL
</div>

すべてのリクエストは次のベースURLを使用します：

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## 認証
</div>

認証には `Authorization` ヘッダーを含める必要があります。ヘッダーの値として `Bearer fc_123456789` を指定し、`fc_123456789` の部分には自分の API キーを設定してください。

```bash
Authorization: Bearer fc_123456789
```

​


<div id="response-codes">
  ## レスポンスコード
</div>

Firecrawl は、リクエストの結果を示すために標準的な HTTP ステータスコードを使用します。

通常、2xx の HTTP ステータスコードは成功を示し、4xx のコードはクライアント側（ユーザー側）のエラーを表し、5xx のコードはインフラストラクチャの問題を示します。

Status | Description
--- | ---
200 | リクエストは正常に完了しました。
400 | パラメータが正しいか確認してください。
401 | API キーが指定されていません。
402 | 支払いが必要です。
404 | 要求されたリソースが見つかりませんでした。
429 | レート制限を超過しました。
5xx | Firecrawl のサーバーエラーを示します。

考えられるすべての API エラーの詳細な説明については、「Error Codes」セクションを参照してください。

​

<div id="rate-limit">
  ## レート制限
</div>

Firecrawl API には、サービスの安定性と信頼性を確保するためのレート制限があります。レート制限はすべてのエンドポイントに適用され、一定時間内のリクエスト数に基づいています。

レート制限を超えると、ステータスコード 429 が返されます。

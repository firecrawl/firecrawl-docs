</div>

まずは [firecrawl.dev](https://firecrawl.dev) で API キーを取得します。次に `FirecrawlApp` を初期化します。以降は `FirecrawlApp::scrape_url` などの関数にアクセスでき、API を利用できます。

</div>

単一のURLをスクレイプするには、`scrape_url` メソッドを使用します。URLを引数に取り、スクレイプ結果を `Document` として返します。

<div id="scraping-with-extract">
  ### Extract でのスクレイピング
</div>

Extract を使うと、任意の URL から構造化データを簡単に抽出できます。`serde_json::json!` マクロを用いて、JSON Schema 形式でスキーマを指定する必要があります。

<LLMExtractRust />

<div id="crawling-a-website">
  ### ウェブサイトのクロール
</div>

ウェブサイトをクロールするには、`crawl_url` メソッドを使用します。クロールが完了するまで待機します。所要時間は開始URLや指定したオプションによっては長くなる場合があります。

<CrawlRustShort />

<div id="crawling-asynchronously">
  #### 非同期でのクロール
</div>

結果を待たずにクロールするには、`crawl_url_async` メソッドを使用します。パラメータは同一ですが、クロールの ID を含む `CrawlAsyncRespone` 構造体を返します。その ID を `check_crawl_status` メソッドで使用して、いつでもステータスを確認できます。完了したクロールは 24 時間後に削除される点にご注意ください。

<div id="error-handling">
  ## エラーハンドリング
</div>

この SDK は、Firecrawl API および依存関係から返されるエラーを処理し、それらを `Error`、`Debug`、`Display` を実装する `FirecrawlError` 列挙体に集約します。すべてのメソッドは `Result<T, FirecrawlError>` を返します。

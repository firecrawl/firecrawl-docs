<Info>
  **従量課金の検索エンドポイント**

  このエンドポイントでは、暗号資産で検索ごとに支払うことで、サブスクリプションなしに Firecrawl の強力な検索機能を利用できます。単発の検索や、サブスクリプション契約前の検証に最適です。
</Info>

<Warning>
  **実験的機能**

  これは実験的なエンドポイントです。機能、料金、提供状況は予告なく変更される場合があります。本番環境では、標準のサブスクリプション型の検索エンドポイントの利用を推奨します。
</Warning>

<div id="authentication">
  ## 認証
</div>

このエンドポイントでは、標準のAPIキーの代わりに、専用の支払いヘッダーが必要です。

```
X-Payment: {{paymentHeader}}
```

<div id="how-it-works">
  ## 仕組み
</div>

On-Demand Search エンドポイントは、ウェブ検索と Firecrawl のスクレイピング機能を組み合わせ、通常の `/search` エンドポイントと同様に任意のクエリに対してページ全体のコンテンツを返しますが、料金は従量課金制です。

各検索結果の完全な markdown コンテンツを取得するには、`scrapeOptions` に `formats: ["markdown"]` を含めてください。指定しない場合は、デフォルトで検索結果（url、title、description）が返されます。

<div id="request-parameters">
  ### リクエストパラメータ
</div>

標準の `/search` エンドポイントのすべてのパラメータが、次の例外を除きサポートされています:

<Note>
  **重要な制限**: ウォレットエンドポイントにおける `limit` パラメータは**最大10件**までです（標準エンドポイントでは100件）。
</Note>

利用可能なパラメータの一覧、サポートされるクエリ演算子、その説明の詳細は、[標準の Search エンドポイントのドキュメント](/ja/api-reference/endpoint/search)を参照してください。

<div id="example-request">
  ## リクエストの例
</div>

```bash
curl -X POST https://api.firecrawl.dev/v1/x402/search \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <token>' \
  -H 'X-Payment: {{paymentHeader}}' \
  -d '{
    "query": "2025/26シーズンのプレミアリーグの試合日程",
    "limit": 10,  # 結果は最大10件
    "scrapeOptions": {
      "formats": ["markdown"],
      "onlyMainContent": true
    }
  }'
```

<div id="response">
  ## レスポンス
</div>

レスポンス形式は標準の `/search` エンドポイントと同じです:

```json
{
  "success": true,
  "data": [
    {
      "title": "ページのタイトル",
      "description": "検索結果のページ説明",
      "url": "https://example.com",
      "markdown": "# ページ内容\n\nスクレイピングをリクエストした場合のMarkdownコンテンツ...",
      "metadata": {
        "title": "ページのタイトル",
        "description": "メタディスクリプション"
        "sourceURL": "https://example.com",
        "statusCode": 200
      }
    }
  ]
}
```

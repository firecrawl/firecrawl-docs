<div id="firecrawl-java-agent-quickstart">
  # Firecrawl Java Agent クイックスタート
</div>

このファイルは、外部エージェント向けの標準的なクイックスタートであり、SDK のソースコードと v2 OpenAPI 仕様に基づいて生成されています。

<div id="install">
  ## インストール
</div>

```gradle
dependencies {
  implementation("com.firecrawl:firecrawl-java:1.2.0")
}
```

<div id="authenticate">
  ## 認証
</div>

```java
import com.firecrawl.client.FirecrawlClient;

FirecrawlClient client = FirecrawlClient.builder()
    .apiKey(System.getenv("FIRECRAWL_API_KEY"))
    .build();
```

<div id="when-to-use-what">
  ## 使い分けの目安
</div>

* `search`: クエリから始めて、対象を見つける必要がある場合に使用します。
* `scrape`: すでにURLがあり、ページのコンテンツを取得したい場合に使用します。
* `interact`: ページ上でのクリックやフォーム入力、またはスクレイピング後のブラウザアクションが必要な場合に使用します。

<div id="search">
  ## 検索
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

`search` を使うと、クエリに関連する URL を見つけ、必要に応じて各結果をスクレイピングできます。特定の URL をスクレイピングする前に、標準の `site:` 演算子 (たとえば `site:docs.firecrawl.dev crawl webhooks`) を使って、クエリを単一のサイトに絞り込むこともできます。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.search(query, options)`

<div id="simple-example">
  ### 簡単な例
</div>

```java
import com.firecrawl.models.SearchData;

SearchData data = client.search("site:docs.firecrawl.dev crawl webhooks");
System.out.println(data.getWeb());
```

<div id="complex-example">
  ### 複雑な例
</div>

```java
import com.firecrawl.models.SearchData;
import com.firecrawl.models.SearchOptions;
import com.firecrawl.models.ScrapeOptions;

SearchData data = client.search(
    "firecrawl pricing",
    SearchOptions.builder()
        .sources(java.util.List.of("web", "news"))
        .categories(java.util.List.of("research"))
        .limit(10)
        .tbs("qdr:w")
        .location("San Francisco,California,United States")
        .ignoreInvalidURLs(true)
        .scrapeOptions(
            ScrapeOptions.builder()
                .formats(java.util.List.of(
                    "markdown",
                    java.util.Map.of("type", "json", "prompt", "Extract pricing tiers and limits.")
                ))
                .onlyMainContent(true)
                .proxy("stealth")
                .build()
        )
        .integration("agent-quickstart")
        .build()
);
```

<div id="parameters">
  ### パラメータ
</div>

* `query`: 検索クエリ文字列。常に必須です。
* `sources`: 検索対象のソース。サポートされる値: `"web"`、`"news"`、`"images"`、または `{type: "web" | "news" | "images"}` のマップ。どの結果配列を返すかを制御するために使用します。
* `categories`: 結果の絞り込み条件。サポートされる値: `"github"`、`"research"`、`"pdf"`、または `{type: "github" | "research" | "pdf"}` のマップ。カテゴリで結果を絞り込むために使用します。
* `limit`: 返す結果の最大数。レスポンスのサイズを抑えるために使用します。
* `tbs`: 時間ベースの検索絞り込み文字列。時間範囲で絞り込むために使用します (例: `qdr:d`) 。
* `location`: ローカライズされた結果のためのロケーション文字列。地域に応じたクエリに使用します。
* `ignoreInvalidURLs`: 結果から無効な URL を除外します。他のエンドポイントにパイプする場合に使用します。
* `timeout`: リクエストのタイムアウト (ミリ秒) 。検索時間を制限するために使用します。
* `scrapeOptions`: 各結果に適用されるスクレイピングのオプション (`scrape` と同じフィールド) 。結果と一緒にコンテンツが必要な場合に使用します。
* `integration`: インテグレーション識別子。追跡や分析に使用します。

<div id="scrape">
  ## スクレイピング
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

URL がすでにわかっていて、特定のフォーマットでコンテンツを取得したい場合は、`scrape` を使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### 簡単な例
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com",
    ScrapeOptions.builder()
        .formats(java.util.List.of("markdown"))
        .build()
);

System.out.println(doc.getMarkdown());
```

### 高度な例

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.LocationConfig;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com/docs",
    ScrapeOptions.builder()
        .formats(java.util.List.of(
            "markdown",
            java.util.Map.of("type", "json", "prompt", "Extract headings and slugs."),
            java.util.Map.of("type", "changeTracking", "modes", java.util.List.of("git-diff"))
        ))
        .onlyMainContent(true)
        .waitFor(1500)
        .actions(java.util.List.of(
            java.util.Map.of("type", "click", "selector", "button#accept"),
            java.util.Map.of("type", "scroll", "direction", "down")
        ))
        .parsers(java.util.List.of(java.util.Map.of("type", "pdf", "mode", "auto", "maxPages", 5)))
        .location(LocationConfig.builder()
            .country("US")
            .languages(java.util.List.of("en-US"))
            .build())
        .proxy("stealth")
        .maxAge(3_600_000L)
        .build()
);
```

<div id="parameters">
  ### パラメータ
</div>

* `url`: スクレイピングする URL。常に必須です。
* `formats`: 含める出力フォーマット。対応するフォーマット文字列: `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`。フォーマットオブジェクト: `json` `{type: "json", prompt?, schema?}`, `changeTracking` `{type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`。
* `headers`: 追加の HTTP ヘッダー。認証、Cookie、または User-Agent に使用します。
* `includeTags`: 指定した HTML タグのコンテンツのみを含めます。抽出対象を絞り込むために使用します。
* `excludeTags`: 指定した HTML タグのコンテンツを除外します。ノイズを取り除くために使用します。
* `onlyMainContent`: メインコンテンツのみを抽出します。ナビゲーションやフッターを除外するために使用します。
* `timeout`: スクレイピングのタイムアウト (ミリ秒) 。実行時間の上限を設定するために使用します。
* `waitFor`: スクレイピング前の待機時間 (ミリ秒) 。JS でレンダリングされるページに使用します。
* `mobile`: モバイルデバイスをエミュレートします。モバイルレイアウトに使用します。
* `parsers`: パーサーのリストまたは設定。対応する PDF 設定: `{type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number}`、または文字列エントリ (例: `"pdf"`) 。
* `actions`: スクレイピング前に実行するブラウザのアクション。対応するアクション: `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`, `click` `{type: "click", selector, all?}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}`。対応フォーマットは `A0`-`A6`、`Letter`、`Legal`、`Tabloid`、`Ledger` です。
* `location`: ロケーション設定 (`country`, `languages`) 。地域固有のページに使用します。
* `skipTlsVerification`: TLS 証明書の確認をスキップします。自己署名証明書に使用します。
* `removeBase64Images`: markdown 出力から base64 画像を削除します。ペイロードサイズを減らすために使用します。
* `blockAds`: 広告と Cookie のブロックを有効にします。ノイズを減らすために使用します。
* `proxy`: プロキシモード `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`、またはカスタムプロキシ URL。
* `maxAge`: この期間 (ms) より新しいキャッシュ済みコンテンツを受け入れます。キャッシュヒットを優先するために使用します。
* `storeInCache`: 結果をキャッシュ/インデックスに保存します。再利用したい場合に使用します。
* `integration`: インテグレーション識別子。追跡/分析に使用します。

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

スクレイピング後に、ページ上でクリックやフォーム入力などのブラウザ操作が必要な場合は、`interact` を使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`client.interact(jobId, code, language, timeout, origin)`

<div id="simple-example">
  ### 簡単な例
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());"
);

System.out.println(exec.getStdout());
```

<div id="complex-example">
  ### 複雑な例
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());",
    "node",
    60,
    "agent-docs"
);
```

<div id="parameters">
  ### パラメータ
</div>

* `jobId`: スクレイピングジョブ ID (`document.metadata.scrapeId` から取得) 。常に必須です。
* `code`: ブラウザサンドボックスで実行するコード。スクリプトによる操作に使用します。
* `language`: ランタイム言語 `"python"`、`"node"`、または `"bash"` (デフォルトは `"node"`) 。コードに合わせて指定します。
* `timeout`: 実行タイムアウト (秒) 。長時間の実行を制限するために使用します。
* `origin`: 任意のオリジンタグ。リクエストの送信元を示すために使用します。

<div id="notes">
  ## 注意
</div>

* Java では、パラメータ名は camelCase です。
* `ScrapeOptions` と `SearchOptions` はビルダーパターンを採用しています。
* 非推奨のエイリアス: `scrapeExecute` と `deleteScrapeBrowser` (代わりに `interact` と `stopInteractiveBrowser` を使用) 。
* Java SDK で提供されるのはコードベースの `interact` のみです (`prompt` パラメータはありません) 。

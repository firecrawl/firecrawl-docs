<div id="firecrawl-elixir-agent-quickstart">
  # Firecrawl Elixir Agent クイックスタート
</div>

このファイルは外部エージェント向けの標準的なクイックスタートで、SDK のソースコードと v2 OpenAPI 仕様から生成されています。

<div id="install">
  ## インストール
</div>

`mix.exs` に依存関係を追加します。

```elixir
def deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

次に、以下を実行します:

```bash
mix deps.get
```

<div id="authenticate">
  ## 認証
</div>

```elixir
# config/runtime.exs または config/config.exs
config :firecrawl, api_key: "fc-your-api-key"
```

または、呼び出しごとに指定します:

```elixir
Firecrawl.scrape_and_extract_from_url([url: "https://example.com"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## どれをいつ使うか
</div>

* `search`: クエリから始めて、情報を見つける必要があるときに使います。
* `scrape`: すでにURLがあり、ページのコンテンツが必要なときに使います。
* `interact`: ページ上でクリックやフォーム操作、またはスクレイピング後のブラウザアクションが必要なときに使います。

<div id="search">
  ## 検索
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

クエリから関連するURLを見つけ、必要に応じて各結果をスクレイピングするには、`search` を使います。特定のURLをスクレイピングする前に、標準の `site:` 演算子 (たとえば `site:docs.firecrawl.dev crawl webhooks`) を使って、クエリを単一のサイトに絞り込むこともできます。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### シンプルな例
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  limit: 5
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### 複雑な使用例
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "firecrawl pricing",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  enterprise: ["anon"],
  scrape_options: [
    formats: [
      "markdown",
      [type: "json", prompt: "Extract pricing tiers and limits."]
    ],
    only_main_content: true,
    proxy: :enhanced
  ],
  timeout: 60000
)

IO.inspect(response.body)
```

<div id="parameters">
  ### パラメータ
</div>

* `query`: 検索クエリ文字列。常に必須です。
* `sources`: 検索対象のソース。サポートされる値: `"web"`、`"news"`、`"images"`、または `[%{type: "web" | "news" | "images"}]` 形式のオブジェクト。どの結果配列を返すかを制御するために使用します。
* `categories`: 結果の絞り込み条件。サポートされる値: `"github"`、`"research"`、`"pdf"`、または `[%{type: "github" | "research" | "pdf"}]` 形式のオブジェクト。カテゴリで結果を絞り込むために使用します。
* `limit`: 返す結果の最大件数。レスポンスサイズを抑えるために使用します。
* `tbs`: 時間ベースの検索フィルタ文字列。時間範囲で絞り込むために使用します (例: `qdr:d`) 。
* `location`: ローカライズされた結果のためのロケーション文字列。地域指定のクエリに使用します。
* `country`: 地理ターゲティングされた検索結果のための ISO 国コード。`location` とあわせて使用します。
* `timeout`: リクエストのタイムアウト (ミリ秒) 。検索時間を制限するために使用します。
* `ignore_invalid_urls`: 無効な URL を結果から除外します。他のエンドポイントにパイプする場合に使用します。
* `enterprise`: エンタープライズ検索のオプション。サポートされる値: `"anon"` (匿名化 ZDR) または `"zdr"` (エンドツーエンド ZDR) 。
* `scrape_options`: 各結果に適用されるスクレイピングのオプション (`scrape` と同じキー) 。結果と一緒にコンテンツも必要な場合に使用します。

<div id="scrape">
  ## スクレイピング
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

URL がすでにわかっていて、特定のフォーマットでコンテンツを取得したい場合は `scrape` を使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.scrape_and_extract_from_url(params, opts)`

### 簡単な例

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com",
  formats: ["markdown"]
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### 複雑な使用例
</div>

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/docs",
  formats: [
    "markdown",
    [type: "json", prompt: "Extract headings and slugs."],
    [type: "changeTracking", modes: ["git-diff"], tag: "docs"],
    [type: "screenshot", full_page: true, viewport: [width: 1280, height: 720]]
  ],
  actions: [
    [type: "click", selector: "button#accept"],
    [type: "scroll", direction: "down"]
  ],
  parsers: [[type: "pdf", mode: "auto", max_pages: 5]],
  location: [country: "US", languages: ["en-US"]],
  proxy: :enhanced,
  profile: [name: "docs-session", save_changes: true],
  max_age: 3_600_000,
  min_age: 0,
  store_in_cache: true,
  zero_data_retention: false
)

IO.inspect(response.body)
```

<div id="parameters">
  ### パラメータ
</div>

* `url`: スクレイピング対象のURL。常に必須です。
* `formats`: 含める出力フォーマット。サポートされるフォーマット文字列: `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`。フォーマットオブジェクト: `json` `[type: "json", prompt?, schema?]`、`changeTracking` `[type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?]`、`screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`。
* `headers`: 追加のHTTPヘッダー。認証、Cookie、User-Agentに使用します。
* `include_tags`: 指定したHTMLタグのコンテンツのみを含めます。抽出対象を絞り込むために使用します。
* `exclude_tags`: 指定したHTMLタグのコンテンツを除外します。ノイズを減らすために使用します。
* `only_main_content`: メインコンテンツのみを抽出します。ナビゲーションやフッターを除外するために使用します。
* `timeout`: スクレイピングのタイムアウト時間 (ミリ秒) 。実行時間を制限するために使用します。
* `wait_for`: スクレイピング前の待機時間 (ミリ秒) 。JSでレンダリングされるページに使用します。
* `mobile`: モバイルデバイスをエミュレートします。モバイル向けレイアウトに使用します。
* `parsers`: パーサーのリストまたは設定。サポートされるPDF設定: `[type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: number]`、または文字列エントリ (例: `"pdf"`) 。
* `actions`: スクレイピング前に実行するブラウザのアクション。サポートされるアクション: `wait` `[type: "wait", milliseconds? | selector?]`、`screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`、`click` `[type: "click", selector, all?]`、`write` `[type: "write", text]`、`press` `[type: "press", key]`、`scroll` `[type: "scroll", direction: "up" | "down", selector?]`、`scrape` `[type: "scrape"]`、`executeJavascript` `[type: "executeJavascript", script]`、`pdf` `[type: "pdf", format?, landscape?, scale?]`。`format` には `A0`-`A6`、`Letter`、`Legal`、`Tabloid`、`Ledger` を指定できます。
* `location`: ロケーション設定 `[country, languages]`。地域別のページに使用します。
* `skip_tls_verification`: TLS証明書の確認をスキップします。自己署名証明書に使用します。
* `remove_base64_images`: markdown出力からbase64画像を削除します。payloadサイズを減らすために使用します。
* `block_ads`: 広告とCookieのブロックを有効にします。ノイズを減らすために使用します。
* `proxy`: プロキシモード `:basic`, `:enhanced`, `:auto`。
* `max_age`: この時間 (ms) 未満のキャッシュ済みコンテンツを許可します。キャッシュヒットを優先するために使用します。
* `min_age`: 少なくともこの時間 (ms) 以上経過したキャッシュ済みコンテンツのみを返します。キャッシュのみの読み取りに使用します。
* `store_in_cache`: 結果をキャッシュ/インデックスに保存します。再利用したい場合に使用します。
* `profile`: 永続的なブラウザプロファイル `[name, save_changes]`。session状態を保持するために使用します。
* `zero_data_retention`: スクレイピングでゼロデータ保持を有効にします。チームでZDRが有効になっている場合に使用します。

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

ページでスクレイピングした後に、クリックやフォーム入力などのブラウザのアクションが必要な場合は、`interact` を使用します。

<div id="preferred-sdk-method">
  ### 推奨SDKメソッド
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

### 簡単な例

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.url())"
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### 複雑な使用例
</div>

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.title())",
  language: :node,
  timeout: 60,
  origin: "agent-docs"
)

IO.inspect(response.body)
```

<div id="parameters">
  ### パラメータ
</div>

* `job_id`: スクレイピングジョブ ID (`metadata.scrapeId` から取得) 。常に必須です。
* `code`: ブラウザサンドボックスで実行するコード。必須です。
* `language`: ランタイム言語 `:python`、`:node`、または `:bash` (デフォルトは `:node`) 。コードに合わせて選択してください。
* `timeout`: 実行タイムアウト (秒) 。長時間の実行を制限するために使用します。
* `origin`: 任意のオリジンタグ。リクエスト元の識別に使用します。

<div id="notes">
  ## 注記
</div>

* パラメータは、`NimbleOptions` で検証される snake&#95;case のキーワードリストとして渡されます。
* 関数は OpenAPI 仕様から自動生成され、名前は OpenAPI のオペレーション ID に従います。
* すべての関数には、エラー時に例外を送出する bang variant があります (例: `search_and_scrape!`) 。
* Elixir SDK では、コードベースの `interact` のみが公開されています (`prompt` パラメータはありません) 。
* すべての関数で、リクエスト単位の上書き用に `api_key` と `base_url` の options を指定できます。

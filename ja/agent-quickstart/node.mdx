<div id="firecrawl-javascripttypescript-agent-quickstart">
  # Firecrawl JavaScript/TypeScript Agent クイックスタート
</div>

このファイルは外部エージェント向けの標準的なクイックスタートで、SDK ソースと v2 OpenAPI 仕様から生成されています。

<div id="install">
  ## インストール
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="authenticate">
  ## 認証
</div>

```ts
import { Firecrawl } from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // セルフホストの場合は任意
});
```

<div id="when-to-use-what">
  ## 使い分けの目安
</div>

* `search`: クエリから始めて、まず対象を見つける必要があるときに使います。
* `scrape`: すでに URL があり、ページのコンテンツを取得したいときに使います。
* `interact`: ページ上でのクリックやフォーム入力、またはスクレイピング後のブラウザアクションが必要なときに使います。

<div id="search">
  ## 検索
</div>

### 利用する理由

`search` を使うと、クエリに関連する URL を見つけ、必要に応じて各結果をスクレイピングできます。特定の URL をスクレイピングする前に、標準の `site:` 演算子 (例: `site:docs.firecrawl.dev crawl webhooks`) を使って、クエリを 1 つのサイトに絞り込むこともできます。

<div id="preferred-sdk-method">
  ### 推奨される SDK メソッド
</div>

`client.search(query, options)`

### 簡単な例

```ts
const results = await client.search(
  "site:docs.firecrawl.dev crawl webhooks",
  { limit: 5 }
);

console.log(results.web?.[0]);
```

### 高度な例

```ts
const results = await client.search("firecrawl pricing", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  scrapeOptions: {
    formats: [
      "markdown",
      { type: "json", prompt: "Extract pricing tiers and limits." }
    ],
    onlyMainContent: true,
    actions: [
      { type: "wait", milliseconds: 1000 },
      { type: "scroll", direction: "down" }
    ],
    proxy: "stealth"
  },
  integration: "agent-quickstart",
  origin: "agent-docs"
});
```

<div id="parameters">
  ### パラメータ
</div>

* `query`: 検索クエリ文字列。常に必須です。
* `sources`: 検索対象のソース。対応する値: `"web"`, `"news"`, `"images"`, または `{ type: "web" | "news" | "images" }`。どの結果配列を返すかを制御するために使用します。
* `categories`: 結果の絞り込み条件。対応する値: `"github"`, `"research"`, `"pdf"`, または `{ type: "github" | "research" | "pdf" }`。カテゴリで結果を絞り込むために使用します。
* `limit`: 返す結果の最大件数。レスポンスサイズを抑えるために使用します。
* `tbs`: 時間ベースの検索絞り込み文字列。時間範囲で絞り込むために使用します (例: `qdr:d`) 。
* `location`: ローカライズされた結果のためのロケーション文字列。地域を指定したクエリに使用します。
* `ignoreInvalidURLs`: 無効な URL を結果から除外します。他のエンドポイントにパイプする場合に使用します。
* `timeout`: リクエストのタイムアウト (ミリ秒) 。検索時間を制限するために使用します。
* `scrapeOptions`: 各結果に適用されるスクレイピングのオプション (`scrape` と同じフィールド) 。結果と一緒にコンテンツも取得したい場合に使用します。
* `integration`: 統合識別子。追跡や分析のために使用します。
* `origin`: 任意の origin タグ。リクエストの帰属情報を付与するために使用します。

<div id="scrape">
  ## スクレイピング
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

URL がすでにわかっており、特定のフォーマットでコンテンツを取得したい場合は、`scrape` を使用します。

### 推奨SDKメソッド

`client.scrape(url, options)`

<div id="simple-example">
  ### 基本的な例
</div>

```ts
const doc = await client.scrape("https://example.com", {
  formats: ["markdown"]
});

console.log(doc.markdown);
```

<div id="complex-example">
  ### 複雑な使用例
</div>

```ts
const doc = await client.scrape("https://example.com/docs", {
  formats: [
    "markdown",
    { type: "json", prompt: "Extract headings and their slugs." },
    { type: "changeTracking", modes: ["git-diff"], tag: "docs" },
    { type: "screenshot", fullPage: true, viewport: { width: 1280, height: 720 } },
    {
      type: "attributes",
      selectors: [{ selector: "a", attribute: "href" }]
    },
    { type: "query", prompt: "What is the pricing model?" }
  ],
  onlyMainContent: true,
  waitFor: 1500,
  actions: [
    { type: "click", selector: "button#accept" },
    { type: "scroll", direction: "down" }
  ],
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  location: { country: "US", languages: ["en-US"] },
  proxy: "stealth",
  profile: { name: "docs-session", saveChanges: true },
  maxAge: 3600000
});
```

<div id="parameters">
  ### パラメータ
</div>

* `url`: スクレイピング対象の URL。常に必須です。
* `formats`: 含める出力フォーマット。サポートされるフォーマット文字列: `markdown`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `json`, `attributes`, `branding`, `audio`。フォーマットオブジェクト: `json` `{ type: "json", prompt?, schema? }` (`prompt` または `schema` が必要) 、`changeTracking` `{ type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag? }`、`screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`、`attributes` `{ type: "attributes", selectors: [{ selector, attribute }] }`、`query` `{ type: "query", prompt }`、および任意の文字列フォーマットに対する `{ type: "<format>" }`。
* `headers`: 追加の HTTP ヘッダー。認証、cookie、または user-agent に使用します。
* `includeTags`: 指定した HTML タグのコンテンツのみを含めます。抽出対象を絞り込むために使用します。
* `excludeTags`: 指定した HTML タグのコンテンツを除外します。ノイズを取り除くために使用します。
* `onlyMainContent`: メインコンテンツのみを抽出します。ナビゲーションやフッターを除外するために使用します。
* `timeout`: スクレイピングのタイムアウト (ミリ秒) 。実行時間の上限を設定するために使用します。
* `waitFor`: スクレイピング前の待機時間 (ミリ秒) 。JS でレンダリングされるページに使用します。
* `mobile`: モバイル端末をエミュレートします。モバイルレイアウトに使用します。
* `parsers`: パーサーのリストまたは設定。サポートされる PDF 設定: `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }` または文字列エントリ (例: `"pdf"`) 。
* `actions`: スクレイピング前に実行するブラウザ アクション。サポートされるアクション: `wait` `{ type: "wait", milliseconds? | selector? }`、`screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`、`click` `{ type: "click", selector }`、`write` `{ type: "write", text }`、`press` `{ type: "press", key }`、`scroll` `{ type: "scroll", direction: "up" | "down", selector? }`、`scrape` `{ type: "scrape" }`、`executeJavascript` `{ type: "executeJavascript", script }`、`pdf` `{ type: "pdf", format?, landscape?, scale? }`。使用できるフォーマットは `A0`-`A6`、`Letter`、`Legal`、`Tabloid`、`Ledger` です。
* `location`: ロケーション設定 `{ country, languages }`。地域固有のページに使用します。
* `skipTlsVerification`: TLS 証明書の確認をスキップします。自己署名証明書に使用します。
* `removeBase64Images`: markdown 出力から base64 画像を削除します。payload サイズを削減するために使用します。
* `fastMode`: 高速モードを有効にします。速度を最優先する場合に使用します。
* `useMock`: 名前付きのモックフィクスチャを使用します (内部/テスト用) 。
* `blockAds`: 広告と cookie のブロックを有効にします。ノイズを減らすために使用します。
* `proxy`: プロキシモード `"basic"`、`"stealth"`、`"enhanced"`、`"auto"`、またはカスタムプロキシ文字列。bot 対策サイトに使用します。
* `maxAge`: この経過時間 (ms) 未満のキャッシュ済みコンテンツを許容します。キャッシュヒットを優先するために使用します。
* `minAge`: 少なくともこの経過時間 (ms) 以上のキャッシュ済みコンテンツのみを返します。キャッシュのみの読み取りに使用します。
* `storeInCache`: 結果をキャッシュ/インデックスに保存します。再利用したい場合に使用します。
* `profile`: 永続的なブラウザ プロファイル `{ name, saveChanges }`。セッション状態を保持するために使用します。
* `integration`: 統合識別子。追跡/分析に使用します。
* `origin`: 任意の origin タグ。リクエスト元の識別に使用します。

<div id="interact">
  ## Interact
</div>

### なぜ使うのか

ページでスクレイピング後にクリックやフォーム入力、その他のブラウザアクションが必要な場合は、`interact` を使用します。

### 推奨のSDKメソッド

`client.interact(jobId, args)`

### 簡単な例

```ts
const run = await client.interact("scrape-job-id", {
  code: "console.log(await page.url())"
});

console.log(run.stdout);
```

### 高度な例

```ts
const run = await client.interact("scrape-job-id", {
  prompt: "Open the pricing page and list the plan names.",
  language: "node",
  timeout: 60,
  origin: "agent-docs"
});

console.log(run.output);
```

<div id="parameters">
  ### パラメータ
</div>

* `jobId`: スクレイピングジョブの ID (`document.metadata.scrapeId` から取得) 。必須です。
* `code`: ブラウザサンドボックスで実行するコード。スクリプトによる操作に使用します。
* `prompt`: ブラウザエージェントへの自然言語の指示。エージェント主導のアクションを使いたい場合に使用します。
* `language`: ランタイム言語 `"python"`、`"node"`、または `"bash"` (デフォルトは `"node"`) 。コードに合わせて使用します。
* `timeout`: 実行タイムアウト (秒単位) 。長時間の実行を制限するために使用します。
* `origin`: 任意の origin タグ。リクエストの送信元の識別に使用します。

<div id="notes">
  ## 注意
</div>

* JS/TS では、パラメータは camelCase です。
* 非推奨エイリアス: `scrapeExecute`, `stopInteractiveBrowser`, `deleteScrapeBrowser` (`interact` と `stopInteraction` を使用) 。
* `Firecrawl` は統合された v2 クライアントです。v1 は従来用途向けに `.v1` 配下で引き続き利用できます。

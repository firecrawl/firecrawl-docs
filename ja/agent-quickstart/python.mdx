<div id="firecrawl-python-agent-quickstart">
  # Firecrawl Pythonエージェント クイックスタート
</div>

このファイルは外部エージェント向けの標準的なクイックスタートで、SDK のソースと v2 OpenAPI 仕様から生成されています。

<div id="install">
  ## インストール
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## 認証
</div>

```python
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])
```

<div id="when-to-use-what">
  ## 使い分けの目安
</div>

* `search`: クエリから始めて、対象を見つけたい場合に使います。
* `scrape`: すでにURLがあり、ページのコンテンツが必要な場合に使います。
* `interact`: ページ上でのクリックやフォーム入力、またはスクレイピング後のブラウザアクションが必要な場合に使います。

<div id="search">
  ## 検索
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

`search` を使うと、クエリから関連するURLを見つけ、必要に応じて各結果をスクレイピングできます。標準の `site:` 演算子 (たとえば `site:docs.firecrawl.dev crawl webhooks`) を使えば、特定のURLをスクレイピングする前に、クエリの対象を単一のサイトに絞り込めます。

<div id="preferred-sdk-method">
  ### 推奨のSDKメソッド
</div>

`client.search(query, **options)`

### 簡単な例

```python
results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    limit=5,
)

print(results.web[0] if results.web else None)
```

### 複雑な例

```python
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "firecrawl pricing",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:w",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            {"type": "json", "prompt": "Extract pricing tiers and limits."},
        ],
        only_main_content=True,
        proxy="stealth",
    ),
    integration="agent-quickstart",
)
```

<div id="parameters">
  ### パラメータ
</div>

* `query`: 検索クエリ文字列。常に必須です。
* `sources`: 検索対象のソース。サポートされる値: `"web"`、`"news"`、`"images"`、または `{"type": ...}` オブジェクト。どの結果配列を返すかを制御するために使用します。
* `categories`: 結果の絞り込み。サポートされる値: `"github"`、`"research"`、`"pdf"`、または `{"type": ...}` オブジェクト。カテゴリで結果を絞り込むために使用します。
* `limit`: 返す結果の最大数。デフォルト: `5`。
* `tbs`: 時間ベースの検索フィルタ文字列。時間範囲で絞り込むために使用します (例: `qdr:d`) 。
* `location`: ローカライズされた結果のためのロケーション文字列。地域固有のクエリに使用します。
* `ignore_invalid_urls`: 結果から無効な URL を除外します。他のエンドポイントにパイプする場合に使用します。
* `timeout`: リクエストのタイムアウト (ミリ秒) 。デフォルト: `300000`。
* `scrape_options`: 各結果に適用されるスクレイピングのオプション (`scrape` と同じフィールド) 。結果と一緒にコンテンツも必要な場合に使用します。
* `integration`: 連携識別子。追跡 / 分析に使用します。

<div id="scrape">
  ## スクレイピング
</div>

<div id="why-use-it">
  ### これを使う理由
</div>

URL がすでにわかっており、特定のフォーマットでコンテンツを取得したい場合は、`scrape` を使います。

### 推奨SDKメソッド

`client.scrape(url, **options)`

<div id="simple-example">
  ### シンプルな例
</div>

```python
doc = client.scrape(
    "https://example.com",
    formats=["markdown"],
)

print(doc.markdown)
```

<div id="complex-example">
  ### 複雑な使用例
</div>

```python
doc = client.scrape(
    "https://example.com/docs",
    formats=[
        "markdown",
        {"type": "json", "prompt": "Extract headings and slugs."},
        {"type": "change_tracking", "modes": ["git-diff"], "tag": "docs"},
        {"type": "screenshot", "full_page": True, "viewport": {"width": 1280, "height": 720}},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
        {"type": "query", "prompt": "What is the pricing model?"},
    ],
    only_main_content=True,
    wait_for=1500,
    actions=[
        {"type": "click", "selector": "button#accept"},
        {"type": "scroll", "direction": "down"},
    ],
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    location={"country": "US", "languages": ["en-US"]},
    proxy="stealth",
    profile={"name": "docs-session", "save_changes": True},
    max_age=3_600_000,
)
```

<div id="parameters">
  ### パラメータ
</div>

* `url`: スクレイピング対象の URL。常に必須です。
* `formats`: 含める出力フォーマット。`ScrapeFormats` オブジェクト (フィールドは `markdown`, `html`, `raw_html`, `summary`, `links`, `images`, `screenshot`, `change_tracking`, `json`) またはフォーマット文字列/オブジェクトのリストを受け付けます。サポートされるフォーマット文字列は `markdown`, `html`, `rawHtml`, `raw_html`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `change_tracking`, `json`, `attributes`, `branding`, `query`, `audio` です。フォーマットオブジェクト: `json` `{type: "json", prompt?, schema?}` (`prompt` または `schema` が必要) 、`changeTracking` `{type: "changeTracking" | "change_tracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `attributes` `{type: "attributes", selectors: [{selector, attribute}]}`, `query` `{type: "query", prompt}`。
* `headers`: 追加の HTTP ヘッダー。認証、Cookie、または user-agent に使用します。
* `include_tags`: 指定した HTML タグのコンテンツのみを含めます。抽出対象を絞り込むために使用します。
* `exclude_tags`: 指定した HTML タグのコンテンツを除外します。ノイズを減らすために使用します。
* `only_main_content`: メインコンテンツのみを抽出します。ナビゲーションやフッターを除外するために使用します。
* `timeout`: スクレイピングのタイムアウト (ミリ秒) 。実行時間の上限を設定するために使用します。
* `wait_for`: スクレイピング前の待機時間 (ミリ秒) 。JS でレンダリングされるページに使用します。
* `mobile`: モバイルデバイスをエミュレートします。モバイル向けレイアウトに使用します。
* `parsers`: パーサーのリストまたは設定。サポートされる PDF 設定: `{type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: int}` または文字列エントリ (例: `"pdf"`) 。
* `actions`: スクレイピング前に実行するブラウザ アクション。サポートされるアクション: `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `click` `{type: "click", selector}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}`。フォーマットは `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger` に対応しています。
* `location`: ロケーション設定 `{country, languages}`。地域固有のページに使用します。
* `skip_tls_verification`: TLS 証明書の確認をスキップします。自己署名証明書に使用します。
* `remove_base64_images`: markdown 出力から base64 画像を削除します。ペイロードサイズを減らすために使用します。
* `fast_mode`: 高速モードを有効にします。速度を最優先する場合に使用します。
* `use_mock`: 名前付きのモックフィクスチャを使用します (内部/テスト用) 。
* `block_ads`: 広告と Cookie のブロックを有効にします。ノイズを減らすために使用します。
* `proxy`: プロキシモード `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`。
* `max_age`: この期間 (ms) 未満のキャッシュ済みコンテンツを受け入れます。キャッシュヒットを優先するために使用します。
* `store_in_cache`: 結果をキャッシュ/インデックスに保存します。再利用したい場合に使用します。
* `profile`: 永続的なブラウザ プロファイル `{name, save_changes}` (`saveChanges` も受け付けます) 。セッション状態を保持するために使用します。
* `integration`: 連携 識別子。追跡/分析に使用します。

<div id="interact">
  ## Interact
</div>

### 使用する理由

ページで、スクレイピング後にクリックやフォーム入力などのブラウザ操作が必要な場合は、`interact` を使います。

<div id="preferred-sdk-method">
  ### 推奨のSDKメソッド
</div>

`client.interact(job_id, code=None, *, prompt=None, **options)`

<div id="simple-example">
  ### シンプルな例
</div>

```python
run = client.interact(
    "scrape-job-id",
    code="print(await page.url())",
)

print(run.stdout)
```

### 複雑な例

```python
run = client.interact(
    "scrape-job-id",
    prompt="Open the pricing page and list the plan names.",
    language="python",
    timeout=60,
    origin="agent-docs",
)

print(run.output)
```

<div id="parameters">
  ### パラメータ
</div>

* `job_id`: スクレイピングジョブ ID (`document.metadata_typed.scrape_id` から取得) 。常に必須です。
* `code`: ブラウザサンドボックスで実行するコード。スクリプトによる操作に使用します。
* `prompt`: ブラウザエージェントへの自然言語による指示。エージェントによるアクションを使いたい場合に使用します。
* `language`: ランタイム言語 `"python"`、`"node"`、または `"bash"` (デフォルトは `"node"`) 。コードに合わせて使用します。
* `timeout`: 実行タイムアウト (秒) 。長時間実行を制限するために使用します。
* `origin`: 任意のオリジンタグ。リクエスト元の識別に使用します。

<div id="notes">
  ## 注意
</div>

* Python ではパラメータ名は snake&#95;case ですが、SDK はペイロードを API の camelCase に正規化します。
* 非推奨のエイリアス: `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (`interact` と `stop_interaction` を使用) 。
* `Firecrawl` は統合された v2 クライアントです。v1 もレガシー用途向けに `.v1` で引き続き利用できます。

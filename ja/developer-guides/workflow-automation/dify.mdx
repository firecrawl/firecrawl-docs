<div id="dify-integration-overview">
  ## Dify 連携の概要
</div>

Dify はオープンソースの LLM アプリ開発プラットフォームです。公式の Firecrawl プラグインにより、AI ワークフロー内でウェブのクロールとスクレイピングを直接実行できます。

  </Card>
</CardGroup>

<div id="firecrawl-tools-in-dify">
  ## Dify における Firecrawl ツール
</div>

<AccordionGroup>
  <Accordion title="Scrape" icon="file-code">

任意の URL をクリーンで構造化されたデータに変換。生の HTML を実用的なインサイトに変換します。

**ユースケース:** 製品データの抽出、記事コンテンツのスクレイピング、JSONモードでの構造化データ取得。

  </Accordion>

  <Accordion title="Crawl" icon="spider">

Web サイトやサブドメインを再帰的にクロールして、広範なコンテンツを収集します。

**ユースケース:** サイト全体のコンテンツ抽出、ドキュメントのスクレイピング、複数ページにわたるデータ収集。

  </Accordion>

  <Accordion title="Map" icon="sitemap">

Web サイト上のすべての URL の完全なマップを生成します。

**ユースケース:** サイト構造の分析、SEO 監査、バッチスクレイピングのための URL 発見。

  </Accordion>

  <Accordion title="Crawl Job" icon="list-check">

ジョブ ID に基づいてスクレイピング結果を取得、または進行中のタスクをキャンセルします。

**ユースケース:** 長時間実行のクロールの監視、非同期スクレイピングのワークフロー管理、必要に応じた操作のキャンセル。

  </Accordion>
</AccordionGroup>

<div id="getting-started">
  ## はじめに
</div>

<Steps>
  <Step title="Firecrawl プラグインをインストール">
    [Dify Plugin Marketplace](https://marketplace.dify.ai/plugins/langgenius/firecrawl) にアクセスし、Firecrawl ツールをインストールします
  </Step>

  <Step title="Firecrawl の API キーを取得">
    [Firecrawl API Keys](https://www.firecrawl.dev/app/api-keys) にアクセスして、新しい API キーを作成します
  </Step>

  <Step title="Dify で認証">
    **Plugins > Firecrawl > To Authorize** に移動し、API キーを入力します
  </Step>

  <Step title="ワークフローに追加">
    Firecrawl ツールを Chatflow、Workflow、または Agent アプリケーションにドラッグ＆ドロップします
  </Step>

  <Step title="設定とテスト">
    パラメーターを設定し、ワークフローをテストします
  </Step>
</Steps>

<div id="usage-patterns">
  ## 利用パターン
</div>

<Tabs>
  <Tab title="Chatflow Apps">
**ビジュアルパイプライン統合**

1. パイプラインに Firecrawl ノードを追加
2. アクションを選択（Map、Crawl、Scrape）
3. 入力変数を定義
4. パイプラインを順次実行

**フロー例:**
```
User Input → Firecrawl (Scrape) → LLM Processing → Response
```
  </Tab>

  <Tab title="Workflow Apps">
**自動データ処理**

次の機能で複数段階のワークフローを構築:
- 定期スクレイピング
- データ変換
- データベース保存
- 通知

**フロー例:**
```
Schedule Trigger → Firecrawl (Crawl) → Data Processing → Storage
```
  </Tab>

  <Tab title="Agent Apps">
**AIによるウェブアクセス**

エージェントにリアルタイムのウェブスクレイピング機能を付与:

1. Agent に Firecrawl ツールを追加
2. エージェントがスクレイプ実行のタイミングを自律的に判断
3. LLM が抽出コンテンツを解析
4. エージェントが根拠に基づく応答を提供

**ユースケース:** ライブドキュメントを参照するカスタマーサポートエージェント
  </Tab>
</Tabs>

<div id="firecrawl-actions">
  ## Firecrawl アクション
</div>

| ツール | 説明 | 最適な用途 |
|------|-------------|----------|
| **Scrape** | 単一ページのデータ抽出 | コンテンツの素早い取得 |
| **Crawl** | 複数ページの再帰的クロール | サイト全体の抽出 |
| **Map** | URL の発見とサイトマップ化 | SEO 分析、URL リスト作成 |
| **Crawl Job** | 非同期ジョブ管理 | 長時間実行タスク |

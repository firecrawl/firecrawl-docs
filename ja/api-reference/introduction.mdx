<div id="base-url">
  ## ベースURL
</div>

すべてのリクエストは次のベースURLを使用します：

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## 認証
</div>

認証には Authorization ヘッダーを含める必要があります。ヘッダーには `Bearer fc-123456789` を指定し、`fc-123456789` はあなたの API キーを指します。

```bash
Authorization: Bearer fc-123456789
```

​

<div id="response-codes">
  ## レスポンスコード
</div>

Firecrawl は、リクエストの結果を示すために一般的な HTTP ステータスコードを使用します。

通常、2xx は成功、4xx はユーザー起因の失敗、5xx はインフラストラクチャ上の問題を示します。

Status | Description
--- | ---
200 | リクエストは成功しました。
400 | パラメータが正しいか確認してください。
401 | API キーが提供されていません。
402 | 支払いが必要です。
404 | 要求されたリソースが見つかりませんでした。
429 | レート制限を超過しました。
5xx | Firecrawl によるサーバーエラーを示します。

発生しうるすべての API エラーの詳細は、Error Codes セクションを参照してください。

<div id="firecrawl-error-codes-5xx">
  ### Firecrawl のエラーコード（5xx）
</div>

5xx エラーが発生した場合、Firecrawl は問題の内容を明確にするため、より具体的なエラーコードを返します。​

Error Code | Status | Description\
--- | --- | ---\
`SCRAPE_ALL_ENGINES_FAILED` | 500 | すべてのスクレイピングエンジンが失敗しました。\
`SCRAPE_SSL_ERROR` | 500 | ページの SSL 証明書が無効です。`skipTlsVerification:true` を使用してこの検証をスキップできます。\
`SCRAPE_SITE_ERROR` | 500 | 回復不能なサイトエラー。\
`SCRAPE_DNS_RESOLUTION_ERROR` | 500 | DNS 解決に失敗しました。\
`SCRAPE_ACTION_ERROR` | 500 | ページのアクション実行中にエラーが発生しました。\
`SCRAPE_PDF_PREFETCH_FAILED` | 500 | PDF のプリフェッチに失敗しました。\
`SCRAPE_PDF_INSUFFICIENT_TIME_ERROR` | 500 | PDF の処理時間が不足しています。\
`SCRAPE_PDF_ANTIBOT_ERROR` | 500 | ボット対策機構により PDF がブロックされました。\
`SCRAPE_ZDR_VIOLATION_ERROR` | 500 | Zero Data Retention の衝突：`zeroDataRetention:true` と指定されている一方で、`screenshot` など一時保存を要する別のオプションが指定されています。\
`SCRAPE_UNSUPPORTED_FILE_ERROR` | 500 | サポートされていないファイル形式です。\
`UNKNOWN_ERROR` | 500 | 一般的または予期しないエラー。

<div id="rate-limit">
  ## レート制限
</div>

Firecrawl API には、サービスの安定性と信頼性を確保するためのレート制限があります。レート制限はすべてのエンドポイントに適用され、特定の時間内に行われるリクエスト数に基づいています。

レート制限を超過すると、レスポンスコード 429 が返されます。

---
title: "エージェントのサインアップをブロック"
description: "エージェントによってプロビジョニングされた API キーをブロックして無効化します。"
---

保留中のエージェントのサインアップをブロックし、API キーを即座に無効化するとともに、サンドボックス化されたアカウントを利用禁止にします。エージェントが許可なくキーを作成した場合に使用します。認証は不要です — 検証トークンがその証明として機能します。

<div id="headers">
  ## ヘッダー
</div>

| ヘッダー | 値 |
|--------|-------|
| `Content-Type` | `application/json` |

<div id="request-body">
  ## リクエストボディ
</div>

| パラメータ                | 型      | 必須     | 説明                     |
| -------------------- | ------ | ------ | ---------------------- |
| `agent_signup_token` | string | **はい** | 確認メール内のリンクに含まれる検証トークン。 |

<div id="response">
  ## レスポンス
</div>

| Field | 型 | 説明 |
|-------|------|-------------|
| `success` | boolean | ブロックが正常に実行されたかどうか |
| `message` | string | 人が読める形式の結果メッセージ |

<div id="example-request">
  ### リクエスト例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/agent-signup/block" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_signup_token": "a1b2c3d4..."
  }'
```

<div id="example-response">
  ### レスポンス例
</div>

```json
{
  "success": true,
  "message": "Agent key has been blocked and disabled."
}
```

<div id="what-happens-on-block">
  ## ブロック時に起こること
</div>

- エージェントの API キーは **削除** されます。
- サンドボックス化されたチームは **禁止** されます。
- sponsor レコードは `"blocked"` としてマークされ、その email では今後エージェントのサインアップができなくなります。
- ブロックされたキーを使用したそれ以降の API リクエストは `403` を返します。

<div id="error-responses">
  ## エラーレスポンス
</div>

| ステータス | エラー | 発生条件 |
|--------|-------|------|
| `400` | 無効なリクエスト | `agent_signup_token` がありません |
| `404` | 無効な検証トークン | トークンが見つかりません |
| `409` | このエージェント キーはすでに確認済みです | すでに確認済みのキーはブロックできません |

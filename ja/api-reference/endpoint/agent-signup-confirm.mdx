---
title: "Agent 登録の確認"
description: "Agent によってプロビジョニングされた API キーを確認し、スポンサーのアカウントに紐付けます。"
---

保留中の Agent 登録を確認します。スポンサーが既存の Firecrawl アカウントを持っている場合、そのキーはそのアカウントに統合されます。持っていない場合は、サンドボックス化されたアカウントがスポンサーのメールアドレスのもとで完全なアカウントに昇格されます。認証は不要で、検証トークンが証明として機能します。

<div id="headers">
  ## ヘッダー
</div>

| ヘッダー           | 値                  |
| -------------- | ------------------ |
| `Content-Type` | `application/json` |

<div id="request-body">
  ## リクエストボディ
</div>

| パラメータ                | 型      | 必須     | 説明                       |
| -------------------- | ------ | ------ | ------------------------ |
| `agent_signup_token` | string | **はい** | 確認メール内のリンクに含まれる検証トークンです。 |

<div id="response">
  ## レスポンス
</div>

| フィールド     | 型       | 説明            |
| --------- | ------- | ------------- |
| `success` | boolean | 確認が成功したかどうか   |
| `message` | string  | 可読性のある結果メッセージ |

<div id="example-request">
  ### リクエスト例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/agent-signup/confirm" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_signup_token": "a1b2c3d4..."
  }'
```


<div id="example-response">
  ### レスポンス例
</div>

```json
{
  "success": true,
  "message": "Agent key confirmed and linked to your account."
}
```

<div id="what-happens-on-confirm">
  ## 確認時に起こること
</div>

- **既存のアカウント**: agent の API キーはスポンサーのチームに移動されます。サンドボックス化された account は無効化されます。
- **アカウントがない場合**: サンドボックス化された account のメールアドレスがスポンサーのメールアドレスに更新され、その人の実際のアカウントになります。
- どちらの場合も、50-credit の上限は解除され、キーはスポンサーの完全なプランを引き継ぎます。

<div id="error-responses">
  ## エラーレスポンス
</div>

| ステータス | エラー | 条件 |
|--------|-------|------|
| `400` | 無効なリクエスト | `agent_signup_token` がありません |
| `403` | 検証期限が過ぎています | 5日間の確認期間が終了しました |
| `403` | この Agent 登録はブロックされています | スポンサーがこのキーをすでにブロックしています |
| `404` | 無効または期限切れの検証トークン | トークンが見つからないか、すでに使用されています |

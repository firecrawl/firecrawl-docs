<div id="base-url">
  ## ベースURL
</div>

すべてのリクエストは次のベースURLを使用します：

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## 認証
</div>

認証には Authorization ヘッダーが必要です。ヘッダーには `Bearer fc-123456789` を指定してください。`fc-123456789` はあなたの API キーです。

```bash
Authorization: Bearer fc-123456789
```

​

<div id="response-codes">
  ## レスポンスコード
</div>

Firecrawl は、リクエスト結果を示すために標準的な HTTP ステータスコードを使用します。

一般に、2xx は成功、4xx はユーザー起因のエラー、5xx はインフラ側の問題を示します。

Status | Description
--- | ---
200 | リクエストは成功しました。
400 | パラメータの正当性を確認してください。
401 | API キーが指定されていません。
402 | 支払いが必要です。
404 | リクエストされたリソースが見つかりませんでした。
429 | レート制限を超えました。
5xx | Firecrawl のサーバーエラーを示します。

考えられるすべての API エラーの詳細は、Error Codes セクションを参照してください。

<div id="rate-limit">
  ## レート制限
</div>

Firecrawl API には、サービスの安定性と信頼性を確保するためのレート制限があります。レート制限はすべてのエンドポイントに適用され、一定時間内のリクエスト数に基づいています。

レート制限を超えると、ステータスコード 429 が返されます。

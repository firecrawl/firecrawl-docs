<InstallGoSDK />

<div id="usage">
  ## 使い方
</div>

1. [firecrawl.dev](https://firecrawl.dev) から API キーを取得します。
2. `FirecrawlApp` 構造体のパラメータとして `API key` を設定します。
3. `API URL` を設定するか、`FirecrawlApp` 構造体にパラメータとして渡します。デフォルトは `https://api.firecrawl.dev` です。
4. `version` を設定するか、`FirecrawlApp` 構造体にパラメータとして渡します。デフォルトは `v1` です。

<div id="scraping-a-url">
  ### URLのスクレイピング
</div>

エラーハンドリング付きで単一のURLをスクレイピングするには、`ScrapeURL` メソッドを使用します。URLを引数に取り、スクレイピング結果をディクショナリ（連想配列）として返します。

<ScrapeGoShort />

<div id="crawling-a-website">
  ### ウェブサイトのクロール
</div>

ウェブサイトをクロールするには、`CrawlUrl` メソッドを使用します。開始 URL と任意のパラメータを引数に取ります。`params` 引数では、クロールする最大ページ数、許可ドメイン、出力フォーマットなど、クロールジョブの追加オプションを指定できます。

<CrawlGoShort />

<div id="checking-crawl-status">
  ### クロールのステータスを確認する
</div>

クロールジョブのステータスを確認するには、`CheckCrawlStatus` メソッドを使用します。ジョブIDを引数に取り、クロールジョブの現在のステータスを返します。

<CheckCrawlStatusGoShort />

<div id="map-a-website">
  ### ウェブサイトをマッピングする
</div>

`MapUrl` を使って、ウェブサイトの URL 一覧を生成します。`params` 引数で、サブドメインの除外やサイトマップの利用など、マッピング処理をカスタマイズできます。

<MapGoShort />

<div id="error-handling">
  ## エラー処理
</div>

SDK は Firecrawl API から返されるエラーを処理し、適切な例外をスローします。リクエスト中にエラーが発生した場合は、わかりやすいエラーメッセージ付きの例外がスローされます。

</div>

<div id="scraping-a-url">
  ### URLをスクレイピングする
</div>

単一のURLをスクレイピングするには、`scrape_url` メソッドを使用します。URLを引数に取り、スクレイプしたデータを `Document` として返します。

<ScrapeRustShort />

<div id="scraping-with-extract">
  ### Extract を使ったスクレイピング
</div>

Extract を使うと、任意の URL から構造化データを簡単に抽出できます。スキーマは `serde_json::json!` マクロを使い、JSON Schema 形式で指定します。

<LLMExtractRust />

<div id="crawling-a-website">
  ### ウェブサイトのクロール
</div>

ウェブサイトをクロールするには、`crawl_url` メソッドを使用します。クロールが完了するまで待機し、開始URLや指定したオプションによっては時間がかかる場合があります。

<CrawlRustShort />

<div id="crawling-asynchronously">
  #### 非同期でクロールする
</div>

結果を待たずにクロールするには、`crawl_url_async` メソッドを使用します。引数は同じですが、クロールの ID を含む `CrawlAsyncRespone` 構造体を返します。その ID を `check_crawl_status` メソッドで使用すると、いつでもステータスを確認できます。完了したクロールは 24 時間後に削除されます。

<div id="error-handling">
  ## エラーハンドリング
</div>

SDK は、Firecrawl API や依存ライブラリから返されるエラーを受け取り、`Error`、`Debug`、`Display` を実装する `FirecrawlError` 列挙型に集約します。すべてのメソッドは `Result<T, FirecrawlError>` を返します。

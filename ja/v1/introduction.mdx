<div id="welcome-to-firecrawl">
  ## Firecrawl へようこそ
</div>

[Firecrawl](https://firecrawl.dev?ref=github) は、URL を受け取ってクロールし、クリーンな Markdown に変換する API サービスです。アクセス可能なサブページをすべてクロールし、各ページごとにクリーンな Markdown を返します。サイトマップは不要です。

<div id="how-to-use-it">
  ## どう使いますか？
</div>

ホスト版の使いやすいAPIをご用意しています。プレイグラウンドとドキュメントは[こちら](https://firecrawl.dev/playground)をご覧ください。必要に応じてバックエンドをセルフホストすることも可能です。

<div id="features">
  ### 機能
</div>

* [**Scrape**](#scraping): URL をスクレイピングし、LLM 向けのフォーマット（markdown、summary、[JSONモード](#json-mode)による構造化データ、screenshot、html）でコンテンツを取得
* [**Crawl**](#crawling): ウェブページ上のすべての URL をスクレイピングし、LLM 向けのフォーマットでコンテンツを返す
* [**Map**](/ja/features/map): ウェブサイトを指定して、そのサイトのすべての URL を高速に取得
* [**Search**](/ja/features/search): ウェブを検索し、結果ページから全文コンテンツを取得
* [**Extract**](/ja/features/extract): 単一ページ、複数ページ、またはサイト全体から AI で構造化データを取得

<div id="powerful-capabilities">
  ### 強力な機能
</div>

- **LLM対応フォーマット**: markdown、summary、構造化データ、スクリーンショット、HTML、リンク、メタデータ
- **難易度の高い処理**: プロキシ、ボット対策、動的コンテンツ（JSレンダリング）、出力のパース、オーケストレーション
- **カスタマイズ性**: タグの除外、カスタムヘッダーのサポート、最大クロール深度など
- **メディア解析**: PDF、DOCX、画像
- **信頼性重視**: どれほど難しくても必要なデータを取得できるよう設計
- **アクション**: データ抽出前にクリック、スクロール、入力、待機などを実行

Firecrawlのすべての機能と使い方は[ドキュメント](https://docs.firecrawl.dev/api-reference/v2-introduction)をご覧ください。

<div id="open-source-vs-cloud">
  ## オープンソース版とクラウド版
</div>

Firecrawl は [AGPL-3.0 ライセンス](https://github.com/firecrawl/firecrawl/blob/main/LICENSE) の下で利用可能なオープンソースです。

最高のプロダクトを提供するため、オープンソース版に加えてホスティングされた Firecrawl も提供しています。クラウド版により、すべてのユーザーに対して継続的な改良と高品質で持続可能なサービスを実現できます。

Firecrawl Cloud は [firecrawl.dev](https://firecrawl.dev) で提供しており、オープンソース版にはないさまざまな機能を備えています:

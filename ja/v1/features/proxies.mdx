Firecrawl は、ボット対策の強度が異なるウェブサイトをスクレイピングできるよう、複数のプロキシタイプを提供しています。プロキシタイプは `proxy` パラメータで指定できます。

> 既定では、プロキシタイプやロケーションを指定しない場合でも、信頼性とアクセス性を確保するために、Firecrawl はすべてのリクエストをプロキシ経由でルーティングします。

<div id="location-based-proxy-selection">
  ## ロケーションに基づくプロキシ選択
</div>

Firecrawlは、指定または検出されたロケーションに基づいて最適なプロキシを自動で選択します。これにより、スクレイピングのパフォーマンスと信頼性が向上します。ただし、現時点ではすべてのロケーションがサポートされているわけではありません。以下のロケーションが利用可能です:

| Country Code | Country Name         | Stealth Mode Support |
|--------------|----------------------|----------------------|
| AE           | United Arab Emirates | No                   |
| AU           | Australia            | No                   |
| BR           | Brazil               | Yes                  |
| CN           | China                | No                   |
| DE           | Germany              | Yes                  |
| FR           | France               | Yes                  |
| GB           | United Kingdom       | No                   |
| JP           | Japan                | No                   |
| QA           | Qatar                | No                   |
| TR           | Turkey               | No                   |
| US           | United States        | Yes                  |
| VN           | Vietnam              | No                   |

<Warning>サポート対象のプロキシロケーション一覧は2025年5月15日に最終更新されました。利用可能状況は今後変更される場合があります。</Warning>

上記にないロケーションでプロキシが必要な場合は、[contact us](mailto:help@firecrawl.com) まで要件をお知らせください。

プロキシやロケーションを指定しない場合、Firecrawlは対象サイトとリクエスト内容に基づいて最適なオプションを自動的に選択します。

<div id="proxy-types">
  ## プロキシタイプ
</div>

Firecrawl は 3 種類のプロキシをサポートしています：

* **basic**: 反ボット対策がない、または基本的な対策のみのサイト向けのプロキシ。高速で、たいていは動作します。
* **stealth**: 高度な反ボット対策があるサイト、または通常のプロキシをブロックするサイト向けのステルスプロキシ。速度は遅いものの、特定のサイトではより信頼性があります。[ステルスモードの詳細はこちら →](/ja/features/stealth-mode)
* **auto**: basic のプロキシが失敗した場合、Firecrawl が自動的にステルスプロキシでのスクレイピングを再試行します。ステルスでの再試行が成功した場合、そのスクレイプには 5 クレジットが請求されます。最初の basic の試行が成功した場合は、通常のコストのみが請求されます。

***

> 注: クレジットのコストやリトライ戦略を含むステルスプロキシの詳細については、[ステルスモードのドキュメント](/ja/features/stealth-mode)を参照してください。

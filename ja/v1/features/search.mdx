Firecrawlの検索APIを使うと、ウェブ検索を実行し、必要に応じて検索結果を同一の操作でスクレイピングできます。

* 出力フォーマットを選択（markdown、HTML、links、screenshots）
* 位置情報などのパラメータを指定してウェブ検索
* 検索結果のページ内容を、必要に応じて各種フォーマットで取得
* 取得件数の制御やタイムアウトの設定が可能

詳細は、[Search Endpoint API Reference](https://docs.firecrawl.dev/api-reference/endpoint/search)を参照してください。

一般的な `tbs` の値:

- `qdr:h` - 過去1時間
- `qdr:d` - 過去24時間
- `qdr:w` - 過去1週間
- `qdr:m` - 過去1か月
- `qdr:y` - 過去1年

より細かな期間指定が必要な場合は、カスタム日付範囲のフォーマットで厳密な日付範囲を指定できます。

<CodeGroup>

```python Python
from firecrawl import FirecrawlApp

# APIキーでクライアントを初期化
app = FirecrawlApp(api_key="fc-YOUR_API_KEY")

# 2024年12月の結果を検索
search_result = app.search(
    "firecrawl updates",
    limit=10,
    tbs="cdr:1,cd_min:12/1/2024,cd_max:12/31/2024"
)
```

```js JavaScript
import FirecrawlApp from '@mendable/firecrawl-js';

// APIキーでクライアントを初期化
const app = new FirecrawlApp({apiKey: "fc-YOUR_API_KEY"});

// 2024年12月の結果を検索
app.search("firecrawl updates", {
  limit: 10,
  tbs: "cdr:1,cd_min:12/1/2024,cd_max:12/31/2024"
})
.then(searchResult => {
  console.log(searchResult.data);
});
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v1/search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR_API_KEY" \
  -d '{
    "query": "firecrawl updates",
    "limit": 10,
    "tbs": "cdr:1,cd_min:12/1/2024,cd_max:12/31/2024"
  }'
```

</CodeGroup>

<div id="scraping-options">
  ## スクレイピングオプション
</div>

検索結果をスクレイピングする際は、複数の出力フォーマットや高度なスクレイピング設定を指定できます。

<CodeGroup>

```python Python
from firecrawl import FirecrawlApp, ScrapeOptions

# APIキーでクライアントを初期化
app = FirecrawlApp(api_key="fc-YOUR_API_KEY")

# 複数フォーマットで検索結果を取得
search_result = app.search(
    "firecrawl features",
    limit=3,
    scrape_options=ScrapeOptions(formats=["markdown", "html", "links", "screenshot"])
)
```

```js JavaScript
import FirecrawlApp from '@mendable/firecrawl-js';

// APIキーでクライアントを初期化
const app = new FirecrawlApp({apiKey: "fc-YOUR_API_KEY"});

// 複数フォーマットで検索結果を取得
app.search("firecrawl features", {
  limit: 3,
  scrapeOptions: {
    formats: ["markdown", "html", "links", "screenshot"]
  }
})
.then(searchResult => {
  // 各フォーマットの結果を処理
  console.log(searchResult.data);
});
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v1/search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR_API_KEY" \
  -d '{
    "query": "firecrawl features",
    "limit": 3,
    "scrapeOptions": {
      "formats": ["markdown", "html", "links", "screenshot"]
    }
  }'
```

</CodeGroup>

利用可能なフォーマット:

- `markdown`: クリーンに整形されたMarkdownコンテンツ
- `html`: 処理済みのHTMLコンテンツ
- `rawHtml`: 未加工のHTMLコンテンツ
- `links`: ページ内で検出されたリンク一覧
- `screenshot`: ページのスクリーンショット
- `screenshot@fullPage`: ページ全体のスクリーンショット
- `extract`: 構造化データの抽出

<div id="cost-implications">
  ## コストへの影響
</div>

検索結果をスクレイプしない場合（スクレイプオプション未指定）、コストは検索結果10件あたり2クレジットです。スクレイピングを有効にした場合でも、標準的なスクレイピング費用に加えて各検索結果の基本スクレイプに対する追加料金は発生しません。

ただし、以下のコスト要因に注意してください:

- **PDF解析**: PDFの1ページあたり1クレジット（複数ページのPDFではコストが大幅に増加する可能性があります）
- **ステルスプロキシモード**: 検索結果1件あたり+4クレジット
- ***JSONモード***: 検索結果1件あたり+4クレジット

コストを抑えるには:

- PDFコンテンツが不要な場合は `parsePDF: false` を設定する
- 可能な限り `"stealth"` の代わりに `proxy: "basic"` を使用する
- `limit` パラメータで検索結果数を制限する

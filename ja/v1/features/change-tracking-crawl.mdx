変更追跡はクロールと組み合わせることで、さらに強力になります。個々のページでの変更追跡がコンテンツの変化を示すのに対し、クロールと併用すればサイト全体の構造を監視でき、新規ページ、削除されたページ、非表示になったページを把握できます。

<div id="basic-usage">
  ## 基本的な使い方
</div>

クロール中に changeTracking を有効にするには、`scrapeOptions` の `formats` 配列に `changeTracking` を追加してください。

```typescript
// JavaScript/TypeScript
const app = new FirecrawlApp({ apiKey: 'your-api-key' });
const result = await app.crawl('https://example.com', {
  scrapeOptions: {
    formats: ['markdown', 'changeTracking']
  }
});
```

```python
# Python
app = FirecrawlApp(api_key='your-api-key')
result = app.crawl('https://firecrawl.dev', {
    'scrapeOptions': {
        'formats': ['markdown', 'changeTracking']
    }
})
```

<div id="understanding-change-status">
  ## 変更ステータスを理解する
</div>

/​crawl で changeTracking を使用する場合、`changeStatus` フィールドが特に有用です:

* `new`: 前回の /​crawl には存在しなかったページ
* `same`: 存在しており、前回の /​crawl 以降に変更されていないページ
* `changed`: 存在しているが、前回の /​crawl 以降に変更されたページ
* `removed`: 前回の /​crawl には存在したが、現在は見つからないページ

<div id="page-visibility">
  ## ページの可視性
</div>

`visibility` フィールドは、ページがどのように発見されるかを把握するのに役立ちます:

* `visible`: ページはリンクまたはサイトマップ経由で発見可能です
* `hidden`: ページ自体は存在しますが、リンクもされておらずサイトマップにも含まれていません

これは次の用途に特に有用です:

* 孤立（オーファン）コンテンツの検出
* ナビゲーションから誤って外されたページの特定
* サイト構造の変更の監視
* 再リンクすべき、または削除すべきコンテンツの洗い出し

<div id="full-diff-support">
  ## 差分の完全対応
</div>

差分を含む詳細な変更追跡には、[Change Tracking for Scrape](/ja/features/change-tracking) のドキュメントで説明されているのと同じオプションを使用できます。

<div id="scrape-endpoint">
  ## /scrape エンドポイント
</div>

更新版の `/scrape` エンドポイントは、信頼性と使いやすさを高めるために再設計されています。新しい `/scrape` のリクエストボディ構造は次のとおりです：

```json
{
  "url": "<string>",
  "formats": ["markdown", "html", "rawHtml", "links", "screenshot", "json"],
  "includeTags": ["<string>"],
  "excludeTags": ["<string>"],
  "headers": { "<key>": "<value>" },
  "waitFor": 123,
  "timeout": 123
}
```

<div id="formats">
  ### フォーマット
</div>

出力のフォーマットを選択できます。複数の出力フォーマットを指定可能です。サポートされているフォーマットは次のとおりです:

* Markdown（markdown）
* HTML（html）
* Raw HTML（rawHtml）（無加工）
* Screenshot（screenshot または screenshot@fullPage）
* Links（links）
* JSON（json）
  デフォルトでは、出力には markdown フォーマットのみが含まれます。

<div id="details-on-the-new-request-body">
  ### 新しいリクエストボディの詳細
</div>

以下の表は、V1 の `/scrape` エンドポイントにおけるリクエストボディのパラメータ変更点を示します。

| Parameter | Change | Description |
| --------- | ------ | ----------- |
| `onlyIncludeTags` | Moved and Renamed | ルートレベルに移動し、`includeTags` に改名。 |
| `removeTags` | Moved and Renamed | ルートレベルに移動し、`excludeTags` に改名。 |
| `onlyMainContent`| Moved | ルートレベルに移動。既定値は `true`。 |
| `waitFor`| Moved | ルートレベルに移動。 |
| `headers`| Moved | ルートレベルに移動。 |
| `parsePDF`| Moved | ルートレベルに移動。 |
| `extractorOptions`| No Change ||
| `timeout`| No Change ||
| `pageOptions` | Removed | `pageOptions` パラメータは不要。スクレイプオプションはルートレベルに移動。 |
| `replaceAllPathsWithAbsolutePaths` | Removed | `replaceAllPathsWithAbsolutePaths` は不要になりました。すべてのパスは既定で絶対パスになります。 |
| `includeHtml`| Removed | 代わりに `formats` に `"html"` を追加。 |
| `includeRawHtml`| Removed | 代わりに `formats` に `"rawHtml"` を追加。 |
| `screenshot`| Removed | 代わりに `formats` に `"screenshot"` を追加。 |
| `fullPageScreenshot`| Removed | 代わりに `formats` に `"screenshot@fullPage"` を追加。 |
| `extractorOptions` | Removed | 代わりに `"json"` フォーマットと `jsonOptions` オブジェクトを使用。 |

新しい `json` フォーマットについては、[llm-extract](/ja/features/extract) セクションを参照してください。

<div id="crawl-endpoint">
  ## /crawl endpoint
</div>

`v1` の `/crawl` エンドポイントも更新しました。改良版のリクエストボディを以下でご確認ください:

```json
{
  "url": "<string>",
  "excludePaths": ["<string>"],
  "includePaths": ["<string>"],
  "maxDepth": 2,
  "ignoreSitemap": true,
  "limit": 10,
  "allowBackwardLinks": true,
  "allowExternalLinks": true,
  "scrapeOptions": {
    // /scrape と同一のオプション
    "formats": ["markdown", "html", "rawHtml", "screenshot", "links"]
    "headers": { "<key>": "<value>" },
    "includeTags": ["<string>"],
    "excludeTags": ["<string>"],
    "onlyMainContent": true,
    "waitFor": 123
  }
}
```

<div id="details-on-the-new-request-body">
  ### 新しいリクエストボディの詳細
</div>

以下の表は、V1 の `/crawl` エンドポイントにおけるリクエストボディのパラメータ変更点を示します。

| Parameter | Change | Description |
| --------- | ------ | ----------- |
| `pageOptions` | Renamed | `scrapeOptions` に名称変更。 |
| `includes` | Moved and Renamed | ルートレベルに移動し、`includePaths` に名称変更。 |
| `excludes` | Moved and Renamed | ルートレベルに移動し、`excludePaths` に名称変更。 |
| `allowBackwardCrawling` | Moved and Renamed | ルートレベルに移動し、`allowBackwardLinks` に名称変更。 |
| `allowExternalLinks` | Moved | ルートレベルに移動。 |
| `maxDepth` | Moved | ルートレベルに移動。 |
| `ignoreSitemap` | Moved | ルートレベルに移動。 |
| `limit` | Moved | ルートレベルに移動。 |
| `crawlerOptions` | Removed | `crawlerOptions` パラメータは不要。クロールオプションはルートレベルへ移動。 |
| `timeout` | Removed | 代わりに `scrapeOptions` の `timeout` を使用。 |

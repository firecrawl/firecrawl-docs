<div id="overview">
  ## 概要
</div>

Firecrawl APIキーをサーバー側で保持するため、Next.js の ルートハンドラー から呼び出します。

<div id="prerequisites">
  ## 前提条件
</div>

* FirecrawlのAPIキー
* Next.js アプリ (App Router)
* Node.js 18+

<div id="install">
  ## インストール
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="configure-environment">
  ## 環境を設定する
</div>

APIキーを`.env.local`に追加し、サーバー側でのみ利用できるようにします。

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## サーバールートを追加する
</div>

app/api/firecrawl/route.ts

```ts
import Firecrawl from '@mendable/firecrawl-js';

export async function POST(request: Request) {
  const { url } = await request.json();
  const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

  const doc = await client.scrape(url, { formats: ['markdown'] });
  return Response.json({ markdown: doc.markdown });
}
```

<div id="call-it-from-your-app">
  ## アプリから呼び出す
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

<div id="notes">
  ## 注意事項
</div>

* APIキーを公開しないため、Firecrawl の呼び出しはサーバー側のルートからのみ行ってください。
* Pages Router を使用している場合は、代わりに `pages/api/firecrawl.ts` を作成してください。

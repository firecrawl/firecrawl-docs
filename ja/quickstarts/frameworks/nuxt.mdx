<div id="overview">
  ## 概要
</div>

Firecrawl APIキーを非公開のまま保持するため、Nuxt のサーバーAPIルートを使用します。

<div id="prerequisites">
  ## 前提条件
</div>

* Firecrawl APIキー
* Nuxt 3アプリ
* Node.js 18+

<div id="install">
  ## インストール
</div>

追加のパッケージは不要です。Nuxt のサーバールートから `fetch` を直接使用できます。

<div id="configure-environment">
  ## 環境を設定する
</div>

`.env` に `APIキー` を追加し、ランタイム設定で利用できるようにします。

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    firecrawlApiKey: process.env.FIRECRAWL_API_KEY,
  },
});
```

<div id="add-a-server-route">
  ## サーバールートを追加
</div>

server/api/firecrawl.post.ts

```ts
export default defineEventHandler(async (event) => {
  const { url } = await readBody(event);
  const config = useRuntimeConfig();

  return await $fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${config.firecrawlApiKey}`,
      'Content-Type': 'application/json',
    },
    body: { url, formats: ['markdown'] },
  });
});
```

<div id="call-it-from-your-app">
  ## アプリから呼び出す
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

<div id="notes">
  ## 注意事項
</div>

* Firecrawl APIキーは`runtimeConfig`に保持し、クライアント側に公開しないでください。

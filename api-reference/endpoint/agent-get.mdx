## Handling max credits

If an agent run hits the `maxCredits` limit, the job ends with a failed status but still returns the most recent results.

- `status` is `failed`
- `error` is `Max credits limit reached`
- `partial` contains the best available output so far

Example response:

```json
{
  "success": true,
  "status": "failed",
  "error": "Max credits limit reached",
  "partial": {
    "companies": [
      {
        "name": "ExampleCo",
        "website": "https://example.com"
      }
    ]
  },
  "expiresAt": "2026-01-21T19:32:12.000Z",
  "creditsUsed": 2500
}
```

---
title: "Batch Scrape Page"
---

Fired each time a URL is scraped in a batch scrape job. The `data` array contains the scraped document.

## Headers

<ParamField header="Content-Type" type="string" required>
  Always `application/json`.
</ParamField>

<ParamField header="X-Firecrawl-Signature" type="string">
  HMAC-SHA256 signature of the raw request body, formatted as `sha256=<hex>`. Present when an HMAC secret is configured in your [account settings](https://www.firecrawl.dev/app/settings?tab=advanced). See [Webhook Security](/webhooks/security) for verification details.
</ParamField>

## Payload

<ParamField body="success" type="boolean" required>
  `true` if the page was scraped successfully, `false` otherwise.
</ParamField>

<ParamField body="type" type="string" required>
  Always `batch_scrape.page`.
</ParamField>

<ParamField body="id" type="string" required>
  The batch scrape job ID.
</ParamField>

<ParamField body="webhookId" type="string" required>
  Unique identifier for this webhook delivery.
</ParamField>

<ParamField body="data" type="array" required>
  Array containing the scraped page document.
</ParamField>

<ParamField body="error" type="string">
  Error message if the page failed to scrape.
</ParamField>

<ParamField body="metadata" type="object">
  The custom metadata object from your webhook configuration, if provided.
</ParamField>

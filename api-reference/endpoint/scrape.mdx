## Browser Actions

You can perform browser interactions before scraping by using the `actions` parameter. This allows you to:
- Wait for elements to load
- Click buttons or links
- Fill form fields
- Press keys
- Scroll pages
- Take screenshots
- Execute custom JavaScript
- Generate PDFs

Supported actions include `wait`, `click`, `write`, `press`, `scroll`, `screenshot`, `scrape`, `executeJavascript`, and `pdf`. For detailed documentation with examples, see the [Advanced Scraping Guide](https://docs.firecrawl.dev/advanced-scraping-guide#actions-actions).

## Persistent Profiles

Pass the `profile` parameter to preserve browser state (cookies, localStorage, sessions) across scrape and [interact](/features/interact) sessions. Scrapes with the same profile name share browser state — useful for staying logged in or preserving preferences.

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/dashboard",
    "profile": {
      "name": "my-session",
      "saveChanges": true
    }
  }'
```

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `profile.name` | string | Yes | — | A name for the profile (1-128 chars). Scrapes with the same name share browser state. |
| `profile.saveChanges` | boolean | No | `true` | When `true`, browser state is saved back to the profile when the interact session stops. Set to `false` to load existing data without writing. |

<Note>
Only one session can save to a profile at a time. If another session is already saving, you'll get a `409` error. You can still open the same profile with `saveChanges: false`, or try again later.
</Note>

For more details, see [Interact — Persistent Profiles](/features/interact#persistent-profiles).

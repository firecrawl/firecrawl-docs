---
title: "Create a webhook"
---

Firecrawl webhooks are configured inline when you create a job — there is no separate CRUD endpoint.

Pass a `webhook` object in the request body of any supported endpoint:

- [`POST /crawl`](/api-reference/endpoint/crawl-post)
- [`POST /batch/scrape`](/api-reference/endpoint/batch-scrape-post)
- [`POST /agent`](/api-reference/endpoint/agent-post)

The webhook is active for the lifetime of that job and cannot be modified after creation.

## Webhook configuration object

<ParamField body="url" type="string" required>
  The HTTPS URL that will receive webhook `POST` requests. Must be publicly accessible.
</ParamField>

<ParamField body="events" type="string[]">
  Array of event types to subscribe to. If omitted, all events for the job type are delivered.

  **Crawl events:** `crawl.started`, `crawl.page`, `crawl.completed`

  **Batch Scrape events:** `batch_scrape.started`, `batch_scrape.page`, `batch_scrape.completed`

  **Agent events:** `agent.started`, `agent.action`, `agent.completed`, `agent.failed`, `agent.cancelled`
</ParamField>

<ParamField body="headers" type="object">
  Custom HTTP headers sent with every webhook delivery. Useful for authentication tokens. The header name `x-firecrawl-signature` is reserved and will be rejected.
</ParamField>

<ParamField body="metadata" type="object">
  Arbitrary key-value pairs echoed back in every webhook payload. Use this to correlate deliveries with your internal records.
</ParamField>

<RequestExample>
```json Example: Crawl with webhook
{
  "url": "https://example.com",
  "webhook": {
    "url": "https://your-server.com/webhooks/firecrawl",
    "events": ["crawl.page", "crawl.completed"],
    "headers": {
      "Authorization": "Bearer your-token"
    },
    "metadata": {
      "internalJobId": "job-123"
    }
  }
}
```
</RequestExample>

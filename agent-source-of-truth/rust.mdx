# Firecrawl Rust Source of Truth

This file is the canonical public source of truth for agents. It is generated from the Firecrawl Rust SDK source and the v2 OpenAPI spec.

## Install

```bash
cargo add firecrawl
```

## Authenticate

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
// let client = Client::new_selfhosted("https://api.firecrawl.dev", Some("fc-your-api-key"))?;
```

## When To Use What

- `search`: start from a query to discover relevant URLs, including `site:` constraints.
- `scrape`: fetch and extract content from a known URL.
- `interact`: run browser actions or code when a page needs clicks, forms, or post-scrape steps.

## Search

### Why use it

Use search when you need discovery from a query and want Firecrawl to return URLs (and optionally scrape each result). You can constrain results to a domain with the standard `site:` operator, for example `site:docs.firecrawl.dev crawl webhooks`.

### Preferred SDK method

`client.search(query, options)`

### Simple Example

```rust
use firecrawl::v2::Client;

let results = client
    .search("site:docs.firecrawl.dev webhook retries", None)
    .await?;
```

### Complex Example

```rust
use firecrawl::v2::{
    Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format, JsonOptions
};

let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:m".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    timeout: Some(60000),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Links, Format::Json]),
        json_options: Some(JsonOptions {
            prompt: Some("Extract title and key endpoints.".to_string()),
            ..Default::default()
        }),
        only_main_content: Some(true),
        ..Default::default()
    }),
    ..Default::default()
};

let results = client
    .search("site:docs.firecrawl.dev crawl webhooks", options)
    .await?;
```

### Parameters

| Parameter | Type / Options | Description |
| --- | --- | --- |
| `query` | `&str` | Search query. Use `site:domain` to restrict results to a site. |
| `options.sources` | `Vec<SearchSource>` (`Web`, `News`, `Images`) | Sources to search. |
| `options.categories` | `Vec<SearchCategory>` (`Github`, `Research`, `Pdf`) | Categories to filter results. |
| `options.limit` | `u32` | Maximum number of results to return. Default 5, max 20 (per SDK docs). |
| `options.tbs` | `String` | Time-based search filter (for example `qdr:d`, `qdr:w`, `sbd:1,qdr:m`). |
| `options.location` | `String` | Location string for search results. |
| `options.ignore_invalid_urls` | `bool` | Exclude invalid URLs that other Firecrawl endpoints cannot scrape. |
| `options.timeout` | `u32` | Timeout in milliseconds. |
| `options.scrape_options` | `ScrapeOptions` | Scrape options applied to each result (same fields as `scrape`). |
| `options.integration` | `String` | Internal only. Reserved for Firecrawl instrumentation. |

## Scrape

### Why use it

Use scrape when you already have a URL and want Firecrawl to return structured content and optional formats.

### Preferred SDK method

`client.scrape(url, options)`

### Simple Example

```rust
use firecrawl::v2::{Client, ScrapeOptions, Format};

let doc = client
    .scrape("https://docs.firecrawl.dev", ScrapeOptions {
        formats: Some(vec![Format::Markdown]),
        ..Default::default()
    })
    .await?;
```

### Complex Example

```rust

    ChangeTrackingMode, AttributeSelector, Action, ScrollDirection, ProxyType, ParserConfig

        Format::Markdown,
        Format::Links,

    ]),
    json_options: Some(JsonOptions {
        prompt: Some("Extract plan names and prices.".to_string()),

    screenshot_options: Some(ScreenshotOptions {
        full_page: Some(true),
        quality: Some(80),
        viewport: None,
    }),
    change_tracking_options: Some(ChangeTrackingOptions {
        modes: Some(vec![ChangeTrackingMode::GitDiff]),
        tag: Some("pricing".to_string()),
        ..Default::default()

    attribute_selectors: Some(vec![AttributeSelector {
        selector: "a".to_string(),
        attribute: "href".to_string(),
    }]),
    headers: None,
    include_tags: Some(vec!["main".to_string(), "article".to_string()]),
    exclude_tags: Some(vec!["nav".to_string(), "footer".to_string()]),
```

## Interact

### Why use it

Use interact when a page requires browser actions or code execution after a scrape starts (clicks, forms, or scripted steps).

### Preferred SDK method

`client.interact(job_id, options)`

### Simple Example

```rust
use firecrawl::v2::{Client, ScrapeOptions, Format, ScrapeExecuteOptions};

let doc = client
    .scrape("https://example.com", ScrapeOptions {
        formats: Some(vec![Format::Markdown]),
        ..Default::default()
    })
    .await?;

let job_id = doc
    .metadata
    .as_ref()
    .and_then(|m| m.scrape_id.clone())
    .expect("Missing scrape_id");

let result = client
    .interact(job_id, ScrapeExecuteOptions {
        prompt: Some("Click the pricing tab and summarize the plans.".to_string()),
        language: None,
        ..Default::default()
    })
    .await?;
```

### Complex Example

```rust
use firecrawl::v2::{Client, ScrapeExecuteOptions, ScrapeExecuteLanguage};

let result = client
    .interact(
        "<scrape_job_id>",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.title());".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            ..Default::default()
        },
    )
    .await?;
```

### Parameters

| Parameter | Type / Options | Description |
| --- | --- | --- |
| `job_id` | `&str` | Scrape job ID from `document.metadata.scrape_id`. |
| `options.code` | `String` | Code to execute in the browser session. Required if `prompt` is not provided. |
| `options.prompt` | `String` | Natural language instruction for the browser agent. Required if `code` is not provided. |
| `options.language` | `ScrapeExecuteLanguage` (`Python`, `Node`, `Bash`) | Execution language. Defaults to `Node` when omitted. |
| `options.timeout` | `u32` | Execution timeout in seconds. |
| `options.origin` | `String` | Internal only. Reserved for Firecrawl instrumentation. |

## Notes

- Deprecated aliases: `scrape_execute`, `stop_interactive_browser`, and `delete_scrape_browser` map to `interact` and `stop_interaction`.
- Rust struct fields are snake_case, but they serialize to the camelCase API fields via serde.

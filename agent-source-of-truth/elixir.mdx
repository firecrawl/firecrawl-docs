# Firecrawl Elixir Source of Truth

This file is the canonical public source of truth for agents. It is generated from the Firecrawl Elixir SDK (OpenAPI-shaped) and the v2 OpenAPI spec.

## Install

```elixir
# mix.exs
defp deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

## Authenticate

Configure your API key in `config/runtime.exs`:

```elixir
config :firecrawl, api_key: System.fetch_env!("FIRECRAWL_API_KEY")
```

Or pass `api_key` per request:

```elixir
{:ok, resp} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-...")
```

## When To Use What

- `search`: start from a query to discover relevant URLs, including `site:` constraints.
- `scrape`: fetch and extract content from a known URL.
- `interact`: run browser actions or code when a page needs clicks, forms, or post-scrape steps.

## Search

### Why use it

Use search when you need discovery from a query and want Firecrawl to return URLs (and optionally scrape each result). You can constrain results to a domain with the standard `site:` operator, for example `site:docs.firecrawl.dev crawl webhooks`.

### Preferred SDK method

`Firecrawl.search_and_scrape(params, opts)`

### Simple Example

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

### Complex Example

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000,
    actions: [
      %{type: "click", selector: "button#accept"},
      %{type: "wait", milliseconds: 750},
      %{type: "scrape"}
    ]
  ]
)
```

### Parameters

| Parameter | Type / Options | Description |
| --- | --- | --- |
| `query` | `String` | Search query. Use `site:domain` to restrict results to a site. |
| `sources` | `List` (`"web"`, `"news"`, `"images"` or maps with `type`) | Sources to search. |
| `categories` | `List` (`"github"`, `"research"`, `"pdf"` or maps with `type`) | Categories to filter results. |
| `limit` | `integer` | Maximum number of results to return. |
| `tbs` | `String` | Time-based search filter (for example `qdr:d`, `qdr:w`, `sbd:1,qdr:m`). |
| `location` | `String` | Location string for search results. |
| `country` | `String` | ISO country code for geo-targeting search results. |
| `enterprise` | `List[String]` (`"anon"`, `"zdr"`) | Enterprise ZDR options when enabled for your team. |
| `ignore_invalid_urls` | `boolean` | Exclude invalid URLs that other Firecrawl endpoints cannot scrape. |
| `timeout` | `integer` | Timeout in milliseconds. |
| `scrape_options` | `keyword` | Scrape options applied to each result (same fields as `scrape`). |

## Scrape

### Why use it

Use scrape when you already have a URL and want Firecrawl to return structured content and optional formats.

### Preferred SDK method

`Firecrawl.scrape_and_extract_from_url(params, opts)`

### Simple Example

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

### Complex Example

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  include_tags: ["main", "article"],
  exclude_tags: ["nav", "footer"],
  only_main_content: true,
  wait_for: 1000,
  mobile: false,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "write", text: "firecrawl"},
    %{type: "press", key: "Enter"},
    %{type: "wait", milliseconds: 1500},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  skip_tls_verification: true,
  remove_base64_images: true,
  block_ads: true,
  proxy: "auto",
  max_age: 86400000,
  store_in_cache: true,
  profile: [name: "docs-session", saveChanges: true]
)
```

### Parameters

| Parameter | Type / Options | Description |
| --- | --- | --- |
| `url` | `String` | URL to scrape. |
| `formats` | `List` | Output formats to include. Strings or objects such as `%{type: "json", prompt, schema}` or `%{type: "screenshot", fullPage, quality, viewport}`. |
| `only_main_content` | `boolean` | Keep only the main content. |
| `include_tags` | `List[String]` | Tags to include. |
| `exclude_tags` | `List[String]` | Tags to exclude. |
| `max_age` | `integer` | Prefer cached data up to this age. |
| `min_age` | `integer` | Only return cached data if it meets the minimum age. |
| `headers` | `map` | Custom headers. |
| `wait_for` | `integer` | Wait time in milliseconds before scraping. |
| `mobile` | `boolean` | Emulate a mobile device. |
| `skip_tls_verification` | `boolean` | Skip TLS certificate verification. |
| `timeout` | `integer` | Timeout in milliseconds. |
| `parsers` | `List` | Parser configuration (PDF options supported). |
| `actions` | `List` | Browser actions to run before scraping. |
| `location` | `keyword` | Location settings for the request. |
| `remove_base64_images` | `boolean` | Strip base64 images from markdown output. |
| `block_ads` | `boolean` | Enable ad and cookie popup blocking. |
| `proxy` | `"basic" | "enhanced" | "auto"` | Proxy mode. |
| `store_in_cache` | `boolean` | Store results in cache. |
| `profile` | `keyword` | Persistent browser profile, for example `[name: "profile", saveChanges: true]`. |
| `zero_data_retention` | `boolean` | Enable zero data retention when your account allows it. |

## Interact

### Why use it

Use interact when a page requires browser actions or code execution after a scrape starts (clicks, forms, or scripted steps).

### Preferred SDK method

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

### Simple Example

```elixir
{:ok, scrape_resp} = Firecrawl.scrape_and_extract_from_url(url: "https://example.com")
job_id = get_in(scrape_resp.body, ["data", "metadata", "scrapeId"])

{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  job_id,
  code: "console.log(document.title)",
  language: :node
)
```

### Complex Example

```elixir
{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  "<scrape_job_id>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

### Parameters

| Parameter | Type / Options | Description |
| --- | --- | --- |
| `job_id` | `String` | Scrape job ID from `data.metadata.scrapeId`. |
| `code` | `String` | Code to execute in the browser session. Required. |
| `language` | `:python | :node | :bash` | Execution language. Defaults to `:node` when omitted. |
| `timeout` | `integer` | Execution timeout in seconds. |
| `origin` | `String` | Internal only. Reserved for Firecrawl instrumentation. |

## Notes

- Elixir client functions are OpenAPI-shaped and accept snake_case keyword params that are converted to camelCase JSON fields.

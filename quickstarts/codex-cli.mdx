Add Firecrawl's keyless Search, Scrape, and Parse tools to [OpenAI Codex CLI](https://github.com/openai/codex) via MCP. Configure credentials later when you need the full tool set.

## Quick Setup

### 1. Add Firecrawl to Codex

Start with the hosted keyless server:

```toml
[mcp_servers.firecrawl]
url = "https://mcp.firecrawl.dev/v2/mcp"
```

This exposes Search, Scrape, and Parse. For full access, configure the same URL with `Authorization: Bearer fc-YOUR_API_KEY` using the header syntax supported by your Codex version, or run the local server with `FIRECRAWL_API_KEY` in its environment.

### 2. Start Codex

```bash
codex
```

Codex discovers the Firecrawl tools on launch. Confirm they are loaded:

```bash
/mcp
```

You should see `firecrawl_search`, `firecrawl_scrape`, and `firecrawl_parse` in keyless mode. Account-gated tools appear after you configure credentials.

## Troubleshooting

- **Codex doesn't see the tools** — run `codex --version` to confirm you're on a version with MCP support, then restart the CLI after editing `config.toml`.
- **`spawn npx ENOENT`** — install Node.js 18+ and ensure `npx` is on your `PATH`, or switch to the remote hosted URL above.
- **Keyless tool unavailable** — follow the returned `connect_account` action when a person is present, or set the returned `Authorization` header for unattended work.
- **Invalid configured key** — regenerate an API key at [firecrawl.dev/app/api-keys](https://www.firecrawl.dev/app/api-keys).

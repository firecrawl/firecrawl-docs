## Overview
Use the Firecrawl HTTP API directly in a Hono route so it works across runtimes.

## Prerequisites
- A Firecrawl API key
- Hono app
- A supported runtime (Node, Bun, Cloudflare, Deno)

## Install
```bash
npm install hono
```

## Configure environment
Expose the API key via your runtime's environment bindings.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

## Add a server route
src/index.ts

```ts
import { Hono } from 'hono';

type Bindings = { FIRECRAWL_API_KEY: string };

const app = new Hono<{ Bindings: Bindings }>();

app.post('/api/firecrawl', async (c) => {
  const { url } = await c.req.json();
  const response = await fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${c.env.FIRECRAWL_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ url, formats: ['markdown'] }),
  });

  const data = await response.json();
  return c.json(data);
});

export default app;
```

## Call it from your app
```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

## Notes
- Configure environment bindings for your chosen runtime (Wrangler, Deno Deploy, etc.).

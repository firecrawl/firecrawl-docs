## Overview
Keep your Firecrawl API key on the server by calling it from a Next.js Route Handler.

## Prerequisites
- A Firecrawl API key
- Next.js app (App Router)
- Node.js 18+

## Install
```bash
npm install @mendable/firecrawl-js
```

## Configure environment
Add the API key to `.env.local` so it is only available on the server.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

## Add a server route
app/api/firecrawl/route.ts

```ts
import Firecrawl from '@mendable/firecrawl-js';

export async function POST(request: Request) {
  const { url } = await request.json();
  const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

  const doc = await client.scrape(url, { formats: ['markdown'] });
  return Response.json({ markdown: doc.markdown });
}
```

## Call it from your app
```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

## Notes
- Call Firecrawl only from server routes to avoid exposing your API key.
- If you use the Pages Router, create `pages/api/firecrawl.ts` instead.

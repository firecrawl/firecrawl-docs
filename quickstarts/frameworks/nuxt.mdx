## Overview
Use a Nuxt server API route so your Firecrawl API key stays private.

## Prerequisites
- A Firecrawl API key
- Nuxt 3 app
- Node.js 18+

## Install
No extra packages required; Nuxt server routes can call fetch directly.

## Configure environment
Add the API key to `.env` and expose it through runtime config.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    firecrawlApiKey: process.env.FIRECRAWL_API_KEY,
  },
});
```

## Add a server route
server/api/firecrawl.post.ts

```ts
export default defineEventHandler(async (event) => {
  const { url } = await readBody(event);
  const config = useRuntimeConfig();

  return await $fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${config.firecrawlApiKey}`,
      'Content-Type': 'application/json',
    },
    body: { url, formats: ['markdown'] },
  });
});
```

## Call it from your app
```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

## Notes
- Keep the Firecrawl API key in `runtimeConfig` and avoid exposing it to the client.

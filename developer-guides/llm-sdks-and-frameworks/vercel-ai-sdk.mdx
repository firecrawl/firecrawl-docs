Firecrawl tools for the [Vercel AI SDK](https://ai-sdk.dev) v6+. Give your AI applications the ability to scrape, search, browse, crawl, and extract web data.

## Install

```bash
npm install firecrawl-aisdk ai @ai-sdk/openai
```

Set your environment variables:

```bash
export FIRECRAWL_API_KEY=fc-your-key   # https://firecrawl.dev
export OPENAI_API_KEY=sk-your-key
```

<Note>
These examples use OpenAI, but Firecrawl tools work with **any** Vercel AI SDK provider — Anthropic, Google, Mistral, and more. See the full list of [supported providers](https://ai-sdk.dev/providers/ai-sdk-providers).
</Note>

## Quick Start

`FirecrawlTools()` bundles **search**, **scrape**, and **browser** with sensible defaults:

```typescript
import { generateText, stepCountIs } from 'ai';
import { openai } from '@ai-sdk/openai';
import { FirecrawlTools } from 'firecrawl-aisdk';

const tools = FirecrawlTools();

const { text } = await generateText({
  model: openai('gpt-4o'),
  tools,
  system: tools.systemPrompt,
  stopWhen: stepCountIs(30),
  prompt: `
    Search for "Firecrawl web scraping".
    Scrape the top result.
    Summarize what you found.
  `,
});

console.log(text);
```

<Tip>
`FirecrawlTools()` auto-generates a `systemPrompt` that guides the model on which tool to use and when. Pass it to the `system` parameter for best results.
</Tip>

You can also import individual tools for fine-grained control — see [Individual Tools](#individual-tools) below.

## Common Patterns

### Search the web

Find relevant pages when you don't know the exact URL.

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { search } from 'firecrawl-aisdk';

const { text } = await generateText({
  model: openai('gpt-4o'),
  prompt: 'Search for the latest Firecrawl features and summarize',
  tools: { search },
});
```

### Scrape a page

Extract content from a known URL.

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { scrape } from 'firecrawl-aisdk';

const { text } = await generateText({
  model: openai('gpt-4o'),
  prompt: 'Scrape https://firecrawl.dev and summarize what it does',
  tools: { scrape },
});
```

### Search + Scrape

Search to discover relevant pages, then scrape for details.

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { search, scrape } from 'firecrawl-aisdk';

const { text } = await generateText({
  model: openai('gpt-4o'),
  tools: { search, scrape },
  prompt: `
    Search for Firecrawl.
    Scrape the top result.
    Explain what it does.
  `,
});
```

### Extract structured data

Use scrape with **JSON format** to extract structured data directly — far more token-efficient than parsing raw markdown.

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { scrape } from 'firecrawl-aisdk';

const { text } = await generateText({
  model: openai('gpt-4o'),
  prompt: `
    Go to https://firecrawl.dev/pricing.
    Extract all pricing plans with their names, prices, and features.
  `,
  tools: { scrape },
});
```

The model will automatically use `formats: [{ type: "json", prompt: "..." }]` when it makes sense. You can also set this as a default via `FirecrawlTools`:

```typescript
const tools = FirecrawlTools({
  scrape: {
    formats: [{ type: 'json', prompt: 'Extract the main content as structured data' }],
  },
});
```

For even more control, pass a JSON schema:

```typescript
// The LLM can pass this in the formats array:
formats: [{
  type: 'json',
  prompt: 'Extract pricing plans',
  schema: {
    type: 'object',
    properties: {
      plans: {
        type: 'array',
        items: {
          type: 'object',
          properties: {
            name: { type: 'string' },
            price: { type: 'string' },
            features: { type: 'array', items: { type: 'string' } },
          },
        },
      },
    },
  },
}]
```

### Browse interactive pages

Give the LLM a cloud browser for pages that need clicking, form-filling, or JS rendering.

```typescript
import { generateText, stepCountIs } from 'ai';
import { openai } from '@ai-sdk/openai';
import { browser, search } from 'firecrawl-aisdk';

const browserTool = browser();
console.log('Live view:', await browserTool.start());

const { text } = await generateText({
  model: openai('gpt-4o'),
  tools: { browser: browserTool, search },
  stopWhen: stepCountIs(25),
  prompt: `
    Go to Hacker News.
    Find the top story.
    Search for more context about it.
  `,
});

await browserTool.close();
```

Call `start()` to get a live view URL for debugging. Always call `close()` when done.

Profiles work like Chrome profiles — they persist cookies, localStorage, and login state across sessions. Use them to avoid re-authenticating every time:

```typescript
const browserTool = browser({
  profile: { name: 'my-profile', saveChanges: true },
});
```

### Crawl a site

Crawl multiple pages from a starting URL. This is an async operation — pair it with `poll`.

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { crawl, poll } from 'firecrawl-aisdk';

const { text } = await generateText({
  model: openai('gpt-4o'),
  prompt: `
    Crawl https://docs.firecrawl.dev with a limit of 5 pages.
    Summarize the documentation structure.
  `,
  tools: { crawl, poll },
});
```

### Scrape multiple URLs at once

Batch scrape is more efficient than calling scrape repeatedly. Also async — use `poll`.

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { batchScrape, poll } from 'firecrawl-aisdk';

const { text } = await generateText({
  model: openai('gpt-4o'),
  prompt: `
    Scrape https://firecrawl.dev and https://docs.firecrawl.dev.
    Compare what each site covers.
  `,
  tools: { batchScrape, poll },
});
```

### Autonomous research

Let Firecrawl Agent handle the research autonomously — just describe what you need.

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { agent, poll } from 'firecrawl-aisdk';

const { text } = await generateText({
  model: openai('gpt-4o'),
  prompt: `
    Find the founders of Firecrawl.
    Get their roles and backgrounds.
  `,
  tools: { agent, poll },
});
```

### Stream responses

Use `streamText` for real-time output:

```typescript
import { streamText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { scrape } from 'firecrawl-aisdk';

const result = streamText({
  model: openai('gpt-4o'),
  prompt: 'Scrape https://firecrawl.dev and explain what it does',
  tools: { scrape },
});

for await (const chunk of result.textStream) {
  process.stdout.write(chunk);
}
```

## Individual Tools

Every tool reads `FIRECRAWL_API_KEY` from the environment. You can also call any tool as a factory for custom config:

```typescript
// Use directly — reads from env
tools: { search, scrape }

// Or call with custom options
tools: { scrape: scrape({ apiKey: 'fc-custom-key', apiUrl: 'https://...' }) }
```

This dual-mode pattern works for `scrape`, `search`, `map`, `crawl`, `batchScrape`, and `agent`.

Each tool accepts the same parameters as its corresponding Firecrawl API endpoint. See the API reference for full details:

- [Scrape](/api-reference/endpoint/scrape) — `url`, `formats`, `onlyMainContent`, `actions`, `headers`, `proxy`, and more
- [Search](/api-reference/endpoint/search) — `query`, `limit`, `sources`, `categories`, `tbs`, `location`, `scrapeOptions`
- [Map](/api-reference/endpoint/map) — `url`, `search`, `limit`, `includeSubdomains`, `sitemap`
- [Crawl](/api-reference/endpoint/crawl-post) — `url`, `limit`, `maxDiscoveryDepth`, `includePaths`, `excludePaths`, `prompt`
- [Batch Scrape](/api-reference/endpoint/batch-scrape) — `urls[]`, plus all scrape parameters
- [Agent](/api-reference/endpoint/agent) — `prompt`, `model` (`spark-1-mini` or `spark-1-pro`), `urls`, `schema`
- [Browser](/features/browser) — `browserId`, `profile`, `firecrawlApiKey`

Async tools (`crawl`, `batchScrape`, `agent`) return a job ID. Pair them with `poll` to get results. Use `status` to check progress and `cancel` to stop a running job.

## FirecrawlTools Configuration

`FirecrawlTools()` is the recommended way to get started. It bundles tools, generates system prompts, and handles defaults.

```typescript
// Defaults: search + scrape + browser
FirecrawlTools();

// Add async tools (poll auto-included)
FirecrawlTools({ crawl: true, agent: true });

// Enable everything
FirecrawlTools({ all: true });

// Disable browser, keep search + scrape
FirecrawlTools({ browser: false });

// Set defaults for bundled tools
FirecrawlTools({
  search: { limit: 3 },
  scrape: { formats: ['markdown'], onlyMainContent: true },
});

// Custom API key
FirecrawlTools({ apiKey: 'fc-custom-key' });

// Customize tool descriptions for the LLM
FirecrawlTools({
  descriptions: {
    search: { append: 'Always search in English.' },
    scrape: { override: 'Fetch a web page and return markdown.' },
  },
});

// Control response size
FirecrawlTools({ maxResponseTokens: 4000 });
```

<Accordion title="All FirecrawlTools options">

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `apiKey` | `string` | `FIRECRAWL_API_KEY` env | Firecrawl API key |
| `apiUrl` | `string` | `https://api.firecrawl.dev` | API base URL |
| `search` | `object \| false` | Included | Search defaults, or `false` to disable |
| `scrape` | `object \| false` | Included | Scrape defaults, or `false` to disable |
| `browser` | `object \| false` | Included | Browser options, or `false` to disable |
| `map` | `boolean` | `false` | Include map tool |
| `crawl` | `boolean` | `false` | Include crawl tool |
| `batchScrape` | `boolean` | `false` | Include batch scrape tool |
| `agent` | `boolean` | `false` | Include agent tool |
| `poll` | `boolean` | Auto | Auto-included when any async tool is enabled |
| `status` | `boolean` | `false` | Include status tool |
| `cancel` | `boolean` | `false` | Include cancel tool |
| `all` | `boolean` | `false` | Enable all opt-in tools |
| `maxResponseTokens` | `number` | Unlimited | Truncate tool responses beyond this limit |
| `descriptions` | `object` | Auto | Customize search/scrape descriptions |
| `systemPrompt` | `string` | Auto-generated | Override the auto-generated system prompt |

</Accordion>

### Custom system prompts

Combine the auto-generated guide with your own instructions:

```typescript
const tools = FirecrawlTools();

const { text } = await generateText({
  model: openai('gpt-4o'),
  tools,
  system: `${tools.systemPrompt}

You are a research assistant. Always cite your sources with URLs.
Prefer JSON extraction mode for structured data.
Keep responses under 500 words.`,
  prompt: 'Research the latest AI SDK features',
});
```

## Logging

Two built-in helpers to track tool calls and token usage:

### stepLogger (detailed)

Accumulates per-tool stats. Call `.summary()` at the end for a breakdown.

```typescript
import { FirecrawlTools, stepLogger } from 'firecrawl-aisdk';

const logger = stepLogger();

const { text, usage } = await generateText({
  model: openai('gpt-4o'),
  tools: FirecrawlTools(),
  onStepFinish: logger.onStep,
  prompt: 'Search for Firecrawl and scrape the homepage',
});

logger.summary(usage);
```

```
› search: Firecrawl → ~1,250 tokens
› scrape: https://firecrawl.dev → ~3,400 tokens
agent: Firecrawl is a web scraping platform...

── stats ──
  search: 1 calls, ~1,250 tokens
  scrape: 1 calls, ~3,400 tokens
  total: 2 tool calls, 3 steps
  tokens: 5,200 in / 450 out / 5,650 total
```

### logStep (simple)

One-liner per tool call, no accumulation:

```typescript
import { logStep } from 'firecrawl-aisdk';

const { text } = await generateText({
  model: openai('gpt-4o'),
  tools: FirecrawlTools(),
  onStepFinish: logStep,
  prompt: 'Search for Firecrawl',
});
```

## Advanced

### Manual polling

For progress callbacks or custom polling logic, use the exported helpers directly:

```typescript
import { pollCrawlJob, pollBatchScrapeJob, pollAgentJob } from 'firecrawl-aisdk';

const result = await pollCrawlJob('job-id-123', {
  pollInterval: 3000,
  timeout: 120000,
  onProgress: (status) => {
    console.log(`Progress: ${status.status}, pages: ${status.total}`);
  },
});
```

### All exports

```typescript
import {
  scrape,             // Scrape a single URL
  search,             // Search the web
  map,                // Discover URLs on a site
  crawl,              // Crawl multiple pages (async → use poll)
  batchScrape,        // Scrape multiple URLs (async → use poll)
  agent,              // Autonomous web research (async → use poll)
  browser,            // Cloud browser (factory — has start/close lifecycle)
  poll,               // Poll async jobs for results
  status,             // Check job status
  cancel,             // Cancel running jobs
  FirecrawlTools,     // All-in-one bundle (recommended)
  stepLogger,         // Detailed token stats per tool call
  logStep,            // Simple one-liner logging
  pollCrawlJob,       // Manual crawl polling
  pollBatchScrapeJob, // Manual batch scrape polling
  pollAgentJob,       // Manual agent polling
} from 'firecrawl-aisdk';
```

### Migration from deprecated imports

If you're using the older `*Tool` naming convention, update your imports:

```typescript
// Before (deprecated)
import { scrapeTool, searchTool, mapTool, crawlTool } from 'firecrawl-aisdk';

// After
import { scrape, search, map, crawl } from 'firecrawl-aisdk';
```

| Deprecated | Current |
| --- | --- |
| `scrapeTool` | `scrape` |
| `searchTool` | `search` |
| `mapTool` | `map` |
| `crawlTool` | `crawl` |
| `batchScrapeTool` | `batchScrape` |
| `agentTool` | `agent` |
| `pollTool` | `poll` |
| `statusTool` | `status` |
| `cancelTool` | `cancel` |

<Note>
`extractTool` has been removed. Use `scrape` with JSON format instead — pass `formats: [{ type: 'json', prompt: '...' }]` for structured extraction.
</Note>

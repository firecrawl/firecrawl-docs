## What the Connector Does

Once enabled, Replit Agent can use Firecrawl while building your app:

| Capability | Use Case |
|------------|----------|
| **Search** | Find relevant live web sources and excerpts |
| **Scrape** | Turn any page into clean, LLM-ready data |
| **Interact** | Click, fill forms, and navigate multi-step flows |
| **Monitor** | Watch pages, sites, or the web for meaningful updates |
| **Crawl** | Discover and scrape reachable pages on a site |
| **Parse** | Convert PDFs and Word docs into Markdown or JSON |

## Quick Start

<CardGroup cols={2}>
  <Card title="Getting Started" icon="play">
    1. Open a Replit workspace on a **Core**, **Pro**, or **Enterprise** plan
    2. Open **Integrations** and search for **Firecrawl**
    3. Connect the Connector to your workspace
    4. Ask Replit Agent to use Firecrawl for the web data you need
  </Card>

  <Card title="Billing Options" icon="credit-card">
    1. **Replit credits** — start without a separate Firecrawl API key
    2. **Bring your own key** — open **Manage → Custom configuration** and add your Firecrawl API key for pass-through billing
  </Card>
</CardGroup>

## FAQs

<AccordionGroup>
  <Accordion title="Do I need a Firecrawl API key?">
    No. By default, usage can be billed through Replit's credit system. If you already have a Firecrawl account, add your API key under **Manage → Custom configuration**.
  </Accordion>

  <Accordion title="Which Replit plans include the Connector?">
    Replit integrations are available on **Core**, **Pro**, or **Enterprise**. Open Integrations in your workspace and search for Firecrawl.
  </Accordion>

  <Accordion title="How is this different from Replit using Firecrawl internally?">
    Replit Agent has used Firecrawl behind the scenes for fresh docs and web content. The official Connector makes Firecrawl available to **your** apps from the Integrations catalog.
  </Accordion>
</AccordionGroup>

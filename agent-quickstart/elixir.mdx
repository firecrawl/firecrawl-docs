# Firecrawl Elixir Agent Quickstart

This file is the canonical quickstart for external agents and is generated from SDK source plus the v2 OpenAPI spec.

## Install

Add the dependency in `mix.exs`:

```elixir
def deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

Then run:

```bash
mix deps.get
```

## Authenticate

```elixir
# config/runtime.exs or config/config.exs
config :firecrawl, api_key: "fc-your-api-key"
```

Or pass per call:

```elixir
Firecrawl.scrape_and_extract_from_url([url: "https://example.com"], api_key: "fc-your-api-key")
```

## When To Use What

- `search`: use when you start with a query and need discovery.
- `scrape`: use when you already have a URL and want page content.
- `interact`: use when the page needs clicks, forms, or post-scrape browser actions.

## Search

### Why use it

Use `search` to discover relevant URLs from a query and optionally scrape each result. You can constrain queries to a single site with the standard `site:` operator (for example, `site:docs.firecrawl.dev crawl webhooks`) before scraping a specific URL.

### Preferred SDK method

`Firecrawl.search_and_scrape(params, opts)`

### Simple Example

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  limit: 5
)

IO.inspect(response.body)
```

### Complex Example

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "firecrawl pricing",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  enterprise: ["anon"],
  scrape_options: [
    formats: [
      "markdown",
      [type: "json", prompt: "Extract pricing tiers and limits."]
    ],
    only_main_content: true,
    proxy: :enhanced
  ],
  timeout: 60000
)

IO.inspect(response.body)
```

### Parameters

- `query`: Search query string. Always required.
- `sources`: Sources to search. Supported values: `"web"`, `"news"`, `"images"`, or `[%{type: "web" | "news" | "images"}]` style objects. Use to control which result arrays are returned.
- `categories`: Result filters. Supported values: `"github"`, `"research"`, `"pdf"`, or `[%{type: "github" | "research" | "pdf"}]` style objects. Use to constrain results by category.
- `limit`: Maximum number of results to return. Use to cap response size.
- `tbs`: Time-based search filter string. Use to limit by time range (e.g., `qdr:d`).
- `location`: Location string for localized results. Use for geo-specific queries.
- `country`: ISO country code for geo-targeted search results. Use alongside `location`.
- `timeout`: Request timeout in milliseconds. Use to cap search time.
- `ignore_invalid_urls`: Exclude invalid URLs from results. Use when piping into other endpoints.
- `enterprise`: Enterprise search options. Supported values: `"anon"` (anonymized ZDR) or `"zdr"` (end-to-end ZDR).
- `scrape_options`: Scrape options applied to each result (same keys as `scrape`). Use when you want content with results.

## Scrape

### Why use it

Use `scrape` when you already know the URL and want content in specific formats.

### Preferred SDK method

`Firecrawl.scrape_and_extract_from_url(params, opts)`

### Simple Example

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com",
  formats: ["markdown"]
)

IO.inspect(response.body)
```

### Complex Example

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/docs",
  formats: [
    "markdown",
    [type: "json", prompt: "Extract headings and slugs."],
    [type: "changeTracking", modes: ["git-diff"], tag: "docs"],
    [type: "screenshot", full_page: true, viewport: [width: 1280, height: 720]]
  ],
  actions: [
    [type: "click", selector: "button#accept"],
    [type: "scroll", direction: "down"]
  ],
  parsers: [[type: "pdf", mode: "auto", max_pages: 5]],
  location: [country: "US", languages: ["en-US"]],
  proxy: :enhanced,
  profile: [name: "docs-session", save_changes: true],
  max_age: 3_600_000,
  min_age: 0,
  store_in_cache: true,
  zero_data_retention: false
)

IO.inspect(response.body)
```

### Parameters

- `url`: URL to scrape. Always required.
- `formats`: Output formats to include. Supported format strings: `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`. Format objects: `json` `[type: "json", prompt?, schema?]`, `changeTracking` `[type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?]`, `screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`.
- `headers`: Additional HTTP headers. Use for auth, cookies, or user-agent.
- `include_tags`: Only include content from these HTML tags. Use to narrow extraction.
- `exclude_tags`: Exclude content from these HTML tags. Use to remove noise.
- `only_main_content`: Extract main content only. Use to drop nav/footer.
- `timeout`: Scrape timeout in milliseconds. Use to cap run time.
- `wait_for`: Delay before scraping in milliseconds. Use for JS-rendered pages.
- `mobile`: Emulate a mobile device. Use for mobile layouts.
- `parsers`: Parser list or configs. Supported PDF config: `[type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: number]` or string entries (e.g., `"pdf"`).
- `actions`: Browser actions to run before scraping. Supported actions: `wait` `[type: "wait", milliseconds? | selector?]`, `screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`, `click` `[type: "click", selector, all?]`, `write` `[type: "write", text]`, `press` `[type: "press", key]`, `scroll` `[type: "scroll", direction: "up" | "down", selector?]`, `scrape` `[type: "scrape"]`, `executeJavascript` `[type: "executeJavascript", script]`, `pdf` `[type: "pdf", format?, landscape?, scale?]` with formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
- `location`: Location config `[country, languages]`. Use for geo-specific pages.
- `skip_tls_verification`: Skip TLS certificate verification. Use for self-signed certs.
- `remove_base64_images`: Strip base64 images from markdown output. Use to reduce payload size.
- `block_ads`: Enable ad and cookie blocking. Use to reduce noise.
- `proxy`: Proxy mode `:basic`, `:enhanced`, `:auto`.
- `max_age`: Accept cached content younger than this age (ms). Use to favor cache hits.
- `min_age`: Only return cached content at least this age (ms). Use for cache-only reads.
- `store_in_cache`: Store the result in cache/index. Use when you want reuse.
- `profile`: Persistent browser profile `[name, save_changes]`. Use to keep session state.
- `zero_data_retention`: Enable zero data retention for the scrape. Use when ZDR is enabled for your team.

## Interact

### Why use it

Use `interact` when the page requires clicks, form fills, or other browser actions after a scrape.

### Preferred SDK method

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

### Simple Example

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.url())"
)

IO.inspect(response.body)
```

### Complex Example

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.title())",
  language: :node,
  timeout: 60,
  origin: "agent-docs"
)

IO.inspect(response.body)
```

### Parameters

- `job_id`: Scrape job ID (from `metadata.scrapeId`). Always required.
- `code`: Code to execute in the browser sandbox. Required.
- `language`: Runtime language `:python`, `:node`, or `:bash` (default `:node`). Use to match your code.
- `timeout`: Execution timeout in seconds. Use to limit long runs.
- `origin`: Optional origin tag. Use for request attribution.

## Notes

- Parameters are passed as snake_case keyword lists validated by `NimbleOptions`.
- Functions are auto-generated from the OpenAPI spec; names follow the OpenAPI operation IDs.
- Every function has a bang variant that raises on error (for example, `search_and_scrape!`).
- The Elixir SDK exposes code-based `interact` only (no prompt parameter).
- All functions accept `api_key` and `base_url` options for per-request overrides.

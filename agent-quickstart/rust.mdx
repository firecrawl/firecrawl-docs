# Firecrawl Rust Agent Quickstart

This file is the canonical quickstart for external agents and is generated from SDK source plus the v2 OpenAPI spec.

## Install

```bash
cargo add firecrawl
```

## Authenticate

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
```

## When To Use What

- `search`: use when you start with a query and need discovery.
- `scrape`: use when you already have a URL and want page content.
- `interact`: use when the page needs clicks, forms, or post-scrape browser actions.

## Search

### Why use it

Use `search` to discover relevant URLs from a query and optionally scrape each result. You can constrain queries to a single site with the standard `site:` operator (for example, `site:docs.firecrawl.dev crawl webhooks`) before scraping a specific URL.

### Preferred SDK method

`client.search(query, options)`

### Simple Example

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
let results = client.search("site:docs.firecrawl.dev crawl webhooks", None).await?;

if let Some(web) = results.data.web {
    println!("{:?}", web.first());
}
```

### Complex Example

```rust
use firecrawl::v2::{Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format};

let client = Client::new("fc-your-api-key")?;
let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:w".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Json]),
        only_main_content: Some(true),
        ..Default::default()
    }),
    integration: Some("agent-quickstart".to_string()),
};

let results = client.search("firecrawl pricing", options).await?;
```

### Parameters

- `query`: Search query string. Always required.
- `limit`: Maximum number of results to return. Use to cap response size.
- `sources`: Sources to search. Supported values: `SearchSource::Web`, `SearchSource::News`, `SearchSource::Images`.
- `categories`: Result filters. Supported values: `SearchCategory::Github`, `SearchCategory::Research`, `SearchCategory::Pdf`.
- `tbs`: Time-based search filter string. Use to limit by time range (e.g., `qdr:d`).
- `location`: Location string for localized results. Use for geo-specific queries.
- `ignore_invalid_urls`: Exclude invalid URLs from results. Use when piping into other endpoints.
- `timeout`: Request timeout in milliseconds. Use to cap search time.
- `scrape_options`: Scrape options applied to each result (same fields as `scrape`). Use when you want content with results.
- `integration`: Integration identifier. Use for tracking/analytics.

## Scrape

### Why use it

Use `scrape` when you already know the URL and want content in specific formats.

### Preferred SDK method

`client.scrape(url, options)`

### Simple Example

```rust
use firecrawl::v2::{Client, ScrapeOptions, Format};

let client = Client::new("fc-your-api-key")?;
let doc = client
    .scrape(
        "https://example.com",
        ScrapeOptions {
            formats: Some(vec![Format::Markdown]),
            ..Default::default()
        },
    )
    .await?;

println!("{:?}", doc.markdown);
```

### Complex Example

```rust
use firecrawl::v2::{
    Action, ChangeTrackingMode, ChangeTrackingOptions, Client, Format, JsonOptions, LocationConfig,
    ParserConfig, ProxyType, ScrapeOptions, ScreenshotOptions, ScrollDirection, Viewport,
};

let client = Client::new("fc-your-api-key")?;
let options = ScrapeOptions {
    formats: Some(vec![Format::Markdown, Format::Json, Format::ChangeTracking, Format::Screenshot]),
    json_options: Some(JsonOptions {
        prompt: Some("Extract headings and slugs.".to_string()),
        ..Default::default()
    }),
    change_tracking_options: Some(ChangeTrackingOptions {
        modes: Some(vec![ChangeTrackingMode::GitDiff]),
        tag: Some("docs".to_string()),
        ..Default::default()
    }),
    screenshot_options: Some(ScreenshotOptions {
        full_page: Some(true),
        viewport: Some(Viewport { width: 1280, height: 720 }),
        ..Default::default()
    }),
    actions: Some(vec![
        Action::Wait {
            milliseconds: Some(1000),
            selector: None,
        },
        Action::Click {
            selector: "button#accept".to_string(),
        },
        Action::Scroll {
            direction: ScrollDirection::Down,
            selector: None,
        },
    ]),
    parsers: Some(vec![ParserConfig::Pdf {
        parser_type: "pdf".to_string(),
        max_pages: Some(5),
    }]),
    location: Some(LocationConfig {
        country: Some("US".to_string()),
        languages: Some(vec!["en-US".to_string()]),
    }),
    proxy: Some(ProxyType::Stealth),
    ..Default::default()
};

let doc = client.scrape("https://example.com/docs", options).await?;
```

### Parameters

- `url`: URL to scrape. Always required.
- `formats`: Output formats to include. Supported values: `Format::Markdown`, `Format::Html`, `Format::RawHtml`, `Format::Links`, `Format::Images`, `Format::Screenshot`, `Format::Summary`, `Format::ChangeTracking`, `Format::Json`, `Format::Attributes`, `Format::Branding`, `Format::Audio`.
- `headers`: Additional HTTP headers. Use for auth, cookies, or user-agent.
- `include_tags`: Only include content from these HTML tags. Use to narrow extraction.
- `exclude_tags`: Exclude content from these HTML tags. Use to remove noise.
- `only_main_content`: Extract main content only. Use to drop nav/footer.
- `timeout`: Scrape timeout in milliseconds. Use to cap run time.
- `wait_for`: Delay before scraping in milliseconds. Use for JS-rendered pages.
- `mobile`: Emulate a mobile device. Use for mobile layouts.
- `parsers`: Parser list or configs. Supported values: `ParserConfig::Simple(String)` or `ParserConfig::Pdf { parser_type: "pdf", max_pages: Option<u32> }`.
- `actions`: Browser actions to run before scraping. Supported actions: `Wait { milliseconds?, selector? }`, `Screenshot { full_page?, quality?, viewport? }`, `Click { selector }`, `Write { text }`, `Press { key }`, `Scroll { direction: Up | Down, selector? }`, `Scrape`, `ExecuteJavascript { script }`, `Pdf { format?, landscape?, scale? }` with formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
- `location`: Location config `{ country, languages }`. Use for geo-specific pages.
- `skip_tls_verification`: Skip TLS certificate verification. Use for self-signed certs.
- `remove_base64_images`: Strip base64 images from markdown output. Use to reduce payload size.
- `block_ads`: Enable ad and cookie blocking. Use to reduce noise.
- `proxy`: Proxy mode `ProxyType::Basic`, `ProxyType::Stealth`, `ProxyType::Enhanced`, `ProxyType::Auto`.
- `max_age`: Accept cached content younger than this age (ms). Use to favor cache hits.
- `min_age`: Only return cached content at least this age (ms). Use for cache-only reads.
- `store_in_cache`: Store the result in cache/index. Use when you want reuse.
- `profile`: Persistent browser profile `ProfileConfig { name, save_changes }`. Use to keep session state.
- `integration`: Integration identifier. Use for tracking/analytics.
- `json_options`: JSON extraction options for the `Format::Json` format (`schema`, `system_prompt`, `prompt`).
- `screenshot_options`: Screenshot options for the `Format::Screenshot` format (`full_page`, `quality`, `viewport`).
- `change_tracking_options`: Change tracking options for the `Format::ChangeTracking` format (`modes`, `schema`, `prompt`, `tag`). Modes: `ChangeTrackingMode::GitDiff`, `ChangeTrackingMode::Json`.
- `attribute_selectors`: Attribute selectors for the `Format::Attributes` format (`selector`, `attribute`).

## Interact

### Why use it

Use `interact` when the page requires clicks, form fills, or other browser actions after a scrape.

### Preferred SDK method

`client.interact(job_id, options)`

### Simple Example

```rust
use firecrawl::v2::{Client, ScrapeExecuteOptions};

let client = Client::new("fc-your-api-key")?;
let run = client
    .interact(
        "scrape-job-id",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.url())".to_string()),
            ..Default::default()
        },
    )
    .await?;

println!("{:?}", run.stdout);
```

### Complex Example

```rust
use firecrawl::v2::{Client, ScrapeExecuteLanguage, ScrapeExecuteOptions};

let client = Client::new("fc-your-api-key")?;
let run = client
    .interact(
        "scrape-job-id",
        ScrapeExecuteOptions {
            prompt: Some("Open the pricing page and list the plan names.".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            origin: Some("agent-docs".to_string()),
            ..Default::default()
        },
    )
    .await?;
```

### Parameters

- `job_id`: Scrape job ID (from `document.metadata.scrape_id`). Always required.
- `code`: Code to execute in the browser sandbox. Use for scripted interactions.
- `prompt`: Natural-language instruction for the browser agent. Use when you prefer agent-driven actions.
- `language`: Runtime language `ScrapeExecuteLanguage::Python`, `ScrapeExecuteLanguage::Node`, or `ScrapeExecuteLanguage::Bash` (defaults to `Node`).
- `timeout`: Execution timeout in seconds. Use to limit long runs.
- `origin`: Optional origin tag. Use for request attribution.

## Notes

- Parameters are snake_case in Rust; JSON is serialized as camelCase.
- Convenience method: `search_and_scrape` performs a search with scraping enabled.
- Deprecated aliases: `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (use `interact` and `stop_interaction`).

# Firecrawl JavaScript/TypeScript Agent Quickstart

This file is the canonical quickstart for external agents and is generated from SDK source plus the v2 OpenAPI spec.

## Install

```bash
npm install @mendable/firecrawl-js
```

## Authenticate

```ts
import { Firecrawl } from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // optional for self-hosted
});
```

## When To Use What

- `search`: use when you start with a query and need discovery.
- `scrape`: use when you already have a URL and want page content.
- `interact`: use when the page needs clicks, forms, or post-scrape browser actions.

## Search

### Why use it

Use `search` to discover relevant URLs from a query and optionally scrape each result. You can constrain queries to a single site with the standard `site:` operator (for example, `site:docs.firecrawl.dev crawl webhooks`) before scraping a specific URL.

### Preferred SDK method

`client.search(query, options)`

### Simple Example

```ts
const results = await client.search(
  "site:docs.firecrawl.dev crawl webhooks",
  { limit: 5 }
);

console.log(results.web?.[0]);
```

### Complex Example

```ts
const results = await client.search("firecrawl pricing", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  scrapeOptions: {
    formats: [
      "markdown",
      { type: "json", prompt: "Extract pricing tiers and limits." }
    ],
    onlyMainContent: true,
    actions: [
      { type: "wait", milliseconds: 1000 },
      { type: "scroll", direction: "down" }
    ],
    proxy: "stealth"
  },
  integration: "agent-quickstart",
  origin: "agent-docs"
});
```

### Parameters

- `query`: Search query string. Always required.
- `sources`: Sources to search. Supported values: `"web"`, `"news"`, `"images"`, or `{ type: "web" | "news" | "images" }`. Use to control which result arrays are returned.
- `categories`: Result filters. Supported values: `"github"`, `"research"`, `"pdf"`, or `{ type: "github" | "research" | "pdf" }`. Use to constrain results by category.
- `limit`: Maximum number of results to return. Use to cap response size.
- `tbs`: Time-based search filter string. Use to limit by time range (e.g., `qdr:d`).
- `location`: Location string for localized results. Use for geo-specific queries.
- `ignoreInvalidURLs`: Exclude invalid URLs from results. Use when piping into other endpoints.
- `timeout`: Request timeout in milliseconds. Use to cap search time.
- `scrapeOptions`: Scrape options applied to each result (same fields as `scrape`). Use when you want content with results.
- `integration`: Integration identifier. Use for tracking/analytics.
- `origin`: Optional origin tag. Use for request attribution.

## Scrape

### Why use it

Use `scrape` when you already know the URL and want content in specific formats.

### Preferred SDK method

`client.scrape(url, options)`

### Simple Example

```ts
const doc = await client.scrape("https://example.com", {
  formats: ["markdown"]
});

console.log(doc.markdown);
```

### Complex Example

```ts
const doc = await client.scrape("https://example.com/docs", {
  formats: [
    "markdown",
    { type: "json", prompt: "Extract headings and their slugs." },
    { type: "changeTracking", modes: ["git-diff"], tag: "docs" },
    { type: "screenshot", fullPage: true, viewport: { width: 1280, height: 720 } },
    {
      type: "attributes",
      selectors: [{ selector: "a", attribute: "href" }]
    },
    { type: "query", prompt: "What is the pricing model?" }
  ],
  onlyMainContent: true,
  waitFor: 1500,
  actions: [
    { type: "click", selector: "button#accept" },
    { type: "scroll", direction: "down" }
  ],
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  location: { country: "US", languages: ["en-US"] },
  proxy: "stealth",
  profile: { name: "docs-session", saveChanges: true },
  maxAge: 3600000
});
```

### Parameters

- `url`: URL to scrape. Always required.
- `formats`: Output formats to include. Supported format strings: `markdown`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `json`, `attributes`, `branding`, `audio`. Format objects: `json` `{ type: "json", prompt?, schema? }` (requires `prompt` or `schema`), `changeTracking` `{ type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag? }`, `screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`, `attributes` `{ type: "attributes", selectors: [{ selector, attribute }] }`, `query` `{ type: "query", prompt }`, plus `{ type: "<format>" }` for any string format.
- `headers`: Additional HTTP headers. Use for auth, cookies, or user-agent.
- `includeTags`: Only include content from these HTML tags. Use to narrow extraction.
- `excludeTags`: Exclude content from these HTML tags. Use to remove noise.
- `onlyMainContent`: Extract main content only. Use to drop nav/footer.
- `timeout`: Scrape timeout in milliseconds. Use to cap run time.
- `waitFor`: Delay before scraping in milliseconds. Use for JS-rendered pages.
- `mobile`: Emulate a mobile device. Use for mobile layouts.
- `parsers`: Parser list or configs. Supported PDF config: `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }` or string entries (e.g., `"pdf"`).
- `actions`: Browser actions to run before scraping. Supported actions: `wait` `{ type: "wait", milliseconds? | selector? }`, `screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`, `click` `{ type: "click", selector }`, `write` `{ type: "write", text }`, `press` `{ type: "press", key }`, `scroll` `{ type: "scroll", direction: "up" | "down", selector? }`, `scrape` `{ type: "scrape" }`, `executeJavascript` `{ type: "executeJavascript", script }`, `pdf` `{ type: "pdf", format?, landscape?, scale? }` with formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
- `location`: Location config `{ country, languages }`. Use for geo-specific pages.
- `skipTlsVerification`: Skip TLS certificate verification. Use for self-signed certs.
- `removeBase64Images`: Strip base64 images from markdown output. Use to reduce payload size.
- `fastMode`: Enable fast mode. Use when speed matters most.
- `useMock`: Use a named mock fixture (internal/testing).
- `blockAds`: Enable ad and cookie blocking. Use to reduce noise.
- `proxy`: Proxy mode `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`, or a custom proxy string. Use for anti-bot sites.
- `maxAge`: Accept cached content younger than this age (ms). Use to favor cache hits.
- `minAge`: Only return cached content at least this age (ms). Use for cache-only reads.
- `storeInCache`: Store the result in cache/index. Use when you want reuse.
- `profile`: Persistent browser profile `{ name, saveChanges }`. Use to keep session state.
- `integration`: Integration identifier. Use for tracking/analytics.
- `origin`: Optional origin tag. Use for request attribution.

## Interact

### Why use it

Use `interact` when the page requires clicks, form fills, or other browser actions after a scrape.

### Preferred SDK method

`client.interact(jobId, args)`

### Simple Example

```ts
const run = await client.interact("scrape-job-id", {
  code: "console.log(await page.url())"
});

console.log(run.stdout);
```

### Complex Example

```ts
const run = await client.interact("scrape-job-id", {
  prompt: "Open the pricing page and list the plan names.",
  language: "node",
  timeout: 60,
  origin: "agent-docs"
});

console.log(run.output);
```

### Parameters

- `jobId`: Scrape job ID (from `document.metadata.scrapeId`). Always required.
- `code`: Code to execute in the browser sandbox. Use for scripted interactions.
- `prompt`: Natural-language instruction for the browser agent. Use when you prefer agent-driven actions.
- `language`: Runtime language `"python"`, `"node"`, or `"bash"` (default `"node"`). Use to match your code.
- `timeout`: Execution timeout in seconds. Use to limit long runs.
- `origin`: Optional origin tag. Use for request attribution.

## Notes

- Parameters are camelCase in JS/TS.
- Deprecated aliases: `scrapeExecute`, `stopInteractiveBrowser`, `deleteScrapeBrowser` (use `interact` and `stopInteraction`).
- `Firecrawl` is the unified v2 client; v1 remains available under `.v1` for legacy use.

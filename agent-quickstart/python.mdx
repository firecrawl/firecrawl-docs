# Firecrawl Python Agent Quickstart

This file is the canonical quickstart for external agents and is generated from SDK source plus the v2 OpenAPI spec.

## Install

```bash
pip install firecrawl-py
```

## Authenticate

```python
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])
```

## When To Use What

- `search`: use when you start with a query and need discovery.
- `scrape`: use when you already have a URL and want page content.
- `interact`: use when the page needs clicks, forms, or post-scrape browser actions.

## Search

### Why use it

Use `search` to discover relevant URLs from a query and optionally scrape each result. You can constrain queries to a single site with the standard `site:` operator (for example, `site:docs.firecrawl.dev crawl webhooks`) before scraping a specific URL.

### Preferred SDK method

`client.search(query, **options)`

### Simple Example

```python
results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    limit=5,
)

print(results.web[0] if results.web else None)
```

### Complex Example

```python
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "firecrawl pricing",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:w",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            {"type": "json", "prompt": "Extract pricing tiers and limits."},
        ],
        only_main_content=True,
        proxy="stealth",
    ),
    integration="agent-quickstart",
)
```

### Parameters

- `query`: Search query string. Always required.
- `sources`: Sources to search. Supported values: `"web"`, `"news"`, `"images"`, or `{"type": ...}` objects. Use to control which result arrays are returned.
- `categories`: Result filters. Supported values: `"github"`, `"research"`, `"pdf"`, or `{"type": ...}` objects. Use to constrain results by category.
- `limit`: Maximum number of results to return. Default: `5`.
- `tbs`: Time-based search filter string. Use to limit by time range (e.g., `qdr:d`).
- `location`: Location string for localized results. Use for geo-specific queries.
- `ignore_invalid_urls`: Exclude invalid URLs from results. Use when piping into other endpoints.
- `timeout`: Request timeout in milliseconds. Default: `300000`.
- `scrape_options`: Scrape options applied to each result (same fields as `scrape`). Use when you want content with results.
- `integration`: Integration identifier. Use for tracking/analytics.

## Scrape

### Why use it

Use `scrape` when you already know the URL and want content in specific formats.

### Preferred SDK method

`client.scrape(url, **options)`

### Simple Example

```python
doc = client.scrape(
    "https://example.com",
    formats=["markdown"],
)

print(doc.markdown)
```

### Complex Example

```python
doc = client.scrape(
    "https://example.com/docs",
    formats=[
        "markdown",
        {"type": "json", "prompt": "Extract headings and slugs."},
        {"type": "change_tracking", "modes": ["git-diff"], "tag": "docs"},
        {"type": "screenshot", "full_page": True, "viewport": {"width": 1280, "height": 720}},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
        {"type": "query", "prompt": "What is the pricing model?"},
    ],
    only_main_content=True,
    wait_for=1500,
    actions=[
        {"type": "click", "selector": "button#accept"},
        {"type": "scroll", "direction": "down"},
    ],
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    location={"country": "US", "languages": ["en-US"]},
    proxy="stealth",
    profile={"name": "docs-session", "save_changes": True},
    max_age=3_600_000,
)
```

### Parameters

- `url`: URL to scrape. Always required.
- `formats`: Output formats to include. Accepts a `ScrapeFormats` object (fields `markdown`, `html`, `raw_html`, `summary`, `links`, `images`, `screenshot`, `change_tracking`, `json`) or a list of format strings/objects. Supported format strings include `markdown`, `html`, `rawHtml`, `raw_html`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `change_tracking`, `json`, `attributes`, `branding`, `query`, `audio`. Format objects: `json` `{type: "json", prompt?, schema?}` (requires `prompt` or `schema`), `changeTracking` `{type: "changeTracking" | "change_tracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `attributes` `{type: "attributes", selectors: [{selector, attribute}]}`, `query` `{type: "query", prompt}`.
- `headers`: Additional HTTP headers. Use for auth, cookies, or user-agent.
- `include_tags`: Only include content from these HTML tags. Use to narrow extraction.
- `exclude_tags`: Exclude content from these HTML tags. Use to remove noise.
- `only_main_content`: Extract main content only. Use to drop nav/footer.
- `timeout`: Scrape timeout in milliseconds. Use to cap run time.
- `wait_for`: Delay before scraping in milliseconds. Use for JS-rendered pages.
- `mobile`: Emulate a mobile device. Use for mobile layouts.
- `parsers`: Parser list or configs. Supported PDF config: `{type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: int}` or string entries (e.g., `"pdf"`).
- `actions`: Browser actions to run before scraping. Supported actions: `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `click` `{type: "click", selector}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}` with formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
- `location`: Location config `{country, languages}`. Use for geo-specific pages.
- `skip_tls_verification`: Skip TLS certificate verification. Use for self-signed certs.
- `remove_base64_images`: Strip base64 images from markdown output. Use to reduce payload size.
- `fast_mode`: Enable fast mode. Use when speed matters most.
- `use_mock`: Use a named mock fixture (internal/testing).
- `block_ads`: Enable ad and cookie blocking. Use to reduce noise.
- `proxy`: Proxy mode `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`.
- `max_age`: Accept cached content younger than this age (ms). Use to favor cache hits.
- `store_in_cache`: Store the result in cache/index. Use when you want reuse.
- `profile`: Persistent browser profile `{name, save_changes}` (also accepts `saveChanges`). Use to keep session state.
- `integration`: Integration identifier. Use for tracking/analytics.

## Interact

### Why use it

Use `interact` when the page requires clicks, form fills, or other browser actions after a scrape.

### Preferred SDK method

`client.interact(job_id, code=None, *, prompt=None, **options)`

### Simple Example

```python
run = client.interact(
    "scrape-job-id",
    code="print(await page.url())",
)

print(run.stdout)
```

### Complex Example

```python
run = client.interact(
    "scrape-job-id",
    prompt="Open the pricing page and list the plan names.",
    language="python",
    timeout=60,
    origin="agent-docs",
)

print(run.output)
```

### Parameters

- `job_id`: Scrape job ID (from `document.metadata_typed.scrape_id`). Always required.
- `code`: Code to execute in the browser sandbox. Use for scripted interactions.
- `prompt`: Natural-language instruction for the browser agent. Use when you prefer agent-driven actions.
- `language`: Runtime language `"python"`, `"node"`, or `"bash"` (default `"node"`). Use to match your code.
- `timeout`: Execution timeout in seconds. Use to limit long runs.
- `origin`: Optional origin tag. Use for request attribution.

## Notes

- Parameters are snake_case in Python; the SDK normalizes payloads to API camelCase.
- Deprecated aliases: `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (use `interact` and `stop_interaction`).
- `Firecrawl` is the unified v2 client; v1 remains available under `.v1` for legacy use.

O menu de navegação é a lista de links em todo site.

Você provavelmente vai atualizar o `mint.json` sempre que adicionar uma nova página. As páginas não aparecem automaticamente.

<div id="navigation-syntax">
  ## Sintaxe de navegação
</div>

Nossa sintaxe de navegação é recursiva, o que significa que você pode criar grupos de navegação aninhados. Você não precisa incluir `.mdx` nos nomes das páginas.

<CodeGroup>

```json Regular Navigation
"navigation": [
    {
        "group": "Getting Started",
        "pages": ["quickstart"]
    }
]
```

```json Nested Navigation
"navigation": [
    {
        "group": "Getting Started",
        "pages": [
            "quickstart",
            {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
            }
        ]
    }
]
```

</CodeGroup>

<div id="folders">
  ## Pastas
</div>

Basta colocar seus arquivos MDX em pastas e atualizar os caminhos em `mint.json`.

Por exemplo, para ter uma página em `https://yoursite.com/your-folder/your-page`, você criaria uma pasta chamada `your-folder` contendo um arquivo MDX chamado `your-page.mdx`.

<Warning>
  Você não pode usar `api` como nome de uma pasta, a menos que ela esteja dentro de outra pasta. O Mintlify usa Next.js, que reserva a pasta `api` no nível superior para chamadas internas do servidor. Um nome de pasta como `api-reference` seria aceito.
</Warning>

```json Navegação com pasta
"navigation": [
    {
        "group": "Nome do grupo",
        "pages": ["your-folder/your-page"]
    }
]
```


<div id="hidden-pages">
  ## Páginas ocultas
</div>

Arquivos MDX que não estiverem incluídos em `mint.json` não aparecerão na barra lateral, mas poderão ser acessados pela barra de pesquisa ou por links diretos.

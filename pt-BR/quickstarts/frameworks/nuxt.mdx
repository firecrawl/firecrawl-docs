<div id="overview">
  ## Visão geral
</div>

Use uma rota de API no servidor do Nuxt para manter sua chave de API do Firecrawl privada.

<div id="prerequisites">
  ## Pré-requisitos
</div>

* Uma chave de API do Firecrawl
* Aplicação Nuxt 3
* Node.js 18+

<div id="install">
  ## Instalação
</div>

Não é necessário nenhum pacote extra; as rotas do servidor do Nuxt podem chamar `fetch` diretamente.

<div id="configure-environment">
  ## Configurar o ambiente
</div>

Adicione a chave de API ao `.env` e disponibilize-a pela configuração de runtime.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    firecrawlApiKey: process.env.FIRECRAWL_API_KEY,
  },
});
```

<div id="add-a-server-route">
  ## Adicione uma rota de servidor
</div>

server/api/firecrawl.post.ts

```ts
export default defineEventHandler(async (event) => {
  const { url } = await readBody(event);
  const config = useRuntimeConfig();

  return await $fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${config.firecrawlApiKey}`,
      'Content-Type': 'application/json',
    },
    body: { url, formats: ['markdown'] },
  });
});
```

<div id="call-it-from-your-app">
  ## Faça a chamada no seu app
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

<div id="notes">
  ## Observações
</div>

* Mantenha a chave de API do Firecrawl em `runtimeConfig` e evite expô-la no cliente.

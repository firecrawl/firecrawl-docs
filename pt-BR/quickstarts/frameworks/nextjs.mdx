<div id="overview">
  ## Visão geral
</div>

Mantenha sua Firecrawl chave de API no servidor fazendo a chamada a partir de um Route Handler do Next.js.

<div id="prerequisites">
  ## Pré-requisitos
</div>

* Uma chave de API do Firecrawl
* App Next.js (App Router)
* Node.js 18+

<div id="install">
  ## Instalação
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="configure-environment">
  ## Configure o ambiente
</div>

Adicione a chave de API ao `.env.local` para que ela fique disponível apenas no servidor.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## Adicione uma rota de servidor
</div>

app/api/firecrawl/route.ts

```ts
import Firecrawl from '@mendable/firecrawl-js';

export async function POST(request: Request) {
  const { url } = await request.json();
  const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

  const doc = await client.scrape(url, { formats: ['markdown'] });
  return Response.json({ markdown: doc.markdown });
}
```

<div id="call-it-from-your-app">
  ## Faça a chamada no seu aplicativo
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

<div id="notes">
  ## Observações
</div>

* Use o Firecrawl apenas em rotas de servidor para evitar expor sua chave de API.
* Se você usar o Pages Router, crie `pages/api/firecrawl.ts` no lugar.

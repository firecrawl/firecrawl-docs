<div id="overview">
  ## Visão geral
</div>

Use a API `fetch` do Deno Deploy para fazer chamadas ao Firecrawl.

<div id="prerequisites">
  ## Pré-requisitos
</div>

* Uma chave de API do Firecrawl
* Um projeto no Deno Deploy

<div id="install">
  ## Instalação
</div>

Não é necessário nenhum pacote extra; o Deno já inclui `fetch`.

<div id="configure-environment">
  ## Configure o ambiente
</div>

Defina a chave de API nas variáveis de ambiente do projeto no Deno Deploy.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## Adicione uma rota no servidor
</div>

main.ts

```ts
Deno.serve(async (req) => {
  const { url } = await req.json();
  const response = await fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${Deno.env.get('FIRECRAWL_API_KEY')}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ url, formats: ['markdown'] }),
  });

  return new Response(await response.text(), {
    headers: { 'Content-Type': 'application/json' },
  });
});
```

<div id="call-it-from-your-app">
  ## Faça a chamada no seu app
</div>

```js
const response = await fetch('https://your-app.deno.dev', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

<div id="notes">
  ## Observações
</div>

* O Deno Deploy só permite requisições `fetch` de saída; mantenha todas as chamadas do Firecrawl no servidor.

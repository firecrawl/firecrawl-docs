<div id="overview">
  ## Visão geral
</div>

Use o SDK Python do Firecrawl em uma view do Django para servir de proxy para solicitações de scraping.

<div id="prerequisites">
  ## Pré-requisitos
</div>

* Uma chave de API do Firecrawl
* Um app Django
* Python 3.8+

<div id="install">
  ## Instalação
</div>

```bash
pip install firecrawl-py
```

<div id="configure-environment">
  ## Configure o ambiente
</div>

Defina a chave de API no seu ambiente do Django.

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="add-a-server-route">
  ## Adicione uma rota ao servidor
</div>

app/views.py

```python
import json
import os
from django.http import JsonResponse
from django.views.decorators.csrf import csrf_exempt
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])

@csrf_exempt
def firecrawl_view(request):
    if request.method != "POST":
        return JsonResponse({"error": "Method not allowed"}, status=405)

    body = json.loads(request.body)
    doc = client.scrape(body["url"], formats=["markdown"])
    return JsonResponse({"markdown": doc.markdown})
```

<div id="call-it-from-your-app">
  ## Chame-o no seu aplicativo
</div>

```python
import requests

response = requests.post(
    "http://localhost:8000/api/firecrawl",
    json={"url": "https://example.com"},
)

print(response.json().get("markdown"))
```

<div id="notes">
  ## Observações
</div>

* Adicione uma rota que mapeie `/api/firecrawl` para `firecrawl_view`.

<div id="overview">
  ## Visão geral
</div>

Use o SDK de Node da Firecrawl em uma rota de servidor do Express.

<div id="prerequisites">
  ## Pré-requisitos
</div>

* Uma chave de API do Firecrawl
* Uma aplicação Express
* Node.js 18+

<div id="install">
  ## Instalar
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="configure-environment">
  ## Configure o ambiente
</div>

Armazene a chave de API no `.env` ou nas variáveis de ambiente do processo.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## Adicione uma rota de servidor
</div>

server.js

```js
import express from 'express';
import Firecrawl from '@mendable/firecrawl-js';

const app = express();
app.use(express.json());

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

app.post('/api/firecrawl', async (req, res) => {
  const { url } = req.body;
  const doc = await client.scrape(url, { formats: ['markdown'] });
  res.json({ markdown: doc.markdown });
});

app.listen(3000, () => console.log('Server listening on :3000'));
```

<div id="call-it-from-your-app">
  ## Chame-o do seu app
</div>

```js
const response = await fetch('http://localhost:3000/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

<div id="notes">
  ## Notas
</div>

* Mantenha as chamadas do Firecrawl no servidor; nunca exponha chaves de API em bundles do lado do cliente.

A Firecrawl oferece diferentes tipos de proxy para ajudar você a fazer scraping de sites com níveis variados de proteção anti-bot. O tipo de proxy pode ser especificado usando o parâmetro `proxy`.

> Por padrão, a Firecrawl encaminha todas as requisições por meio de proxies para garantir confiabilidade e acesso, mesmo que você não especifique um tipo de proxy ou localização.

<div id="location-based-proxy-selection">
  ## Seleção de proxy por localização
</div>

O Firecrawl seleciona automaticamente o melhor proxy com base na sua localização especificada ou detectada, otimizando o desempenho e a confiabilidade da coleta. Porém, nem todas as regiões são compatíveis no momento. As seguintes localidades estão disponíveis:

| Country Code | Country Name         | Stealth Mode Support |
|--------------|----------------------|----------------------|
| AE           | United Arab Emirates | No                   |
| AU           | Australia            | No                   |
| BR           | Brazil               | Yes                  |
| CN           | China                | No                   |
| DE           | Germany              | Yes                  |
| FR           | France               | Yes                  |
| GB           | United Kingdom       | No                   |
| JP           | Japan                | No                   |
| QA           | Qatar                | No                   |
| TR           | Turkey               | No                   |
| US           | United States        | Yes                  |
| VN           | Vietnam              | No                   |

<Warning>A lista de localidades de proxy compatíveis foi atualizada pela última vez em 15 de maio de 2025. A disponibilidade pode mudar ao longo do tempo.</Warning>

Se você precisar de proxies em uma localidade não listada acima, [entre em contato](mailto:help@firecrawl.com) e informe seus requisitos.

Se você não especificar um proxy ou uma localidade, o Firecrawl selecionará automaticamente a melhor opção com base no site de destino e na sua solicitação.

<div id="how-to-specify-proxy-location">
  ## Como especificar a localização do proxy
</div>

Você pode solicitar uma localização específica de proxy definindo o parâmetro `location.country` na sua requisição. Por exemplo, para usar um proxy no Brasil, defina `location.country` como `BR`.

Para mais detalhes, consulte a [referência da API para `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location-country).

<div id="proxy-types">
  ## Tipos de proxy
</div>

Firecrawl oferece suporte a três tipos de proxy:

* **basic**: Proxy para scraping de sites com pouca ou nenhuma proteção anti-bot. Rápido e geralmente funciona.
* **stealth**: Proxy furtivo para scraping de sites com proteções anti-bot avançadas ou que bloqueiam proxies comuns. Mais lento, porém mais confiável em certos sites. [Saiba mais sobre o Modo Stealth →](/pt-BR/features/stealth-mode)
* **auto**: O Firecrawl tentará automaticamente usar proxy stealth se o proxy basic falhar. Se a nova tentativa com stealth tiver sucesso, 5 créditos serão cobrados pelo scraping. Se a primeira tentativa com basic for bem-sucedida, apenas o custo regular será cobrado.

***

> **Nota:** Para informações detalhadas sobre o uso de proxies stealth, incluindo custos de crédito e estratégias de nova tentativa, consulte a [documentação do Modo Stealth](/pt-BR/features/stealth-mode).

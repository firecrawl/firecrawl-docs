A Firecrawl oferece diferentes tipos de proxy para ajudar você a fazer scraping de sites com níveis variados de complexidade. O tipo de proxy pode ser especificado usando o parâmetro `proxy`.

> Por padrão, a Firecrawl encaminha todas as requisições por meio de proxies para garantir confiabilidade e acesso, mesmo que você não especifique um tipo de proxy ou localização.

<div id="how-to-specify-proxy-location">
  ## Como especificar a localização do proxy
</div>

Você pode solicitar uma localização específica de proxy definindo o parâmetro `location.country` na sua requisição. Por exemplo, para usar um proxy no Brasil, defina `location.country` como `BR`.

Para mais detalhes, consulte a [referência da API para `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location-country).

<div id="proxy-types">
  ## Tipos de proxy
</div>

Firecrawl oferece suporte a três tipos de proxy:

* **basic**: Proxy para scraping da maioria dos sites. Rápido e geralmente funciona.
* **enhanced**: Proxy aprimorado para scraping de sites complexos, mantendo a privacidade. Mais lento, porém mais confiável em certos sites. [Saiba mais sobre o Modo Enhanced →](/pt-BR/features/enhanced-mode)
* **auto**: O Firecrawl tentará automaticamente usar proxy enhanced se o proxy basic falhar. Se a nova tentativa com enhanced tiver sucesso, 5 créditos serão cobrados pelo scraping. Se a primeira tentativa com basic for bem-sucedida, apenas o custo regular será cobrado.

***

> **Nota:** Para informações detalhadas sobre o uso de proxies enhanced, incluindo custos de crédito e estratégias de nova tentativa, consulte a [documentação do Modo Enhanced](/pt-BR/features/enhanced-mode).

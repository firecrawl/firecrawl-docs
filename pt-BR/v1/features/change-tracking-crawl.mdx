O rastreamento de mudanças fica ainda mais poderoso quando combinado com o crawling. Enquanto o rastreamento em páginas individuais mostra alterações de conteúdo, usá-lo com o crawl permite monitorar toda a estrutura do seu site — exibindo páginas novas, removidas e que se tornaram ocultas.

<div id="basic-usage">
  ## Uso básico
</div>

Para ativar o rastreio de mudanças durante um crawl, inclua-o no array `formats` em `scrapeOptions`:

```typescript
// JavaScript/TypeScript
const app = new FirecrawlApp({ apiKey: 'sua-chave-de-api' });
const result = await app.crawl('https://example.com', {
  scrapeOptions: {
    formats: ['markdown', 'rastreioDeMudanças']
  }
});
```

```python
# Python
app = FirecrawlApp(api_key='sua-api-key')
result = app.crawl('https://firecrawl.dev', {
    'scrapeOptions': {
        'formats': ['markdown', 'changeTracking']
    }
})
```

<div id="understanding-change-status">
  ## Entendendo o status de mudanças
</div>

Ao usar o rastreioDeMudanças com o crawl, o campo `changeStatus` se torna especialmente útil:

* `new`: Uma página que não existia no seu crawl anterior
* `same`: Uma página que existe e não mudou desde o seu último crawl
* `changed`: Uma página que existe, mas foi modificada desde o seu último crawl
* `removed`: Uma página que existia no seu crawl anterior, mas não é mais encontrada

<div id="page-visibility">
  ## Visibilidade da página
</div>

O campo `visibility` ajuda você a entender como as páginas são descobertas:

* `visible`: A página pode ser encontrada por meio de links ou do sitemap
* `hidden`: A página ainda existe, mas não está mais vinculada nem presente no sitemap

Isso é particularmente útil para:

* Detectar conteúdo órfão
* Encontrar páginas removidas acidentalmente da navegação
* Monitorar mudanças na estrutura do site
* Identificar conteúdo que deve ser novamente vinculado ou removido

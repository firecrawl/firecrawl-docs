<Info>
  **Endpoint de busca com pagamento por uso**

  Este endpoint permite usar os recursos avançados de busca do Firecrawl sem precisar de assinatura, pagando por busca com criptomoedas. Ideal para buscas pontuais ou testes antes de optar por uma assinatura.
</Info>

<Warning>
  **Recurso experimental**

  Este é um endpoint experimental. Recursos, preços e disponibilidade podem mudar sem aviso prévio. Recomendamos usar o endpoint de busca padrão, baseado em assinatura, para casos de uso em produção.
</Warning>

<div id="authentication">
  ## Autenticação
</div>

Este endpoint exige um cabeçalho de pagamento especial em vez da chave de API padrão:

```
X-Pagamento: {{paymentHeader}}
```

<div id="how-it-works">
  ## Como funciona
</div>

O endpoint de Busca Sob Demanda combina a pesquisa na web (SERP) com os recursos de scraping do Firecrawl para retornar o conteúdo completo da página para qualquer consulta, assim como o endpoint `/search` convencional, mas com cobrança por uso.

Inclua `scrapeOptions` com `formats: ["markdown"]` para obter o conteúdo completo em markdown de cada resultado da pesquisa; caso contrário, por padrão você receberá apenas os resultados do SERP (url, title, description).

<div id="request-parameters">
  ### Parâmetros da requisição
</div>

Todos os parâmetros do endpoint `/search` padrão são aceitos, com a seguinte exceção:

<Note>
  **Limitação importante**: O parâmetro `limit` é limitado a **10 resultados no máximo** para o endpoint de wallet (em comparação a 100 no endpoint padrão).
</Note>

Para a lista completa de parâmetros disponíveis, operadores de consulta compatíveis e suas descrições, consulte a [documentação do endpoint Search padrão](/pt-BR/api-reference/endpoint/search).

<div id="example-request">
  ## Exemplo de requisição
</div>

```bash
curl -X POST https://api.firecrawl.dev/v1/x402/search \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <token>' \
  -H 'X-Payment: {{paymentHeader}}' \
  -d '{
    "query": "Jogos da Premier League para a temporada 2025/26",
    "limit": 10,  # Máximo de 10 resultados permitido
    "scrapeOptions": {
      "formats": ["markdown"],
      "onlyMainContent": true
    }
  }'
```

<div id="response">
  ## Resposta
</div>

O formato da resposta é idêntico ao do endpoint `/search` padrão:

```json
{
  "success": true,
  "data": [
    {
      "title": "Título da página",
      "description": "Descrição da página no resultado de busca",
      "url": "https://example.com",
      "markdown": "# Conteúdo da página\n\nConteúdo em Markdown se a raspagem tiver sido solicitada...",
      "metadata": {
        "title": "Título da página",
        "description": "Descrição meta"
        "sourceURL": "https://example.com",
        "statusCode": 200
      }
    }
  ]
}
```

  </Step>
</Steps>

<div id="firecrawl-actions-overview">
  ## Visão geral das ações do Firecrawl
</div>

| Módulo | Caso de uso | Melhor para |
|--------|-------------|-------------|
| **Scrape a Website** | Extração de dados de uma única página | Captura rápida de dados |
| **Crawl a Website** | Coleta de conteúdo de várias páginas | Raspagem completa do site |
| **Extract a Website** | Extração estruturada com IA | Demandas de dados complexas |
| **Search a Website** | SERP + conteúdo completo | Automação de pesquisa |
| **Map a Website** | Descoberta de URLs | Análise de SEO |

<div id="best-practices">
  ## Melhores práticas
</div>

<CardGroup cols={2}>
  <Card title="Desempenho" icon="gauge-high">
    - Use **Scrape** para páginas individuais (mais rápido)
    - Use **Crawl** com limites para sites grandes
    - Agende de forma adequada para evitar rate limits
    - Adicione módulos de tratamento de erros
  </Card>

  <Card title="Otimização de custos" icon="piggy-bank">
    - Agende de forma estratégica (de hora em hora/diário/semanal)
    - Use filtros para evitar execuções desnecessárias
    - Defina limites de crawl para controlar o uso da API
    - Teste primeiro no playground do Firecrawl
  </Card>
</CardGroup>

<div id="make-vs-zapier-vs-n8n">
  ## Make vs Zapier vs n8n
</div>

| Recurso | Make | Zapier | n8n |
|---------|------|--------|-----|
| **Configuração** | Construtor visual, nuvem | No-code, nuvem | Auto-hospedado ou nuvem |
| **Preços** | Baseado em operações | Preço por tarefa | Mensal fixo |
| **Integrações** | 3.000+ apps | 8.000+ apps | 400+ integrações |
| **Complexidade** | Workflows avançados | Workflows simples | Workflows complexos |
| **Melhor Para** | Automação visual, complexidade média | Automação rápida | Controle para desenvolvedores |
| **Curva de Aprendizado** | Moderada | Fácil | Moderada a avançada |

<Tip>
**Dica pro:** O Make se destaca no design visual de workflows e automações complexas. Ideal para equipes que precisam de mais controle do que o Zapier, mas preferem a construção visual em vez da abordagem code-first do n8n.
</Tip>

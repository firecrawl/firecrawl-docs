<div id="browser-actions">
  ## Ações do navegador
</div>

Você pode realizar interações no navegador antes do scraping usando o parâmetro `actions`. Isso permite que você:

* Aguarde o carregamento de elementos
* Clique em botões ou links
* Preencha campos de formulários
* Pressione teclas
* Role páginas
* Tire capturas de tela
* Execute JavaScript personalizado
* Gere PDFs

As ações compatíveis incluem `wait`, `click`, `write`, `press`, `scroll`, `screenshot`, `scrape`, `executeJavascript` e `pdf`. Para ver a documentação detalhada com exemplos, consulte o [Guia Avançado de Scraping](https://docs.firecrawl.dev/advanced-scraping-guide#actions-actions).

<div id="persistent-profiles">
  ## Perfis persistentes
</div>

Passe o parâmetro `profile` para preservar o estado do navegador (cookies, localStorage, sessões) ao longo de sessões de scraping e [interagir](/pt-BR/features/interact). Execuções de scraping com o mesmo nome de perfil compartilham o estado do navegador — útil para continuar conectado ou preservar preferências.

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/dashboard",
    "profile": {
      "name": "my-session",
      "saveChanges": true
    }
  }'
```

| Parâmetro             | Tipo    | Obrigatório | Padrão | Descrição                                                                                                                                                                |
| --------------------- | ------- | ----------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `profile.name`        | string  | Sim         | —      | Um nome para o perfil (1 a 128 caracteres). Solicitações de scraping com o mesmo nome compartilham o estado do navegador.                                                |
| `profile.saveChanges` | boolean | Não         | `true` | Quando `true`, o estado do navegador é salvo novamente no perfil quando a sessão do Interact é encerrada. Defina como `false` para carregar dados existentes sem gravar. |

<Note>
  Apenas uma sessão pode salvar em um perfil por vez. Se outra sessão já estiver salvando, você receberá um erro `409`. Você ainda pode abrir o mesmo perfil com `saveChanges: false` ou tentar novamente mais tarde.
</Note>

Para mais detalhes, consulte [Interact — Perfis persistentes](/pt-BR/features/interact#persistent-profiles).

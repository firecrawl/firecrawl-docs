<div id="firecrawl-elixir-source-of-truth">
  # Firecrawl Elixir fonte oficial
</div>

Este arquivo é a fonte pública canônica oficial para agentes. Ele é gerado a partir do SDK Elixir do Firecrawl (no formato OpenAPI) e da especificação OpenAPI v2.

<div id="install">
  ## Instalação
</div>

```elixir
# mix.exs
defp deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

<div id="authenticate">
  ## Autenticação
</div>

Configure sua API key em `config/runtime.exs`:

```elixir
config :firecrawl, api_key: System.fetch_env!("FIRECRAWL_API_KEY")
```

Ou passe `api_key` em cada requisição:

```elixir
{:ok, resp} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-...")
```

<div id="when-to-use-what">
  ## Quando usar cada opção
</div>

* `search`: comece com uma consulta para encontrar URLs relevantes, incluindo filtros `site:`.
* `scrape`: obtenha e extraia conteúdo de uma URL conhecida.
* `interact`: execute ações no navegador ou código quando uma página exigir cliques, formulários ou etapas após o scraping.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use a busca quando precisar descobrir resultados com base em uma consulta e quiser que o Firecrawl retorne URLs (e, opcionalmente, faça scraping de cada resultado). Você pode restringir os resultados a um domínio com o operador padrão `site:`, por exemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`Firecrawl.search_and_scrape(params, opts)`

### Exemplo básico

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000,
    actions: [
      %{type: "click", selector: "button#accept"},
      %{type: "wait", milliseconds: 750},
      %{type: "scrape"}
    ]
  ]
)
```

<div id="parameters">
  ### Parâmetros
</div>

| Parâmetro             | Tipo / Opções                                                 | Descrição                                                                                     |
| --------------------- | ------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| `query`               | `String`                                                      | Consulta de busca. Use `site:domain` para restringir os resultados a um site específico.      |
| `sources`             | `List` (`"web"`, `"news"`, `"images"` ou maps com `type`)     | Fontes para pesquisar.                                                                        |
| `categories`          | `List` (`"github"`, `"research"`, `"pdf"` ou maps com `type`) | Categorias para filtrar os resultados.                                                        |
| `limit`               | `integer`                                                     | Número máximo de resultados retornados.                                                       |
| `tbs`                 | `String`                                                      | Filtro de busca por tempo (por exemplo, `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).                     |
| `location`            | `String`                                                      | Localização dos resultados de busca.                                                          |
| `country`             | `String`                                                      | Código de país ISO para segmentação geográfica dos resultados de busca.                       |
| `enterprise`          | `List[String]` (`"anon"`, `"zdr"`)                            | Opções do Enterprise ZDR, quando habilitadas para sua equipe.                                 |
| `ignore_invalid_urls` | `boolean`                                                     | Exclui URLs inválidas que outros endpoints do Firecrawl não conseguem processar com scraping. |
| `timeout`             | `integer`                                                     | Tempo limite em milissegundos.                                                                |
| `scrape_options`      | `keyword`                                                     | Opções de scraping aplicadas a cada resultado (os mesmos campos de `scrape`).                 |

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o scraping quando você já tiver uma URL e quiser que o Firecrawl retorne conteúdo estruturado e formatos opcionais.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`Firecrawl.scrape_and_extract_from_url(params, opts)`

<div id="simple-example">
  ### Exemplo simples
</div>

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  include_tags: ["main", "article"],
  exclude_tags: ["nav", "footer"],
  only_main_content: true,
  wait_for: 1000,
  mobile: false,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "write", text: "firecrawl"},
    %{type: "press", key: "Enter"},
    %{type: "wait", milliseconds: 1500},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  skip_tls_verification: true,
  remove_base64_images: true,
  block_ads: true,
  proxy: "auto",
  max_age: 86400000,
  store_in_cache: true,
  profile: [name: "docs-session", saveChanges: true]
)
```

<div id="parameters">
  ### Parâmetros
</div>

| Parâmetro               | Tipo / Opções      | Descrição                                                                                                                                           |                   |                |
| ----------------------- | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------- | -------------- |
| `url`                   | `String`           | URL para scraping.                                                                                                                                  |                   |                |
| `formats`               | `List`             | Formatos de resultado a incluir. Strings ou objetos como `%{type: "json", prompt, schema}` ou `%{type: "screenshot", fullPage, quality, viewport}`. |                   |                |
| `only_main_content`     | `boolean`          | Manter apenas o conteúdo principal.                                                                                                                 |                   |                |
| `include_tags`          | `List[String]`     | Tags a incluir.                                                                                                                                     |                   |                |
| `exclude_tags`          | `List[String]`     | Tags a excluir.                                                                                                                                     |                   |                |
| `max_age`               | `integer`          | Dar preferência a dados em cache até esta idade.                                                                                                    |                   |                |
| `min_age`               | `integer`          | Retornar dados em cache apenas se atenderem à idade mínima.                                                                                         |                   |                |
| `headers`               | `map`              | Cabeçalhos personalizados.                                                                                                                          |                   |                |
| `wait_for`              | `integer`          | Tempo de espera em milissegundos antes do scraping.                                                                                                 |                   |                |
| `mobile`                | `boolean`          | Emular um dispositivo móvel.                                                                                                                        |                   |                |
| `skip_tls_verification` | `boolean`          | Ignorar a verificação do certificado TLS.                                                                                                           |                   |                |
| `timeout`               | `integer`          | Tempo limite em milissegundos.                                                                                                                      |                   |                |
| `parsers`               | `List`             | Configuração de parser (com suporte a opções de PDF).                                                                                               |                   |                |
| `actions`               | `List`             | Ações do navegador a executar antes do scraping.                                                                                                    |                   |                |
| `location`              | `keyword`          | Configurações de localidade para a requisição.                                                                                                      |                   |                |
| `remove_base64_images`  | `boolean`          | Remover imagens em base64 da saída em markdown.                                                                                                     |                   |                |
| `block_ads`             | `boolean`          | Ativar o bloqueio de anúncios e pop-ups de cookies.                                                                                                 |                   |                |
| `proxy`                 | `&quot;basic&quot; | &quot;enhanced&quot;                                                                                                                                | &quot;auto&quot;` | Modo de proxy. |
| `store_in_cache`        | `boolean`          | Armazenar resultados em cache.                                                                                                                      |                   |                |
| `profile`               | `keyword`          | Perfil persistente do navegador, por exemplo `[name: "profile", saveChanges: true]`.                                                                |                   |                |
| `zero_data_retention`   | `boolean`          | Ativar zero data retention quando sua conta permitir.                                                                                               |                   |                |

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interact` quando uma página exigir ações no navegador ou execução de código após o início do scraping (cliques, formulários ou etapas com script).

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

<div id="simple-example">
  ### Exemplo simples
</div>

```elixir
{:ok, scrape_resp} = Firecrawl.scrape_and_extract_from_url(url: "https://example.com")
job_id = get_in(scrape_resp.body, ["data", "metadata", "scrapeId"])

{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  job_id,
  code: "console.log(document.title)",
  language: :node
)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  "<scrape_job_id>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="parameters">
  ### Parâmetros
</div>

| Parâmetro  | Tipo / Opções | Descrição                                                            |        |                                                                 |
| ---------- | ------------- | -------------------------------------------------------------------- | ------ | --------------------------------------------------------------- |
| `job_id`   | `String`      | ID do job de scraping em `data.metadata.scrapeId`.                   |        |                                                                 |
| `code`     | `String`      | Código a ser executado na sessão do navegador. Obrigatório.          |        |                                                                 |
| `language` | `:python      | :node                                                                | :bash` | Linguagem de execução. O valor padrão é `:node` quando omitido. |
| `timeout`  | `integer`     | Tempo limite de execução em segundos.                                |        |                                                                 |
| `origin`   | `String`      | Apenas para uso interno. Reservado para instrumentação do Firecrawl. |        |                                                                 |

<div id="notes">
  ## Observações
</div>

* As funções do cliente Elixir seguem o padrão do OpenAPI e aceitam parâmetros nomeados em snake&#95;case, que são convertidos em campos JSON em camelCase.

<div id="firecrawl-python-source-of-truth">
  # fonte oficial do Firecrawl Python
</div>

Este arquivo é a referência pública canônica para agentes. Ele é gerado a partir do código-fonte do SDK Python do Firecrawl e da especificação OpenAPI v2.

<div id="install">
  ## Instalação
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## Autenticação
</div>

```py
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ.get("FIRECRAWL_API_KEY"))
# client = Firecrawl(api_key="fc-...", api_url="https://api.firecrawl.dev")
```

<div id="when-to-use-what">
  ## Quando usar cada um
</div>

* `search`: parta de uma consulta para encontrar URLs relevantes, incluindo restrições `site:`.
* `scrape`: busque e extraia conteúdo de uma URL conhecida.
* `interact`: execute ações do navegador ou código quando uma página exigir cliques, formulários ou etapas após o scraping.

<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use a busca quando precisar descobrir resultados a partir de uma consulta e quiser que o Firecrawl retorne URLs (e, opcionalmente, faça scraping de cada resultado). Você pode restringir os resultados a um domínio com o operador padrão `site:`, por exemplo `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.search(query, **options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```py
results = client.search("site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```py
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:m",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    timeout=300000,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            "links",
            {"type": "json", "prompt": "Extract title and key endpoints."}
        ],
        only_main_content=True,
        include_tags=["main", "article"],
        exclude_tags=["nav", "footer"],
        wait_for=1000,
        actions=[
            {"type": "click", "selector": "button#accept"},
            {"type": "wait", "milliseconds": 750},
            {"type": "scrape"}
        ]
    )
)
```

<div id="parameters">
  ### Parâmetros
</div>

| Parâmetro             | Tipo / Opções                                                  | Descrição                                                                                   |
| --------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| `query`               | `str`                                                          | Consulta de busca. Use `site:domain` para restringir os resultados a um site.               |
| `sources`             | `lista de nomes de fontes ou objetos de fonte tipados`         | Fontes a pesquisar.                                                                         |
| `categories`          | `lista de nomes de categorias ou objetos de categoria tipados` | Categorias para filtrar os resultados.                                                      |
| `limit`               | `int`                                                          | Número máximo de resultados a retornar. O padrão é `5` no modelo do SDK.                    |
| `tbs`                 | `str`                                                          | Filtro de busca baseado em tempo (por exemplo `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).             |
| `location`            | `str`                                                          | String de localização para os resultados de busca.                                          |
| `ignore_invalid_urls` | `bool`                                                         | Exclui URLs inválidas que outros endpoints do Firecrawl não conseguem fazer scraping.       |
| `timeout`             | `int`                                                          | Tempo limite em milissegundos. O padrão é `300000` no modelo do SDK.                        |
| `scrape_options`      | `ScrapeOptions`                                                | Opções de scraping aplicadas a cada resultado (mesmos campos que `scrape`, mais `min_age`). |
| `integration`         | `str`                                                          | Somente para uso interno. Reservado para instrumentação do Firecrawl.                       |

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o `scrape` quando você já tiver uma URL e quiser que o Firecrawl retorne conteúdo estruturado e formatos opcionais.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.scrape(url, **options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```py
doc = client.scrape("https://docs.firecrawl.dev", formats=["markdown"])
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```py
doc = client.scrape(
    "https://example.com/pricing",
    formats=[
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract plan names and prices."},
        {"type": "screenshot", "full_page": True, "quality": 80, "viewport": {"width": 1280, "height": 720}},
        {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    headers={"User-Agent": "FirecrawlDocsBot/1.0"},
    include_tags=["main", "article"],
    exclude_tags=["nav", "footer"],
    only_main_content=True,
    wait_for=1000,
    mobile=False,
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    actions=[
        {"type": "click", "selector": "#accept"},
        {"type": "write", "text": "firecrawl"},
        {"type": "press", "key": "Enter"},
        {"type": "wait", "milliseconds": 1500},
        {"type": "scrape"}
    ],
    location={"country": "US", "languages": ["en-US"]},
    skip_tls_verification=True,
    remove_base64_images=True,
    fast_mode=True,
    block_ads=True,
    proxy="auto",
    max_age=86400000,
    store_in_cache=True,
    profile={"name": "docs-session", "save_changes": True}
)
```

  ### Parâmetros
</div>

| Parâmetro                | Tipo / Opções                                                                                                                                                              | Descrição                                                                                                                          |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `url`                    | `str`                                                                                                                                                                      | URL para scraping.                                                                                                                 |

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use o interact quando uma página exigir ações no navegador ou execução de código depois que o scraping começar (cliques, formulários ou etapas com script).

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.interact(job_id, code=None, prompt=None, language="node", timeout=None)`

<div id="simple-example">
  ### Exemplo simples
</div>

```py
doc = client.scrape("https://example.com", formats=["markdown"])
job_id = doc.metadata.scrape_id if doc.metadata else None
if not job_id:
    raise ValueError("Missing scrape_id from scrape response")

result = client.interact(
    job_id,
    prompt="Click the pricing tab and summarize the plans.",
    language="node"
)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```py
result = client.interact(
    "<scrape_job_id>",
    code="print(await page.title())",
    language="python",
    timeout=60
)
```

<div id="parameters">
  ### Parâmetros
</div>

| Parâmetro  | Tipo / Opções          | Descrição                                                                                           |
| ---------- | ---------------------- | --------------------------------------------------------------------------------------------------- |
| `job_id`   | `str`                  | ID do job de scraping em `document.metadata.scrape_id`.                                             |
| `code`     | `str`                  | Código a ser executado na sessão do navegador. Obrigatório se `prompt` não for fornecido.           |
| `prompt`   | `str`                  | Instrução em linguagem natural para o agente do navegador. Obrigatório se `code` não for fornecido. |
| `language` | `python, node ou bash` | Linguagem de execução. O padrão é `node`.                                                           |
| `timeout`  | `int`                  | Tempo limite de execução em segundos (1-300).                                                       |
| `origin`   | `str`                  | Apenas para uso interno. Reservado para a instrumentação do Firecrawl.                              |

<div id="notes">
  ## Notas
</div>

* Aliases descontinuados: `scrape_execute`, `stop_interactive_browser` e `delete_scrape_browser` correspondem a `interact` e `stop_interaction`.
* `ScrapeOptions` inclui `min_age`, mas o método de conveniência `scrape` não o disponibiliza. Use `ScrapeOptions` por meio de `search(..., scrape_options=...)` se precisar dele.
* O cliente principal `Firecrawl` expõe os métodos v2 diretamente; a versão v1 legada está disponível em `client.v1`.

<div id="firecrawl-rust-agent-quickstart">
  # Guia de início rápido do agente Rust do Firecrawl
</div>

Este arquivo é o guia de início rápido oficial para agentes externos e é gerado a partir do código-fonte do SDK e da especificação OpenAPI v2.

<div id="install">
  ## Instalar
</div>

```bash
cargo add firecrawl
```

<div id="authenticate">
  ## Autenticação
</div>

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
```

<div id="when-to-use-what">
  ## Quando usar cada um
</div>

* `search`: use quando você começa com uma consulta e precisa descobrir resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou ações no navegador após o scraping.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `search` para encontrar URLs relevantes a partir de uma consulta e, opcionalmente, fazer scraping de cada resultado. Você pode restringir a consulta a um único site com o operador padrão `site:` (por exemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de fazer scraping de uma URL específica.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.search(query, options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
let results = client.search("site:docs.firecrawl.dev crawl webhooks", None).await?;

if let Some(web) = results.data.web {
    println!("{:?}", web.first());
}
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```rust
use firecrawl::v2::{Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format};

let client = Client::new("fc-your-api-key")?;
let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:w".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Json]),
        only_main_content: Some(true),
        ..Default::default()
    }),
    integration: Some("agent-quickstart".to_string()),
};

let results = client.search("firecrawl pricing", options).await?;
```

<div id="parameters">
  ### Parâmetros
</div>

* `query`: String de consulta de busca. Sempre obrigatória.
* `limit`: Número máximo de resultados a retornar. Use para limitar o tamanho da resposta.
* `sources`: Fontes para busca. Valores compatíveis: `SearchSource::Web`, `SearchSource::News`, `SearchSource::Images`.
* `categories`: Filtros de resultado. Valores compatíveis: `SearchCategory::Github`, `SearchCategory::Research`, `SearchCategory::Pdf`.
* `tbs`: String de filtro de busca baseado em tempo. Use para limitar por intervalo de tempo (por exemplo, `qdr:d`).
* `location`: String de localização para resultados localizados. Use em consultas específicas por região.
* `ignore_invalid_urls`: Exclui URLs inválidas dos resultados. Use ao encaminhar para outros endpoints.
* `timeout`: Tempo limite da requisição em milissegundos. Use para limitar o tempo de busca.
* `scrape_options`: Opções de scraping aplicadas a cada resultado (mesmos campos de `scrape`). Use quando quiser incluir conteúdo nos resultados.
* `integration`: Identificador da integração. Use para rastreamento/análise.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `scrape` quando você já souber a URL e quiser o conteúdo em formatos específicos.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```rust
use firecrawl::v2::{Client, ScrapeOptions, Format};

let client = Client::new("fc-your-api-key")?;
let doc = client
    .scrape(
        "https://example.com",
        ScrapeOptions {
            formats: Some(vec![Format::Markdown]),
            ..Default::default()
        },
    )
    .await?;

println!("{:?}", doc.markdown);
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```rust
use firecrawl::v2::{
    Action, ChangeTrackingMode, ChangeTrackingOptions, Client, Format, JsonOptions, LocationConfig,
    ParserConfig, ProxyType, ScrapeOptions, ScreenshotOptions, ScrollDirection, Viewport,
};

let client = Client::new("fc-your-api-key")?;
let options = ScrapeOptions {
    formats: Some(vec![Format::Markdown, Format::Json, Format::ChangeTracking, Format::Screenshot]),
    json_options: Some(JsonOptions {
        prompt: Some("Extract headings and slugs.".to_string()),
        ..Default::default()
    }),
    change_tracking_options: Some(ChangeTrackingOptions {
        modes: Some(vec![ChangeTrackingMode::GitDiff]),
        tag: Some("docs".to_string()),
        ..Default::default()
    }),
    screenshot_options: Some(ScreenshotOptions {
        full_page: Some(true),
        viewport: Some(Viewport { width: 1280, height: 720 }),
        ..Default::default()
    }),
    actions: Some(vec![
        Action::Wait {
            milliseconds: Some(1000),
            selector: None,
        },
        Action::Click {
            selector: "button#accept".to_string(),
        },
        Action::Scroll {
            direction: ScrollDirection::Down,
            selector: None,
        },
    ]),
    parsers: Some(vec![ParserConfig::Pdf {
        parser_type: "pdf".to_string(),
        max_pages: Some(5),
    }]),
    location: Some(LocationConfig {
        country: Some("US".to_string()),
        languages: Some(vec!["en-US".to_string()]),
    }),
    proxy: Some(ProxyType::Stealth),
    ..Default::default()
};

let doc = client.scrape("https://example.com/docs", options).await?;
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`: URL para fazer scraping. Sempre obrigatória.
* `formats`: Formatos de resultado a serem incluídos. Valores compatíveis: `Format::Markdown`, `Format::Html`, `Format::RawHtml`, `Format::Links`, `Format::Images`, `Format::Screenshot`, `Format::Summary`, `Format::ChangeTracking`, `Format::Json`, `Format::Attributes`, `Format::Branding`, `Format::Audio`.
* `headers`: Cabeçalhos HTTP adicionais. Use para autenticação, cookies ou user-agent.
* `include_tags`: Inclua apenas conteúdo destas tags HTML. Use para restringir a extração.
* `exclude_tags`: Exclua conteúdo destas tags HTML. Use para remover ruído.
* `only_main_content`: Extraia apenas o conteúdo principal. Use para remover navegação/rodapé.
* `timeout`: Tempo limite do scraping em milissegundos. Use para limitar o tempo de execução.
* `wait_for`: Atraso antes do scraping em milissegundos. Use para páginas renderizadas com JS.
* `mobile`: Emule um dispositivo móvel. Use para layouts móveis.
* `parsers`: Lista de parsers ou configurações. Valores compatíveis: `ParserConfig::Simple(String)` ou `ParserConfig::Pdf { parser_type: "pdf", max_pages: Option<u32> }`.
* `actions`: Ações do navegador a executar antes do scraping. Ações compatíveis: `Wait { milliseconds?, selector? }`, `Screenshot { full_page?, quality?, viewport? }`, `Click { selector }`, `Write { text }`, `Press { key }`, `Scroll { direction: Up | Down, selector? }`, `Scrape`, `ExecuteJavascript { script }`, `Pdf { format?, landscape?, scale? }` com formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuração de localização `{ country, languages }`. Use para páginas específicas por região.
* `skip_tls_verification`: Ignore a verificação do certificado TLS. Use para certificados autoassinados.
* `remove_base64_images`: Remova imagens em base64 do resultado em markdown. Use para reduzir o tamanho do payload.
* `block_ads`: Ative o bloqueio de anúncios e cookies. Use para reduzir ruído.
* `proxy`: Modo de proxy `ProxyType::Basic`, `ProxyType::Stealth`, `ProxyType::Enhanced`, `ProxyType::Auto`.
* `max_age`: Aceite conteúdo em cache com idade inferior a este valor (ms). Use para favorecer acertos de cache.
* `min_age`: Retorne apenas conteúdo em cache com pelo menos esta idade (ms). Use para leituras somente de cache.
* `store_in_cache`: Armazene o resultado em cache/índice. Use quando quiser reutilizá-lo.
* `profile`: Perfil persistente do navegador `ProfileConfig { name, save_changes }`. Use para manter o estado da sessão.
* `integration`: Identificador da integração. Use para rastreamento/analytics.
* `json_options`: Opções de extração JSON para o formato `Format::Json` (`schema`, `system_prompt`, `prompt`).
* `screenshot_options`: Opções de captura de tela para o formato `Format::Screenshot` (`full_page`, `quality`, `viewport`).
* `change_tracking_options`: Opções de rastreamento de mudanças para o formato `Format::ChangeTracking` (`modes`, `schema`, `prompt`, `tag`). Modos: `ChangeTrackingMode::GitDiff`, `ChangeTrackingMode::Json`.
* `attribute_selectors`: Seletores de atributos para o formato `Format::Attributes` (`selector`, `attribute`).

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interact` quando a página exigir cliques, preenchimento de formulários ou outras ações no navegador após o scraping.

### Método preferido do SDK

`client.interact(job_id, options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```rust
use firecrawl::v2::{Client, ScrapeExecuteOptions};

let client = Client::new("fc-your-api-key")?;
let run = client
    .interact(
        "scrape-job-id",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.url())".to_string()),
            ..Default::default()
        },
    )
    .await?;

println!("{:?}", run.stdout);
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```rust
use firecrawl::v2::{Client, ScrapeExecuteLanguage, ScrapeExecuteOptions};

let client = Client::new("fc-your-api-key")?;
let run = client
    .interact(
        "scrape-job-id",
        ScrapeExecuteOptions {
            prompt: Some("Open the pricing page and list the plan names.".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            origin: Some("agent-docs".to_string()),
            ..Default::default()
        },
    )
    .await?;
```

<div id="parameters">
  ### Parâmetros
</div>

* `job_id`: ID do job de scraping (de `document.metadata.scrape_id`). Sempre obrigatório.
* `code`: Código a ser executado no sandbox do navegador. Use para interações via script.
* `prompt`: Instrução em linguagem natural para o agente do navegador. Use quando preferir ações orientadas por agente.
* `language`: Linguagem de execução `ScrapeExecuteLanguage::Python`, `ScrapeExecuteLanguage::Node` ou `ScrapeExecuteLanguage::Bash` (o padrão é `Node`).
* `timeout`: Tempo limite de execução em segundos. Use para limitar execuções longas.
* `origin`: Tag de origem opcional. Use para atribuição da requisição.

<div id="notes">
  ## Notas
</div>

* Os parâmetros usam `snake_case` em Rust; o JSON é serializado como `camelCase`.
* Método de conveniência: `search_and_scrape` executa uma busca com scraping habilitado.
* Aliases descontinuados: `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (use `interact` e `stop_interaction`).

<div id="firecrawl-elixir-agent-quickstart">
  # Início rápido do agente Elixir da Firecrawl
</div>

Este arquivo é o guia oficial de início rápido para agentes externos e é gerado a partir do código-fonte do SDK e da especificação OpenAPI v2.

<div id="install">
  ## Instalação
</div>

Adicione a dependência no `mix.exs`:

```elixir
def deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

Em seguida, execute:

```bash
mix deps.get
```

<div id="authenticate">
  ## Autentique-se
</div>

```elixir
# config/runtime.exs ou config/config.exs
config :firecrawl, api_key: "fc-your-api-key"
```

Ou informe em cada chamada:

```elixir
Firecrawl.scrape_and_extract_from_url([url: "https://example.com"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## Quando Usar Cada Um
</div>

* `search`: use quando você parte de uma consulta e precisa descobrir resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou ações no navegador após o scraping.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `search` para descobrir URLs relevantes com base em uma consulta e, opcionalmente, fazer scraping de cada resultado. Você pode limitar as consultas a um único site com o operador padrão `site:` (por exemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de fazer scraping de uma URL específica.

<div id="preferred-sdk-method">
  ### método recomendado do SDK
</div>

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### Exemplo simples
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  limit: 5
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "firecrawl pricing",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  enterprise: ["anon"],
  scrape_options: [
    formats: [
      "markdown",
      [type: "json", prompt: "Extract pricing tiers and limits."]
    ],
    only_main_content: true,
    proxy: :enhanced
  ],
  timeout: 60000
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Parâmetros
</div>

* `query`: String da consulta de busca. Sempre obrigatória.
* `sources`: Fontes para buscar. Valores compatíveis: `"web"`, `"news"`, `"images"` ou objetos no estilo `[%{type: "web" | "news" | "images"}]`. Use para controlar quais arrays de resultados são retornados.
* `categories`: Filtros de resultado. Valores compatíveis: `"github"`, `"research"`, `"pdf"` ou objetos no estilo `[%{type: "github" | "research" | "pdf"}]`. Use para restringir os resultados por categoria.
* `limit`: Número máximo de resultados a retornar. Use para limitar o tamanho da resposta.
* `tbs`: String de filtro de busca baseada em tempo. Use para limitar por intervalo de tempo (por exemplo, `qdr:d`).
* `location`: String de localização para resultados regionalizados. Use para consultas geoespecíficas.
* `country`: Código de país ISO para resultados de busca segmentados geograficamente. Use junto com `location`.
* `timeout`: Tempo limite da requisição em milissegundos. Use para limitar o tempo de busca.
* `ignore_invalid_urls`: Exclui URLs inválidas dos resultados. Use ao encadear para outros endpoints.
* `enterprise`: Opções de busca Enterprise. Valores compatíveis: `"anon"` (ZDR anonimizado) ou `"zdr"` (ZDR de ponta a ponta).
* `scrape_options`: Opções de scraping aplicadas a cada resultado (mesmas chaves que `scrape`). Use quando quiser conteúdo com os resultados.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `scrape` quando já souber a URL e quiser o conteúdo em formatos específicos.

### Método recomendado do SDK

`Firecrawl.scrape_and_extract_from_url(params, opts)`

### Exemplo básico

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com",
  formats: ["markdown"]
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/docs",
  formats: [
    "markdown",
    [type: "json", prompt: "Extract headings and slugs."],
    [type: "changeTracking", modes: ["git-diff"], tag: "docs"],
    [type: "screenshot", full_page: true, viewport: [width: 1280, height: 720]]
  ],
  actions: [
    [type: "click", selector: "button#accept"],
    [type: "scroll", direction: "down"]
  ],
  parsers: [[type: "pdf", mode: "auto", max_pages: 5]],
  location: [country: "US", languages: ["en-US"]],
  proxy: :enhanced,
  profile: [name: "docs-session", save_changes: true],
  max_age: 3_600_000,
  min_age: 0,
  store_in_cache: true,
  zero_data_retention: false
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`: URL para scraping. Sempre obrigatória.
* `formats`: Formatos de resultado a incluir. Strings de formato compatíveis: `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`. Objetos de formato: `json` `[type: "json", prompt?, schema?]`, `changeTracking` `[type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?]`, `screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`.
* `headers`: Cabeçalhos HTTP adicionais. Use para autenticação, cookies ou user-agent.
* `include_tags`: Inclua apenas conteúdo destas tags HTML. Use para restringir a extração.
* `exclude_tags`: Exclua conteúdo destas tags HTML. Use para remover ruído.
* `only_main_content`: Extraia apenas o conteúdo principal. Use para remover navegação/rodapé.
* `timeout`: Tempo limite do scraping em milissegundos. Use para limitar o tempo de execução.
* `wait_for`: Tempo de espera antes do scraping, em milissegundos. Use para páginas renderizadas com JS.
* `mobile`: Emule um dispositivo móvel. Use para layouts móveis.
* `parsers`: Lista de parsers ou configurações. Configuração de PDF compatível: `[type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: number]` ou entradas como string (por exemplo, `"pdf"`).
* `actions`: Ações do navegador a serem executadas antes do scraping. Ações compatíveis: `wait` `[type: "wait", milliseconds? | selector?]`, `screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`, `click` `[type: "click", selector, all?]`, `write` `[type: "write", text]`, `press` `[type: "press", key]`, `scroll` `[type: "scroll", direction: "up" | "down", selector?]`, `scrape` `[type: "scrape"]`, `executeJavascript` `[type: "executeJavascript", script]`, `pdf` `[type: "pdf", format?, landscape?, scale?]` com formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuração de localização `[country, languages]`. Use para páginas específicas por região.
* `skip_tls_verification`: Ignore a verificação do certificado TLS. Use para certificados autoassinados.
* `remove_base64_images`: Remova imagens em base64 do resultado em markdown. Use para reduzir o tamanho do payload.
* `block_ads`: Ative o bloqueio de anúncios e cookies. Use para reduzir ruído.
* `proxy`: Modo de proxy `:basic`, `:enhanced`, `:auto`.
* `max_age`: Aceite conteúdo em cache mais recente do que esta idade (ms). Use para favorecer acertos de cache.
* `min_age`: Retorne apenas conteúdo em cache com pelo menos esta idade (ms). Use para leituras somente de cache.
* `store_in_cache`: Armazene o resultado no cache/índice. Use quando quiser reutilizá-lo.
* `profile`: Perfil persistente do navegador `[name, save_changes]`. Use para manter o estado da sessão.
* `zero_data_retention`: Ative a retenção zero de dados para o scraping. Use quando o ZDR estiver ativado para sua equipe.

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interact` quando a página exigir cliques, preenchimento de formulários ou outras ações no navegador após o scraping.

### Método recomendado do SDK

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

<div id="simple-example">
  ### Exemplo simples
</div>

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.url())"
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.title())",
  language: :node,
  timeout: 60,
  origin: "agent-docs"
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Parâmetros
</div>

* `job_id`: ID do job de scraping (de `metadata.scrapeId`). Sempre obrigatório.
* `code`: Código para executar no sandbox do navegador. Obrigatório.
* `language`: Linguagem de runtime `:python`, `:node` ou `:bash` (padrão: `:node`). Use para combinar com seu código.
* `timeout`: Tempo limite de execução em segundos. Use para limitar execuções longas.
* `origin`: Tag de origem opcional. Use para atribuir a origem da request.

<div id="notes">
  ## Notas
</div>

* Os parâmetros são passados como listas de palavras-chave em snake&#95;case, validadas por `NimbleOptions`.
* As funções são geradas automaticamente a partir da especificação OpenAPI; os nomes seguem os IDs de operação da OpenAPI.
* Cada função tem uma variante com bang que lança exceção em caso de erro (por exemplo, `search_and_scrape!`).
* O SDK Elixir expõe apenas `interact` via código (sem o parâmetro `prompt`).
* Todas as funções aceitam as opções `api_key` e `base_url` para sobrescritas por requisição.

<div id="firecrawl-python-agent-quickstart">
  # Guia de início rápido do agente Python do Firecrawl
</div>

Este arquivo é o guia de início rápido oficial para agentes externos e é gerado a partir do código-fonte do SDK e da especificação OpenAPI v2.

<div id="install">
  ## Instalar
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## Autenticação
</div>

```python
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])
```

<div id="when-to-use-what">
  ## Quando Usar Cada Um
</div>

* `search`: use quando você começa com uma consulta e precisa descobrir resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou ações no navegador após o scraping.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `search` para encontrar URLs relevantes com base em uma consulta e, opcionalmente, fazer scraping de cada resultado. Você pode restringir as consultas a um único site com o operador padrão `site:` (por exemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de fazer scraping de uma URL específica.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.search(query, **options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```python
results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    limit=5,
)

print(results.web[0] if results.web else None)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```python
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "firecrawl pricing",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:w",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            {"type": "json", "prompt": "Extract pricing tiers and limits."},
        ],
        only_main_content=True,
        proxy="stealth",
    ),
    integration="agent-quickstart",
)
```

<div id="parameters">
  ### Parâmetros
</div>

* `query`: String da consulta de busca. Obrigatória.
* `sources`: Fontes da busca. Valores compatíveis: `"web"`, `"news"`, `"images"` ou objetos `{"type": ...}`. Use para controlar quais arrays de resultados são retornados.
* `categories`: Filtros de resultados. Valores compatíveis: `"github"`, `"research"`, `"pdf"` ou objetos `{"type": ...}`. Use para restringir os resultados por categoria.
* `limit`: Número máximo de resultados retornados. Padrão: `5`.
* `tbs`: String de filtro de busca por tempo. Use para limitar por intervalo de tempo (por exemplo, `qdr:d`).
* `location`: String de localização para resultados regionalizados. Use em consultas específicas por região.
* `ignore_invalid_urls`: Exclui URLs inválidas dos resultados. Use ao encaminhar para outros endpoints.
* `timeout`: Tempo limite da solicitação em milissegundos. Padrão: `300000`.
* `scrape_options`: Opções de scraping aplicadas a cada resultado (mesmos campos de `scrape`). Use quando quiser incluir o conteúdo nos resultados.
* `integration`: Identificador da integração. Use para acompanhamento/análises.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `scrape` quando você já conhecer a URL e quiser o conteúdo em formatos específicos.

### Método preferido do SDK

`client.scrape(url, **options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```python
doc = client.scrape(
    "https://example.com",
    formats=["markdown"],
)

print(doc.markdown)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```python
doc = client.scrape(
    "https://example.com/docs",
    formats=[
        "markdown",
        {"type": "json", "prompt": "Extract headings and slugs."},
        {"type": "change_tracking", "modes": ["git-diff"], "tag": "docs"},
        {"type": "screenshot", "full_page": True, "viewport": {"width": 1280, "height": 720}},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
        {"type": "query", "prompt": "What is the pricing model?"},
    ],
    only_main_content=True,
    wait_for=1500,
    actions=[
        {"type": "click", "selector": "button#accept"},
        {"type": "scroll", "direction": "down"},
    ],
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    location={"country": "US", "languages": ["en-US"]},
    proxy="stealth",
    profile={"name": "docs-session", "save_changes": True},
    max_age=3_600_000,
)
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`: URL para scraping. Sempre obrigatória.
* `formats`: Formatos de resultado a incluir. Aceita um objeto `ScrapeFormats` (campos `markdown`, `html`, `raw_html`, `summary`, `links`, `images`, `screenshot`, `change_tracking`, `json`) ou uma lista de strings/objetos de formato. As strings de formato compatíveis incluem `markdown`, `html`, `rawHtml`, `raw_html`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `change_tracking`, `json`, `attributes`, `branding`, `query`, `audio`. Objetos de formato: `json` `{type: "json", prompt?, schema?}` (requer `prompt` ou `schema`), `changeTracking` `{type: "changeTracking" | "change_tracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `attributes` `{type: "attributes", selectors: [{selector, attribute}]}`, `query` `{type: "query", prompt}`.
* `headers`: Cabeçalhos HTTP adicionais. Use para autenticação, cookies ou user-agent.
* `include_tags`: Inclua apenas conteúdo dessas tags HTML. Use para restringir a extração.
* `exclude_tags`: Exclua conteúdo dessas tags HTML. Use para remover ruído.
* `only_main_content`: Extraia apenas o conteúdo principal. Use para remover navegação/rodapé.
* `timeout`: Tempo limite do scraping em milissegundos. Use para limitar o tempo de execução.
* `wait_for`: Atraso antes do scraping, em milissegundos. Use para páginas renderizadas por JS.
* `mobile`: Emule um dispositivo móvel. Use para layouts móveis.
* `parsers`: Lista de parsers ou configurações. Configuração de PDF compatível: `{type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: int}` ou entradas em string (por exemplo, `"pdf"`).
* `actions`: Ações do navegador a executar antes do scraping. Ações compatíveis: `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `click` `{type: "click", selector}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}` com formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuração de localização `{country, languages}`. Use para páginas específicas por região.
* `skip_tls_verification`: Ignore a verificação do certificado TLS. Use para certificados autoassinados.
* `remove_base64_images`: Remova imagens em base64 do resultado em markdown. Use para reduzir o tamanho do payload.
* `fast_mode`: Ative o modo rápido. Use quando a velocidade for o mais importante.
* `use_mock`: Use um mock fixture nomeado (interno/testes).
* `block_ads`: Ative o bloqueio de anúncios e cookies. Use para reduzir ruído.
* `proxy`: Modo de proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`.
* `max_age`: Aceite conteúdo em cache mais novo do que esta idade (ms). Use para favorecer acertos de cache.
* `store_in_cache`: Armazene o resultado em cache/índice. Use quando quiser reutilizá-lo.
* `profile`: Perfil persistente do navegador `{name, save_changes}` (também aceita `saveChanges`). Use para manter o estado da sessão.
* `integration`: Identificador da integração. Use para acompanhamento/análises.

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interact` quando a página exigir cliques, preenchimento de formulários ou outras ações no navegador após o scraping.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.interact(job_id, code=None, *, prompt=None, **options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```python
run = client.interact(
    "scrape-job-id",
    code="print(await page.url())",
)

print(run.stdout)
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```python
run = client.interact(
    "scrape-job-id",
    prompt="Open the pricing page and list the plan names.",
    language="python",
    timeout=60,
    origin="agent-docs",
)

print(run.output)
```

<div id="parameters">
  ### Parâmetros
</div>

* `job_id`: ID do job de scraping (de `document.metadata_typed.scrape_id`). Sempre obrigatório.
* `code`: Código a ser executado no navegador sandbox. Use para interações via script.
* `prompt`: Instrução em linguagem natural para o agente do navegador. Use quando preferir ações conduzidas pelo agente.
* `language`: Linguagem de runtime `"python"`, `"node"` ou `"bash"` (default `"node"`). Use para combinar com seu código.
* `timeout`: Tempo limite de execução em segundos. Use para limitar execuções longas.
* `origin`: Tag de origem opcional. Use para atribuição da requisição.

<div id="notes">
  ## Observações
</div>

* Os parâmetros usam `snake_case` em Python; o SDK normaliza os payloads para o `camelCase` da API.
* Aliases descontinuados: `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (use `interact` e `stop_interaction`).
* `Firecrawl` é o cliente unificado da v2; a v1 continua disponível em `.v1` para uso legado.

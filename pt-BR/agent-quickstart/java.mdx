<div id="firecrawl-java-agent-quickstart">
  # Início rápido do agente Java do Firecrawl
</div>

Este arquivo é o guia oficial de início rápido para agentes externos e é gerado a partir do código-fonte do SDK e da especificação OpenAPI v2.

<div id="install">
  ## Instalar
</div>

```gradle
dependencies {
  implementation("com.firecrawl:firecrawl-java:1.2.0")
}
```

<div id="authenticate">
  ## Autentique-se
</div>

```java
import com.firecrawl.client.FirecrawlClient;

FirecrawlClient client = FirecrawlClient.builder()
    .apiKey(System.getenv("FIRECRAWL_API_KEY"))
    .build();
```

<div id="when-to-use-what">
  ## Quando Usar Cada Um
</div>

* `search`: use quando você começa com uma consulta e precisa descobrir resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou ações do navegador após o scraping.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `search` para descobrir URLs relevantes com base em uma consulta e, opcionalmente, fazer scraping de cada resultado. Você pode restringir as consultas a um único site com o operador padrão `site:` (por exemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de fazer scraping de uma URL específica.

### Método preferido da SDK

`client.search(query, options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```java
import com.firecrawl.models.SearchData;

SearchData data = client.search("site:docs.firecrawl.dev crawl webhooks");
System.out.println(data.getWeb());
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```java
import com.firecrawl.models.SearchData;
import com.firecrawl.models.SearchOptions;
import com.firecrawl.models.ScrapeOptions;

SearchData data = client.search(
    "firecrawl pricing",
    SearchOptions.builder()
        .sources(java.util.List.of("web", "news"))
        .categories(java.util.List.of("research"))
        .limit(10)
        .tbs("qdr:w")
        .location("San Francisco,California,United States")
        .ignoreInvalidURLs(true)
        .scrapeOptions(
            ScrapeOptions.builder()
                .formats(java.util.List.of(
                    "markdown",
                    java.util.Map.of("type", "json", "prompt", "Extract pricing tiers and limits.")
                ))
                .onlyMainContent(true)
                .proxy("stealth")
                .build()
        )
        .integration("agent-quickstart")
        .build()
);
```

<div id="parameters">
  ### Parâmetros
</div>

* `query`: String da consulta de busca. Sempre obrigatória.
* `sources`: Fontes para buscar. Valores compatíveis: `"web"`, `"news"`, `"images"` ou maps `{type: "web" | "news" | "images"}`. Use para controlar quais arrays de resultados são retornados.
* `categories`: Filtros de resultados. Valores compatíveis: `"github"`, `"research"`, `"pdf"` ou maps `{type: "github" | "research" | "pdf"}`. Use para restringir os resultados por categoria.
* `limit`: Número máximo de resultados a retornar. Use para limitar o tamanho da resposta.
* `tbs`: String de filtro de busca baseado em tempo. Use para limitar por intervalo de tempo (por exemplo, `qdr:d`).
* `location`: String de localização para resultados regionalizados. Use em consultas específicas por região.
* `ignoreInvalidURLs`: Exclui URLs inválidas dos resultados. Use ao encaminhar para outros endpoints.
* `timeout`: Tempo limite da request em milissegundos. Use para limitar o tempo da busca.
* `scrapeOptions`: Opções de scraping aplicadas a cada resultado (mesmos campos de `scrape`). Use quando quiser conteúdo junto com os resultados.
* `integration`: Identificador da integração. Use para acompanhamento/análises.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `scrape` quando você já souber a URL e quiser o conteúdo em formatos específicos.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com",
    ScrapeOptions.builder()
        .formats(java.util.List.of("markdown"))
        .build()
);

System.out.println(doc.getMarkdown());
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.LocationConfig;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com/docs",
    ScrapeOptions.builder()
        .formats(java.util.List.of(
            "markdown",
            java.util.Map.of("type", "json", "prompt", "Extract headings and slugs."),
            java.util.Map.of("type", "changeTracking", "modes", java.util.List.of("git-diff"))
        ))
        .onlyMainContent(true)
        .waitFor(1500)
        .actions(java.util.List.of(
            java.util.Map.of("type", "click", "selector", "button#accept"),
            java.util.Map.of("type", "scroll", "direction", "down")
        ))
        .parsers(java.util.List.of(java.util.Map.of("type", "pdf", "mode", "auto", "maxPages", 5)))
        .location(LocationConfig.builder()
            .country("US")
            .languages(java.util.List.of("en-US"))
            .build())
        .proxy("stealth")
        .maxAge(3_600_000L)
        .build()
);
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`: URL para scraping. Sempre obrigatória.
* `formats`: Formatos de resultado a incluir. Strings de formato compatíveis: `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`. Objetos de formato: `json` `{type: "json", prompt?, schema?}`, `changeTracking` `{type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`.
* `headers`: Cabeçalhos HTTP adicionais. Use para autenticação, cookies ou user-agent.
* `includeTags`: Inclui apenas conteúdo destas tags HTML. Use para restringir a extração.
* `excludeTags`: Exclui conteúdo destas tags HTML. Use para remover ruído.
* `onlyMainContent`: Extrai apenas o conteúdo principal. Use para remover navegação/rodapé.
* `timeout`: Tempo limite do scraping em milissegundos. Use para limitar o tempo de execução.
* `waitFor`: Atraso antes do scraping em milissegundos. Use para páginas renderizadas por JS.
* `mobile`: Emula um dispositivo móvel. Use para layouts móveis.
* `parsers`: Lista de parsers ou configurações. Configuração de PDF compatível: `{type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number}` ou entradas como string (por exemplo, `"pdf"`).
* `actions`: Ações do navegador executadas antes do scraping. Ações compatíveis: `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`, `click` `{type: "click", selector, all?}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}` com formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuração de localização (`country`, `languages`). Use para páginas específicas por região.
* `skipTlsVerification`: Ignora a verificação do certificado TLS. Use para certificados autoassinados.
* `removeBase64Images`: Remove imagens em base64 do resultado em markdown. Use para reduzir o tamanho do payload.
* `blockAds`: Ativa o bloqueio de anúncios e cookies. Use para reduzir ruído.
* `proxy`: Modo de proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"` ou uma URL de proxy personalizada.
* `maxAge`: Aceita conteúdo em cache mais recente do que esta idade (ms). Use para favorecer acertos de cache.
* `storeInCache`: Armazena o resultado no cache/índice. Use quando quiser reutilizá-lo.
* `integration`: Identificador da integração. Use para acompanhamento/análises.

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interact` quando a página exigir cliques, preenchimento de formulários ou outras ações no navegador após o scraping.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.interact(jobId, code, language, timeout, origin)`

<div id="simple-example">
  ### Exemplo simples
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());"
);

System.out.println(exec.getStdout());
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());",
    "node",
    60,
    "agent-docs"
);
```

<div id="parameters">
  ### Parâmetros
</div>

* `jobId`: ID do job de scraping (de `document.metadata.scrapeId`). Sempre obrigatório.
* `code`: Código a ser executado no sandbox do navegador. Use para interações via script.
* `language`: Linguagem de execução `"python"`, `"node"` ou `"bash"` (padrão: `"node"`). Use para corresponder ao seu código.
* `timeout`: Tempo limite de execução em segundos. Use para limitar execuções longas.
* `origin`: Tag de origem opcional. Use para atribuição da requisição.

<div id="notes">
  ## Observações
</div>

* Os parâmetros usam camelCase em Java.
* `ScrapeOptions` e `SearchOptions` usam o padrão builder.
* Aliases descontinuados: `scrapeExecute` e `deleteScrapeBrowser` (use `interact` e `stopInteractiveBrowser`).
* O SDK Java expõe apenas `interact` via código (sem o parâmetro `prompt`).

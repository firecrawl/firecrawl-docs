<div id="firecrawl-javascripttypescript-agent-quickstart">
  # Guia de início rápido do agente JavaScript/TypeScript do Firecrawl
</div>

Este arquivo é o guia rápido oficial para agentes externos e é gerado a partir do código-fonte do SDK e da especificação OpenAPI v2.

<div id="install">
  ## Instalação
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="authenticate">
  ## Autentique-se
</div>

```ts
import { Firecrawl } from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // opcional para self-hosted
});
```

<div id="when-to-use-what">
  ## Quando Usar Cada Um
</div>

* `search`: use quando você começa com uma consulta e precisa encontrar resultados.
* `scrape`: use quando você já tem uma URL e quer o conteúdo da página.
* `interact`: use quando a página exige cliques, formulários ou ações do navegador após o scraping.

<div id="search">
  ## Busca
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `search` para encontrar URLs relevantes com base em uma consulta e, opcionalmente, fazer scraping de cada resultado. Você pode restringir as consultas a um único site com o operador padrão `site:` (por exemplo, `site:docs.firecrawl.dev crawl webhooks`) antes de fazer scraping de uma URL específica.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.search(query, options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```ts
const results = await client.search(
  "site:docs.firecrawl.dev crawl webhooks",
  { limit: 5 }
);

console.log(results.web?.[0]);
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```ts
const results = await client.search("firecrawl pricing", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  scrapeOptions: {
    formats: [
      "markdown",
      { type: "json", prompt: "Extract pricing tiers and limits." }
    ],
    onlyMainContent: true,
    actions: [
      { type: "wait", milliseconds: 1000 },
      { type: "scroll", direction: "down" }
    ],
    proxy: "stealth"
  },
  integration: "agent-quickstart",
  origin: "agent-docs"
});
```

<div id="parameters">
  ### Parâmetros
</div>

* `query`: String da consulta de busca. Sempre obrigatório.
* `sources`: Fontes para busca. Valores compatíveis: `"web"`, `"news"`, `"images"` ou `{ type: "web" | "news" | "images" }`. Use para controlar quais arrays de resultados são retornados.
* `categories`: Filtros de resultado. Valores compatíveis: `"github"`, `"research"`, `"pdf"` ou `{ type: "github" | "research" | "pdf" }`. Use para restringir os resultados por categoria.
* `limit`: Número máximo de resultados a retornar. Use para limitar o tamanho da resposta.
* `tbs`: String de filtro de busca por tempo. Use para limitar por intervalo de tempo (por exemplo, `qdr:d`).
* `location`: String de localização para resultados regionalizados. Use para consultas geograficamente específicas.
* `ignoreInvalidURLs`: Exclui URLs inválidas dos resultados. Use ao encaminhar para outros endpoints.
* `timeout`: Tempo limite da solicitação em milissegundos. Use para limitar o tempo de busca.
* `scrapeOptions`: Opções de scraping aplicadas a cada resultado (mesmos campos de `scrape`). Use quando quiser incluir o conteúdo nos resultados.
* `integration`: Identificador da integração. Use para rastreamento/análises.
* `origin`: Tag de origem opcional. Use para atribuição da solicitação.

<div id="scrape">
  ## Scraping
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `scrape` quando você já conhece a URL e quer o conteúdo em formatos específicos.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### Exemplo simples
</div>

```ts
const doc = await client.scrape("https://example.com", {
  formats: ["markdown"]
});

console.log(doc.markdown);
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```ts
const doc = await client.scrape("https://example.com/docs", {
  formats: [
    "markdown",
    { type: "json", prompt: "Extract headings and their slugs." },
    { type: "changeTracking", modes: ["git-diff"], tag: "docs" },
    { type: "screenshot", fullPage: true, viewport: { width: 1280, height: 720 } },
    {
      type: "attributes",
      selectors: [{ selector: "a", attribute: "href" }]
    },
    { type: "query", prompt: "What is the pricing model?" }
  ],
  onlyMainContent: true,
  waitFor: 1500,
  actions: [
    { type: "click", selector: "button#accept" },
    { type: "scroll", direction: "down" }
  ],
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  location: { country: "US", languages: ["en-US"] },
  proxy: "stealth",
  profile: { name: "docs-session", saveChanges: true },
  maxAge: 3600000
});
```

<div id="parameters">
  ### Parâmetros
</div>

* `url`: URL para fazer scraping. Sempre obrigatória.
* `formats`: Formatos de resultado a incluir. Strings de formato compatíveis: `markdown`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `json`, `attributes`, `branding`, `audio`. Objetos de formato: `json` `{ type: "json", prompt?, schema? }` (requer `prompt` ou `schema`), `changeTracking` `{ type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag? }`, `screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`, `attributes` `{ type: "attributes", selectors: [{ selector, attribute }] }`, `query` `{ type: "query", prompt }`, além de `{ type: "<format>" }` para qualquer formato em string.
* `headers`: Cabeçalhos HTTP adicionais. Use para autenticação, cookies ou user-agent.
* `includeTags`: Inclua apenas conteúdo dessas tags HTML. Use para restringir a extração.
* `excludeTags`: Exclua conteúdo dessas tags HTML. Use para remover ruído.
* `onlyMainContent`: Extraia apenas o conteúdo principal. Use para remover navegação/rodapé.
* `timeout`: Tempo limite do scraping em milissegundos. Use para limitar o tempo de execução.
* `waitFor`: Tempo de espera antes do scraping, em milissegundos. Use para páginas renderizadas com JS.
* `mobile`: Emule um dispositivo móvel. Use para layouts mobile.
* `parsers`: Lista de parsers ou configurações. Configuração de PDF compatível: `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }` ou entradas em string (por exemplo, `"pdf"`).
* `actions`: Ações do navegador a serem executadas antes do scraping. Ações compatíveis: `wait` `{ type: "wait", milliseconds? | selector? }`, `screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`, `click` `{ type: "click", selector }`, `write` `{ type: "write", text }`, `press` `{ type: "press", key }`, `scroll` `{ type: "scroll", direction: "up" | "down", selector? }`, `scrape` `{ type: "scrape" }`, `executeJavascript` `{ type: "executeJavascript", script }`, `pdf` `{ type: "pdf", format?, landscape?, scale? }` com formatos `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location`: Configuração de localização `{ country, languages }`. Use para páginas específicas por geolocalização.
* `skipTlsVerification`: Ignore a verificação do certificado TLS. Use para certificados autoassinados.
* `removeBase64Images`: Remova imagens base64 da saída em markdown. Use para reduzir o tamanho do payload.
* `fastMode`: Ative o modo rápido. Use quando a velocidade for o mais importante.
* `useMock`: Use um mock fixture nomeado (interno/testes).
* `blockAds`: Ative o bloqueio de anúncios e cookies. Use para reduzir ruído.
* `proxy`: Modo de proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"` ou uma string de proxy personalizada. Use para sites com proteção anti-bot.
* `maxAge`: Aceite conteúdo em cache mais novo do que esta idade (ms). Use para favorecer acertos de cache.
* `minAge`: Retorne apenas conteúdo em cache com pelo menos esta idade (ms). Use para leituras somente de cache.
* `storeInCache`: Armazene o resultado no cache/índice. Use quando quiser reutilizá-lo.
* `profile`: Perfil persistente do navegador `{ name, saveChanges }`. Use para manter o estado da sessão.
* `integration`: Identificador da integração. Use para rastreamento/análises.
* `origin`: Tag de origem opcional. Use para atribuição da request.

<div id="interact">
  ## Interagir
</div>

<div id="why-use-it">
  ### Por que usar
</div>

Use `interact` quando a página exigir cliques, preenchimento de formulários ou outras ações no navegador após o scraping.

<div id="preferred-sdk-method">
  ### Método recomendado do SDK
</div>

`client.interact(jobId, args)`

<div id="simple-example">
  ### Exemplo simples
</div>

```ts
const run = await client.interact("scrape-job-id", {
  code: "console.log(await page.url())"
});

console.log(run.stdout);
```

<div id="complex-example">
  ### Exemplo complexo
</div>

```ts
const run = await client.interact("scrape-job-id", {
  prompt: "Open the pricing page and list the plan names.",
  language: "node",
  timeout: 60,
  origin: "agent-docs"
});

console.log(run.output);
```

<div id="parameters">
  ### Parâmetros
</div>

* `jobId`: ID do job de scraping (de `document.metadata.scrapeId`). Sempre obrigatório.
* `code`: Código a ser executado no sandbox do navegador. Use para interações via script.
* `prompt`: Instrução em linguagem natural para o agente do navegador. Use quando preferir ações orientadas pelo agente.
* `language`: Linguagem de execução `"python"`, `"node"` ou `"bash"` (padrão: `"node"`). Use para corresponder ao seu código.
* `timeout`: Tempo limite de execução em segundos. Use para limitar execuções longas.
* `origin`: Tag de origem opcional. Use para atribuir a origem da requisição.

<div id="notes">
  ## Observações
</div>

* Os parâmetros usam camelCase em JS/TS.
* Aliases descontinuados: `scrapeExecute`, `stopInteractiveBrowser`, `deleteScrapeBrowser` (use `interact` e `stopInteraction`).
* `Firecrawl` é o cliente v2 unificado; a v1 continua disponível em `.v1` para compatibilidade legada.

A Firecrawl oferece diferentes tipos de proxy para ajudar você a fazer scraping de sites com diversos níveis de proteção anti-bot. O tipo de proxy pode ser especificado usando o parâmetro `proxy`.

> Por padrão, a Firecrawl roteia todas as requisições por meio de proxies para garantir confiabilidade e acesso, mesmo que você não especifique um tipo de proxy ou localização.

<div id="location-based-proxy-selection">

<div id="how-to-specify-proxy-location">
  ## Como especificar a localização do proxy
</div>

Você pode solicitar uma localização específica de proxy definindo o parâmetro `location.country` na sua requisição. Por exemplo, para usar um proxy no Brasil, defina `location.country` como `BR`.

Para ver todos os detalhes, consulte a [referência da API de `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location).

<div id="proxy-types">
  ## Tipos de Proxy
</div>

Firecrawl oferece suporte a três tipos de proxies:

* **basic**: Proxies para coletar dados de sites com nenhuma ou poucas soluções anti-bot. Rápidos e geralmente funcionam.
* **stealth**: Proxies stealth para coletar dados de sites com soluções anti-bot avançadas ou para sites que bloqueiam proxies comuns. Mais lentos, mas mais confiáveis em certos sites. [Saiba mais sobre o Modo Stealth →](/pt-BR/features/stealth-mode)
* **auto**: O Firecrawl tentará automaticamente novamente a coleta com proxies stealth se o proxy basic falhar. Se a nova tentativa com stealth for bem-sucedida, 5 créditos serão cobrados pela coleta. Se a primeira tentativa com basic for bem-sucedida, apenas o custo padrão será cobrado.

***

> **Observação:** Para informações detalhadas sobre o uso de proxies stealth, incluindo custos de créditos e estratégias de nova tentativa, consulte a [documentação do Modo Stealth](/pt-BR/features/stealth-mode).

<div id="installation">
  ## Instalação
</div>

Para instalar o SDK do Firecrawl para Node, você pode usar o npm:

```bash
npm install @mendable/firecrawl-js@0.0.36
```

<div id="usage">
  ## Uso
</div>

1. Obtenha uma chave de API em [firecrawl.dev](https://firecrawl.dev)
2. Configure a chave de API como uma variável de ambiente chamada `FIRECRAWL_API_KEY` ou passe-a como parâmetro para a classe `FirecrawlApp`.

Veja um exemplo de como usar o SDK com tratamento de erros:

```js
import FirecrawlApp from '@mendable/firecrawl-js';

// Inicialize o FirecrawlApp com sua chave de API
const app = new FirecrawlApp({ apiKey: "YOUR_API_KEY" });

// Faça o scrape de uma única URL
const url = 'https://docs.firecrawl.dev';
const scrapedData = await app.scrapeUrl(url);

// Faça o crawl de um site
const crawlUrl = 'https://docs.firecrawl.dev';
const params = {
  crawlerOptions: {
    excludes: ['blog/'],
    includes: [], // deixe vazio para incluir todas as páginas
    limit: 1000,
  },
  pageOptions: {
      onlyMainContent: true
  }
};

const crawlResult = await app.crawlUrl(crawlUrl, params);
```

<div id="scraping-a-url">
  ### Fazendo scraping de uma URL
</div>

Para extrair o conteúdo de uma única URL com tratamento de erros, use o método `scrapeUrl`. Ele recebe a URL como parâmetro e retorna os dados coletados como um dicionário.

```js
const url = 'https://exemplo.com';
const dadosRaspados = await app.scrapeUrl(url);
```

<div id="crawling-a-website">
  ### Rastreando um site
</div>

Para rastrear um site com tratamento de erros, use o método `crawlUrl`. Ele recebe a URL inicial e parâmetros opcionais como argumentos. O parâmetro `params` permite especificar opções adicionais para a tarefa de rastreamento, como o número máximo de páginas a rastrear, os domínios permitidos e o formato de saída.

```js
const crawlUrl = 'https://example.com';

const params = {
  crawlerOptions: {
    excludes: ['blog/'],
    includes: [], // deixe em branco para todas as páginas
    limit: 1000,
  },
  pageOptions: {
    onlyMainContent: true
  }
};

const waitUntilDone = true;
const pollInterval = 5;

const crawlResult = await app.crawlUrl(
  crawlUrl,
  params,
  waitUntilDone,
  pollInterval
);
```

<div id="checking-crawl-status">
  ### Verificando o status do crawl
</div>

Para verificar o status de uma tarefa de crawl com tratamento de erros, use o método `checkCrawlStatus`. Ele recebe o ID da tarefa como parâmetro e retorna o status atual da tarefa de crawl.

```js
const status = await app.checkCrawlStatus(jobId);
```

<div id="extracting-structured-data-from-a-url">
  ### Extraindo dados estruturados de uma URL
</div>

Com a extração via LLM, você pode extrair dados estruturados de qualquer URL com facilidade. Também oferecemos suporte ao schema Zod para simplificar o processo. Veja como usar:

```js
import FirecrawlApp from "@mendable/firecrawl-js";
import { z } from "zod";

const app = new FirecrawlApp({
  apiKey: "fc-YOUR_API_KEY",
});

// Defina o esquema para extrair o conteúdo
const schema = z.object({
  top: z
    .array(
      z.object({
        title: z.string(),
        points: z.number(),
        by: z.string(),
        commentsURL: z.string(),
      })
    )
    .length(5)
    .describe("Top 5 notícias do Hacker News"),
});

const scrapeResult = await app.scrapeUrl("https://firecrawl.dev", {
  extractorOptions: { extractionSchema: schema },
});

console.log(scrapeResult.data["llm_extraction"]);
```

<div id="search-for-a-query">
  ### Pesquisar por uma consulta
</div>

Com o método `search`, você pode buscar uma consulta em um mecanismo de pesquisa e obter os principais resultados junto com o conteúdo de cada página retornada. O método recebe a consulta como parâmetro e retorna os resultados da pesquisa.

```js
const query = 'o que é a Mendable?';
const searchResults = await app.search(query, {
  pageOptions: {
    fetchPageContent: true // Buscar o conteúdo da página para cada resultado da pesquisa
  }
});
```

<div id="error-handling">
  ## Tratamento de erros
</div>

O SDK trata os erros retornados pela API do Firecrawl e lança exceções apropriadas. Se ocorrer um erro durante uma requisição, uma exceção será lançada com uma mensagem descritiva. Os exemplos acima mostram como lidar com esses erros usando blocos `try/catch`.

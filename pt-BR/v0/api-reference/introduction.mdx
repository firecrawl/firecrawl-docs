<div id="base-url">
  ## URL base
</div>

Todas as solicitações usam a seguinte URL base:

```bash
https://api.firecrawl.dev 
```


<div id="authentication">
  ## Autenticação
</div>

Para autenticar, inclua o cabeçalho Authorization. Ele deve conter `Bearer fc_123456789`, em que `fc_123456789` é a sua chave de API.

```bash
Authorization: Bearer fc_123456789
```

​


<div id="response-codes">
  ## Códigos de resposta
</div>

O Firecrawl utiliza códigos de status HTTP convencionais para indicar o resultado das suas solicitações.

Em geral, códigos HTTP 2xx indicam sucesso, 4xx representam erros do cliente e 5xx sinalizam problemas de infraestrutura.

Status | Descrição
--- | ---
200 | Solicitação bem-sucedida.
400 | Verifique se os parâmetros estão corretos.
401 | A chave da API não foi fornecida.
402 | Pagamento necessário.
404 | O recurso solicitado não foi encontrado.
429 | O limite de taxa foi excedido.
5xx | Indica um erro de servidor no Firecrawl.

Consulte a seção Códigos de erro para uma explicação detalhada de todos os possíveis erros da API.

​

<div id="rate-limit">
  ## Limite de taxa
</div>

A API do Firecrawl possui um limite de taxa para garantir a estabilidade e a confiabilidade do serviço. Esse limite é aplicado a todos os endpoints e se baseia no número de solicitações feitas em um intervalo de tempo específico.

Ao exceder o limite de taxa, você receberá o código de status 429.

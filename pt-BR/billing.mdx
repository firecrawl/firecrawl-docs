<div id="overview">
  ## Visão geral
</div>

Firecrawl utiliza um **sistema de cobrança baseado em créditos**. Cada chamada de API que você faz consome créditos, e a quantidade de créditos consumidos depende do endpoint e das opções que você utiliza. Você recebe uma cota mensal de créditos com base no seu plano e pode comprar créditos adicionais por meio de pacotes de recarga automática, caso precise de mais.

Para ver os preços atuais dos planos, visite a [página de preços do Firecrawl](https://www.firecrawl.dev/pricing).

<Note>
  Todas as faturas do Firecrawl são emitidas em **dólares americanos (USD)**, independentemente do seu endereço de cobrança ou método de pagamento.
</Note>

<div id="credits">
  ## Créditos
</div>

Créditos são a unidade de consumo no Firecrawl. Cada plano inclui uma cota mensal de créditos que é renovada no início de cada ciclo de cobrança. Diferentes endpoints da API consomem quantidades diferentes de créditos.

<div id="credit-costs-per-endpoint">
  ### Custos de crédito por endpoint
</div>

| Endpoint    | Custo de crédito                 | Observações                                                                                                                                                                                                                                                                                                                                                                       |
| ----------- | -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Scrape**  | 1 crédito / página               | Converte uma única URL em markdown limpo, HTML ou dados estruturados. Créditos adicionais se aplicam ao usar opções de scraping (consulte abaixo).                                                                                                                                                                                                                                |
| **Crawl**   | 1 crédito / página               | Faz o scraping de um site inteiro seguindo links a partir de uma URL inicial. Os mesmos custos das opções de scraping por página se aplicam a cada página rastreada.                                                                                                                                                                                                              |
| **Map**     | 1 crédito / chamada              | Descobre todas as URLs em um site sem fazer scraping do conteúdo.                                                                                                                                                                                                                                                                                                                 |
| **Search**  | 2 créditos / 10 resultados       | Faz uma busca na web e, opcionalmente, realiza o scraping dos resultados. Arredondado para cima a cada 10 resultados (por exemplo, 11 resultados = 4 créditos). Custos adicionais de scraping por página se aplicam a cada resultado que for raspado. A busca ZDR Enterprise custa 10 créditos / 10 resultados (consulte [ZDR Search](/pt-BR/features/search#zero-data-retention-zdr)). |
| **Browser** | 2 créditos / minuto de navegador | Sessão interativa de sandbox de navegador, cobrada por minuto.                                                                                                                                                                                                                                                                                                                    |
| **Agent**   | Dinâmico                         | Agente autônomo de pesquisa na web. 5 execuções diárias gratuitas; cobrança baseada em uso após isso.                                                                                                                                                                                                                                                                             |

<div id="additional-credit-costs-for-scrape-options">
  ### Custos adicionais de crédito para opções de scraping
</div>

Certas opções de scraping adicionam créditos além do custo base por página:

| Opção | Custo adicional | Descrição |
|--------|-----------------|-----------|
| PDF parsing | +1 crédito / página de PDF | Extrair conteúdo de documentos PDF |
| JSON format (LLM extraction) | +4 créditos / página | Usar um LLM para extrair dados JSON estruturados da página |
| Enhanced Mode | +4 créditos / página | Scraping aprimorado para páginas que são difíceis de acessar |
| Zero Data Retention (ZDR) | +1 crédito / página | Garante que nenhum dado seja persistido além da requisição (veja [Scrape ZDR](/pt-BR/features/scrape#zero-data-retention-zdr)) |

Esses modificadores se acumulam. Por exemplo, fazer scraping de uma página com JSON format e Enhanced Mode custa **1 + 4 + 4 = 9 créditos** por página. Esses mesmos modificadores se aplicam aos endpoints Crawl e Search, já que eles usam scraping internamente para cada página.

<div id="when-credits-are-charged">
  ### Quando os créditos são debitados
</div>

Os créditos são debitados sempre que a infraestrutura do Firecrawl processa uma request, mesmo que o site de destino retorne um código de status de erro HTTP, como 403 Forbidden ou 404 Not Found. Isso acontece porque a infraestrutura de scraping (renderização no navegador, proxy etc.) é totalmente utilizada, independentemente da resposta do site de destino. Você pode verificar o campo `metadata.statusCode` na resposta da API para detectar esses casos e evitar tentar novamente URLs que são bloqueadas de forma consistente.

Para jobs de **batch scrape** e **rastreamento**, os créditos são debitados de forma assíncrona à medida que cada página conclui o processamento — não quando o job é submetido. Isso significa que pode haver um atraso entre a submissão de um job e ver o custo total de créditos refletido na sua conta. Se um batch contiver muitas URLs ou se as páginas forem enfileiradas durante períodos de alto tráfego, os créditos podem continuar aparecendo minutos ou horas após a submissão. Realizar polling ou verificar o status do batch não consome créditos.

<Note>
  **Verificação prévia de créditos do rastreamento:** Antes de um job de rastreamento começar, o Firecrawl verifica se o saldo de créditos restante é suficiente para cobrir todo o parâmetro `limit` solicitado. Se o seu saldo for menor que `limit`, a request retorna um 402 mesmo que o rastreamento tivesse descoberto menos páginas. O `limit` padrão é **10.000**, portanto omiti-lo exige que 10.000 créditos estejam disponíveis antecipadamente. Para evitar isso, passe um `limit` explícito que corresponda ao número de páginas que você realmente pretende rastrear (por exemplo, `limit: 100`).
</Note>

<div id="tracking-your-usage">
  ### Monitorando seu uso
</div>

Você pode acompanhar o uso de créditos de duas maneiras:

* **Dashboard**: Veja seu uso atual e histórico em [firecrawl.dev/app](https://www.firecrawl.dev/app)
* **API**: Use os endpoints [Credit Usage](/pt-BR/api-reference/endpoint/credit-usage) e [Credit Usage Historical](/pt-BR/api-reference/endpoint/credit-usage-historical) para verificar seu uso programaticamente

<Note>
  Estamos trabalhando ativamente em melhorias para tornar o uso de créditos mais fácil de entender. Fique de olho nas próximas atualizações.
</Note>

<div id="plans">
  ## Planos
</div>

Firecrawl oferece planos mensais por assinatura. Não há uma opção pura de pague‑conforme‑o‑uso (pay-as-you-go), mas a recarga automática (descrita abaixo) oferece escalabilidade flexível adicional ao seu plano base.

<div id="available-plans">
  ### Planos disponíveis
</div>

| Plano        | Créditos mensais | Navegadores concorrentes |
| ------------ | ---------------- | -----------------------: |
| **Free**     | 1,000            |                       2 |
| **Hobby**    | 5,000            |                       5 |
| **Standard** | 100,000          |                      50 |
| **Growth**   | 500,000          |                     100 |
| **Scale**    | 1,000,000        |                     150 |

<Note>
  Para necessidades além do plano Scale, Firecrawl oferece planos **Enterprise** com créditos personalizados, suporte dedicado, SLAs, descontos em volume, sem retenção de dados e SSO. Visite a [página Enterprise](https://www.firecrawl.dev/enterprise) para mais detalhes.
</Note>

Todos os planos pagos estão disponíveis com cobrança **mensal** ou **anual**. A cobrança anual oferece um desconto em comparação ao pagamento mês a mês. Para preços atualizados de cada plano, visite a [página de preços](https://www.firecrawl.dev/pricing).

<div id="choosing-a-credit-tier-with-the-pricing-slider">
  ### Escolhendo um nível de créditos com o slider de preços
</div>

A [página de preços](https://www.firecrawl.dev/pricing) usa um único slider que percorre todas as opções de créditos, do plano Free até Scale e Custom. Além das faixas principais dos planos, cada plano pago oferece níveis intermediários de créditos para que você possa ajustar sua assinatura mais de acordo com o seu uso real, sem precisar passar para o plano seguinte:

| Plano        | Níveis de créditos disponíveis |
| ------------ | ------------------------------ |
| **Hobby**    | 5,000 · 6,500 · 8,000          |
| **Standard** | 100,000 · 130,000 · 160,000    |
| **Growth**   | 500,000 · 650,000              |
| **Scale**    | 1,000,000                      |

Navegadores concorrentes, limites de taxa e nível de suporte são definidos pelo plano, não pelo nível específico de créditos — escolher 6,500 créditos no Hobby oferece a mesma concorrência que escolher 5,000. Use o slider para ajustar a quantidade de créditos e depois assine esse nível.

<div id="concurrent-browsers">
  ### Navegadores simultâneos
</div>

Navegadores simultâneos representam quantas páginas da web o Firecrawl pode processar para você ao mesmo tempo. Seu plano determina esse limite. Se você o exceder, tarefas adicionais ficam em uma fila de espera até que uma vaga seja liberada. Consulte [Limites de Taxa](/pt-BR/rate-limits) para ver todos os detalhes sobre concorrência e limites de taxa da API.

<div id="handling-usage-spikes">
  ## Como lidar com picos de uso
</div>

O Firecrawl oferece dois mecanismos opcionais para manter as requisições em andamento quando você fica sem créditos no meio do ciclo: **Recarga Automática** e **Upgrade Inteligente**. Ambos são configurados nas suas [configurações de cobrança](https://www.firecrawl.dev/app/settings?tab=billing) e são **mutuamente exclusivos** — sua organização usa um ou outro, não ambos.

|                      | Recarga Automática                                                     | Upgrade Inteligente                                                 |
| -------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------- |
| O que faz            | Compra um pacote avulso de créditos e o adiciona ao seu saldo          | Move você para o próximo nível de créditos na estrutura de preços   |
| Quando é acionado    | O saldo cai até o seu limite (padrão: 0)                               | O saldo chega a 0                                                   |
| Cobrança             | Preço fixo do pacote                                                   | Diferença proporcional entre seu nível atual e o próximo            |
| Créditos adicionados | O pacote permanece disponível por 1 ano e não é redefinido mensalmente | Cota completa do novo nível, substituindo o nível anterior          |
| Melhor para          | Picos ocasionais além de um plano estável                              | Crescimento constante — subindo de nível à medida que o uso aumenta |

<div id="auto-recharge">
  ### Recarga automática
</div>

Quando seus créditos restantes caírem abaixo de um limite, Firecrawl compra automaticamente um pacote adicional de créditos e o adiciona ao seu saldo. Por padrão, o limite é de **0 créditos**, mas você pode personalizá-lo nas suas [configurações de cobrança](https://www.firecrawl.dev/app/settings?tab=billing) para acionar as recargas mais cedo — por exemplo, definir um limite mais alto garante que você sempre tenha uma reserva de créditos disponível, o que é útil se você executar grandes rastreamentos. Observe que essa verificação ocorre quando você faz uma requisição à API — seu saldo não será recarregado até que sua próxima requisição acione a recarga.

* Pacotes de recarga automática estão disponíveis em todos os planos pagos
* Os tamanhos dos pacotes e os preços variam conforme o plano (visíveis na [página de preços](https://www.firecrawl.dev/pricing))
* Você pode ativar ou desativar a recarga automática a qualquer momento
* A recarga automática é limitada a **4 pacotes por mês**
* Os créditos dos pacotes de recarga automática **não são restaurados mensalmente** — eles permanecem na sua conta entre ciclos de cobrança e expiram **1 ano** após a compra, ao contrário dos créditos mensais do seu plano, que são restaurados a cada período.

A recarga automática é ideal para lidar com picos pontuais de uso. Se você perceber que está excedendo de forma consistente a cota do seu plano, fazer upgrade para um plano superior — ou ativar o Smart Upgrade — oferecerá um melhor custo-benefício por crédito.

<div id="smart-upgrade">
  ### Upgrade Inteligente
</div>

O Upgrade Inteligente é um modo de cobrança opcional que move você automaticamente para o próximo nível de créditos sempre que seu saldo zera, para que seu uso não seja interrompido. Em vez de comprar um pacote avulso de créditos, você passa para o próximo nível de assinatura no slider (por exemplo, Hobby 5k → Hobby 6.5k → Hobby 8k → Standard 100k → … → Scale 1M).

Como funciona:

* **Acionamento:** o Upgrade Inteligente é acionado quando seu saldo restante de créditos chega a 0. Se você ainda tiver qualquer crédito disponível — incluindo créditos de recarga automática não utilizados — nenhum upgrade ocorrerá.
* **Cobrança:** é cobrado de você um valor proporcional equivalente à diferença entre seu nível atual e o próximo nível. Nos planos **mensais**, isso corresponde à diferença de preço mensal; nos planos **anuais**, é `(meses restantes no ciclo ÷ 12) × diferença de preço anual`. O cálculo proporcional usa meses corridos completos desde o início do seu período de cobrança atual.
* **Créditos concedidos:** você recebe a franquia total de créditos do novo nível, menos a franquia do seu nível anterior (ou seja, você paga apenas pela diferença).
* **Notificações:** cada vez que o Upgrade Inteligente é acionado, você recebe um e-mail confirmando o upgrade, o valor cobrado e uma prévia da próxima etapa da progressão. Desative o Upgrade Inteligente em [configurações de cobrança](https://www.firecrawl.dev/app/settings?tab=billing) a qualquer momento para interromper upgrades futuros.
* **Topo da progressão:** ao chegar ao nível Scale (1.000.000 de créditos/mês), o Upgrade Inteligente não terá mais níveis seguintes. Para aumentar os créditos além de Scale, [fale com vendas](https://www.firecrawl.dev/enterprise) sobre um plano Enterprise personalizado.

O Upgrade Inteligente é ativado no momento em que você assina ou muda de plano (você verá o botão na confirmação do checkout) e pode ser ativado ou desativado depois nas suas configurações de cobrança. Ele está disponível para organizações com uma única equipe em planos padrão; organizações com contratos Enterprise personalizados continuam usando seu modelo de cobrança negociado.

O Upgrade Inteligente é ideal se seu uso estiver crescendo de forma constante e você preferir subir a escada de planos automaticamente em vez de complementar com pacotes de créditos. Use a Recarga Automática se seu plano base for o mais adequado e você só precisar de recargas avulsas ocasionais.

<div id="coupons">
  ## Cupons
</div>

A Firecrawl oferece dois tipos de cupons:

* **Cupons de assinatura** aplicam um desconto à assinatura do seu plano (por exemplo, uma porcentagem de desconto no valor mensal ou anual). Eles **só** podem ser aplicados durante o fluxo de checkout do Stripe, quando você assina um plano pago pela primeira vez ou muda de plano. Não é possível aplicar um cupom de assinatura depois que o checkout for concluído.
* **Cupons de crédito** adicionam créditos bônus à sua conta. Eles podem ser resgatados na seção **cobrança** do seu dashboard em [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing). Procure o campo de cupom na página de cobrança para aplicar seu código. Os créditos bônus de cupons de crédito são separados da cota mensal do seu plano e permanecem mesmo se você fizer upgrade ou downgrade do seu plano.

Se você tiver um código de cupom e não souber de que tipo ele é, tente aplicá-lo primeiro na seção de cobrança do seu dashboard. Se for um cupom de assinatura, você precisará usá-lo na página de checkout do Stripe.

<div id="billing-cycle">
  ## Ciclo de cobrança
</div>

* **Planos mensais**: Os créditos são restaurados na sua data de renovação mensal
* **Planos anuais**: A cobrança é anual, mas os créditos ainda são restaurados todo mês na sua data de renovação mensal virtual
* **Créditos do plano não utilizados não são transferidos por padrão** — sua cota mensal é restaurada a cada mês. **Planos anuais Scale transferem créditos do plano não utilizados por 1 mês**, e **planos anuais Enterprise os transferem por 2 meses**. Créditos de pacotes de recarga automática não estão vinculados ao seu ciclo de cobrança — eles permanecem e expiram **1 ano** a partir da data da compra.

<div id="upgrading-and-downgrading">
  ## Upgrades e downgrades de plano
</div>

* **Upgrades** entram em vigor imediatamente. O valor integral do novo plano é cobrado hoje (sem cobrança proporcional), e seu ciclo de cobrança é reiniciado — sua próxima renovação será em um mês ou um ano a partir da data do upgrade. Quaisquer créditos não utilizados do seu plano anterior são transferidos, e sua nova franquia de créditos e limites de concorrência se aplicam na hora.
* **Downgrades** são agendados para entrar em vigor na sua próxima data de renovação. Você mantém os créditos e limites do seu plano atual até lá, e o tempo não utilizado do seu plano atual não gera créditos nem reembolso. Você pode desfazer um downgrade agendado nas suas [configurações de cobrança](https://www.firecrawl.dev/app/settings?tab=billing) a qualquer momento antes da data de vigência.

<div id="switching-between-monthly-and-yearly-billing">
  ### Alternar entre cobrança mensal e anual
</div>

* **Mensal → Anual** no mesmo nível de créditos ou em um nível superior é considerado um upgrade imediato.
* **Anual → Mensal** é considerado um upgrade imediato apenas se você mudar para um nível de créditos estritamente superior.

Os pacotes de recarga automática que você já comprou não são afetados por mudanças de plano — eles mantêm a validade original de 1 ano, independentemente de upgrades ou downgrades.

<div id="what-happens-when-you-run-out-of-credits">
  ## O que acontece quando você fica sem créditos
</div>

Se você esgotar sua cota de créditos e não tiver a recarga automática ou o upgrade inteligente ativados, as solicitações de API que consomem créditos retornarão um erro **HTTP 402 (Pagamento Necessário)**.

* Com a **recarga automática** ativada, o uso continua enquanto pacotes de recarga são comprados automaticamente — mas, se o limite mensal de pacotes (4) for atingido ou uma recarga falhar, seu saldo poderá ficar negativo até o próximo ciclo de cobrança.
* Com o **upgrade inteligente** ativado, sua assinatura é movida automaticamente para o próximo nível de créditos no seletor, e você recebe imediatamente a cota de créditos do novo nível. O upgrade inteligente para quando você atinge o nível Scale.

Para retomar o uso após uma parada total, você pode:

1. Ativar a [recarga automática](#auto-recharge) ou o [upgrade inteligente](#smart-upgrade)
2. Migrar para um plano superior
3. Aguardar a redefinição dos seus créditos no próximo ciclo de cobrança

<div id="free-plan">
  ## Plano Gratuito
</div>

O plano gratuito oferece **1.000 créditos por mês**, sem necessidade de cartão de crédito. Os créditos são redefinidos todo mês na sua data de renovação mensal — depois que forem usados, você pode aguardar a próxima redefinição ou fazer upgrade para um plano pago para obter uma cota maior. O plano gratuito também possui limites de taxa e de concorrência menores em comparação aos planos pagos (consulte [Limites de Taxa](/pt-BR/rate-limits)).

  ## Perguntas frequentes
</div>

<AccordionGroup>
  <Accordion title="Créditos não utilizados são transferidos para o próximo mês?">
    **Créditos do plano** não são transferidos por padrão — sua cota mensal é redefinida a cada mês. **Planos anuais Scale transferem créditos do plano não utilizados por 1 mês**, e **planos anuais Enterprise os transferem por 2 meses**. Créditos comprados por meio de **pacotes de recarga automática** não estão vinculados ao seu ciclo de cobrança e permanecem disponíveis em qualquer plano. Créditos de recarga automática expiram **1 ano** após a data da compra.
  </Accordion>

  <Accordion title="Qual é a diferença entre recarga automática e upgrade inteligente?">
    **recarga automática** compra um pacote avulso de créditos quando seu saldo atinge um limite, mantendo você no seu plano atual. **upgrade inteligente** move você para o próximo nível de créditos na escala de preços (por exemplo, Hobby 5k → 6.5k → 8k → Standard 100k → …) quando seu saldo chega a 0, cobrando a diferença proporcional. Eles são mutuamente exclusivos — sua organização usa um ou outro. Use a recarga automática para picos ocasionais e o upgrade inteligente se o seu uso estiver crescendo de forma constante e você preferir subir de nível automaticamente.
  </Accordion>

  <Accordion title="Como a cobrança do upgrade inteligente é calculada?">
    O upgrade inteligente cobra a diferença proporcional entre o seu nível atual e o próximo nível. Nos planos **mensais**, isso é simplesmente a diferença de preço mensal. Nos planos **anuais**, é `(meses restantes no seu ciclo de cobrança ÷ 12) × diferença de preço anual`, usando meses corridos completos a partir do início do seu período atual. Você também recebe a diferença de créditos entre os dois níveis, então paga apenas pelo que ganha.
  </Accordion>

  <Accordion title="Posso escolher uma quantidade de créditos entre dois planos?">
    Sim. O [slider de preços](https://www.firecrawl.dev/pricing) mostra níveis intermediários de créditos dentro de cada plano — Hobby tem 5k/6.5k/8k, Standard tem 100k/130k/160k e Growth tem 500k/650k. Esses níveis intermediários têm o mesmo nível de concorrência e suporte do nível principal do plano; apenas a cota de créditos e o preço mudam.
  </Accordion>

  <Accordion title="Como eu sei quantos créditos ainda tenho?">
    Verifique o painel em [firecrawl.dev/app](https://www.firecrawl.dev/app), ou acesse o [endpoint da API de uso de créditos](/pt-BR/api-reference/endpoint/credit-usage) programaticamente.
  </Accordion>

  <Accordion title="Qual é a diferença entre créditos e tokens?">
    Créditos são a unidade de cobrança para chamadas à API do Firecrawl. Tokens se referem a tokens de LLM usados internamente por endpoints como Extract e Agent. Cada crédito é equivalente a 15 tokens. Para a maioria dos usuários, você só precisa pensar em termos de créditos.
  </Accordion>

  <Accordion title="Onde aplico um código de cupom?">
    Depende do tipo de cupom. **Cupons de crédito** são aplicados na seção cobrança do seu painel. **Cupons de assinatura** (descontos no preço do seu plano) só podem ser aplicados na página de checkout do Stripe ao assinar ou mudar de plano.
  </Accordion>

  </Accordion>

  <Accordion title="Em qual moeda as faturas são cobradas?">

  </Accordion>

  <Accordion title="Como adiciono um número de VAT, nome da empresa ou endereço de cobrança às minhas faturas?">
    Acesse suas [configurações de cobrança](https://www.firecrawl.dev/app/settings?tab=billing), clique em **Manage Subscriptions** e atualize seu endereço de cobrança, nome da empresa e número de VAT no portal da Stripe. As próximas faturas incluirão automaticamente as informações atualizadas. Se precisar que uma fatura anterior seja reemitida com os novos dados, entre em contato com [help@firecrawl.dev](mailto:help@firecrawl.dev).
  </Accordion>

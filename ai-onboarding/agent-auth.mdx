## Agent registration

**Can you mint a WorkOS ID-JAG for `aud=https://api.firecrawl.dev/`?**

If **yes**, fetch [`auth.md`](https://www.firecrawl.dev/auth.md) for the full ID-JAG claim requirements, discovery steps, registration body, and error handling. Do not skip ahead in that doc.

If you **cannot** mint a WorkOS ID-JAG, a human should create an API key in the [Firecrawl dashboard](https://www.firecrawl.dev/app/api-keys) and provide it to you. For CLI install, skills, and MCP setup after you have a key, see [Skills + CLI](/ai-onboarding#skills-cli).

## How discovery works

Agents that hit the Firecrawl API without credentials can learn what to do next:

1. **`401` from the API** — `api.firecrawl.dev` returns `WWW-Authenticate` pointing at Protected Resource Metadata (PRM).
2. **PRM** — `https://www.firecrawl.dev/.well-known/oauth-protected-resource`
3. **Authorization Server metadata** — `https://www.firecrawl.dev/.well-known/oauth-authorization-server` includes an `agent_auth` block with `skill` → `auth.md` and `register_uri` → `https://www.firecrawl.dev/agent/auth`

If you already have a direct link to [`auth.md`](https://www.firecrawl.dev/auth.md), fetch it and follow it — you do not need this page.

## What happens next

Follow [`auth.md`](https://www.firecrawl.dev/auth.md) Steps 1–4: discover → register at `POST /agent/auth` → use the returned API key as a bearer token against `https://api.firecrawl.dev`.

## Unsupported today

Firecrawl does **not** currently support through `/agent/auth`:

- Anonymous registration
- Verified-email-only registration (without ID-JAG)
- Claim ceremony endpoints
- Agent-facing revocation endpoints

If a previously working API key returns `401`, drop it and restart discovery per `auth.md`.

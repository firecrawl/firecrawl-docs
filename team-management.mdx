## Overview

Firecrawl lets you invite teammates to collaborate under a shared account. From the **Team Settings** page in your [dashboard](https://www.firecrawl.dev/app), you can invite members, assign roles, and manage your team. Each team supports up to **20 seats**.

## Roles

Every team member is assigned one of two roles: **Admin** or **Member**. You choose the role when sending an invitation.

| Capability | Admin | Member |
|---|:---:|:---:|
| Use the team's API keys and shared resources | ✓ | ✓ |
| View the team member list | ✓ | ✓ (read-only) |
| Invite new team members | ✓ | ✗ |
| Remove team members | ✓ | ✗ |
| Change a member's role | ✓ | ✗ |
| Revoke pending invitations | ✓ | ✗ |
| Edit the team name | ✓ | ✗ |
| Access team settings | ✓ | ✗ |
| Leave the team voluntarily | ✓ | ✓ |

In short, **Admins** have full control over team management, while **Members** can use the team's resources but cannot make changes to the team itself.

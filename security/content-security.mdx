Firecrawl applies different levels of content security depending on whether you use LLM-powered features or raw content formats.

## LLM-Powered Features

When using Firecrawl's LLM-powered features (such as the [Agent](/features/agent) endpoint or [JSON extraction](/features/scrape#extract-structured-data)), prompt injection defenses are applied internally:

- **Input sanitization** — All scraped content interpolated into LLM prompts is sanitized. Control characters are stripped, newlines are flattened, backticks are removed, quotes are escaped, and content is length-limited.
- **System prompt hardening** — System prompts include explicit instructions that page content is untrusted and that any embedded directives (e.g., "ignore previous instructions") should be treated as part of the page data, not as real instructions.

These defenses protect against prompt injection attacks originating from scraped web content.

## Raw Content Formats

When using raw content formats (`markdown`, `html`, `rawHtml`), Firecrawl does **not** perform security filtering on the returned content. The scraping pipeline converts and cleans page content for readability, but it does not:

- Strip or flag executable tags (e.g., `<script>` tags in `rawHtml`)
- Detect or remove prompt injection patterns
- Filter malicious embedded payloads
- Remove requests to call tools, APIs, or external links

Raw content is returned as-is after format conversion. This is by design, since different consumers have different threat models and sanitization requirements.

## Recommendations

If you are building a pipeline that passes scraped content to your own LLMs or displays it to users, consider adding your own security layer:

- **Prompt injection filtering** — Sanitize scraped content before passing it to your own LLMs.
- **HTML sanitization** — Strip dangerous tags if you are processing `rawHtml`.
- **Link validation** — Validate any URLs extracted from scraped content before following them.
- **Input limits** — Apply length and complexity limits appropriate to your downstream systems.

If you are using Firecrawl's Agent or JSON extraction features, prompt injection defenses are already in place on our side, but adding your own layer as defense-in-depth is still recommended.

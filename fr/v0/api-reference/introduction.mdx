<div id="base-url">
  ## URL de base
</div>

Toutes les requêtes contiennent l’URL de base suivante :

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## Authentification
</div>

Pour l’authentification, vous devez inclure un en-tête HTTP `Authorization`. Cet en-tête doit contenir `Bearer fc_123456789`, où `fc_123456789` correspond à votre clé API.

```bash
Authorization: Bearer fc_123456789
```

​

<div id="response-codes">
  ## Codes de réponse
</div>

Firecrawl utilise des codes d'état HTTP conventionnels pour indiquer le résultat de vos requêtes.

En général, les codes d'état HTTP 2xx indiquent une réussite, les codes 4xx représentent des erreurs liées à l'utilisateur, et les codes 5xx signalent des problèmes d'infrastructure.

Status | Description
--- | ---
200 | La requête a réussi.
400 | Vérifiez la validité des paramètres.
401 | La clé d'API n'a pas été fournie.
402 | Paiement requis.
404 | La ressource demandée est introuvable.
429 | La limite de débit a été dépassée.
5xx | Indique une erreur de serveur avec Firecrawl.

Consultez la section Codes d’erreur pour une explication détaillée de toutes les erreurs API possibles.

​

<div id="rate-limit">
  ## Limite de débit
</div>

L’API Firecrawl impose une limite de débit afin d’assurer la stabilité et la fiabilité du service. Cette limite s’applique à tous les points de terminaison et se base sur le nombre de requêtes effectuées dans une période donnée.

Si vous dépassez cette limite, vous recevrez un code de réponse 429.

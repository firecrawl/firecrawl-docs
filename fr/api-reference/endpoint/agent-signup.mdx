---
title: "Inscription Agent"
description: "Créez une clé API Firecrawl en bac à sable au nom d’un sponsor humain."
---

Crée immédiatement une nouvelle clé API, en bac à sable avec 50 credits. Un e-mail de vérification est envoyé au sponsor. Aucune authentification requise.

Pour un aperçu complet du flux, consultez le [guide Agent Signup](/fr/features/agent-signup).

<div id="headers">
  ## En-têtes
</div>

| En-tête        | Valeur             |
| -------------- | ------------------ |
| `Content-Type` | `application/json` |

<div id="request-body">
  ## Corps de la requête
</div>

| Parameter      | Type    | Obligatoire | Description                                                                                                                |
| -------------- | ------- | ----------- | -------------------------------------------------------------------------------------------------------------------------- |
| `email`        | string  | **Oui**     | Adresse e-mail du sponsor humain. Doit être une adresse e-mail valide.                                                     |
| `agent_name`   | string  | **Oui**     | Nom de l’agent qui crée la clé (1 à 100 caractères). Affiché dans l’e-mail de vérification.                                |
| `accept_terms` | boolean | **Oui**     | Doit être défini sur `true`. Indique l’acceptation des [Conditions d’utilisation](https://firecrawl.dev/terms-of-service). |

<div id="response">
  ## Réponse
</div>

| Champ | Type | Description |
|-------|------|-------------|
| `success` | boolean | Indique si la clé a été créée |
| `api_key` | string | La clé API fournie (commence par `fc-`) |
| `sponsor_status` | string | Toujours `"pending"` lors de la création |
| `credit_limit` | number | Nombre maximal de crédits disponibles avant vérification (50) |
| `credits_remaining` | number | Crédits restants (50 lors de la création) |
| `verification_deadline_at` | string | Horodatage ISO 8601. Le sponsor doit confirmer avant cette échéance (5 jours). |
| `tos_url` | string | URL des Conditions d’utilisation acceptées |

<div id="example-request">
  ### Exemple de requête
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/agent-signup" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@company.com",
    "agent_name": "Research Assistant",
    "accept_terms": true
  }'
```

<div id="example-response">
  ### Exemple de réponse
</div>

```json
{
  "success": true,
  "api_key": "fc-abc123...",
  "sponsor_status": "pending",
  "credit_limit": 50,
  "credits_remaining": 50,
  "verification_deadline_at": "2026-03-23T12:00:00.000Z",
  "tos_url": "https://firecrawl.dev/terms-of-service"
}
```

<div id="error-responses">
  ## Réponses d’erreur
</div>

| État  | Erreur                                                               | Quand                                                              |
| ----- | -------------------------------------------------------------------- | ------------------------------------------------------------------ |
| `400` | Corps de la requête invalide                                         | Champs manquants ou invalides                                      |
| `403` | Les inscriptions d’agent sont bloquées pour cet e-mail               | Le sponsor a précédemment bloqué les inscriptions d’agent          |
| `403` | La vérification d’une précédente inscription d’agent a expiré        | Une inscription précédente a expiré ; le sponsor doit se connecter |
| `409` | Une confirmation d’inscription d’agent en attente a déjà été envoyée | Une clé est déjà en attente pour cet e-mail                        |

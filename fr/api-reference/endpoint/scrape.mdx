<div id="browser-actions">
  ## Actions du navigateur
</div>

Vous pouvez interagir avec le navigateur avant le scraping en utilisant le paramètre `actions`. Cela vous permet de :

* Attendre le chargement des éléments
* Cliquer sur des boutons ou des liens
* Remplir des champs de formulaire
* Appuyer sur des touches
* Faire défiler des pages
* Prendre des captures d’écran
* Exécuter du JavaScript personnalisé
* Générer des PDF

Les actions prises en charge incluent `wait`, `click`, `write`, `press`, `scroll`, `screenshot`, `scrape`, `executeJavascript` et `pdf`. Pour une documentation détaillée avec des exemples, consultez le [Guide de scraping avancé](https://docs.firecrawl.dev/advanced-scraping-guide#actions-actions).

<div id="persistent-profiles">
  ## Profils persistants
</div>

Passez le paramètre `profile` pour conserver l’état du navigateur (`cookies`, `localStorage`, sessions) entre les sessions de scrape et d’[Interact](/fr/features/interact). Les opérations de scrape utilisant le même nom de profil partagent l’état du navigateur — pratique pour rester connecté ou conserver des préférences.

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/dashboard",
    "profile": {
      "name": "my-session",
      "saveChanges": true
    }
  }'
```

| Paramètre             | Type    | Obligatoire | Par défaut | Description                                                                                                                                                                     |
| --------------------- | ------- | ----------- | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `profile.name`        | string  | Oui         | —          | Nom du profil (1 à 128 caractères). Les scrapes portant le même nom partagent l’état du navigateur.                                                                             |
| `profile.saveChanges` | boolean | Non         | `true`     | Lorsque `true`, l’état du navigateur est réenregistré dans le profil à la fin de la session Interact. Définissez `false` pour charger les données existantes sans les réécrire. |

<Note>
  Une seule session peut enregistrer un profil à la fois. Si une autre session est déjà en cours d’enregistrement, vous recevrez une erreur `409`. Vous pouvez quand même ouvrir le même profil avec `saveChanges: false`, ou réessayer plus tard.
</Note>

Pour en savoir plus, consultez [Interact — Profils persistants](/fr/features/interact#persistent-profiles).

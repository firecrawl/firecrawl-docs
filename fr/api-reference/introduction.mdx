<div id="base-url">
  ## URL de base
</div>

Toutes les requêtes utilisent l’URL de base suivante :

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## Authentification
</div>

Pour s’authentifier, vous devez inclure un en-tête Authorization. Cet en-tête doit contenir `Bearer fc-123456789`, où `fc-123456789` correspond à votre clé API.

```bash
Authorization : Bearer fc-123456789
```

​

<div id="response-codes">
  ## Codes de réponse
</div>

Firecrawl utilise des codes d’état HTTP conventionnels pour indiquer l’issue de vos requêtes.

En général, les codes d’état HTTP 2xx indiquent une réussite, les codes 4xx des erreurs imputables au client, et les codes 5xx des problèmes d’infrastructure.

Status | Description
--- | ---
200 | Requête réussie.
400 | Vérifiez la validité des paramètres.
401 | Clé d’API manquante.
402 | Paiement requis.
404 | La ressource demandée est introuvable.
429 | La limite de débit a été dépassée.
5xx | Indique une erreur serveur côté Firecrawl.

Reportez-vous à la section Codes d’erreur pour une explication détaillée de toutes les erreurs d’API possibles.

<div id="firecrawl-error-codes-5xx">
  ### Codes d’erreur Firecrawl (5xx)
</div>

Lorsqu’une erreur 5xx se produit, Firecrawl fournit des codes d’erreur plus précis pour indiquer ce qui s’est mal passé.​

Error Code | Status | Description\
--- | --- | ---\
`SCRAPE_ALL_ENGINES_FAILED` | 500 | Échec de tous les moteurs de scraping.\
`SCRAPE_SSL_ERROR` | 500 | Le certificat SSL de la page est invalide. Vous pouvez utiliser `skipTlsVerification:true` pour contourner cette vérification.\
`SCRAPE_SITE_ERROR` | 500 | Erreur de site non récupérable.\
`SCRAPE_DNS_RESOLUTION_ERROR` | 500 | Échec de la résolution DNS.\
`SCRAPE_ACTION_ERROR` | 500 | Erreur lors de l’exécution d’une action sur la page.\
`SCRAPE_PDF_PREFETCH_FAILED` | 500 | Échec du préchargement du PDF.\
`SCRAPE_PDF_INSUFFICIENT_TIME_ERROR` | 500 | Temps insuffisant pour traiter le PDF.\
`SCRAPE_PDF_ANTIBOT_ERROR` | 500 | PDF bloqué par des mécanismes antibot.\
`SCRAPE_ZDR_VIOLATION_ERROR` | 500 | Conflit Zero Data Retention : survient lorsque `zeroDataRetention:true` est activé mais qu’une autre option (p. ex. `screenshot`) nécessite un stockage temporaire.\
`SCRAPE_UNSUPPORTED_FILE_ERROR` | 500 | Type de fichier non pris en charge.\
`UNKNOWN_ERROR` | 500 | Erreur générique ou inattendue.

<div id="rate-limit">
  ## Limite de débit
</div>

L’API Firecrawl applique une limite de débit afin d’assurer la stabilité et la fiabilité du service. Cette limite s’applique à tous les points de terminaison et dépend du nombre de requêtes effectuées sur une période donnée.

Si vous dépassez la limite de débit, vous recevrez un code de réponse 429.

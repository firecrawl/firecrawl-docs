---
title: "Générer LLMs.txt"
openapi: "v1-openapi POST /llmstxt"
---

Le point de terminaison de génération LLMs.txt crée les fichiers LLMs.txt et LLMs-full.txt pour n’importe quel site web. Ces fichiers proposent un format structuré, optimisé pour les LLM, du contenu du site, afin de faciliter la compréhension et le traitement des informations par les modèles de langage.

Vous cherchez le point de terminaison de statut ? Consultez le point de terminaison [LLMs.txt Status](/fr/api-reference/endpoint/llmstxt-get).

<div id="response-structure">
  ### Structure de la réponse
</div>

La réponse comprend :

* **success** : booléen indiquant si la requête a réussi
* **id** : identifiant unique du job de génération

<div id="output-format">
  ### Format de sortie
</div>

1. **LLMs.txt**

   * Résumé concis et structuré du site web
   * Contient des liens essentiels et leurs descriptions
   * Mis en forme en Markdown pour un traitement/parsing facile
   * Exemple :

     ```markdown
     # http://example.com llms.txt

     - [Titre de la page](https://example.com/page) : Brève description
     - [Autre page](https://example.com/another) : Autre description
     ```

2. **LLMs-full.txt** (lorsque showFullText est à true)

   * Contient l’intégralité du contenu des pages traitées
   * Conserve la structure hiérarchique
   * Inclut des informations plus détaillées
   * Exemple :

     ```markdown
     # http://example.com llms-full.txt

     ## Titre de la page

     Contenu complet de la page…

     ## Autre page

     Contenu plus détaillé…
     ```

<div id="billing">
  ### Facturation
</div>

La facturation repose sur les appels à l’API et les URL traitées :

* Coût de base : 1 crédit par appel à l’API
* Supplément : 1 crédit par URL traitée
* Maîtrisez le coût des URL avec le paramètre `maxUrls`

<div id="base-url">
  ## URL de base
</div>

Toutes les requêtes utilisent l’URL de base suivante :

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## Authentification
</div>

Pour l’authentification, vous devez inclure un en-tête Authorization. Cet en-tête doit contenir `Bearer fc-123456789`, où `fc-123456789` correspond à votre clé d’API.

```bash
Authorization : Bearer fc-123456789
```

​

<div id="response-codes">
  ## Codes de réponse
</div>

Firecrawl utilise des codes d’état HTTP classiques pour indiquer l’issue de vos requêtes.

En règle générale, les codes 2xx indiquent un succès, les 4xx des erreurs imputables à l’utilisateur, et les 5xx des problèmes d’infrastructure.

Status | Description
--- | ---
200 | Requête réussie.
400 | Vérifiez la validité des paramètres.
401 | La clé d’API est manquante.
402 | Paiement requis
404 | La ressource demandée est introuvable.
429 | La limite de débit a été dépassée.
5xx | Indique une erreur serveur côté Firecrawl.

Reportez-vous à la section Codes d’erreur pour une explication détaillée de toutes les erreurs d’API possibles.

<div id="rate-limit">
  ## Limite de débit
</div>

L’API Firecrawl applique une limite de débit pour garantir la stabilité et la fiabilité du service. Cette limite s’applique à tous les points de terminaison et se base sur le nombre de requêtes effectuées dans une période donnée.

Si vous dépassez la limite de débit, vous recevrez un code de réponse 429.

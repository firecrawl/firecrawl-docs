---
title: "Générer LLMs.txt"
openapi: "v1-openapi POST /llmstxt"
---

Le point de terminaison de génération de LLMs.txt crée les fichiers LLMs.txt et LLMs-full.txt pour n’importe quel site web. Ces fichiers offrent un format structuré et compatible avec les LLM pour le contenu du site, afin de faciliter la compréhension et le traitement de l’information par les modèles de langage.

Vous cherchez le point de terminaison de statut ? Consultez le point de terminaison [LLMs.txt Status](/fr/api-reference/endpoint/llmstxt-get).

<div id="response-structure">
  ### Structure de la réponse
</div>

La réponse comprend :

* **success** : booléen indiquant si la requête a réussi
* **id** : identifiant unique du job de génération

<div id="output-format">
  ### Format de sortie
</div>

1. **LLMs.txt**

   * Résumé concis et structuré du site web
   * Contient les liens clés et leurs descriptions
   * Mis en forme en Markdown pour un traitement facile
   * Exemple :

     ```markdown
     # http://example.com llms.txt

     - [Page Title](https://example.com/page): Brief description
     - [Another Page](https://example.com/another): Another description
     ```

2. **LLMs-full.txt** (lorsque showFullText est true)

   * Contient l’intégralité du contenu des pages traitées
   * Préserve la structure hiérarchique
   * Inclut des informations plus détaillées
   * Exemple :

     ```markdown
     # http://example.com llms-full.txt

     ## Page Title

     Full content of the page...

     ## Another Page

     More detailed content...
     ```

<div id="billing">
  ### Facturation
</div>

La facturation est basée sur les appels à l’API et les URL traitées :

* Coût de base : 1 crédit par appel à l’API
* Supplément : 1 crédit par URL traitée
* Contrôlez le coût des URL avec le paramètre `maxUrls`

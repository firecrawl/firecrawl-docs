Obtenez le statut et les résultats d’une génération de LLMs.txt. Ce point de terminaison vous permet de vérifier si la génération est terminée et de récupérer le contenu produit.

<div id="response-structure">
  ### Structure de la réponse
</div>

La réponse comprend :

- **success**: Booléen indiquant si la requête a réussi
- **status**: État actuel de la tâche de génération :

  - `processing`: La tâche est toujours en cours d’exécution
  - `completed`: La tâche s’est terminée avec succès
  - `failed`: La tâche a rencontré une erreur

- **data**: Contenu généré (lorsque l’état est `completed`) :

  - `llmstxt`: Contenu LLMs.txt généré
  - `llmsfulltxt`: Texte intégral (si showFullText était true)

- **expiresAt**: Horodatage ISO indiquant la date d’expiration des résultats

<div id="status-examples">
  ### Exemples de statuts
</div>

1. **Statut : en cours de traitement**

   ```json
   {
     "success": true,
     "status": "processing",
     "data": {
       "llmstxt": "# Firecrawl.dev llms.txt\n\n- [Web Data Extraction Tool](https://www.firecrawl.dev/)...",
       "llmsfulltxt": "# Firecrawl.dev llms-full.txt\n\n"
     },
     "expiresAt": "2025-03-03T23:19:18.000Z"
   }
   ```

2. **Statut : terminé**
   ```json
   {
     "success": true,
     "status": "completed",
     "data": {
       "llmstxt": "# http://firecrawl.dev llms.txt\n\n- [Web Data Extraction Tool](https://www.firecrawl.dev/): Transform websites into clean, LLM-ready data effortlessly.\n- [Flexible Web Scraping Pricing](https://www.firecrawl.dev/pricing): Flexible pricing plans for web scraping and data extraction.",
       "llmsfulltxt": "# http://firecrawl.dev llms-full.txt\n\n## Web Data Extraction Tool\nIntroducing /extract - Get web data with a prompt..."
     },
     "expiresAt": "2025-03-03T22:45:50.000Z"
   }
   ```

<div id="error-handling">
  ### Gestion des erreurs
</div>

Si la tâche de génération échoue ou est introuvable, vous recevrez une réponse d’erreur appropriée :

```json
{
  "success": false,
  "error": "Tâche de génération de LLMs.txt introuvable"
}
```

<div id="polling-recommendations">
  ### Recommandations de polling
</div>

1. Interroger toutes les 2 à 3 secondes au départ
2. Augmenter l’intervalle si le traitement est toujours en cours après 30 secondes
3. Arrêter le polling si le statut est `completed` ou `failed`
4. Mettre en place un backoff exponentiel pour éviter les limites de débit

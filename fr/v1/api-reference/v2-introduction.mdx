<div id="base-url">
  ## URL de base
</div>

Toutes les requêtes incluent l’URL de base suivante :

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## Authentification
</div>

Pour l’authentification, vous devez inclure un en-tête Authorization. Cet en-tête doit contenir `Bearer fc-123456789`, où `fc-123456789` correspond à votre clé API.

```bash
Authorization : Bearer fc-123456789
```

​

<div id="response-codes">
  ## Codes de réponse
</div>

Firecrawl utilise des codes d’état HTTP classiques pour indiquer le résultat de vos requêtes.

En général, les codes HTTP 2xx indiquent une réussite, les codes 4xx des erreurs imputables à l’utilisateur, et les codes 5xx des problèmes d’infrastructure.

Status | Description
--- | ---
200 | Requête réussie.
400 | Vérifiez l’exactitude des paramètres.
401 | Clé API non fournie.
402 | Paiement requis.
404 | Ressource introuvable.
429 | La limite de débit a été dépassée.
5xx | Indique une erreur serveur côté Firecrawl.

Reportez-vous à la section Codes d’erreur pour une explication détaillée de toutes les erreurs API possibles.

<div id="rate-limit">
  ## Limite de débit
</div>

L’API Firecrawl impose une limite de débit pour garantir la stabilité et la fiabilité du service. Elle s’applique à tous les points de terminaison et se base sur le nombre de requêtes effectuées sur une période donnée.

Si vous dépassez cette limite, vous recevrez un code de statut 429.

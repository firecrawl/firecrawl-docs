Firecrawl propose différents types de proxy pour vous aider à extraire des sites web présentant divers niveaux de complexité. Le type de proxy peut être défini à l’aide du paramètre `proxy`.

> Par défaut, Firecrawl achemine toutes les requêtes via des proxy afin d’assurer la fiabilité et l’accès, même si vous ne précisez pas de type de proxy ni d’emplacement.

<div id="how-to-specify-proxy-location">
  ## Comment spécifier l’emplacement du proxy
</div>

Vous pouvez demander un emplacement de proxy précis en définissant le paramètre `location.country` dans votre requête. Par exemple, pour utiliser un proxy au Brésil, définissez `location.country` sur `BR`.

Pour plus de détails, consultez la [référence de l’API pour `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location-country).

<div id="proxy-types">
  ## Types de proxy
</div>

Firecrawl prend en charge trois types de proxy :

* **basic** : Proxies pour le scraping de la plupart des sites. Rapides et généralement efficaces.
* **enhanced** : Proxies améliorés pour le scraping de sites complexes tout en préservant la confidentialité. Plus lents, mais plus fiables sur certains sites. [En savoir plus sur le mode Enhanced →](/fr/features/enhanced-mode)
* **auto** : Firecrawl réessaie automatiquement le scraping avec des proxies améliorés si le proxy basic échoue. Si le réessai avec enhanced réussit, 5 crédits seront facturés pour le scrape. Si la première tentative avec basic réussit, seul le coût standard sera facturé.

***

> **Remarque :** Pour des informations détaillées sur l’utilisation des proxies améliorés, y compris les coûts en crédits et les stratégies de réessai, consultez la [documentation du mode Enhanced](/fr/features/enhanced-mode).

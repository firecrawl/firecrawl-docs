Firecrawl propose différents types de proxy pour vous aider à extraire des sites web présentant divers niveaux de protection anti-bot. Le type de proxy peut être défini à l’aide du paramètre `proxy`.

> Par défaut, Firecrawl achemine toutes les requêtes via des proxy afin d’assurer la fiabilité et l’accès, même si vous ne précisez pas de type de proxy ni d’emplacement.

<div id="location-based-proxy-selection">
  ## Sélection de proxy selon la localisation
</div>

Firecrawl choisit automatiquement le meilleur proxy en fonction de votre localisation spécifiée ou détectée. Cela optimise les performances et la fiabilité du scraping. Cependant, toutes les localisations ne sont pas encore prises en charge. Les localisations suivantes sont disponibles :

| Code pays | Nom du pays           | Prise en charge du mode stealth |
|-----------|------------------------|---------------------------------|
| AE        | Émirats arabes unis    | Non                             |
| AU        | Australie              | Non                             |
| BR        | Brésil                 | Oui                             |
| CN        | Chine                  | Non                             |
| DE        | Allemagne              | Oui                             |
| FR        | France                 | Oui                             |
| GB        | Royaume-Uni            | Non                             |
| JP        | Japon                  | Non                             |
| QA        | Qatar                  | Non                             |
| TR        | Turquie                | Non                             |
| US        | États-Unis             | Oui                             |
| VN        | Viêt Nam               | Non                             |

<Warning>La liste des localisations de proxy prises en charge a été mise à jour pour la dernière fois le 15 mai 2025. La disponibilité peut évoluer au fil du temps.</Warning>

Si vous avez besoin de proxys dans une localisation non répertoriée ci-dessus, veuillez [nous contacter](mailto:help@firecrawl.com) et nous indiquer vos besoins.

Si vous ne spécifiez pas de proxy ni de localisation, Firecrawl sélectionnera automatiquement la meilleure option en fonction du site ciblé et de votre requête.

<div id="how-to-specify-proxy-location">
  ## Comment spécifier l’emplacement du proxy
</div>

Vous pouvez demander un emplacement de proxy précis en définissant le paramètre `location.country` dans votre requête. Par exemple, pour utiliser un proxy au Brésil, définissez `location.country` sur `BR`.

Pour plus de détails, consultez la [référence de l’API pour `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location-country).

<div id="proxy-types">
  ## Types de proxy
</div>

Firecrawl prend en charge trois types de proxy :

* **basic** : Proxies pour le scraping de sites avec des solutions anti-bot inexistantes ou basiques. Rapides et généralement efficaces.
* **stealth** : Proxies furtifs pour le scraping de sites dotés de solutions anti-bot avancées, ou pour les sites qui bloquent les proxies classiques. Plus lents, mais plus fiables sur certains sites. [En savoir plus sur le mode Stealth →](/fr/features/stealth-mode)
* **auto** : Firecrawl réessaie automatiquement le scraping avec des proxies furtifs si le proxy basic échoue. Si le réessai avec stealth réussit, 5 crédits seront facturés pour le scrape. Si la première tentative avec basic réussit, seul le coût standard sera facturé.

***

> **Remarque :** Pour des informations détaillées sur l’utilisation des proxies furtifs, y compris les coûts en crédits et les stratégies de réessai, consultez la [documentation du mode Stealth](/fr/features/stealth-mode).

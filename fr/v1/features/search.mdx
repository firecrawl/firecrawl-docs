L’API de recherche de Firecrawl vous permet d’effectuer des recherches sur le web et, en option, de récupérer et d’extraire les résultats en une seule opération.

* Choisissez des formats de sortie précis (markdown, HTML, liens, captures d’écran)
* Lancez des recherches web avec des paramètres personnalisables (localisation, etc.)
* Récupérez en option le contenu des résultats de recherche dans différents formats
* Contrôlez le nombre de résultats et définissez des délais d’attente

Pour plus de détails, consultez la [référence du point de terminaison /search](https://docs.firecrawl.dev/api-reference/endpoint/search).

Valeurs `tbs` courantes :

- `qdr:h` - Dernière heure
- `qdr:d` - Dernières 24 heures
- `qdr:w` - Dernière semaine
- `qdr:m` - Dernier mois
- `qdr:y` - Dernière année

Pour un filtrage temporel plus précis, vous pouvez indiquer des plages de dates exactes en utilisant le format de plage personnalisé :

<CodeGroup>

```python Python
from firecrawl import FirecrawlApp

# Initialize the client with your API key
app = FirecrawlApp(api_key="fc-YOUR_API_KEY")

# Search for results from December 2024
search_result = app.search(
    "firecrawl updates",
    limit=10,
    tbs="cdr:1,cd_min:12/1/2024,cd_max:12/31/2024"
)
```

```js JavaScript
import FirecrawlApp from '@mendable/firecrawl-js';

// Initialize the client with your API key
const app = new FirecrawlApp({apiKey: "fc-YOUR_API_KEY"});

// Search for results from December 2024
app.search("firecrawl updates", {
  limit: 10,
  tbs: "cdr:1,cd_min:12/1/2024,cd_max:12/31/2024"
})
.then(searchResult => {
  console.log(searchResult.data);
});
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v1/search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR_API_KEY" \
  -d '{
    "query": "firecrawl updates",
    "limit": 10,
    "tbs": "cdr:1,cd_min:12/1/2024,cd_max:12/31/2024"
  }'
```

</CodeGroup>

<div id="custom-timeout">
  ### Délai d’attente personnalisé
</div>

Définissez un délai d’attente personnalisé pour les opérations de recherche :

<CodeGroup>

```python Python
from firecrawl import FirecrawlApp

# Initialisez le client avec votre clé API
app = FirecrawlApp(api_key="fc-YOUR_API_KEY")

# Définissez un délai d’attente de 30 secondes
search_result = app.search(
    "complex search query",
    limit=10,
    timeout=30000  # 30 secondes en millisecondes
)
```

```js JavaScript
import FirecrawlApp from '@mendable/firecrawl-js';

// Initialisez le client avec votre clé API
const app = new FirecrawlApp({apiKey: "fc-YOUR_API_KEY"});

// Définissez un délai d’attente de 30 secondes
app.search("complex search query", {
  limit: 10,
  timeout: 30000  // 30 secondes en millisecondes
})
.then(searchResult => {
  // Traitez les résultats
  console.log(searchResult.data);
});
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v1/search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR_API_KEY" \
  -d '{
    "query": "complex search query",
```

<div id="scraping-options">
  ## Options de scraping
</div>

Lors du scraping de résultats de recherche, vous pouvez spécifier plusieurs formats de sortie et des options de scraping avancées :

<CodeGroup>

```python Python
from firecrawl import FirecrawlApp, ScrapeOptions

# Initialize the client with your API key
app = FirecrawlApp(api_key="fc-YOUR_API_KEY")

# Get search results with multiple formats
search_result = app.search(
    "firecrawl features",
    limit=3,
    scrape_options=ScrapeOptions(formats=["markdown", "html", "links", "screenshot"])
)
```

```js JavaScript
import FirecrawlApp from '@mendable/firecrawl-js';

// Initialize the client with your API key
const app = new FirecrawlApp({apiKey: "fc-YOUR_API_KEY"});

// Get search results with multiple formats
app.search("firecrawl features", {
  limit: 3,
  scrapeOptions: {
    formats: ["markdown", "html", "links", "screenshot"]
  }
})
.then(searchResult => {
  // Process results with various formats
  console.log(searchResult.data);
});
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v1/search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR_API_KEY" \
  -d '{
    "query": "firecrawl features",
    "limit": 3,
    "scrapeOptions": {
      "formats": ["markdown", "html", "links", "screenshot"]
    }
  }'
```

</CodeGroup>

Formats disponibles :

- `markdown` : Contenu Markdown propre et mis en forme
- `html` : Contenu HTML traité
- `rawHtml` : Contenu HTML non modifié
- `links` : Liste des liens trouvés sur la page
- `screenshot` : Capture d’écran de la page
- `screenshot@fullPage` : Capture d’écran de la page entière
- `extract` : Extraction de données structurées

<div id="cost-implications">
  ## Incidences sur les coûts
</div>

Lorsque les résultats de recherche ne sont pas extraits (aucune option d’extraction spécifiée), le coût est de 2 crédits pour 10 résultats. Lorsque l’extraction est activée, il n’y a pas de frais supplémentaires pour les extractions de base de chaque résultat au-delà des coûts d’extraction standard.

Cependant, tenez compte des facteurs de coût suivants :

- **Analyse de PDF** : 1 crédit par page PDF (peut augmenter significativement les coûts pour les PDF multi‑pages)
- **Mode proxy furtif** : +4 crédits supplémentaires par résultat de recherche
- ***mode JSON*** : +4 crédits supplémentaires par résultat de recherche

Pour maîtriser les coûts :

- Définissez `parsePDF: false` si vous n’avez pas besoin du contenu des PDF
- Utilisez `proxy: "basic"` plutôt que `"stealth"` lorsque possible
- Limitez le nombre de résultats de recherche avec le paramètre `limit`

<div id="old-version">
  # Ancienne version
</div>

Avec le nouveau format `changeTracking`, vous pouvez surveiller les modifications sur un site web.

Le format `changeTracking` renvoie les champs suivants dans la réponse :

* `previousScrapeAt` : L’horodatage du précédent scrape auquel la page actuelle est comparée. (`null` s’il n’y a pas de précédent scrape)
* `changeStatus` : Le résultat de la comparaison entre les deux versions de la page
  * `new` : Cette page n’existait pas ou n’avait pas été découverte auparavant. (a généralement un `previousScrapeAt` à `null`)
  * `same` : Le contenu de cette page n’a pas changé depuis le dernier scrape.
  * `changed` : Le contenu de cette page a changé depuis le dernier scrape.
  * `removed` : Cette page a été supprimée depuis le dernier scrape.
* `visibility` : La visibilité de la page/URL actuelle.
  * `visible` : Cette page est visible, ce qui signifie que son URL a été découverte par un parcours organique (via des liens sur d’autres pages visibles ou le sitemap).
  * `hidden` : Cette page n’est pas visible, ce qui signifie que son URL a été retrouvée via la mémoire de précédents crawls de la même URL.

Quelques points utiles à connaître :

* Les scrapes sont toujours comparés à partir de leur réponse en markdown.
  * Le format `markdown` doit également être spécifié lors de l’utilisation du format `changeTracking`. D’autres formats peuvent aussi être ajoutés en complément.
  * L’algorithme de comparaison est robuste aux changements d’espaces et d’ordre du contenu. Les URL de source d’iframe sont actuellement ignorées pour mieux résister aux captchas et aux antibots qui randomisent les URL.
* Les scrapes précédents utilisés pour la comparaison sont actuellement appariés uniquement sur l’URL source, l’ID d’équipe et le format `markdown`, et non sur d’autres options de scrape ou de crawl.
  * Pour une comparaison efficace, l’URL d’entrée doit être exactement la même que lors de la requête précédente pour le même contenu. C’est une limite que nous travaillons activement à lever.
  * Crawler les mêmes URL avec des `includePaths`/`excludePaths` différents entraînera des incohérences avec `changeTracking`.
  * Scraper les mêmes URL avec des `includeTags`/`excludeTags`/`onlyMainContent` différents entraînera des incohérences avec `changeTracking`.
  * Les pages seront également comparées à des scrapes précédents qui n’avaient que le format `markdown`, sans le format `changeTracking`.
  * Les comparaisons sont limitées à votre équipe. Si vous scrapez une URL pour la première fois avec votre clé API, son `changeStatus` sera toujours `new`, même si d’autres utilisateurs de Firecrawl l’ont déjà scrappée.
* Pendant la bêta, il est recommandé de surveiller le champ `warning` du document résultant et de gérer le cas où l’objet `changeTracking` pourrait manquer dans la réponse. Nous surveillons le système pour que cela se produise le moins possible.
  * Cela peut survenir si la recherche en base visant à trouver le précédent scrape à comparer expire. Nous suivons cette situation et déploierons des améliorations et optimisations avant le lancement officiel de la fonctionnalité.
* Nous ne renvoyons actuellement pas de diff du contenu effectivement modifié de la page.
  * Nous ne savons pas encore quel serait le format optimal. Si vous souhaitez obtenir le diff, vous savez probablement dans quel format vous le voulez. Contactez-nous à [mogery@firecrawl.com](mailto:mogery@firecrawl.com), et nous l’ajouterons !

<div id="examples">
  ## Exemples
</div>

<div id="scrape">
  ### Scraper
</div>

```json
{
    "url": "https://firecrawl.dev",
    "formats": ["markdown", "suiviDesModifications"]
}
```

```json
{
  "success": true,
  "data": {
    "markdown": "...",
    "metadata": {...},
    "changeTracking": {
      "previousScrapeAt": "2025-03-30T15:07:17.543071+00:00",
      "changeStatus": "same",
      "visibility": "visible"
    }
  }
}
```

<div id="crawl">
  ### Crawl
</div>

```json
{
    "url": "https://firecrawl.dev",
    "scrapeOptions": {
        "formats": ["markdown", "suiviDesModifications"]
    }
}
```

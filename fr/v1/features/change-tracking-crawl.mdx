Le suivi des modifications devient encore plus puissant lorsqu’il est combiné à l’exploration. Alors que le suivi sur des pages individuelles met en évidence les changements de contenu, l’utiliser avec l’exploration vous permet de surveiller toute la structure de votre site : nouvelles pages, pages supprimées et pages désormais masquées.

<div id="basic-usage">
  ## Utilisation de base
</div>

Pour activer le suivi des modifications lors d’un crawl, ajoutez-le au tableau `formats` de vos `scrapeOptions` :

```typescript
// JavaScript/TypeScript
const app = new FirecrawlApp({ apiKey: 'votre-clé-API' });
const result = await app.crawl('https://example.com', {
  scrapeOptions: {
    formats: ['markdown', 'suiviDesModifications']
  }
});
```

```python
# Python
app = FirecrawlApp(api_key='votre-clé-api')
result = app.crawl('https://firecrawl.dev', {
    'scrapeOptions': {
        'formats': ['markdown', 'suiviDesModifications']
    }
})
```

<div id="understanding-change-status">
  ## Comprendre le statut des modifications
</div>

Lors de l’utilisation du suivi des modifications avec le crawl, le champ `changeStatus` est particulièrement utile :

* `new` : Une page qui n’existait pas lors de votre précédent crawl
* `same` : Une page existante qui n’a pas changé depuis votre dernier crawl
* `changed` : Une page existante qui a été modifiée depuis votre dernier crawl
* `removed` : Une page qui existait lors de votre précédent crawl mais qui n’est plus trouvée

<div id="page-visibility">
  ## Visibilité des pages
</div>

Le champ `visibility` vous aide à comprendre comment les pages sont découvertes :

* `visible` : la page est accessible via des liens ou le plan de site (sitemap)
* `hidden` : la page existe toujours mais n’est plus liée ni présente dans le plan de site (sitemap)

C’est particulièrement utile pour :

* Détecter le contenu orphelin
* Trouver des pages retirées accidentellement de la navigation
* Surveiller les changements de structure du site
* Identifier le contenu à relier de nouveau ou à supprimer

<div id="full-diff-support">
  ## Prise en charge complète des diffs
</div>

Pour un suivi détaillé des modifications avec diffs, vous pouvez utiliser les mêmes options que celles décrites dans la documentation [Suivi des modifications pour Scrape](/fr/features/change-tracking).

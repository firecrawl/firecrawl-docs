<div id="introducing-map">
  ## Présentation de /map
</div>

La façon la plus simple de passer d’une seule URL à une cartographie de l’ensemble du site. C’est extrêmement utile pour :

- Lorsque vous devez demander à l’utilisateur final de choisir quels liens scraper
- Quand vous avez besoin d’identifier rapidement les liens d’un site
- Quand vous devez scraper des pages d’un site liées à un sujet précis (utilisez le paramètre `search`)
- Quand vous ne devez scraper que des pages spécifiques d’un site

<div id="map-with-search">
  #### Cartographier avec recherche
</div>

La cartographie avec le paramètre `search` vous permet de rechercher des URL spécifiques au sein d’un site web.

```bash cURL
curl -X POST https://api.firecrawl.dev/v1/map \
    -H 'Content-Type: application/json' \
    -H 'Authorization: Bearer VOTRE_CLÉ_API' \
    -d '{
      "url": "https://firecrawl.dev",
      "search": "docs"
    }'
```

La réponse sera une liste ordonnée, de la plus pertinente à la moins pertinente.

```json
{
  "status": "réussi",
  "links": [
    "https://docs.firecrawl.dev",
    "https://docs.firecrawl.dev/sdks/python",
    "https://docs.firecrawl.dev/learn/rag-llama3",
  ]
}
```

<div id="considerations">
  ## Considérations
</div>

Ce point de terminaison privilégie la vitesse, il se peut donc qu’il ne capture pas tous les liens du site. Nous travaillons à des améliorations. Vos retours et suggestions sont les bienvenus.

Si vous développez avec l’IA, Firecrawl propose plusieurs ressources pour améliorer votre expérience. Firecrawl inclut des **skills** — des blocs de connaissances autonomes que les agents IA de développement découvrent et utilisent automatiquement. Une seule commande d’installation fournit aux agents des outils CLI pour interagir avec le web en temps réel, ainsi que des skills pour intégrer Firecrawl au code de votre application. Des agents comme Claude Code, Cursor, Antigravity et OpenCode peuvent s’auto-intégrer avec une seule commande — aucune configuration manuelle n’est nécessaire une fois la clé API créée.

<div id="prerequisite-create-an-api-key">
  ## Prérequis : créer une clé API
</div>

Pour l’instant, la création d’un compte Firecrawl doit être effectuée par une personne. Une fois votre compte créé, vous devrez [créer une clé API](https://www.firecrawl.dev/app/api-keys). Avec une clé API, votre agent peut s’occuper du reste — installer les skills, authentifier le CLI, configurer MCP et effectuer des appels en votre nom.

<Card title="Obtenir votre clé API" icon="key" href="https://www.firecrawl.dev/app/api-keys">
  Inscrivez-vous et récupérez une clé API pour commencer à utiliser Firecrawl.
</Card>

<div id="skills-cli">
  ## Skills + CLI
</div>

Le [Firecrawl CLI](/fr/sdks/cli) permet à votre agent de rechercher, scraper, interagir, lancer un crawl, faire de la cartographie, extraire des données et exécuter des jobs d’agent depuis le terminal. Il est conçu pour les humains, les agents IA et les pipelines CI/CD.

Les **skills** Firecrawl sont des ensembles de connaissances autonomes que les agents IA de développement comme Claude Code, Antigravity et OpenCode découvrent et utilisent automatiquement. Une seule commande d’installation configure tout — les outils CLI pour travailler sur le web en direct et les skills Build pour intégrer Firecrawl dans le code de l’application :

```bash
npx -y firecrawl-cli@latest init --all --browser
```

* `--all` installe les skills Firecrawl sur tous les agents IA de code détectés sur la machine
* `--browser` ouvre automatiquement le navigateur pour l’authentification Firecrawl

Après l’installation, vérifiez que tout fonctionne :

```bash
firecrawl --status
firecrawl scrape "https://firecrawl.dev"
```

<div id="what-the-install-gives-you">
  ### Ce que l’installation inclut
</div>

L’installation met en place deux catégories de skills qui couvrent toutes les façons dont un agent utilise Firecrawl :

**Skills CLI** — pour les opérations web en direct pendant une session d’agent :

| Skill                | Objectif                                                         |
| -------------------- | ---------------------------------------------------------------- |
| `firecrawl/cli`      | Workflow global des commandes CLI                                |
| `firecrawl-search`   | Rechercher sur le web et découvrir des pages                     |
| `firecrawl-scrape`   | Extraire un contenu propre depuis une URL connue                 |
| `firecrawl-interact` | Interagir avec des pages scrapées à l’aide de prompts ou de code |
| `firecrawl-crawl`    | Extraire en masse le contenu d’un site entier                    |
| `firecrawl-map`      | Découvrir toutes les URL d’un domaine                            |
| `firecrawl-agent`    | Exécuter une collecte autonome de données web avec un job        |

**Skills Build** — pour intégrer Firecrawl au code de l’application :

| Skill                        | Objectif                                                             |
| ---------------------------- | -------------------------------------------------------------------- |
| `firecrawl-build`            | Choisir le bon point de terminaison Firecrawl pour votre produit     |
| `firecrawl-build-onboarding` | Authentification et configuration du projet                          |
| `firecrawl-build-scrape`     | Implémenter le scraping dans le code de l’application                |
| `firecrawl-build-search`     | Implémenter la recherche dans le code de l’application               |
| `firecrawl-build-interact`   | Implémenter l’interaction avec la page dans le code de l’application |
| `firecrawl-build-crawl`      | Implémenter le crawling dans le code de l’application                |
| `firecrawl-build-map`        | Implémenter la découverte d’URL dans le code de l’application        |

<div id="choose-your-path">
  ### Choisissez votre voie
</div>

Les deux catégories de skills utilisent la même installation. La différence se joue ensuite :

<Steps>
  <Step title="Outils web en temps réel (skills CLI)">
    Utilisez ceci lorsque vous avez besoin de données web pendant votre session en cours — recherche sur le web, scraping d&#39;URL connues, interaction avec des pages scrapées, crawl de documentation, cartographie d&#39;un site ou exécution d&#39;un job d&#39;agent.

    Le flux par défaut :

    1. Commencez par **recherche** lorsque vous avez besoin de découvrir
    2. Passez à **scrape** lorsque vous avez une URL
    3. Utilisez **interact** lorsque la page scrapée nécessite des actions supplémentaires
    4. Utilisez **cartographie** ou **crawl** lorsque vous avez besoin de nombreuses URL ou pages
    5. Utilisez **agent** lorsque la tâche est ouverte et nécessite une découverte autonome

    ```bash
    # Rechercher sur le web
    firecrawl search "best open-source web crawlers"

    # Scraper une page en markdown propre
    firecrawl scrape https://docs.firecrawl.dev

    # Crawler un site entier
    firecrawl crawl https://docs.firecrawl.dev
    ```
  </Step>

  <Step title="Intégration applicative (skills Build)">
    Utilisez ceci lorsque vous créez une application, un agent ou un workflow qui appelle l&#39;API Firecrawl depuis du code. Les skills Build aident à choisir le bon point de terminaison, à intégrer le SDK et à exécuter un test rapide.

    L&#39;agent répond à une question clé — *que doit faire Firecrawl dans le produit ?* — et les skills Build orientent ensuite vers `/search`, `/scrape`, `/interact`, `/crawl`, `/map` ou `/agent` selon le cas.
  </Step>

  <Step title="API REST (aucune installation nécessaire)">
    Si vous préférez ne rien installer, les agents peuvent appeler directement l&#39;API REST de Firecrawl. Définissez la clé API et appelez les points de terminaison :

    * `POST https://api.firecrawl.dev/v2/search` — découvrir des pages à partir d&#39;une requête
    * `POST https://api.firecrawl.dev/v2/scrape` — extraire un markdown propre depuis une URL
    * `POST https://api.firecrawl.dev/v2/interact` — interagir avec une page scrapée
    * `POST https://api.firecrawl.dev/v2/crawl` — extraire en masse un site entier
    * `POST https://api.firecrawl.dev/v2/map` — découvrir des URL sur un domaine
    * `POST https://api.firecrawl.dev/v2/agent` — exécuter une collecte autonome de données web

    En-tête d&#39;authentification : `Authorization: Bearer VOTRE_CLÉ_API`
  </Step>
</Steps>

La définition complète de l&#39;onboarding est disponible sur [`firecrawl.dev/agent-onboarding/SKILL.md`](https://www.firecrawl.dev/agent-onboarding/SKILL.md) — les agents peuvent la récupérer directement pour s&#39;initialiser automatiquement.

<div id="using-firecrawl-as-a-tool">
  ## Utiliser Firecrawl comme outil
</div>

Firecrawl met à la disposition des agents cinq outils principaux pour travailler avec le web. Chaque outil correspond à un point de terminaison d’API et à une commande CLI. Les agents choisissent l’outil adapté à leurs besoins :

<AccordionGroup>
  <Accordion title="recherche — découvrir des pages à partir d’une requête" icon="magnifying-glass">
    Commencez ici si vous n’avez pas encore d’URL. recherche renvoie des pages web pertinentes pour une requête en langage naturel, avec en option le contenu complet des pages dans les résultats.

    ```bash
    # CLI
    firecrawl search "latest OpenAI API pricing"
    ```

    ```bash
    # API REST
    curl -X POST https://api.firecrawl.dev/v2/search \
      -H "Authorization: Bearer fc-YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"query": "latest OpenAI API pricing"}'
    ```

    **Quand l’utiliser :** recherche, localisation de documentation, analyse concurrentielle, réponse à des questions nécessitant des informations web à jour.
  </Accordion>

  <Accordion title="Scrape — extraire le contenu d’une URL" icon="file-lines">
    Utilisez cet outil lorsque vous avez déjà une URL et que vous avez besoin d’un contenu propre, prêt pour les LLM. Scrape convertit n’importe quelle page web en markdown, HTML ou données structurées, en gérant automatiquement le rendu JavaScript, les mesures anti-bot et le HTML désordonné.

    ```bash
    # CLI
    firecrawl scrape https://docs.stripe.com/api/charges
    ```

    ```bash
    # API REST
    curl -X POST https://api.firecrawl.dev/v2/scrape \
      -H "Authorization: Bearer fc-YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"url": "https://docs.stripe.com/api/charges"}'
    ```

    **Quand l’utiliser :** lecture de documentation, extraction du contenu d’articles, récupération de données depuis une page connue, conversion de pages web en contexte pour les LLM.
  </Accordion>

  <Accordion title="Crawl — extraire en masse un site entier" icon="spider">
    Crawl suit récursivement les liens à partir d’une URL de départ et scrape chaque page trouvée. Utilisez-le lorsque vous avez besoin du contenu d’un site entier ou d’un ensemble de documentation, et non d’une seule page.

    ```bash
    # CLI
    firecrawl crawl https://docs.firecrawl.dev --limit 50
    ```

    ```bash
    # API REST
    curl -X POST https://api.firecrawl.dev/v2/crawl \
      -H "Authorization: Bearer fc-YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"url": "https://docs.firecrawl.dev", "limit": 50}'
    ```

    **Quand l’utiliser :** ingestion de sites de documentation complets, création de bases de connaissances, migration de contenu, collecte de données d’entraînement.
  </Accordion>

  <Accordion title="cartographie — découvrir toutes les URL d’un domaine" icon="sitemap">
    cartographie découvre rapidement toutes les URL indexées d’un domaine sans en scraper le contenu. Utilisez-le lorsque vous avez besoin de comprendre la structure d’un site ou de trouver des pages spécifiques avant de les scraper.

    ```bash
    # CLI
    firecrawl map https://docs.firecrawl.dev
    ```

    ```bash
    # API REST
    curl -X POST https://api.firecrawl.dev/v2/map \
      -H "Authorization: Bearer fc-YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"url": "https://docs.firecrawl.dev"}'
    ```

    **Quand l’utiliser :** audits de site, recherche de pages spécifiques sur un site volumineux, compréhension de la structure d’un site avant un crawl ciblé.
  </Accordion>

  <Accordion title="Interact — travailler avec une page scrapée" icon="hand-pointer">
    Interact permet aux agents de poursuivre à partir d’un scrape à l’aide de prompts ou de code. Utilisez-le lorsqu’une page scrapée nécessite des clics, le remplissage de formulaires, de la navigation ou une extraction complémentaire.

    ```bash
    # CLI
    firecrawl scrape https://example.com
    firecrawl interact "Click the pricing tab and extract the plan names"
    ```

    ```bash
    # API REST
    curl -X POST https://api.firecrawl.dev/v2/interact \
      -H "Authorization: Bearer fc-YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"scrapeId": "scrape-id-from-scrape", "prompt": "Click the pricing tab and extract the plan names"}'
    ```

    **Quand l’utiliser :** poursuite après un scrape, navigation dans des pages dynamiques, remplissage de formulaires et extraction de données après des actions sur la page.
  </Accordion>
</AccordionGroup>

<div id="how-agents-chain-tools-together">
  ### Comment les agents enchaînent les outils
</div>

La plupart des workflows d’agents combinent plusieurs outils. Schéma typique :

1. **Recherche** de pages pertinentes → obtenir une liste d’URL
2. **Scrape** des URL les plus pertinentes → obtenir un contenu propre
3. **Interact** lorsque la page scrapée nécessite des actions de suivi
4. **Agent** lorsque la tâche nécessite une découverte autonome ou une extraction structurée sur plusieurs pages

Pour les traitements en masse, les agents utilisent d’abord **Cartographie** pour découvrir des URL, puis **Crawl** ou **Scrape** de manière sélective sur les pages dont ils ont besoin.

<div id="agentic-debugging">
  ## Débogage agentique
</div>

Lorsqu&#39;un appel à Firecrawl échoue ou renvoie des résultats inattendus, votre agent n&#39;a pas besoin de faire appel à un humain. Le point de terminaison [`/support/ask`](/fr/api-reference/endpoint/ask) est un agent de support IA conçu pour la communication **d’agent à agent** — il diagnostique les problèmes liés à vos jobs, à votre compte et à votre utilisation de l’API, puis renvoie une réponse vérifiée avec des paramètres de correction lisibles par machine que votre agent peut appliquer directement.

Intégrez-le au flux de gestion des erreurs de votre agent afin qu’il puisse se rétablir automatiquement après des échecs de scraping, des problèmes de crawl et des erreurs de configuration — généralement en 15 à 30 secondes, sans intervention humaine.

<div id="how-it-works">
  ### Comment cela fonctionne
</div>

1. **Votre agent décrit le problème** — une question en langage naturel qui décrit le problème.
2. **L&#39;agent de support enquête** — il examine les logs de tâche, l&#39;état du compte, la documentation et le code source.
3. **L&#39;agent de support valide** — lorsque c&#39;est possible, il teste un correctif sur l&#39;API Firecrawl en conditions réelles (par exemple, en relançant un scrape avec des paramètres ajustés).
4. **Votre agent obtient une réponse vérifiée** — une `answer` rédigée en langage naturel, des `fixParameters` exploitables par machine à appliquer directement, et des résultats de `validation` indiquant si le correctif a été testé.

<div id="example">
  ### Exemple
</div>

Envoyez une question, ainsi qu’un `rationale` facultatif pour donner à l’agent de support plus de contexte sur l’objectif de votre utilisateur final :

```bash
curl -X POST https://api.firecrawl.dev/v2/support/ask \
  -H "Authorization: Bearer fc-YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "my crawl returned 3 pages but I expected 50",
    "rationale": "user is on their third failed crawl attempt today"
  }'
```

La réponse inclut une `answer`, un niveau de `confidence`, des `fixParameters` facultatifs (par ex. `{"waitFor": 5000}`) que votre agent peut transmettre lors de l’appel suivant, ainsi qu’une `validation` indiquant si le correctif a été testé sur l’API en production.

<div id="firecrawl-mcp-server">
  ## Serveur MCP Firecrawl
</div>

MCP est un protocole ouvert qui standardise la façon dont les applications fournissent du contexte aux LLM. Parmi ses autres avantages, il donne aux LLM des outils pour agir en votre nom. Notre [serveur MCP](https://github.com/firecrawl/firecrawl-mcp-server) est open source et couvre l’ensemble de notre API — recherche, scrape, interact, crawl, cartographie, extraction et agent.

Utilisez l’URL hébergée à distance :

```
https://mcp.firecrawl.dev/{FIRECRAWL_API_KEY}/v2/mcp
```

Ou ajoutez le serveur local dans n’importe quel client MCP :

```json
{
  "mcpServers": {
    "firecrawl": {
      "command": "npx",
      "args": ["-y", "firecrawl-mcp"],
      "env": {
        "FIRECRAWL_API_KEY": "fc-YOUR-API-KEY"
      }
    }
  }
}
```

<div id="firecrawl-docs-for-agents">
  ## Docs Firecrawl pour les agents
</div>

Vous pouvez fournir à votre agent la documentation Firecrawl à jour de façon contextualisée. Les agents peuvent se configurer eux-mêmes en récupérant directement ces ressources — sans intervention humaine.

<Steps>
  <Step title="Docs Markdown">
    Chaque page dispose d&#39;une version markdown. Ajoutez `.md` à n&#39;importe quelle URL de la documentation, ou utilisez le menu d&#39;action de la page pour la copier au format markdown.

    ```
    Docs for this page: https://docs.firecrawl.dev/ai-onboarding.md
    ```
  </Step>

  <Step title="llms.txt complet">
    Fournissez à votre agent l&#39;ensemble de notre documentation dans un seul fichier.

    ```
    Here are the Firecrawl docs: https://docs.firecrawl.dev/llms-full.txt
    ```

    Une version d&#39;index plus courte est également disponible à `https://docs.firecrawl.dev/llms.txt`.
  </Step>

  <Step title="Serveur de docs MCP">
    Pour une approche structurée avec les outils MCP, connectez le serveur MCP de Firecrawl dans n&#39;importe quel client MCP (Cursor, Claude Code, Claude Desktop, Windsurf). Consultez la page [MCP Server](/fr/mcp-server) pour les commandes d&#39;installation.
  </Step>

  <Step title="Copier vers ChatGPT / Claude">
    Chaque page inclut un menu d&#39;action contextuel (copier, consulter en markdown, ouvrir dans ChatGPT, ouvrir dans Claude), afin que les agents comme les humains puissent déplacer des pages entre les outils en un clic.
  </Step>
</Steps>

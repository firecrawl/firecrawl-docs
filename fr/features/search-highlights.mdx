Les extraits de recherche remplacent, pour chaque résultat, la simple description du site web par des passages de la page pertinents pour votre query. Ils sont activés par défaut sur `/v2/search` ; aucun paramètre supplémentaire, format de sortie ou `scrapeOptions` n&#39;est requis.

Les extraits conservent l&#39;URL, le titre, la position et le classement du résultat de recherche. Pour les résultats web, le texte mis en évidence est renvoyé dans `description` ; pour les résultats d&#39;actualités, il est renvoyé dans `snippet`.

<Note>
  Si Firecrawl ne peut pas générer d&#39;extrait pour un résultat, la description simple du site web ou le snippet est conservé. Une page indisponible n&#39;empêchera pas le renvoi des autres résultats de recherche.
</Note>

<div id="highlights-are-enabled-by-default">
  ## Les extraits sont activés par défaut
</div>

Utilisez recherche normalement et Firecrawl renverra des extraits dès que du contenu de page pertinent est disponible.

<CodeGroup>
  ```python Python
  from firecrawl import Firecrawl

  firecrawl = Firecrawl(api_key="fc-YOUR-API-KEY")

  results = firecrawl.search(
      query="how does firecrawl handle javascript rendering",
      limit=5,
  )

  for result in results.web or []:
      print(result.description)
  ```

  ```js Node
  import { Firecrawl } from 'firecrawl';

  const firecrawl = new Firecrawl({ apiKey: 'fc-YOUR-API-KEY' });

  const results = await firecrawl.search(
    'how does firecrawl handle javascript rendering',
    {
      limit: 5,
    }
  );

  for (const result of results.web ?? []) {
    console.log(result.description);
  }
  ```

  ```bash cURL
  curl -X POST https://api.firecrawl.dev/v2/search \
    -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "query": "how does firecrawl handle javascript rendering",
      "limit": 5
    }'
  ```

  ```bash CLI
  firecrawl search "how does firecrawl handle javascript rendering" \
    --limit 5
  ```
</CodeGroup>

<div id="mcp">
  ### MCP
</div>

L’outil MCP `firecrawl_search` renvoie également des extraits par défaut :

```json MCP

  "query": "how does firecrawl handle javascript rendering",
```

<div id="response">
  ## Réponse
</div>

Les extraits utilisent les champs de description existants ; la structure de la réponse reste donc la même que celle d’une réponse de recherche normale.

````json
{
  "success": true,
  "data": {
    "web": [
      {
        "url": "https://www.firecrawl.dev/blog/javascript-web-scraping",
        "title": "Web Scraping With JavaScript: Step-by-Step Guide",
        "description": "# Web Scraping With JavaScript: Step-by-Step Guide\n## When should you use scraping APIs instead of DIY tools?\n### Setting up Firecrawl\n```\nnpm install firecrawl\n```\n\n```\nFIRECRAWL_API_KEY=fc-your-api-key-here\n```\n\n### Solving the JavaScript quotes problem\nYou describe what you want, and Firecrawl handles extraction and validation.",
        "position": 1
      }
    ]
  }
}
````

Les extraits peuvent contenir du Markdown lorsque le contenu pertinent de la page comporte des titres, des listes, des tableaux ou du code. Affichez ou traitez ce champ au format Markdown si vous souhaitez préserver cette structure.

<div id="disable-highlights">
  ## Désactiver extraits
</div>

Définissez `highlights` sur `false` si vous préférez obtenir, pour chaque site web, une description simple ou un snippet.

<CodeGroup>
  ```python Python
  results = firecrawl.search(
      query="Firecrawl search API",
      highlights=False,
  )
  ```

  ```js Node
  const results = await firecrawl.search('Firecrawl search API', {
    highlights: false,
  });
  ```

  ```bash cURL
  curl -X POST https://api.firecrawl.dev/v2/search \
    -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "query": "Firecrawl search API",
      "highlights": false
    }'
  ```

  ```bash CLI
  firecrawl search "Firecrawl search API" --no-highlights
  ```
</CodeGroup>

<div id="behavior-by-result-type">
  ## Comportement par type de résultat
</div>

* **Web :** Remplace `description` par un contenu de page pertinent pour la requête, lorsqu’il est disponible.
* **News :** Remplace `snippet` par un contenu de page pertinent pour la requête, lorsqu’il est disponible.
* **Images :** Les résultats d’image sont renvoyés tels quels.
* **recherche with scraping :** Les extraits affectent la description ou le snippet de recherche. Le contenu demandé via `scrapeOptions` est renvoyé séparément et n’est pas remplacé.
* **Zero Data Retention :** Les recherches ZDR conservent les descriptions et snippets bruts des sites web.

Pour chaque paramètre de recherche et le schéma de réponse complet, consultez la [référence de l’API recherche](/fr/api-reference/endpoint/search).

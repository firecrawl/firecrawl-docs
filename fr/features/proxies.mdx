Firecrawl propose différents types de proxy pour vous aider à scraper des sites web soumis à des niveaux variés de protection anti-bot. Le type de proxy peut être spécifié à l&#39;aide du paramètre `proxy`.

> Par défaut, Firecrawl achemine toutes les requêtes via des proxy afin d’assurer la fiabilité et l’accessibilité, même si vous ne spécifiez pas de type de proxy ni de localisation.

<div id="location-based-proxy-selection">

</div>

Si vous ne spécifiez pas de proxy ni d’emplacement, Firecrawl sélectionnera automatiquement la meilleure option en fonction du site cible et de votre requête.

<div id="how-to-specify-proxy-location">
  ## Comment spécifier l’emplacement du proxy
</div>

Vous pouvez demander un emplacement de proxy spécifique en définissant le paramètre `location.country` dans votre requête. Par exemple, pour utiliser un proxy au Brésil, définissez `location.country` sur `BR`.

Pour plus de détails, consultez la [référence de l’API pour `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location).

<div id="proxy-types">
  ## Types de proxy
</div>

Firecrawl prend en charge trois types de proxy :

* **basic** : Proxy pour le scraping de sites avec peu ou pas de solutions antibot. Rapide et généralement efficace.
* **stealth** : Proxys furtifs pour le scraping de sites dotés de solutions antibot avancées ou bloquant les proxys classiques. Plus lents, mais plus fiables sur certains sites. [En savoir plus sur le mode Stealth →](/fr/features/stealth-mode)
* **auto** : Firecrawl réessaie automatiquement avec des proxys furtifs si le proxy basic échoue. Si le réessai en mode stealth réussit, 5 crédits sont facturés pour le scrape. Si la première tentative avec basic réussit, seul le coût standard est facturé.

***

> **Remarque :** Pour des informations détaillées sur l’utilisation des proxys furtifs, y compris les coûts en crédits et les stratégies de réessai, consultez la [documentation du mode Stealth](/fr/features/stealth-mode).

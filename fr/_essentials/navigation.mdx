Le menu de navigation est la liste des liens présents sur chaque site Web.

Vous devrez probablement mettre à jour `mint.json` chaque fois que vous ajoutez une nouvelle page. Les pages n’apparaissent pas automatiquement.

<div id="navigation-syntax">
  ## Syntaxe de navigation
</div>

Notre syntaxe de navigation est récursive, ce qui signifie que vous pouvez créer des groupes de navigation imbriqués. Il n’est pas nécessaire d’inclure `.mdx` dans les noms de pages.

<CodeGroup>

```json Regular Navigation
"navigation": [
    {
        "group": "Getting Started",
        "pages": ["quickstart"]
    }
]
```

```json Nested Navigation
"navigation": [
    {
        "group": "Getting Started",
        "pages": [
            "quickstart",
            {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
            }
        ]
    }
]
```

</CodeGroup>

<div id="folders">
  ## Dossiers
</div>

Placez simplement vos fichiers MDX dans des dossiers et mettez à jour les chemins dans `mint.json`.

Par exemple, pour avoir une page à l’adresse `https://yoursite.com/your-folder/your-page`, vous créeriez un dossier nommé `your-folder` contenant un fichier MDX nommé `your-page.mdx`.

<Warning>
  Vous ne pouvez pas utiliser `api` comme nom de dossier, sauf si vous l’imbriquez dans un autre dossier. Mintlify utilise Next.js, qui réserve le dossier racine `api` pour les appels internes au serveur. Un nom de dossier tel que `api-reference` serait accepté.
</Warning>

```json Navigation avec dossier
"navigation": [
    {
        "group": "Nom du groupe",
        "pages": ["your-folder/your-page"]
    }
]
```

<div id="hidden-pages">
  ## Pages cachées
</div>

Les fichiers MDX non inclus dans `mint.json` n’apparaîtront pas dans la barre latérale, mais restent accessibles via la barre de recherche et par des liens directs.

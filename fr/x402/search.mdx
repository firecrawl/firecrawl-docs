<Info>
  **Point de terminaison de recherche à l&#39;acte**

  Ce point de terminaison vous permet d&#39;utiliser les puissantes capacités de recherche de Firecrawl sans abonnement, en payant chaque recherche en cryptomonnaie. Idéal pour des recherches ponctuelles ou pour tester avant de souscrire un abonnement.
</Info>

<Warning>
  **Fonctionnalité expérimentale**

  Il s&#39;agit d&#39;un point de terminaison expérimental. Les fonctionnalités, la tarification et la disponibilité peuvent changer sans préavis. Nous recommandons d&#39;utiliser le point de terminaison de recherche standard, basé sur un abonnement, pour les cas d&#39;usage en production.
</Warning>

<div id="authentication">
  ## Authentification
</div>

Ce point de terminaison requiert un en-tête de paiement spécifique au lieu de la clé API standard :

```
X-Payment : {{paymentHeader}}
```

<div id="how-it-works">
  ## Fonctionnement
</div>

Le point de terminaison On-Demand Search combine la recherche web (SERP) avec les capacités de scraping de Firecrawl pour renvoyer le contenu complet des pages pour toute requête, comme le point de terminaison `/search` classique, mais avec une tarification à l’usage.

Incluez `scrapeOptions` avec `formats: ["markdown"]` pour obtenir le contenu Markdown complet pour chaque résultat de recherche ; sinon, par défaut, vous ne recevrez que les résultats du SERP (url, title, description).

<div id="request-parameters">
  ### Paramètres de requête
</div>

Tous les paramètres du point de terminaison standard `/search` sont pris en charge, à l’exception suivante :

<Note>
  **Limitation importante** : le paramètre `limit` est limité à **10 résultats maximum** pour le point de terminaison wallet (contre 100 pour le point de terminaison standard).
</Note>

Pour la liste complète des paramètres disponibles, des opérateurs de requête pris en charge et de leurs descriptions, veuillez consulter la [documentation du point de terminaison Search standard](/fr/api-reference/endpoint/search).

<div id="example-request">
  ## Exemple de requête
</div>

```bash
curl -X POST https://api.firecrawl.dev/v1/x402/search \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <token>' \
  -H 'X-Payment: {{paymentHeader}}' \
  -d '{
    "query": "Calendrier des matches de Premier League pour la saison 2025-2026",
    "limit": 10,  # Maximum de 10 résultats autorisés
    "scrapeOptions": {
      "formats": ["markdown"],
      "onlyMainContent": true
    }
  }'
```

<div id="response">
  ## Réponse
</div>

Le format de la réponse est identique à celui du point de terminaison `/search` standard :

```json
{
  "success": true,
  "data": [
    {
      "title": "Titre de la page",
      "description": "Description de la page issue du résultat de recherche",
      "url": "https://example.com",
      "markdown": "# Contenu de la page\n\nContenu Markdown si l’extraction a été demandée...",
      "metadata": {
        "title": "Titre de la page",
        "description": "Méta-description"
        "sourceURL": "https://example.com",
        "statusCode": 200
      }
    }
  ]
}
```

<div id="pricing">
  ## Tarification
</div>

**Prix actuel : 0,01 $ par requête**

<Warning>
  **Avis de tarification** : Le prix actuel de 0,01 $ par requête peut être modifié à tout moment sans préavis. Veuillez vérifier la tarification la plus récente lors de la génération de votre en-tête de paiement.
</Warning>

Chaque requête de recherche est facturée à l’usage, en cryptomonnaie, via le système d’en-tête de paiement.

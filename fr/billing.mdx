<div id="overview">
  ## Vue d&#39;ensemble
</div>

Firecrawl utilise un **système de facturation basé sur des crédits**. Chaque appel à l&#39;API que vous effectuez consomme des crédits, et le nombre de crédits consommés dépend du point de terminaison et des options que vous utilisez. Vous disposez d&#39;un quota mensuel de crédits en fonction de votre offre, et vous pouvez acheter des crédits supplémentaires via des packs de recharge automatique si vous avez besoin de plus.

Pour consulter les tarifs actuels, rendez-vous sur la [page de tarification Firecrawl](https://www.firecrawl.dev/pricing).

<Note>
  Toutes les factures Firecrawl sont émises en **dollars américains (USD)**, quelle que soit votre adresse de facturation ou votre mode de paiement.
</Note>

<div id="credits">
  ## Crédits
</div>

Les crédits sont l’unité d’utilisation dans Firecrawl. Chaque offre inclut un quota mensuel de crédits qui est réinitialisé au début de chaque cycle de facturation. Différents endpoints d’API consomment un nombre de crédits différent.

<div id="credit-costs-per-endpoint">
  ### Coût en crédits par point de terminaison
</div>

| Point de terminaison | Coût en crédits                  | Notes                                                                                                                                                                                                                                                                                                                                                                                          |
| -------------------- | -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Scrape**           | 1 crédit / page                  | Convertit une URL unique en markdown propre, HTML ou données structurées. Des crédits supplémentaires s’appliquent lors de l’utilisation des options d’extraction (voir ci-dessous).                                                                                                                                                                                                           |
| **Crawl**            | 1 crédit / page                  | Scrape un site web entier en suivant les liens à partir d’une URL de départ. Les mêmes coûts des options d’extraction par page s’appliquent à chaque page explorée.                                                                                                                                                                                                                            |
| **Map**              | 1 crédit / appel                 | Découvre toutes les URL d’un site web sans scraper leur contenu.                                                                                                                                                                                                                                                                                                                               |
| **Search**           | 2 crédits / 10 résultats         | Recherche sur le web et, en option, scrape les résultats. Arrondi à la tranche supérieure par groupe de 10 résultats (par ex., 11 résultats = 4 crédits). Des coûts supplémentaires de scraping par page s’appliquent à chaque résultat qui est scrapé. Les coûts de recherche ZDR Enterprise sont de 10 crédits / 10 résultats (voir [ZDR Search](/fr/features/search#zero-data-retention-zdr)). |
| **Browser**          | 2 crédits / minute de navigateur | Session de bac à sable navigateur interactive, facturée à la minute.                                                                                                                                                                                                                                                                                                                           |
| **Agent**            | Dynamique                        | Agent autonome de recherche web. 5 exécutions quotidiennes gratuites ; tarification à l’usage au-delà de ce quota.                                                                                                                                                                                                                                                                             |

<div id="additional-credit-costs-for-scrape-options">
  ### Coûts de crédits supplémentaires pour les options d’extraction
</div>

Certaines options d’extraction ajoutent des crédits en plus du coût de base par page :

| Option | Coût supplémentaire | Description |
|--------|----------------|-------------|
| PDF parsing | +1 crédit / page PDF | Extraire le contenu de documents PDF |
| JSON format (LLM extraction) | +4 crédits / page | Utiliser un LLM pour extraire des données JSON structurées à partir de la page |
| Enhanced Mode | +4 crédits / page | Scrape amélioré pour les pages difficiles d’accès |
| Zero Data Retention (ZDR) | +1 crédit / page | Garantit qu’aucune donnée n’est conservée au-delà de la requête (voir [Scrape ZDR](/fr/features/scrape#zero-data-retention-zdr)) |

Ces modificateurs se cumulent. Par exemple, scraper une page avec à la fois le format JSON et le mode Enhanced coûte **1 + 4 + 4 = 9 crédits** par page. Ces mêmes modificateurs s’appliquent aux points de terminaison Crawl et Search, puisqu’ils utilisent `scrape` en interne pour chaque page.

<div id="when-credits-are-charged">
  ### Quand les crédits sont débités
</div>

Les crédits sont débités chaque fois que l’infrastructure de Firecrawl traite une requête, même si le site cible renvoie un code d’erreur HTTP tel que 403 Forbidden ou 404 Not Found. En effet, l’infrastructure de scraping (rendu du navigateur, proxy, etc.) est entièrement mobilisée, quelle que soit la réponse du site cible. Vous pouvez vérifier le champ `metadata.statusCode` dans la réponse de l’API pour détecter ces cas et éviter de relancer des URL systématiquement bloquées.

Pour les tâches de **batch scrape** et de **crawl**, les crédits sont facturés de manière asynchrone à mesure que chaque page est traitée — et non au moment où la tâche est soumise. Cela signifie qu’il peut y avoir un délai entre l’envoi d’une tâche et le moment où le coût total en crédits apparaît sur votre compte. Si un batch contient de nombreuses URL ou si des pages sont mises en file d’attente pendant des périodes de fort trafic, des crédits peuvent continuer à être débités et à apparaître minutes ou heures après la soumission. Le polling ou la consultation du statut d’un batch ne consomment pas de crédits.

<Note>
  **Vérification préalable des crédits pour le crawl :** Avant qu’une tâche de crawl ne démarre, Firecrawl vérifie que votre solde de crédits restant peut couvrir l’intégralité du paramètre `limit` demandé. Si votre solde est inférieur à `limit`, la requête renvoie un 402, même si le crawl aurait découvert moins de pages. La valeur par défaut de `limit` est **10,000** ; l’omettre exige donc de disposer de 10,000 crédits à l’avance. Pour éviter cela, transmettez un `limit` explicite correspondant au nombre de pages que vous avez réellement l’intention de parcourir (par exemple, `limit: 100`).
</Note>

<div id="tracking-your-usage">
  ### Suivi de votre consommation de crédits
</div>

Vous pouvez suivre votre consommation de crédits de deux manières :

* **Tableau de bord** : consultez votre consommation actuelle et historique sur [firecrawl.dev/app](https://www.firecrawl.dev/app)
* **API** : utilisez les points de terminaison [Credit Usage](/fr/api-reference/endpoint/credit-usage) et [Credit Usage Historical](/fr/api-reference/endpoint/credit-usage-historical) pour vérifier votre consommation par programmation

<Note>
  Nous travaillons activement sur des améliorations pour rendre la consommation de crédits plus facile à comprendre. Restez informé des prochaines mises à jour.
</Note>

En plus des principaux niveaux d’offre, chaque offre payante propose des paliers de crédits intermédiaires afin que vous puissiez ajuster votre abonnement au plus près de votre utilisation réelle, sans devoir passer à l’offre supérieure :

| ------------ | ------------------------------ |
| **Hobby**    | 5,000 · 6,500 · 8,000          |
| **Standard** | 100,000 · 130,000 · 160,000    |
| **Growth**   | 500,000 · 650,000              |

Les navigateurs concurrents, les limites de débit et le niveau de support dépendent de l’offre, et non du palier de crédits choisi — choisir 6,500 crédits sur Hobby vous donne le même niveau de concurrence que 5,000. Utilisez le curseur pour régler le nombre de crédits, puis abonnez-vous au palier correspondant.

<div id="concurrent-browsers">
  ### Navigateurs simultanés
</div>

Les navigateurs simultanés correspondent au nombre de pages web que Firecrawl peut traiter en parallèle pour vous. Votre offre détermine cette limite. Si vous la dépassez, les tâches supplémentaires attendent dans une file d’attente jusqu’à ce qu’un créneau se libère. Consultez [Limites de débit](/fr/rate-limits) pour tous les détails sur la concurrence et les limites de débit de l’API.

<div id="handling-usage-spikes">
  ## Gérer les pics d’utilisation
</div>

Firecrawl propose deux mécanismes optionnels pour maintenir les requêtes en cours lorsque vous tombez à court de crédits en cours de cycle : **Recharge automatique** et **Smart Upgrade**. Tous deux se configurent depuis vos [paramètres de facturation](https://www.firecrawl.dev/app/settings?tab=billing) et sont **mutuellement exclusifs** — votre organisation utilise l’un ou l’autre, jamais les deux.

|                         | Recharge automatique                                             | Smart Upgrade                                                                  |
| ----------------------- | ---------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Ce que cela fait        | Achète un pack de crédits ponctuel et l’ajoute à votre solde     | Vous fait passer au palier de crédits supérieur dans la grille tarifaire       |
| Quand cela se déclenche | Le solde tombe jusqu’à votre seuil (par défaut : 0)              | Le solde atteint 0                                                             |
| Facturation             | Prix fixe du pack                                                | Différence au prorata entre votre palier actuel et le palier supérieur         |
| Crédits ajoutés         | Le pack reste valable 1 an et n’est pas réinitialisé chaque mois | Quota complet du nouveau palier, qui remplace l’ancien                         |
| Idéal pour              | Des pics ponctuels sur une offre stable                          | Une croissance régulière — montée en gamme à mesure que l’utilisation augmente |

<div id="auto-recharge">
  ### Recharge automatique
</div>

Lorsque vos crédits restants passent sous un seuil, Firecrawl achète automatiquement un pack de crédits supplémentaire et l&#39;ajoute à votre solde. Par défaut, ce seuil est de **0 crédit**, mais vous pouvez le personnaliser dans vos [paramètres de facturation](https://www.firecrawl.dev/app/settings?tab=billing) pour déclencher les recharges plus tôt — par exemple, définir un seuil plus élevé vous garantit de toujours disposer d&#39;une réserve de crédits, ce qui est utile si vous lancez de gros crawls. Notez que cette vérification a lieu lorsque vous effectuez une requête API — votre solde ne sera pas réapprovisionné tant qu&#39;une nouvelle requête ne déclenchera pas la recharge.

* Les packs de recharge automatique sont disponibles avec toutes les offres payantes
* La taille et le prix des packs varient selon l&#39;offre (consultables sur la [page de tarification](https://www.firecrawl.dev/pricing))
* Vous pouvez activer ou désactiver la recharge automatique à tout moment
* La recharge automatique est limitée à **4 packs par mois**
* Les crédits des packs de recharge automatique **ne sont pas réinitialisés chaque mois** — ils sont conservés d&#39;un cycle de facturation à l&#39;autre et expirent **1 an** après l&#39;achat, contrairement aux crédits mensuels de votre offre qui sont réinitialisés à chaque période.

La recharge automatique est idéale pour gérer des pics d&#39;utilisation ponctuels. Si vous dépassez régulièrement le quota inclus dans votre offre, passer à une offre supérieure — ou activer Smart Upgrade — vous offrira un meilleur tarif par crédit.

<div id="smart-upgrade">
  ### Smart Upgrade
</div>

Smart Upgrade est un mode de facturation optionnel qui vous fait automatiquement passer au palier de crédits supérieur dès que votre solde tombe à zéro, afin que votre utilisation ne soit pas interrompue. Au lieu d&#39;acheter un pack de crédits ponctuel, vous passez automatiquement à l&#39;offre d&#39;abonnement suivante dans le sélecteur (par ex. Hobby 5k → Hobby 6.5k → Hobby 8k → Standard 100k → … → Scale 1M).

Fonctionnement :

* **Déclenchement :** Smart Upgrade se déclenche lorsque votre solde de crédits restants atteint 0. S&#39;il vous reste encore des crédits — y compris des crédits d&#39;auto-recharge inutilisés — aucune mise à niveau n&#39;a lieu.
* **Facturation :** un montant au prorata vous est facturé, correspondant à la différence entre votre niveau actuel et le niveau supérieur. Sur les offres **mensuelles**, il s&#39;agit de la différence de prix mensuelle ; sur les offres **annuelles**, il s&#39;agit de `(mois restants dans le cycle ÷ 12) × différence de prix annuel`. Le prorata est calculé sur la base de mois calendaires complets à partir du début de votre période de facturation actuelle.
* **Crédits accordés :** vous recevez le quota total de crédits du nouveau niveau, moins le quota de votre niveau précédent (vous ne payez donc que l&#39;écart).
* **Notifications :** chaque fois que Smart Upgrade se déclenche, vous recevez un e-mail confirmant la mise à niveau, le montant facturé et un aperçu du prochain palier. Désactivez Smart Upgrade dans les [paramètres de facturation](https://www.firecrawl.dev/app/settings?tab=billing) à tout moment pour empêcher toute nouvelle mise à niveau.
* **Sommet de l&#39;échelle :** une fois le niveau Scale atteint (1 000 000 crédits/mois), Smart Upgrade n&#39;a plus de palier supérieur. Pour augmenter le nombre de crédits au-delà de Scale, [contactez l&#39;équipe commerciale](https://www.firecrawl.dev/enterprise) au sujet d&#39;une offre Enterprise personnalisée.

Smart Upgrade est activé lorsque vous vous abonnez ou changez d&#39;offre (vous verrez l&#39;option dans la confirmation de commande), puis peut être activé ou désactivé plus tard depuis vos paramètres de facturation. Il est disponible pour les organisations ne comptant qu&#39;une seule équipe sur des offres standard ; les organisations ayant des contrats Enterprise personnalisés continuent d&#39;utiliser leur modèle de facturation négocié.

Smart Upgrade est particulièrement adapté si votre utilisation augmente régulièrement et que vous préférez monter automatiquement dans l&#39;échelle des offres plutôt que d&#39;acheter des packs de crédits. Utilisez Auto-Recharge si votre offre de référence vous convient et que vous n&#39;avez besoin que de recharges ponctuelles occasionnelles.

<div id="coupons">
  ## Coupons
</div>

Firecrawl prend en charge deux types de coupons :

* **Les coupons d’abonnement** appliquent une réduction à votre abonnement (par ex. un pourcentage de réduction sur votre tarif mensuel ou annuel). Ils peuvent **uniquement** être appliqués pendant le processus de paiement Stripe, lorsque vous souscrivez pour la première fois à une offre payante ou changez d’offre. Vous ne pouvez pas appliquer un coupon d’abonnement une fois le paiement terminé.
* **Les coupons de crédit** ajoutent des crédits bonus à votre compte. Ils peuvent être utilisés depuis la section **Facturation** de votre tableau de bord à l’adresse [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing). Recherchez le champ de saisie du coupon sur la page de facturation pour appliquer votre code. Les crédits bonus issus des coupons de crédit sont distincts du quota mensuel de votre offre et sont conservés même si vous passez à une offre supérieure ou inférieure.

Si vous avez un code promo et ne savez pas de quel type il s’agit, essayez d’abord de l’appliquer dans la section de facturation de votre tableau de bord. S’il s’agit d’un coupon d’abonnement, vous devrez l’utiliser à la place sur la page de paiement Stripe.

<div id="billing-cycle">
  ## Cycle de facturation
</div>

* **Offres mensuelles** : les crédits sont réinitialisés à votre date de renouvellement mensuelle
* **Offres annuelles** : vous êtes facturé une fois par an, mais les crédits sont tout de même réinitialisés chaque mois à votre date de renouvellement mensuelle virtuelle
* **Par défaut, les crédits d’offre inutilisés ne sont pas reportés** — votre quota mensuel est réinitialisé chaque mois. **Les offres annuelles Scale reportent les crédits d’offre inutilisés pendant 1 mois**, et **les offres annuelles Enterprise les reportent pendant 2 mois**. Les crédits des packs de recharge automatique ne sont pas liés à votre cycle de facturation — ils restent disponibles et expirent **1 an** à compter de la date d’achat.

<div id="upgrading-and-downgrading">
  ## Mise à niveau et rétrogradation
</div>

* Les **mises à niveau** prennent effet immédiatement. Le prix complet de la nouvelle offre vous est facturé aujourd’hui (sans prorata), et votre cycle de facturation est réinitialisé — votre prochain renouvellement aura lieu un mois ou un an après la date de mise à niveau. Tous les crédits inutilisés de votre offre précédente sont reportés, et votre nouveau quota de crédits ainsi que vos limites de concurrence s’appliquent immédiatement.
* Les **rétrogradations** sont programmées pour prendre effet à votre prochaine date de renouvellement. Vous conservez les crédits et limites de votre offre actuelle jusqu’à cette date, et le temps inutilisé sur votre offre actuelle n’est ni crédité ni remboursé. Vous pouvez annuler une rétrogradation programmée depuis vos [paramètres de facturation](https://www.firecrawl.dev/app/settings?tab=billing) à tout moment avant la date d’effet.

<div id="switching-between-monthly-and-yearly-billing">
  ### Basculer entre la facturation mensuelle et annuelle
</div>

* **Mensuel → Annuel** au même niveau de crédits ou à un niveau supérieur est considéré comme une mise à niveau immédiate.
* **Annuel → Mensuel** est considéré comme une mise à niveau immédiate uniquement si vous passez à un niveau de crédits strictement supérieur.

Les packs de recharge automatique que vous avez déjà achetés ne sont pas affectés par les changements d&#39;offre — ils conservent leur date d&#39;expiration initiale d&#39;un an, quelles que soient les mises à niveau ou les passages à une offre inférieure.

<div id="what-happens-when-you-run-out-of-credits">
  ## Que se passe-t-il lorsque vous n’avez plus de crédits
</div>

Si vous épuisez votre quota de crédits et que ni recharge automatique ni Smart Upgrade ne sont activés, les requêtes API qui consomment des crédits renverront une erreur **HTTP 402 (Payment Required)**.

* Avec **recharge automatique** activé, l’utilisation continue pendant que des packs de recharge sont achetés automatiquement — mais si la limite mensuelle de packs (4) est atteinte ou si une recharge échoue, votre solde peut devenir négatif jusqu’au prochain cycle de facturation.
* Avec **Smart Upgrade** activé, votre abonnement passe automatiquement au palier de crédits suivant sur le curseur, et vous recevez immédiatement le quota de crédits du nouveau palier. Smart Upgrade s’arrête une fois le palier Scale atteint.

Pour reprendre l’utilisation après un arrêt forcé, vous pouvez :

1. Activer [recharge automatique](#auto-recharge) ou [Smart Upgrade](#smart-upgrade)
2. Passer à une offre supérieure
3. Attendre que vos crédits soient réinitialisés lors du prochain cycle de facturation

<div id="free-plan">
  ## Offre gratuite
</div>

L&#39;offre gratuite propose **1 000 crédits par mois** sans nécessiter de carte de crédit. Les crédits sont réinitialisés chaque mois à votre date de renouvellement mensuel : une fois qu&#39;ils sont épuisés, vous pouvez attendre la prochaine réinitialisation ou passer à une offre payante pour un quota plus élevé. L&#39;offre gratuite impose également des limites de débit et de concurrence plus basses que les offres payantes (voir [limites de débit](/fr/rate-limits)).

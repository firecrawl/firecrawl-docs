<div id="overview">
  ## Aperçu
</div>

Utilisez le SDK Python Firecrawl dans une vue Django pour servir de proxy aux requêtes de scraping.

<div id="prerequisites">
  ## Prérequis
</div>

* Une clé d’API Firecrawl
* Une application Django
* Python 3.8+

<div id="install">
  ## Installation
</div>

```bash
pip install firecrawl-py
```

<div id="configure-environment">
  ## Configurer l’environnement
</div>

Définissez la clé API dans votre environnement Django.

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="add-a-server-route">
  ## Ajouter une route côté serveur
</div>

app/views.py

```python
import json
import os
from django.http import JsonResponse
from django.views.decorators.csrf import csrf_exempt
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])

@csrf_exempt
def firecrawl_view(request):
    if request.method != "POST":
        return JsonResponse({"error": "Method not allowed"}, status=405)

    body = json.loads(request.body)
    doc = client.scrape(body["url"], formats=["markdown"])
    return JsonResponse({"markdown": doc.markdown})
```

<div id="call-it-from-your-app">
  ## Appelez-le depuis votre application
</div>

```python
import requests

response = requests.post(
    "http://localhost:8000/api/firecrawl",
    json={"url": "https://example.com"},
)

print(response.json().get("markdown"))
```

<div id="notes">
  ## Remarques
</div>

* Ajoutez une route URL qui associe `/api/firecrawl` à `firecrawl_view`.

<div id="overview">
  ## Vue d’ensemble
</div>

Utilisez le SDK Node de Firecrawl dans une route Express côté serveur.

<div id="prerequisites">
  ## Prérequis
</div>

* Une clé API Firecrawl
* Une application Express
* Node.js 18+

<div id="install">
  ## Installation
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="configure-environment">
  ## Configurer l’environnement
</div>

Stockez la clé API dans `.env` ou dans les variables d’environnement du processus.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## Ajouter une route côté serveur
</div>

server.js

```js
import express from 'express';
import Firecrawl from '@mendable/firecrawl-js';

const app = express();
app.use(express.json());

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

app.post('/api/firecrawl', async (req, res) => {
  const { url } = req.body;
  const doc = await client.scrape(url, { formats: ['markdown'] });
  res.json({ markdown: doc.markdown });
});

app.listen(3000, () => console.log('Server listening on :3000'));
```

<div id="call-it-from-your-app">
  ## Appelez-le depuis votre application
</div>

```js
const response = await fetch('http://localhost:3000/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

<div id="notes">
  ## Remarques
</div>

* Gardez les appels à Firecrawl côté serveur ; n’exposez jamais les clés API dans les bundles clients.

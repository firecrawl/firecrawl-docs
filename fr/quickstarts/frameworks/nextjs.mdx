<div id="overview">
  ## Vue d’ensemble
</div>

Conservez votre clé API Firecrawl sur le serveur en effectuant les appels depuis un Route Handler Next.js.

<div id="prerequisites">
  ## Prérequis
</div>

* Une clé API Firecrawl
* Une application Next.js (App Router)
* Node.js 18+

<div id="install">
  ## Installation
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="configure-environment">
  ## Configurer l’environnement
</div>

Ajoutez la clé API dans `.env.local` afin qu’elle ne soit accessible que sur le serveur.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## Ajouter une route côté serveur
</div>

app/api/firecrawl/route.ts

```ts
import Firecrawl from '@mendable/firecrawl-js';

export async function POST(request: Request) {
  const { url } = await request.json();
  const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

  const doc = await client.scrape(url, { formats: ['markdown'] });
  return Response.json({ markdown: doc.markdown });
}
```

<div id="call-it-from-your-app">
  ## Appelez-le depuis votre application
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

<div id="notes">
  ## Remarques
</div>

* Appelez Firecrawl uniquement depuis des routes côté serveur pour éviter d’exposer votre clé API.
* Si vous utilisez le Pages Router, créez plutôt `pages/api/firecrawl.ts`.

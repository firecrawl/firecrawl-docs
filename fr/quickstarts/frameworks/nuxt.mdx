<div id="overview">
  ## Vue d’ensemble
</div>

Utilisez une route API côté serveur de Nuxt afin que votre clé API Firecrawl reste privée.

<div id="prerequisites">
  ## Prérequis
</div>

* Une clé API Firecrawl
* Une application Nuxt 3
* Node.js 18+

<div id="install">
  ## Installation
</div>

Aucun package supplémentaire n’est nécessaire ; les routes serveur de Nuxt peuvent appeler fetch directement.

<div id="configure-environment">
  ## Configurer l’environnement
</div>

Ajoutez la clé API à `.env` et exposez-la via la configuration d’exécution.

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    firecrawlApiKey: process.env.FIRECRAWL_API_KEY,
  },
});
```

<div id="add-a-server-route">
  ## Ajouter une route serveur
</div>

server/api/firecrawl.post.ts

```ts
export default defineEventHandler(async (event) => {
  const { url } = await readBody(event);
  const config = useRuntimeConfig();

  return await $fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${config.firecrawlApiKey}`,
      'Content-Type': 'application/json',
    },
    body: { url, formats: ['markdown'] },
  });
});
```

<div id="call-it-from-your-app">
  ## Appelez-le depuis votre application
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

<div id="notes">
  ## Remarques
</div>

* Conservez la clé API Firecrawl dans `runtimeConfig` et évitez de l’exposer au client.

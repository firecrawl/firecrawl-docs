<div id="firecrawl-rust-source-of-truth">
  # Source de référence Rust de Firecrawl
</div>

Ce fichier est la référence publique faisant autorité pour les agents. Il est généré à partir du code source du SDK Rust de Firecrawl et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

```bash
cargo add firecrawl
```

<div id="authenticate">
  ## S’authentifier
</div>

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
// let client = Client::new_selfhosted("https://api.firecrawl.dev", Some("fc-your-api-key"))?;
```

<div id="when-to-use-what">
  ## Quand utiliser chaque option
</div>

* `search` : partir d’une requête pour trouver des URL pertinentes, y compris avec des contraintes `site:`.
* `scrape` : récupérer et extraire le contenu d’une URL connue.
* `interact` : exécuter des actions du navigateur ou du code lorsqu’une page nécessite des clics, des formulaires ou des étapes post-scraping.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez la recherche lorsque vous avez besoin d’obtenir des résultats à partir d’une requête et que vous souhaitez que Firecrawl renvoie des URL (et, éventuellement, extraie le contenu de chaque résultat). Vous pouvez limiter les résultats à un domaine avec l’opérateur standard `site:`, par exemple `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.search(query, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```rust
use firecrawl::v2::Client;

let results = client
    .search("site:docs.firecrawl.dev webhook retries", None)
    .await?;
```

<div id="complex-example">
  ### Exemple complexe
</div>

```rust
use firecrawl::v2::{
    Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format, JsonOptions
};

let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:m".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    timeout: Some(60000),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Links, Format::Json]),
        json_options: Some(JsonOptions {
            prompt: Some("Extract title and key endpoints.".to_string()),
            ..Default::default()
        }),
        only_main_content: Some(true),
        ..Default::default()
    }),
    ..Default::default()
};

let results = client
    .search("site:docs.firecrawl.dev crawl webhooks", options)
    .await?;
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre                     | Type / Options                                      | Description                                                                                                  |
| ----------------------------- | --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| `query`                       | `&str`                                              | Requête de recherche. Utilisez `site:domain` pour limiter les résultats à un site.                           |
| `options.sources`             | `Vec<SearchSource>` (`Web`, `News`, `Images`)       | Sources dans lesquelles effectuer la recherche.                                                              |
| `options.categories`          | `Vec<SearchCategory>` (`Github`, `Research`, `Pdf`) | Catégories utilisées pour filtrer les résultats.                                                             |
| `options.limit`               | `u32`                                               | Nombre maximal de résultats à renvoyer. Valeur par défaut : 5, maximum : 20 (selon la documentation du SDK). |
| `options.tbs`                 | `String`                                            | Filtre de recherche temporel (par exemple `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).                                  |
| `options.location`            | `String`                                            | Chaîne de localisation des résultats de recherche.                                                           |
| `options.ignore_invalid_urls` | `bool`                                              | Exclut les URL invalides que les autres points de terminaison Firecrawl ne peuvent pas scraper.              |
| `options.timeout`             | `u32`                                               | Délai d’expiration en millisecondes.                                                                         |
| `options.scrape_options`      | `ScrapeOptions`                                     | Options de scrape appliquées à chaque résultat (mêmes champs que `scrape`).                                  |
| `options.integration`         | `String`                                            | Interne uniquement. Réservé à l’instrumentation Firecrawl.                                                   |

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez scrape lorsque vous avez déjà une URL et que vous souhaitez que Firecrawl renvoie un contenu structuré et des formats optionnels.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```rust
use firecrawl::v2::{Client, ScrapeOptions, Format};

let doc = client
    .scrape("https://docs.firecrawl.dev", ScrapeOptions {
        formats: Some(vec![Format::Markdown]),
        ..Default::default()
    })
    .await?;
```

<div id="complex-example">
  ### Exemple complexe
</div>

```rust
use firecrawl::v2::{
    Client, ScrapeOptions, Format, JsonOptions, ScreenshotOptions, ChangeTrackingOptions,
    ChangeTrackingMode, AttributeSelector, Action, ScrollDirection, ProxyType, ParserConfig
};

let options = ScrapeOptions {
    formats: Some(vec![
        Format::Markdown,
        Format::Links,
        Format::Json,
        Format::Screenshot,
        Format::ChangeTracking,
        Format::Attributes,
    ]),
    json_options: Some(JsonOptions {
        prompt: Some("Extract plan names and prices.".to_string()),
        ..Default::default()
    }),
    screenshot_options: Some(ScreenshotOptions {
        full_page: Some(true),
        quality: Some(80),
        viewport: None,
    }),
    change_tracking_options: Some(ChangeTrackingOptions {
        modes: Some(vec![ChangeTrackingMode::GitDiff]),
        tag: Some("pricing".to_string()),
        ..Default::default()
    }),
    attribute_selectors: Some(vec![AttributeSelector {
        selector: "a".to_string(),
        attribute: "href".to_string(),
    }]),
    headers: None,
    include_tags: Some(vec!["main".to_string(), "article".to_string()]),
    exclude_tags: Some(vec!["nav".to_string(), "footer".to_string()]),
    only_main_content: Some(true),
    wait_for: Some(1000),
    parsers: Some(vec![ParserConfig::Pdf {
        parser_type: "pdf".to_string(),
        max_pages: Some(5),
    }]),
    actions: Some(vec![
        Action::Click { selector: "#accept".to_string() },
        Action::Write { text: "firecrawl".to_string() },
        Action::Press { key: "Enter".to_string() },
        Action::Wait { milliseconds: Some(1500), selector: None },
        Action::Scroll { direction: ScrollDirection::Down, selector: None },
        Action::Scrape,
    ]),
    skip_tls_verification: Some(true),
    remove_base64_images: Some(true),
    fast_mode: Some(true),
    block_ads: Some(true),
    proxy: Some(ProxyType::Auto),
    max_age: Some(86400000),
    min_age: None,
    store_in_cache: Some(true),
    ..Default::default()
};

let doc = client.scrape("https://example.com/pricing", options).await?;
```

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez interact lorsqu’une page nécessite des actions du navigateur ou l’exécution de code une fois le scrape lancé (clics, formulaires ou étapes scriptées).

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.interact(job_id, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```rust
use firecrawl::v2::{Client, ScrapeOptions, Format, ScrapeExecuteOptions};

let doc = client
    .scrape("https://example.com", ScrapeOptions {
        formats: Some(vec![Format::Markdown]),
        ..Default::default()
    })
    .await?;

let job_id = doc
    .metadata
    .as_ref()
    .and_then(|m| m.scrape_id.clone())
    .expect("Missing scrape_id");

let result = client
    .interact(job_id, ScrapeExecuteOptions {
        prompt: Some("Click the pricing tab and summarize the plans.".to_string()),
        language: None,
        ..Default::default()
    })
    .await?;
```

<div id="complex-example">
  ### Exemple complexe
</div>

```rust
use firecrawl::v2::{Client, ScrapeExecuteOptions, ScrapeExecuteLanguage};

let result = client
    .interact(
        "<scrape_job_id>",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.title());".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            ..Default::default()
        },
    )
    .await?;
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre          | Type / Options                                     | Description                                                                                        |
| ------------------ | -------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `job_id`           | `&str`                                             | ID de la tâche de scraping provenant de `document.metadata.scrape_id`.                             |
| `options.code`     | `String`                                           | Code à exécuter dans la session de navigateur. Obligatoire si `prompt` n’est pas fourni.           |
| `options.prompt`   | `String`                                           | Instruction en langage naturel pour l’agent du navigateur. Obligatoire si `code` n’est pas fourni. |
| `options.language` | `ScrapeExecuteLanguage` (`Python`, `Node`, `Bash`) | Langage d’exécution. Par défaut, `Node` si omis.                                                   |
| `options.timeout`  | `u32`                                              | Temps limite d’exécution, en secondes.                                                             |
| `options.origin`   | `String`                                           | Usage interne uniquement. Réservé à l’instrumentation de Firecrawl.                                |

<div id="notes">
  ## Remarques
</div>

* Alias dépréciés : `scrape_execute`, `stop_interactive_browser` et `delete_scrape_browser` renvoient à `interact` et `stop_interaction`.
* Les champs des structs Rust sont en snake&#95;case, mais sont sérialisés en champs d’API camelCase via serde.

<div id="firecrawl-python-source-of-truth">
  # Source de référence de Firecrawl Python
</div>

Ce fichier est la source publique de référence pour les agents. Il est généré à partir du code source du SDK Python de Firecrawl et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## Authentification
</div>

```py
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ.get("FIRECRAWL_API_KEY"))
# client = Firecrawl(api_key="fc-...", api_url="https://api.firecrawl.dev")
```

<div id="when-to-use-what">
  ## Quand utiliser chaque option
</div>

* `search` : partir d’une requête pour découvrir des URL pertinentes, y compris avec des contraintes `site:`.
* `scrape` : récupérer et extraire le contenu d’une URL connue.
* `interact` : exécuter des actions du navigateur ou du code lorsqu’une page nécessite des clics, des formulaires ou des étapes après le scraping.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l&#39;utiliser
</div>

Utilisez la recherche lorsque vous souhaitez explorer des résultats à partir d&#39;une requête et que vous voulez que Firecrawl renvoie des URLs (et scrape optionnellement chaque résultat). Vous pouvez restreindre les résultats à un domaine avec l&#39;opérateur standard `site:`, par exemple `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.search(query, **options)`

<div id="simple-example">
  ### Exemple simple
</div>

```py
results = client.search("site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Exemple complexe
</div>

```py
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:m",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    timeout=300000,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            "links",
            {"type": "json", "prompt": "Extract title and key endpoints."}
        ],
        only_main_content=True,
        include_tags=["main", "article"],
        exclude_tags=["nav", "footer"],
        wait_for=1000,
        actions=[
            {"type": "click", "selector": "button#accept"},
            {"type": "wait", "milliseconds": 750},
            {"type": "scrape"}
        ]
    )
)
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre             | Type / Options                                            | Description                                                                                      |
| --------------------- | --------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| `query`               | `str`                                                     | Requête de recherche. Utilisez `site:domain` pour restreindre les résultats à un site.           |
| `sources`             | `liste de noms de sources ou d'objets source typés`       | Sources à interroger.                                                                            |
| `categories`          | `liste de noms de catégories ou d'objets catégorie typés` | Catégories pour filtrer les résultats.                                                           |
| `limit`               | `int`                                                     | Nombre maximum de résultats à retourner. La valeur par défaut est `5` dans le modèle SDK.        |
| `tbs`                 | `str`                                                     | Filtre de recherche temporel (par exemple `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).                      |
| `location`            | `str`                                                     | Chaîne de localisation pour les résultats de recherche.                                          |
| `ignore_invalid_urls` | `bool`                                                    | Exclut les URLs invalides que les autres points de terminaison Firecrawl ne peuvent pas scraper. |
| `timeout`             | `int`                                                     | Timeout en millisecondes. La valeur par défaut est `300000` dans le modèle SDK.                  |
| `scrape_options`      | `ScrapeOptions`                                           | Options de scrape appliquées à chaque résultat (mêmes champs que `scrape`, plus `min_age`).      |
| `integration`         | `str`                                                     | Usage interne uniquement. Réservé à l&#39;instrumentation Firecrawl.                             |

<div id="scrape">
  ## Scrape
</div>

### Pourquoi l’utiliser

Utilisez scrape si vous avez déjà une URL et souhaitez que Firecrawl renvoie un contenu structuré ainsi que des formats en option.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.scrape(url, **options)`

<div id="simple-example">
  ### Exemple simple
</div>

```py
doc = client.scrape("https://docs.firecrawl.dev", formats=["markdown"])
```

<div id="complex-example">
  ### Exemple complexe
</div>

```py
doc = client.scrape(
    "https://example.com/pricing",
    formats=[
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract plan names and prices."},
        {"type": "screenshot", "full_page": True, "quality": 80, "viewport": {"width": 1280, "height": 720}},
        {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    headers={"User-Agent": "FirecrawlDocsBot/1.0"},
    include_tags=["main", "article"],
    exclude_tags=["nav", "footer"],
    only_main_content=True,
    wait_for=1000,
    mobile=False,
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    actions=[
        {"type": "click", "selector": "#accept"},
        {"type": "write", "text": "firecrawl"},
        {"type": "press", "key": "Enter"},
        {"type": "wait", "milliseconds": 1500},
        {"type": "scrape"}
    ],
    location={"country": "US", "languages": ["en-US"]},
    skip_tls_verification=True,
    remove_base64_images=True,
    fast_mode=True,
    block_ads=True,
    proxy="auto",
    max_age=86400000,
    store_in_cache=True,
    profile={"name": "docs-session", "save_changes": True}
)
```

<div id="interact">
  ## Interact
</div>

### Pourquoi l’utiliser

Utilisez interact lorsqu’une page nécessite des actions dans le navigateur ou l’exécution de code une fois le scrape lancé (clics, formulaires ou étapes scriptées).

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.interact(job_id, code=None, prompt=None, language="node", timeout=None)`

<div id="simple-example">
  ### Exemple simple
</div>

```py
doc = client.scrape("https://example.com", formats=["markdown"])
job_id = doc.metadata.scrape_id if doc.metadata else None
if not job_id:
    raise ValueError("Missing scrape_id from scrape response")

result = client.interact(
    job_id,
    prompt="Click the pricing tab and summarize the plans.",
    language="node"
)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```py
result = client.interact(
    "<scrape_job_id>",
    code="print(await page.title())",
    language="python",
    timeout=60
)
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre  | Type / options             | Description                                                                                                |
| ---------- | -------------------------- | ---------------------------------------------------------------------------------------------------------- |
| `job_id`   | `str`                      | ID de la tâche de scraping issu de `document.metadata.scrape_id`.                                          |
| `code`     | `str`                      | Code à exécuter dans la session de navigateur. Obligatoire si `prompt` n&#39;est pas fourni.               |
| `prompt`   | `str`                      | Instruction en langage naturel pour l&#39;agent de navigateur. Obligatoire si `code` n&#39;est pas fourni. |
| `language` | `python`, `node` ou `bash` | Langage d&#39;exécution. Par défaut : `node`.                                                              |
| `timeout`  | `int`                      | Délai maximal d&#39;exécution en secondes (1-300).                                                         |
| `origin`   | `str`                      | Usage interne uniquement. Réservé à l&#39;instrumentation de Firecrawl.                                    |

<div id="notes">
  ## Remarques
</div>

* Alias dépréciés : `scrape_execute`, `stop_interactive_browser` et `delete_scrape_browser` renvoient à `interact` et `stop_interaction`.
* `ScrapeOptions` inclut `min_age`, mais la méthode utilitaire `scrape` ne l’expose pas. Utilisez `ScrapeOptions` via `search(..., scrape_options=...)` si vous en avez besoin.
* Le client `Firecrawl` principal expose directement les méthodes v2 ; l’ancienne version v1 est disponible sous `client.v1`.

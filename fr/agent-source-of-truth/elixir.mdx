<div id="firecrawl-elixir-source-of-truth">
  # Source de référence de Firecrawl Elixir
</div>

Ce fichier constitue la source publique de référence pour les agents. Il est généré à partir du SDK Firecrawl Elixir (au format OpenAPI) et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

```elixir
# mix.exs
defp deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

<div id="authenticate">
  ## Authentification
</div>

Configurez votre clé API dans `config/runtime.exs` :

```elixir
config :firecrawl, api_key: System.fetch_env!("FIRECRAWL_API_KEY")
```

Ou transmettez `api_key` à chaque requête :

```elixir
{:ok, resp} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-...")
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : partir d’une requête pour trouver des URL pertinentes, y compris avec des contraintes `site:`.
* `scrape` : récupérer et extraire le contenu d’une URL connue.
* `interact` : exécuter des actions de navigateur ou du code lorsqu’une page nécessite des clics, des formulaires ou des étapes après le scrape.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez la recherche lorsque vous avez besoin de découvrir du contenu à partir d’une requête et que vous souhaitez que Firecrawl renvoie des URL (et, éventuellement, extraire chaque résultat). Vous pouvez limiter les résultats à un domaine avec l’opérateur standard `site:`, par exemple `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000,
    actions: [
      %{type: "click", selector: "button#accept"},
      %{type: "wait", milliseconds: 750},
      %{type: "scrape"}
    ]
  ]
)
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre             | Type / Options                                                 | Description                                                                                       |
| --------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| `query`               | `String`                                                       | Requête de recherche. Utilisez `site:domain` pour limiter les résultats à un site.                |
| `sources`             | `List` (`"web"`, `"news"`, `"images"` or maps with `type`)     | Sources dans lesquelles effectuer la recherche.                                                   |
| `categories`          | `List` (`"github"`, `"research"`, `"pdf"` or maps with `type`) | Catégories permettant de filtrer les résultats.                                                   |
| `limit`               | `integer`                                                      | Nombre maximal de résultats à renvoyer.                                                           |
| `tbs`                 | `String`                                                       | Filtre de recherche temporel (par exemple `qdr:d`, `qdr:w`, `sbd:1,qdr:m`).                       |
| `location`            | `String`                                                       | Chaîne de localisation pour les résultats de recherche.                                           |
| `country`             | `String`                                                       | Code pays ISO pour le ciblage géographique des résultats de recherche.                            |
| `enterprise`          | `List[String]` (`"anon"`, `"zdr"`)                             | Options Enterprise ZDR lorsqu’elles sont activées pour votre équipe.                              |
| `ignore_invalid_urls` | `boolean`                                                      | Exclut les URL non valides que les autres points de terminaison Firecrawl ne peuvent pas scraper. |
| `timeout`             | `integer`                                                      | Délai d’expiration en millisecondes.                                                              |
| `scrape_options`      | `keyword`                                                      | Options de scrape appliquées à chaque résultat (mêmes champs que `scrape`).                       |

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `scrape` lorsque vous avez déjà une URL et que vous souhaitez que Firecrawl renvoie du contenu structuré et des formats optionnels.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`Firecrawl.scrape_and_extract_from_url(params, opts)`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  include_tags: ["main", "article"],
  exclude_tags: ["nav", "footer"],
  only_main_content: true,
  wait_for: 1000,
  mobile: false,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "write", text: "firecrawl"},
    %{type: "press", key: "Enter"},
    %{type: "wait", milliseconds: 1500},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  skip_tls_verification: true,
  remove_base64_images: true,
  block_ads: true,
  proxy: "auto",
  max_age: 86400000,
  store_in_cache: true,
  profile: [name: "docs-session", saveChanges: true]
)
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre               | Type / Options     | Description                                                                                                                                        |                   |             |
| ----------------------- | ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------- | ----------- |
| `url`                   | `String`           | URL à scraper.                                                                                                                                     |                   |             |
| `formats`               | `List`             | Formats de sortie à inclure. Chaînes ou objets tels que `%{type: "json", prompt, schema}` ou `%{type: "screenshot", fullPage, quality, viewport}`. |                   |             |
| `only_main_content`     | `boolean`          | Conserver uniquement le contenu principal.                                                                                                         |                   |             |
| `include_tags`          | `List[String]`     | Balises à inclure.                                                                                                                                 |                   |             |
| `exclude_tags`          | `List[String]`     | Balises à exclure.                                                                                                                                 |                   |             |
| `max_age`               | `integer`          | Préférer les données en cache jusqu’à cet âge.                                                                                                     |                   |             |
| `min_age`               | `integer`          | Retourner les données en cache uniquement si elles ont au moins l’âge minimum.                                                                     |                   |             |
| `headers`               | `map`              | En-têtes personnalisés.                                                                                                                            |                   |             |
| `wait_for`              | `integer`          | Temps d’attente en millisecondes avant le scraping.                                                                                                |                   |             |
| `mobile`                | `boolean`          | Émuler un appareil mobile.                                                                                                                         |                   |             |
| `skip_tls_verification` | `boolean`          | Ignorer la vérification du certificat TLS.                                                                                                         |                   |             |
| `timeout`               | `integer`          | Délai d’expiration en millisecondes.                                                                                                               |                   |             |
| `parsers`               | `List`             | Configuration des parseurs (options PDF prises en charge).                                                                                         |                   |             |
| `actions`               | `List`             | Actions du Browser à exécuter avant le scraping.                                                                                                   |                   |             |
| `location`              | `keyword`          | Paramètres de localisation pour la requête.                                                                                                        |                   |             |
| `remove_base64_images`  | `boolean`          | Supprimer les images base64 de la sortie markdown.                                                                                                 |                   |             |
| `block_ads`             | `boolean`          | Activer le blocage des publicités et des fenêtres contextuelles de cookies.                                                                        |                   |             |
| `proxy`                 | `&quot;basic&quot; | &quot;enhanced&quot;                                                                                                                               | &quot;auto&quot;` | Mode proxy. |
| `store_in_cache`        | `boolean`          | Stocker les résultats dans le cache.                                                                                                               |                   |             |
| `profile`               | `keyword`          | Profil persistant du Browser, par exemple `[name: "profile", saveChanges: true]`.                                                                  |                   |             |
| `zero_data_retention`   | `boolean`          | Activer la Zero Data Retention si votre compte le permet.                                                                                          |                   |             |

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez Interact lorsqu’une page nécessite des actions dans le navigateur ou l’exécution de code une fois le scrape lancé (clics, formulaires ou étapes scriptées).

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, scrape_resp} = Firecrawl.scrape_and_extract_from_url(url: "https://example.com")
job_id = get_in(scrape_resp.body, ["data", "metadata", "scrapeId"])

{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  job_id,
  code: "console.log(document.title)",
  language: :node
)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  "<scrape_job_id>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre  | Type / options | Description                                                         |        |                                                              |
| ---------- | -------------- | ------------------------------------------------------------------- | ------ | ------------------------------------------------------------ |
| `job_id`   | `String`       | ID de la tâche de scraping issue de `data.metadata.scrapeId`.       |        |                                                              |
| `code`     | `String`       | Code à exécuter dans la session de navigateur. Obligatoire.         |        |                                                              |
| `language` | `:python       | :node                                                               | :bash` | Langage d’exécution. `:node` est utilisé par défaut si omis. |
| `timeout`  | `integer`      | Délai d’exécution maximal, en secondes.                             |        |                                                              |
| `origin`   | `String`       | Usage interne uniquement. Réservé à l’instrumentation de Firecrawl. |        |                                                              |

<div id="notes">
  ## Remarques
</div>

* Les fonctions du client Elixir sont structurées selon OpenAPI et acceptent des paramètres nommés en snake&#95;case, convertis en champs JSON en camelCase.

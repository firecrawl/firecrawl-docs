<div id="firecrawl-nodejs-source-of-truth">
  # Source de vérité Firecrawl Node.js
</div>

Ce fichier est la source publique de référence pour les agents. Il est généré à partir du code source du SDK JS Firecrawl et de la spécification OpenAPI v2.



<div id="install">
  ## Installation
</div>

```bash
npm install @mendable/firecrawl-js
```



<div id="authenticate">
  ## Authentification
</div>

```ts
import Firecrawl from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // remplacement optionnel pour self-hosted
});
```



<div id="when-to-use-what">
  ## Quand utiliser chaque option
</div>

* `search` : partez d’une requête pour trouver des URL pertinentes, y compris avec des contraintes `site:`.
* `scrape` : récupérez et extrayez le contenu d’une URL connue.
* `interact` : exécutez des actions dans le navigateur ou du code lorsqu’une page nécessite des clics, des formulaires ou des étapes après le `scrape`.



<div id="search">
  ## Search
</div>

<div id="why-use-it">
  ### Pourquoi l'utiliser
</div>

Utilisez la recherche lorsque vous avez besoin de découvrir des résultats à partir d'une requête et souhaitez que Firecrawl renvoie des URLs (et scrape optionnellement chaque résultat). Vous pouvez restreindre les résultats à un domaine avec l'opérateur standard `site:`, par exemple `site:docs.firecrawl.dev crawl webhooks`.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.search(query, options?)`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev crawl webhooks", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  timeout: 60000,
  scrapeOptions: {
    formats: [
      "markdown",
      "links",
      { type: "json", prompt: "Extract title, summary, and key endpoints." }
    ],
    onlyMainContent: true,
    includeTags: ["main", "article"],
    excludeTags: ["nav", "footer"],
    waitFor: 1000,
    actions: [
      { type: "click", selector: "button#accept" },
      { type: "wait", milliseconds: 750 },
      { type: "scrape" }
    ]
  }
})
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre                   | Type / Options  | Description                                                             |
| --------------------------- | --------------- | ----------------------------------------------------------------------- | -------- | ---------------- | ---------- | ------------ | ----------------------------- |
| `query`                     | `string`        | Requête de recherche. Utilisez `site:domain` pour restreindre les résultats à un site. |
| `options.sources`           | `Array<"web"    | "news"                                                                  | "images" | { type: "web"    | "news"     | "images" }>` | Sources à rechercher.         |
| `options.categories`        | `Array<"github" | "research"                                                              | "pdf"    | { type: "github" | "research" | "pdf" }>`    | Catégories pour filtrer les résultats. |
| `options.limit`             | `number`        | Nombre maximum de résultats à retourner.                                |
| `options.tbs`               | `string`        | Filtre de recherche temporel (par exemple `qdr:d`, `qdr:w`, `sbd:1,qdr:m`). |
| `options.location`          | `string`        | Chaîne de localisation pour les résultats de recherche.                 |
| `options.ignoreInvalidURLs` | `boolean`       | Exclut les URLs invalides que les autres points de terminaison Firecrawl ne peuvent pas scraper. |
| `options.timeout`           | `number`        | Timeout en millisecondes.                                               |
| `options.scrapeOptions`     | `ScrapeOptions` | Options de scrape appliquées à chaque résultat (mêmes champs que `scrape`). |
| `options.integration`       | `string`        | Usage interne uniquement. Réservé à l'instrumentation Firecrawl.        |
| `options.origin`            | `string`        | Usage interne uniquement. Réservé à l'instrumentation Firecrawl.        |


<div id="scrape">
  ## Scrape
</div>

### Pourquoi l’utiliser

Utilisez scrape lorsque vous avez déjà une URL et souhaitez que Firecrawl renvoie un contenu structuré ainsi que des formats optionnels.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.scrape(url, options?)`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const doc = await client.scrape("https://docs.firecrawl.dev", {
  formats: ["markdown"]
});
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const doc = await client.scrape("https://example.com/pricing", {
  formats: [
    "markdown",
    "links",
    { type: "json", prompt: "Extract plan names and prices." },
    { type: "screenshot", fullPage: true, quality: 80, viewport: { width: 1280, height: 720 } },
    { type: "changeTracking", modes: ["git-diff"], tag: "pricing" }
  ],
  headers: {
    "User-Agent": "FirecrawlDocsBot/1.0"
  },
  includeTags: ["main", "article"],
  excludeTags: ["nav", "footer"],
  onlyMainContent: true,
  waitFor: 1000,
  mobile: false,
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  actions: [
    { type: "click", selector: "#accept" },
    { type: "write", text: "firecrawl" },
    { type: "press", key: "Enter" },
    { type: "wait", milliseconds: 1500 },
    { type: "scrape" }
  ],
  location: { country: "US", languages: ["en-US"] },
  skipTlsVerification: true,
  removeBase64Images: true,
  fastMode: true,
  blockAds: true,
  proxy: "auto",
  maxAge: 86400000,
  storeInCache: true,
  profile: { name: "docs-session", saveChanges: true }
});
```

<div id="interact">
  ## Interact
</div>

### Pourquoi l’utiliser

Utilisez Interact lorsqu’une page nécessite des actions dans le navigateur ou l’exécution de code une fois le scrape démarré (clics, formulaires ou étapes scriptées).

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.interact(jobId, args)`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const doc = await client.scrape("https://example.com", { formats: ["markdown"] });
const jobId = doc.metadata?.scrapeId;
if (!jobId) throw new Error("Missing scrapeId from scrape response");

const result = await client.interact(jobId, {
  prompt: "Click the pricing tab and summarize the plans.",
  language: "node"
});
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const result = await client.interact("<scrapeJobId>", {
  code: "console.log(await page.title());",
  language: "node",
  timeout: 60
});
```

<div id="parameters">
  ### Paramètres
</div>

| Paramètre       | Type / Options      | Description                                                                                |                   |                                                             |
| --------------- | ------------------- | ------------------------------------------------------------------------------------------ | ----------------- | ----------------------------------------------------------- |
| `jobId`         | `string`            | ID de la tâche de scraping issu de `document.metadata.scrapeId`.                           |                   |                                                             |
| `args.code`     | `string`            | Code à exécuter dans la session de navigateur. Requis si `prompt` n’est pas fourni.        |                   |                                                             |
| `args.prompt`   | `string`            | Instruction en langage naturel pour l’agent navigateur. Requis si `code` n’est pas fourni. |                   |                                                             |
| `args.language` | `&quot;python&quot; | &quot;node&quot;                                                                           | &quot;bash&quot;` | Langage d’exécution. `node` est utilisé par défaut si omis. |
| `args.timeout`  | `number`            | Temps limite d’exécution en secondes.                                                      |                   |                                                             |
| `args.origin`   | `string`            | Interne uniquement. Réservé à l’instrumentation de Firecrawl.                              |                   |                                                             |



<div id="notes">
  ## Remarques
</div>

* Alias obsolètes : `scrapeExecute`, `stopInteractiveBrowser` et `deleteScrapeBrowser` renvoient à `interact` et `stopInteraction`.
* Le client `Firecrawl` principal expose directement les méthodes v2 ; l’ancienne version v1 reste disponible via `client.v1`.

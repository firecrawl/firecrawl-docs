<div id="how-it-works">
  ## Comment ça fonctionne
</div>

Intégrez Firecrawl directement dans votre flux de développement IA. Recherchez la documentation, récupérez les spécifications d’API et accédez aux données du web sans quitter votre environnement de développement grâce au Model Context Protocol.

<div id="faqs">
  ## FAQ
</div>

<AccordionGroup>
  <Accordion title="Quels assistants IA prennent en charge le MCP ?">
    Actuellement, Claude Desktop et Cursor prennent en charge nativement le MCP. D&#39;autres assistants IA ajoutent régulièrement cette prise en charge. Vous pouvez également utiliser le SDK MCP pour créer des intégrations personnalisées.
  </Accordion>

  <Accordion title="Puis-je utiliser le MCP dans VS Code ou d'autres IDE ?">
    VS Code et d&#39;autres IDE peuvent utiliser le MCP via des extensions communautaires ou des intégrations via le terminal. La prise en charge native varie selon l&#39;IDE. Consultez notre [référentiel GitHub](https://github.com/firecrawl/firecrawl-mcp-server) pour des guides de configuration spécifiques aux IDE.
  </Accordion>

  <Accordion title="Comment mettre en cache les documents fréquemment consultés ?">
    Le serveur MCP met automatiquement en cache les réponses pendant 15 minutes. Vous pouvez configurer la durée de conservation du cache dans les paramètres de votre serveur MCP ou implémenter une logique de mise en cache personnalisée.
  </Accordion>

  <Accordion title="Existe-t-il une limite de débit pour les requêtes MCP ?">
    Les requêtes MCP utilisent vos limites de débit standard de l&#39;API Firecrawl. Nous recommandons de regrouper les requêtes liées et d&#39;activer la mise en cache pour la documentation fréquemment consultée.
  </Accordion>

  <Accordion title="Comment configurer le MCP avec ma clé API Firecrawl ?">
    Suivez notre [guide de configuration](https://github.com/firecrawl/firecrawl-mcp-server#installation) pour paramétrer le MCP. Vous devrez ajouter votre clé API Firecrawl à votre fichier de configuration MCP. L&#39;opération ne prend que quelques minutes.
  </Accordion>
</AccordionGroup>

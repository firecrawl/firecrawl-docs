<div id="firecrawl-rust-agent-quickstart">
  # Démarrage rapide de l’agent Rust Firecrawl
</div>

Ce fichier est le guide de démarrage rapide de référence pour les agents externes. Il est généré à partir du code source du SDK et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

```bash
cargo add firecrawl
```

<div id="authenticate">
  ## S’authentifier
</div>

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : à utiliser si vous partez d’une requête et avez besoin de découvrir des résultats.
* `scrape` : à utiliser si vous avez déjà une URL et souhaitez récupérer le contenu de la page.
* `interact` : à utiliser si la page nécessite des clics, des formulaires ou des actions du navigateur après le scraping.

<div id="search">
  ## Recherche
</div>

### Pourquoi l’utiliser

Utilisez `search` pour trouver des URL pertinentes à partir d’une requête et, si besoin, scraper chaque résultat. Vous pouvez limiter les requêtes à un seul site avec l’opérateur standard `site:` (par exemple, `site:docs.firecrawl.dev crawl webhooks`) avant de scraper une URL spécifique.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.search(query, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```rust
use firecrawl::v2::Client;

let client = Client::new("fc-your-api-key")?;
let results = client.search("site:docs.firecrawl.dev crawl webhooks", None).await?;

if let Some(web) = results.data.web {
    println!("{:?}", web.first());
}
```

<div id="complex-example">
  ### Exemple complexe
</div>

```rust
use firecrawl::v2::{Client, SearchOptions, SearchSource, SearchCategory, ScrapeOptions, Format};

let client = Client::new("fc-your-api-key")?;
let options = SearchOptions {
    sources: Some(vec![SearchSource::Web, SearchSource::News]),
    categories: Some(vec![SearchCategory::Research]),
    limit: Some(10),
    tbs: Some("qdr:w".to_string()),
    location: Some("San Francisco,California,United States".to_string()),
    ignore_invalid_urls: Some(true),
    scrape_options: Some(ScrapeOptions {
        formats: Some(vec![Format::Markdown, Format::Json]),
        only_main_content: Some(true),
        ..Default::default()
    }),
    integration: Some("agent-quickstart".to_string()),
};

let results = client.search("firecrawl pricing", options).await?;
```

<div id="parameters">
  ### Paramètres
</div>

* `query` : Chaîne de requête de recherche. Toujours obligatoire.
* `limit` : Nombre maximal de résultats à renvoyer. Permet de limiter la taille de la réponse.
* `sources` : Sources dans lesquelles effectuer la recherche. Valeurs prises en charge : `SearchSource::Web`, `SearchSource::News`, `SearchSource::Images`.
* `categories` : Filtres de résultats. Valeurs prises en charge : `SearchCategory::Github`, `SearchCategory::Research`, `SearchCategory::Pdf`.
* `tbs` : Chaîne de filtre de recherche temporel. Permet de limiter la recherche à une plage de temps donnée (par ex. : `qdr:d`).
* `location` : Chaîne de localisation pour des résultats localisés. À utiliser pour des requêtes géographiquement ciblées.
* `ignore_invalid_urls` : Exclut les URL non valides des résultats. À utiliser lors du chaînage vers d’autres points de terminaison.
* `timeout` : Délai d’expiration de la requête, en millisecondes. Permet de limiter la durée de la recherche.
* `scrape_options` : Options de scrape appliquées à chaque résultat (mêmes champs que `scrape`). À utiliser si vous souhaitez obtenir le contenu avec les résultats.
* `integration` : Identifiant d’intégration. À utiliser pour le suivi et l’analyse.

<div id="scrape">
  ## Scrape
</div>

### Pourquoi l’utiliser

Utilisez `scrape` lorsque vous connaissez déjà l’URL et souhaitez obtenir le contenu dans des formats précis.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```rust
use firecrawl::v2::{Client, ScrapeOptions, Format};

let client = Client::new("fc-your-api-key")?;
let doc = client
    .scrape(
        "https://example.com",
        ScrapeOptions {
            formats: Some(vec![Format::Markdown]),
            ..Default::default()
        },
    )
    .await?;

println!("{:?}", doc.markdown);
```

<div id="complex-example">
  ### Exemple complexe
</div>

```rust
use firecrawl::v2::{
    Action, ChangeTrackingMode, ChangeTrackingOptions, Client, Format, JsonOptions, LocationConfig,
    ParserConfig, ProxyType, ScrapeOptions, ScreenshotOptions, ScrollDirection, Viewport,
};

let client = Client::new("fc-your-api-key")?;
let options = ScrapeOptions {
    formats: Some(vec![Format::Markdown, Format::Json, Format::ChangeTracking, Format::Screenshot]),
    json_options: Some(JsonOptions {
        prompt: Some("Extract headings and slugs.".to_string()),
        ..Default::default()
    }),
    change_tracking_options: Some(ChangeTrackingOptions {
        modes: Some(vec![ChangeTrackingMode::GitDiff]),
        tag: Some("docs".to_string()),
        ..Default::default()
    }),
    screenshot_options: Some(ScreenshotOptions {
        full_page: Some(true),
        viewport: Some(Viewport { width: 1280, height: 720 }),
        ..Default::default()
    }),
    actions: Some(vec![
        Action::Wait {
            milliseconds: Some(1000),
            selector: None,
        },
        Action::Click {
            selector: "button#accept".to_string(),
        },
        Action::Scroll {
            direction: ScrollDirection::Down,
            selector: None,
        },
    ]),
    parsers: Some(vec![ParserConfig::Pdf {
        parser_type: "pdf".to_string(),
        max_pages: Some(5),
    }]),
    location: Some(LocationConfig {
        country: Some("US".to_string()),
        languages: Some(vec!["en-US".to_string()]),
    }),
    proxy: Some(ProxyType::Stealth),
    ..Default::default()
};

let doc = client.scrape("https://example.com/docs", options).await?;
```

<div id="parameters">
  ### Paramètres
</div>

* `url` : URL à scraper. Toujours obligatoire.
* `formats` : Formats de sortie à inclure. Valeurs prises en charge : `Format::Markdown`, `Format::Html`, `Format::RawHtml`, `Format::Links`, `Format::Images`, `Format::Screenshot`, `Format::Summary`, `Format::ChangeTracking`, `Format::Json`, `Format::Attributes`, `Format::Branding`, `Format::Audio`.
* `headers` : En-têtes HTTP supplémentaires. À utiliser pour l&#39;authentification, les cookies ou le user-agent.
* `include_tags` : Inclut uniquement le contenu de ces balises HTML. À utiliser pour cibler l&#39;extraction.
* `exclude_tags` : Exclut le contenu de ces balises HTML. À utiliser pour supprimer les éléments parasites.
* `only_main_content` : Extrait uniquement le contenu principal. À utiliser pour exclure la navigation et le pied de page.
* `timeout` : Délai d&#39;expiration du scrape en millisecondes. À utiliser pour limiter le temps d&#39;exécution.
* `wait_for` : Délai avant le scraping en millisecondes. À utiliser pour les pages rendues en JS.
* `mobile` : Émule un appareil mobile. À utiliser pour les mises en page mobiles.
* `parsers` : Liste de parseurs ou configurations de parseur. Valeurs prises en charge : `ParserConfig::Simple(String)` ou `ParserConfig::Pdf { parser_type: "pdf", max_pages: Option<u32> }`.
* `actions` : actions Browser à exécuter avant le scraping. Actions prises en charge : `Wait { milliseconds?, selector? }`, `Screenshot { full_page?, quality?, viewport? }`, `Click { selector }`, `Write { text }`, `Press { key }`, `Scroll { direction: Up | Down, selector? }`, `Scrape`, `ExecuteJavascript { script }`, `Pdf { format?, landscape?, scale? }` avec les formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location` : Configuration de localisation `{ country, languages }`. À utiliser pour les pages spécifiques à une zone géographique.
* `skip_tls_verification` : Ignore la vérification du certificat TLS. À utiliser pour les certificats auto-signés.
* `remove_base64_images` : Supprime les images base64 de la sortie markdown. À utiliser pour réduire la taille du payload.
* `block_ads` : Active le blocage des publicités et des cookies. À utiliser pour réduire les éléments parasites.
* `proxy` : Mode proxy `ProxyType::Basic`, `ProxyType::Stealth`, `ProxyType::Enhanced`, `ProxyType::Auto`.
* `max_age` : Accepte du contenu en cache plus récent que cet âge (ms). À utiliser pour privilégier les résultats du cache.
* `min_age` : Retourne uniquement du contenu en cache datant d&#39;au moins cet âge (ms). À utiliser pour les lectures exclusivement depuis le cache.
* `store_in_cache` : Stocke le résultat dans le cache/l&#39;index. À utiliser lorsque vous souhaitez le réutiliser.
* `profile` : Profil Browser persistant `ProfileConfig { name, save_changes }`. À utiliser pour conserver l&#39;état de la session.
* `integration` : Identifiant d&#39;intégration. À utiliser pour le suivi et l&#39;analytique.
* `json_options` : Options d&#39;extraction JSON pour le format `Format::Json` (`schema`, `system_prompt`, `prompt`).
* `screenshot_options` : Options de capture d&#39;écran pour le format `Format::Screenshot` (`full_page`, `quality`, `viewport`).
* `change_tracking_options` : Options de suivi des modifications pour le format `Format::ChangeTracking` (`modes`, `schema`, `prompt`, `tag`). Modes : `ChangeTrackingMode::GitDiff`, `ChangeTrackingMode::Json`.
* `attribute_selectors` : Sélecteurs d&#39;attributs pour le format `Format::Attributes` (`selector`, `attribute`).

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Pourquoi l&#39;utiliser
</div>

Utilisez `interact` lorsque la page nécessite des clics, le remplissage de formulaires ou d&#39;autres actions du navigateur après le scraping.

<div id="preferred-sdk-method">
  ### Méthode recommandée du SDK
</div>

`client.interact(job_id, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```rust
use firecrawl::v2::{Client, ScrapeExecuteOptions};

let client = Client::new("fc-your-api-key")?;
let run = client
    .interact(
        "scrape-job-id",
        ScrapeExecuteOptions {
            code: Some("console.log(await page.url())".to_string()),
            ..Default::default()
        },
    )
    .await?;

println!("{:?}", run.stdout);
```

<div id="complex-example">
  ### Exemple complexe
</div>

```rust
use firecrawl::v2::{Client, ScrapeExecuteLanguage, ScrapeExecuteOptions};

let client = Client::new("fc-your-api-key")?;
let run = client
    .interact(
        "scrape-job-id",
        ScrapeExecuteOptions {
            prompt: Some("Open the pricing page and list the plan names.".to_string()),
            language: Some(ScrapeExecuteLanguage::Node),
            timeout: Some(60),
            origin: Some("agent-docs".to_string()),
            ..Default::default()
        },
    )
    .await?;
```

<div id="parameters">
  ### Paramètres
</div>

* `job_id` : ID de tâche de scraping (issu de `document.metadata.scrape_id`). Toujours obligatoire.
* `code` : Code à exécuter dans le sandbox du navigateur. À utiliser pour les interactions scriptées.
* `prompt` : Instruction en langage naturel pour l’agent du navigateur. À utiliser si vous préférez des actions pilotées par l’agent.
* `language` : Langage d’exécution `ScrapeExecuteLanguage::Python`, `ScrapeExecuteLanguage::Node` ou `ScrapeExecuteLanguage::Bash` (valeur par défaut : `Node`).
* `timeout` : Délai d’exécution maximal, en secondes. À utiliser pour limiter les exécutions longues.
* `origin` : Tag d’origine facultatif. À utiliser pour l’attribution des requêtes.

<div id="notes">
  ## Remarques
</div>

* Les paramètres sont en snake&#95;case en Rust ; le JSON est sérialisé en camelCase.
* Méthode utilitaire : `search_and_scrape` effectue une recherche avec le scraping activé.
* Alias dépréciés : `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (utilisez `interact` et `stop_interaction`).

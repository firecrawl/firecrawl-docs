<div id="firecrawl-python-agent-quickstart">
  # Guide de démarrage rapide de l’agent Python Firecrawl
</div>

Ce fichier constitue le guide de démarrage rapide de référence pour les agents externes et est généré à partir du code source du SDK et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

```bash
pip install firecrawl-py
```

<div id="authenticate">
  ## S’authentifier
</div>

```python
import os
from firecrawl import Firecrawl

client = Firecrawl(api_key=os.environ["FIRECRAWL_API_KEY"])
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : à utiliser si vous partez d’une requête et avez besoin de découvrir des résultats.
* `scrape` : à utiliser si vous avez déjà une URL et voulez le contenu de la page.
* `interact` : à utiliser si la page nécessite des clics, des formulaires ou des actions du navigateur après le scraping.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `search` pour trouver des URL pertinentes à partir d’une requête et, si besoin, scraper chaque résultat. Vous pouvez limiter les requêtes à un seul site avec l’opérateur standard `site:` (par exemple, `site:docs.firecrawl.dev crawl webhooks`) avant de scraper une URL spécifique.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`client.search(query, **options)`

<div id="simple-example">
  ### Exemple simple
</div>

```python
results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    limit=5,
)

print(results.web[0] if results.web else None)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```python
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "firecrawl pricing",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:w",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            {"type": "json", "prompt": "Extract pricing tiers and limits."},
        ],
        only_main_content=True,
        proxy="stealth",
    ),
    integration="agent-quickstart",
)
```

<div id="parameters">
  ### Paramètres
</div>

* `query` : Chaîne de requête de recherche. Obligatoire.
* `sources` : Sources dans lesquelles effectuer la recherche. Valeurs prises en charge : `"web"`, `"news"`, `"images"` ou des objets `{"type": ...}`. Permet de contrôler les tableaux de résultats renvoyés.
* `categories` : Filtres de résultats. Valeurs prises en charge : `"github"`, `"research"`, `"pdf"` ou des objets `{"type": ...}`. Permet de limiter les résultats par catégorie.
* `limit` : Nombre maximal de résultats à renvoyer. Par défaut : `5`.
* `tbs` : Chaîne de filtre de recherche temporel. Permet de limiter la recherche à une plage de temps (par exemple, `qdr:d`).
* `location` : Chaîne de localisation pour des résultats localisés. À utiliser pour des requêtes géographiques spécifiques.
* `ignore_invalid_urls` : Exclut les URL non valides des résultats. À utiliser lors du chaînage vers d’autres points de terminaison.
* `timeout` : Délai d’expiration de la requête, en millisecondes. Par défaut : `300000`.
* `scrape_options` : Options de scrape appliquées à chaque résultat (mêmes champs que `scrape`). À utiliser si vous souhaitez inclure le contenu dans les résultats.
* `integration` : Identifiant d’intégration. À utiliser pour le suivi et l’analyse.

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `scrape` lorsque vous connaissez déjà l’URL et que vous souhaitez obtenir le contenu dans des formats spécifiques.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`client.scrape(url, **options)`

<div id="simple-example">
  ### Exemple simple
</div>

```python
doc = client.scrape(
    "https://example.com",
    formats=["markdown"],
)

print(doc.markdown)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```python
doc = client.scrape(
    "https://example.com/docs",
    formats=[
        "markdown",
        {"type": "json", "prompt": "Extract headings and slugs."},
        {"type": "change_tracking", "modes": ["git-diff"], "tag": "docs"},
        {"type": "screenshot", "full_page": True, "viewport": {"width": 1280, "height": 720}},
        {"type": "attributes", "selectors": [{"selector": "a", "attribute": "href"}]},
        {"type": "query", "prompt": "What is the pricing model?"},
    ],
    only_main_content=True,
    wait_for=1500,
    actions=[
        {"type": "click", "selector": "button#accept"},
        {"type": "scroll", "direction": "down"},
    ],
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    location={"country": "US", "languages": ["en-US"]},
    proxy="stealth",
    profile={"name": "docs-session", "save_changes": True},
    max_age=3_600_000,
)
```

<div id="parameters">
  ### Paramètres
</div>

* `url` : URL à scraper. Toujours obligatoire.
* `formats` : Formats de sortie à inclure. Accepte un objet `ScrapeFormats` (champs `markdown`, `html`, `raw_html`, `summary`, `links`, `images`, `screenshot`, `change_tracking`, `json`) ou une liste de chaînes/objets de format. Les chaînes de format prises en charge incluent `markdown`, `html`, `rawHtml`, `raw_html`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `change_tracking`, `json`, `attributes`, `branding`, `query`, `audio`. Objets de format : `json` `{type: "json", prompt?, schema?}` (requiert `prompt` ou `schema`), `changeTracking` `{type: "changeTracking" | "change_tracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `attributes` `{type: "attributes", selectors: [{selector, attribute}]}`, `query` `{type: "query", prompt}`.
* `headers` : En-têtes HTTP supplémentaires. À utiliser pour l&#39;authentification, les cookies ou le user-agent.
* `include_tags` : Inclure uniquement le contenu de ces balises HTML. À utiliser pour restreindre l&#39;extraction.
* `exclude_tags` : Exclure le contenu de ces balises HTML. À utiliser pour supprimer les éléments parasites.
* `only_main_content` : Extraire uniquement le contenu principal. À utiliser pour exclure la navigation et le pied de page.
* `timeout` : Délai d&#39;expiration du scrape en millisecondes. À utiliser pour limiter la durée d&#39;exécution.
* `wait_for` : Délai avant le scraping en millisecondes. À utiliser pour les pages rendues en JS.
* `mobile` : Émuler un appareil mobile. À utiliser pour les mises en page mobiles.
* `parsers` : Liste ou configurations d&#39;analyseurs. Configuration PDF prise en charge : `{type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: int}` ou entrées sous forme de chaîne (par exemple, `"pdf"`).
* `actions` : Actions du Browser à exécuter avant le scraping. Actions prises en charge : `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", full_page?, quality?, viewport?}`, `click` `{type: "click", selector}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}` avec les formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location` : Configuration de localisation `{country, languages}`. À utiliser pour les pages spécifiques à une zone géographique.
* `skip_tls_verification` : Ignorer la vérification du certificat TLS. À utiliser pour les certificats auto-signés.
* `remove_base64_images` : Supprimer les images base64 de la sortie markdown. À utiliser pour réduire la taille du payload.
* `fast_mode` : Activer le mode rapide. À utiliser lorsque la vitesse prime.
* `use_mock` : Utiliser une fixture mock nommée (interne/test).
* `block_ads` : Activer le blocage des publicités et des cookies. À utiliser pour réduire les éléments parasites.
* `proxy` : Mode proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`.
* `max_age` : Accepter du contenu mis en cache dont l&#39;âge est inférieur à cette valeur (ms). À utiliser pour favoriser les réponses en cache.
* `store_in_cache` : Stocker le résultat dans le cache/l&#39;index. À utiliser si vous souhaitez le réutiliser.
* `profile` : Profil Browser persistant `{name, save_changes}` (accepte aussi `saveChanges`). À utiliser pour conserver l&#39;état de la session.
* `integration` : Identifiant d&#39;intégration. À utiliser pour le suivi et l&#39;analytique.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `interact` lorsque la page nécessite des clics, le remplissage de formulaires ou d’autres actions dans le navigateur après le scraping.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`client.interact(job_id, code=None, *, prompt=None, **options)`

<div id="simple-example">
  ### Exemple simple
</div>

```python
run = client.interact(
    "scrape-job-id",
    code="print(await page.url())",
)

print(run.stdout)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```python
run = client.interact(
    "scrape-job-id",
    prompt="Open the pricing page and list the plan names.",
    language="python",
    timeout=60,
    origin="agent-docs",
)

print(run.output)
```

<div id="parameters">
  ### Paramètres
</div>

* `job_id` : ID de la tâche de scraping (issu de `document.metadata_typed.scrape_id`). Toujours requis.
* `code` : Code à exécuter dans le sandbox du navigateur. À utiliser pour les interactions scriptées.
* `prompt` : Instruction en langage naturel pour l’agent du navigateur. À utiliser si vous préférez des actions pilotées par l’agent.
* `language` : Langage d’exécution `"python"`, `"node"` ou `"bash"` (par défaut `"node"`). À utiliser pour l’adapter à votre code.
* `timeout` : Délai d’exécution en secondes. À utiliser pour limiter les exécutions longues.
* `origin` : Balise d’origine facultative. À utiliser pour l’attribution des requêtes.

<div id="notes">
  ## Remarques
</div>

* Les paramètres sont en `snake_case` en Python ; le SDK normalise les payloads au format `camelCase` pour l’API.
* Alias obsolètes : `scrape_execute`, `stop_interactive_browser`, `delete_scrape_browser` (utilisez `interact` et `stop_interaction`).
* `Firecrawl` est le client v2 unifié ; la v1 reste disponible sous `.v1` pour des raisons de compatibilité.

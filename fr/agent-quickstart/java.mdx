<div id="firecrawl-java-agent-quickstart">
  # Guide de démarrage rapide de l&#39;agent Java Firecrawl
</div>

Ce fichier est le guide de démarrage rapide de référence pour les agents externes. Il est généré à partir du code source du SDK et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

```gradle
dependencies {
  implementation("com.firecrawl:firecrawl-java:1.2.0")
}
```

<div id="authenticate">
  ## Authentification
</div>

```java
import com.firecrawl.client.FirecrawlClient;

FirecrawlClient client = FirecrawlClient.builder()
    .apiKey(System.getenv("FIRECRAWL_API_KEY"))
    .build();
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : à utiliser lorsque vous partez d&#39;une requête et devez découvrir des résultats.
* `scrape` : à utiliser lorsque vous avez déjà une URL et voulez récupérer le contenu de la page.
* `interact` : à utiliser lorsque la page nécessite des clics, des formulaires ou des actions dans le navigateur après le scraping.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `search` pour trouver des URL pertinentes à partir d’une requête et, si besoin, scraper chaque résultat. Vous pouvez limiter les requêtes à un seul site avec l’opérateur standard `site:` (par exemple, `site:docs.firecrawl.dev crawl webhooks`) avant de scraper une URL spécifique.

### Méthode SDK recommandée

`client.search(query, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```java
import com.firecrawl.models.SearchData;

SearchData data = client.search("site:docs.firecrawl.dev crawl webhooks");
System.out.println(data.getWeb());
```

<div id="complex-example">
  ### Exemple complexe
</div>

```java
import com.firecrawl.models.SearchData;
import com.firecrawl.models.SearchOptions;
import com.firecrawl.models.ScrapeOptions;

SearchData data = client.search(
    "firecrawl pricing",
    SearchOptions.builder()
        .sources(java.util.List.of("web", "news"))
        .categories(java.util.List.of("research"))
        .limit(10)
        .tbs("qdr:w")
        .location("San Francisco,California,United States")
        .ignoreInvalidURLs(true)
        .scrapeOptions(
            ScrapeOptions.builder()
                .formats(java.util.List.of(
                    "markdown",
                    java.util.Map.of("type", "json", "prompt", "Extract pricing tiers and limits.")
                ))
                .onlyMainContent(true)
                .proxy("stealth")
                .build()
        )
        .integration("agent-quickstart")
        .build()
);
```

<div id="parameters">
  ### Paramètres
</div>

* `query`: Chaîne de requête de recherche. Toujours requise.
* `sources`: Sources à rechercher. Valeurs prises en charge : `"web"`, `"news"`, `"images"` ou objets `{type: "web" | "news" | "images"}`. À utiliser pour déterminer quels tableaux de résultats sont renvoyés.
* `categories`: Filtres de résultats. Valeurs prises en charge : `"github"`, `"research"`, `"pdf"` ou objets `{type: "github" | "research" | "pdf"}`. À utiliser pour restreindre les résultats par catégorie.
* `limit`: Nombre maximal de résultats à renvoyer. À utiliser pour limiter la taille de la réponse.
* `tbs`: Chaîne de filtre de recherche basée sur le temps. À utiliser pour restreindre les résultats à une période donnée (par ex., `qdr:d`).
* `location`: Chaîne de localisation pour des résultats localisés. À utiliser pour des requêtes géographiquement ciblées.
* `ignoreInvalidURLs`: Exclut les URL non valides des résultats. À utiliser lors de l’envoi vers d’autres points de terminaison.
* `timeout`: Délai d’expiration de la requête en millisecondes. À utiliser pour limiter le temps de recherche.
* `scrapeOptions`: Options de scraping appliquées à chaque résultat (mêmes champs que `scrape`). À utiliser lorsque vous souhaitez inclure le contenu dans les résultats.
* `integration`: Identifiant d’intégration. À utiliser pour le suivi et l’analytique.

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `scrape` lorsque vous connaissez déjà l’URL et souhaitez obtenir le contenu dans des formats précis.

### Méthode SDK recommandée

`client.scrape(url, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com",
    ScrapeOptions.builder()
        .formats(java.util.List.of("markdown"))
        .build()
);

System.out.println(doc.getMarkdown());
```

<div id="complex-example">
  ### Exemple complexe
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.LocationConfig;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com/docs",
    ScrapeOptions.builder()
        .formats(java.util.List.of(
            "markdown",
            java.util.Map.of("type", "json", "prompt", "Extract headings and slugs."),
            java.util.Map.of("type", "changeTracking", "modes", java.util.List.of("git-diff"))
        ))
        .onlyMainContent(true)
        .waitFor(1500)
        .actions(java.util.List.of(
            java.util.Map.of("type", "click", "selector", "button#accept"),
            java.util.Map.of("type", "scroll", "direction", "down")
        ))
        .parsers(java.util.List.of(java.util.Map.of("type", "pdf", "mode", "auto", "maxPages", 5)))
        .location(LocationConfig.builder()
            .country("US")
            .languages(java.util.List.of("en-US"))
            .build())
        .proxy("stealth")
        .maxAge(3_600_000L)
        .build()
);
```

<div id="parameters">
  ### Paramètres
</div>

* `url` : URL à scraper. Toujours requise.
* `formats` : Formats de sortie à inclure. Chaînes de format prises en charge : `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`. Objets de format : `json` `{type: "json", prompt?, schema?}`, `changeTracking` `{type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`, `screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`.
* `headers` : En-têtes HTTP supplémentaires. À utiliser pour l&#39;authentification, les cookies ou le user-agent.
* `includeTags` : Inclure uniquement le contenu de ces balises HTML. À utiliser pour affiner l&#39;extraction.
* `excludeTags` : Exclure le contenu de ces balises HTML. À utiliser pour supprimer le bruit.
* `onlyMainContent` : Extraire uniquement le contenu principal. À utiliser pour supprimer la navigation et le pied de page.
* `timeout` : Délai d&#39;expiration du scraping en millisecondes. À utiliser pour limiter le temps d&#39;exécution.
* `waitFor` : Délai avant le scraping en millisecondes. À utiliser pour les pages rendues en JS.
* `mobile` : Émuler un appareil mobile. À utiliser pour les mises en page mobiles.
* `parsers` : Liste ou configurations de parseurs. Configuration PDF prise en charge : `{type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number}` ou entrées sous forme de chaîne (par ex. `"pdf"`).
* `actions` : actions Browser à exécuter avant le scraping. actions prises en charge : `wait` `{type: "wait", milliseconds? | selector?}`, `screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`, `click` `{type: "click", selector, all?}`, `write` `{type: "write", text}`, `press` `{type: "press", key}`, `scroll` `{type: "scroll", direction: "up" | "down", selector?}`, `scrape` `{type: "scrape"}`, `executeJavascript` `{type: "executeJavascript", script}`, `pdf` `{type: "pdf", format?, landscape?, scale?}` avec les formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location` : Configuration de localisation (`country`, `languages`). À utiliser pour les pages spécifiques à une zone géographique.
* `skipTlsVerification` : Ignorer la vérification du certificat TLS. À utiliser pour les certificats autosignés.
* `removeBase64Images` : Supprimer les images base64 de la sortie markdown. À utiliser pour réduire la taille du payload.
* `blockAds` : Activer le blocage des publicités et des cookies. À utiliser pour réduire le bruit.
* `proxy` : Mode proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"` ou URL de proxy personnalisée.
* `maxAge` : Accepter le contenu en cache dont l&#39;âge est inférieur à cette valeur (ms). À utiliser pour privilégier les résultats en cache.
* `storeInCache` : Stocker le résultat dans le cache/l&#39;index. À utiliser si vous souhaitez le réutiliser.
* `integration` : Identifiant d&#39;intégration. À utiliser pour le suivi et l&#39;analyse.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `interact` lorsque la page nécessite des clics, le remplissage de formulaires ou d’autres actions dans le navigateur après un scrape.

<div id="preferred-sdk-method">
  ### Méthode du SDK recommandée
</div>

`client.interact(jobId, code, language, timeout, origin)`

<div id="simple-example">
  ### Exemple simple
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());"
);

System.out.println(exec.getStdout());
```

<div id="complex-example">
  ### Exemple complexe
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());",
    "node",
    60,
    "agent-docs"
);
```

<div id="parameters">
  ### Paramètres
</div>

* `jobId` : ID de tâche de scraping (à partir de `document.metadata.scrapeId`). Toujours requis.
* `code` : Code à exécuter dans le sandbox du navigateur. À utiliser pour les interactions scriptées.
* `language` : Langage d’exécution `"python"`, `"node"` ou `"bash"` (par défaut : `"node"`). À utiliser pour l’adapter à votre code.
* `timeout` : Délai d’exécution maximal, en secondes. À utiliser pour limiter les exécutions trop longues.
* `origin` : Tag d’origine facultatif. À utiliser pour l’attribution des requêtes.

<div id="notes">
  ## Remarques
</div>

* Les paramètres sont en camelCase en Java.
* `ScrapeOptions` et `SearchOptions` utilisent le pattern builder.
* Alias dépréciés : `scrapeExecute` et `deleteScrapeBrowser` (utilisez `interact` et `stopInteractiveBrowser`).
* Le SDK Java expose uniquement `interact` via du code (sans paramètre `prompt`).

<div id="firecrawl-javascripttypescript-agent-quickstart">
  # Guide de démarrage rapide de l’agent Firecrawl JavaScript/TypeScript
</div>

Ce fichier est le guide de démarrage rapide de référence pour les agents externes. Il est généré à partir du code source du SDK et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="authenticate">
  ## S’authentifier
</div>

```ts
import { Firecrawl } from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // optionnel pour self-hosted
});
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : à utiliser si vous partez d’une requête et avez besoin de découvrir des résultats.
* `scrape` : à utiliser si vous avez déjà une URL et voulez récupérer le contenu de la page.
* `interact` : à utiliser si la page nécessite des clics, des formulaires ou des actions dans le navigateur après le scraping.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `search` pour trouver des URL pertinentes à partir d’une requête et, si vous le souhaitez, scraper chaque résultat. Vous pouvez limiter les requêtes à un seul site avec l’opérateur standard `site:` (par exemple, `site:docs.firecrawl.dev crawl webhooks`) avant de scraper une URL spécifique.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`client.search(query, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const results = await client.search(
  "site:docs.firecrawl.dev crawl webhooks",
  { limit: 5 }
);

console.log(results.web?.[0]);
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const results = await client.search("firecrawl pricing", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  scrapeOptions: {
    formats: [
      "markdown",
      { type: "json", prompt: "Extract pricing tiers and limits." }
    ],
    onlyMainContent: true,
    actions: [
      { type: "wait", milliseconds: 1000 },
      { type: "scroll", direction: "down" }
    ],
    proxy: "stealth"
  },
  integration: "agent-quickstart",
  origin: "agent-docs"
});
```

<div id="parameters">
  ### Paramètres
</div>

* `query` : Chaîne de requête de recherche. Toujours obligatoire.
* `sources` : Sources à rechercher. Valeurs prises en charge : `"web"`, `"news"`, `"images"` ou `{ type: "web" | "news" | "images" }`. Permet de contrôler quels tableaux de résultats sont renvoyés.
* `categories` : Filtres de résultats. Valeurs prises en charge : `"github"`, `"research"`, `"pdf"` ou `{ type: "github" | "research" | "pdf" }`. Permet de restreindre les résultats par catégorie.
* `limit` : Nombre maximal de résultats à renvoyer. Permet de limiter la taille de la réponse.
* `tbs` : Chaîne de filtre de recherche temporel. Permet de limiter les résultats à une plage de temps donnée (par ex., `qdr:d`).
* `location` : Chaîne d’emplacement pour des résultats localisés. À utiliser pour des requêtes géolocalisées.
* `ignoreInvalidURLs` : Exclut les URL non valides des résultats. À utiliser lors du chaînage vers d’autres points de terminaison.
* `timeout` : Délai d’expiration de la requête en millisecondes. Permet de limiter la durée de la recherche.
* `scrapeOptions` : Options de scrape appliquées à chaque résultat (mêmes champs que `scrape`). À utiliser si vous souhaitez inclure le contenu dans les résultats.
* `integration` : Identifiant d’intégration. À utiliser pour le suivi et l’analytique.
* `origin` : Balise d’origine facultative. À utiliser pour l’attribution de la requête.

<div id="scrape">
  ## Extraction
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `scrape` si vous connaissez déjà l’URL et souhaitez obtenir le contenu dans des formats spécifiques.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const doc = await client.scrape("https://example.com", {
  formats: ["markdown"]
});

console.log(doc.markdown);
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const doc = await client.scrape("https://example.com/docs", {
  formats: [
    "markdown",
    { type: "json", prompt: "Extract headings and their slugs." },
    { type: "changeTracking", modes: ["git-diff"], tag: "docs" },
    { type: "screenshot", fullPage: true, viewport: { width: 1280, height: 720 } },
    {
      type: "attributes",
      selectors: [{ selector: "a", attribute: "href" }]
    },
    { type: "query", prompt: "What is the pricing model?" }
  ],
  onlyMainContent: true,
  waitFor: 1500,
  actions: [
    { type: "click", selector: "button#accept" },
    { type: "scroll", direction: "down" }
  ],
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  location: { country: "US", languages: ["en-US"] },
  proxy: "stealth",
  profile: { name: "docs-session", saveChanges: true },
  maxAge: 3600000
});
```

<div id="parameters">
  ### Paramètres
</div>

* `url` : URL à scraper. Toujours obligatoire.
* `formats` : Formats de sortie à inclure. Chaînes de format prises en charge : `markdown`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `summary`, `changeTracking`, `json`, `attributes`, `branding`, `audio`. Objets de format : `json` `{ type: "json", prompt?, schema? }` (nécessite `prompt` ou `schema`), `changeTracking` `{ type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag? }`, `screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`, `attributes` `{ type: "attributes", selectors: [{ selector, attribute }] }`, `query` `{ type: "query", prompt }`, plus `{ type: "<format>" }` pour n’importe quel format sous forme de chaîne.
* `headers` : En-têtes HTTP supplémentaires. À utiliser pour l’authentification, les cookies ou le user-agent.
* `includeTags` : Inclure uniquement le contenu de ces balises HTML. À utiliser pour affiner l’extraction.
* `excludeTags` : Exclure le contenu de ces balises HTML. À utiliser pour supprimer le bruit.
* `onlyMainContent` : Extraire uniquement le contenu principal. À utiliser pour exclure la navigation et le pied de page.
* `timeout` : Délai d’expiration du scrape en millisecondes. À utiliser pour limiter le temps d’exécution.
* `waitFor` : Délai avant le scraping en millisecondes. À utiliser pour les pages rendues en JS.
* `mobile` : Émuler un appareil mobile. À utiliser pour les affichages mobiles.
* `parsers` : Liste ou configurations de parseurs. Configuration PDF prise en charge : `{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }` ou entrées sous forme de chaîne (par ex. : `"pdf"`).
* `actions` : actions du navigateur à exécuter avant le scraping. actions prises en charge : `wait` `{ type: "wait", milliseconds? | selector? }`, `screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`, `click` `{ type: "click", selector }`, `write` `{ type: "write", text }`, `press` `{ type: "press", key }`, `scroll` `{ type: "scroll", direction: "up" | "down", selector? }`, `scrape` `{ type: "scrape" }`, `executeJavascript` `{ type: "executeJavascript", script }`, `pdf` `{ type: "pdf", format?, landscape?, scale? }` avec les formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location` : Configuration de localisation `{ country, languages }`. À utiliser pour les pages spécifiques à une zone géographique.
* `skipTlsVerification` : Ignorer la vérification du certificat TLS. À utiliser pour les certificats auto-signés.
* `removeBase64Images` : Supprimer les images base64 de la sortie markdown. À utiliser pour réduire la taille du payload.
* `fastMode` : Activer le mode rapide. À utiliser lorsque la vitesse est prioritaire.
* `useMock` : Utiliser une fixture mock nommée (interne/test).
* `blockAds` : Activer le blocage des publicités et des cookies. À utiliser pour réduire le bruit.
* `proxy` : Mode proxy `"basic"`, `"stealth"`, `"enhanced"`, `"auto"`, ou une chaîne de proxy personnalisée. À utiliser pour les sites avec protection anti-bot.
* `maxAge` : Accepter du contenu en cache plus récent que cet âge (ms). À utiliser pour privilégier les accès au cache.
* `minAge` : Retourner uniquement du contenu en cache datant d’au moins cet âge (ms). À utiliser pour les lectures depuis le cache uniquement.
* `storeInCache` : Stocker le résultat dans le cache/l’index. À utiliser si vous souhaitez le réutiliser.
* `profile` : Profil Browser persistant `{ name, saveChanges }`. À utiliser pour conserver l’état de la session.
* `integration` : Identifiant d’intégration. À utiliser pour le suivi/les analyses.
* `origin` : Balise d’origine facultative. À utiliser pour l’attribution de la request.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `interact` lorsque la page nécessite des clics, la saisie de formulaires ou d’autres actions du navigateur après un scrape.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`client.interact(jobId, args)`

<div id="simple-example">
  ### Exemple simple
</div>

```ts
const run = await client.interact("scrape-job-id", {
  code: "console.log(await page.url())"
});

console.log(run.stdout);
```

<div id="complex-example">
  ### Exemple complexe
</div>

```ts
const run = await client.interact("scrape-job-id", {
  prompt: "Open the pricing page and list the plan names.",
  language: "node",
  timeout: 60,
  origin: "agent-docs"
});

console.log(run.output);
```

<div id="parameters">
  ### Paramètres
</div>

* `jobId`: ID de la tâche de scraping (issu de `document.metadata.scrapeId`). Toujours requis.
* `code`: Code à exécuter dans le sandbox du navigateur. Utilisez-le pour les interactions scriptées.
* `prompt`: Instruction en langage naturel pour l&#39;agent du navigateur. Utilisez-la si vous préférez des actions pilotées par l&#39;agent.
* `language`: Langage d&#39;exécution `"python"`, `"node"` ou `"bash"` (par défaut `"node"`). Utilisez-le pour l&#39;adapter à votre code.
* `timeout`: Délai d&#39;exécution maximal, en secondes. Utilisez-le pour limiter les exécutions trop longues.
* `origin`: Tag d&#39;origine facultatif. Utilisez-le pour l&#39;attribution des requêtes.

<div id="notes">
  ## Remarques
</div>

* Les paramètres sont en camelCase en JS/TS.
* Alias obsolètes : `scrapeExecute`, `stopInteractiveBrowser`, `deleteScrapeBrowser` (utilisez `interact` et `stopInteraction`).
* `Firecrawl` est le client v2 unifié ; la v1 reste disponible sous `.v1` pour assurer la compatibilité avec l’existant.

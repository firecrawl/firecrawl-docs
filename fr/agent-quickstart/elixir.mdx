<div id="firecrawl-elixir-agent-quickstart">
  # Démarrage rapide de l’agent Elixir Firecrawl
</div>

Ce fichier est le guide de démarrage rapide de référence pour les agents externes. Il est généré à partir du code source du SDK et de la spécification OpenAPI v2.

<div id="install">
  ## Installation
</div>

Ajoutez la dépendance à `mix.exs` :

```elixir
def deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

Ensuite, exécutez :

```bash
mix deps.get
```

<div id="authenticate">
  ## S&#39;authentifier
</div>

```elixir
# config/runtime.exs or config/config.exs
config :firecrawl, api_key: "fc-your-api-key"
```

Ou le fournir à chaque appel :

```elixir
Firecrawl.scrape_and_extract_from_url([url: "https://example.com"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## Quand utiliser quoi
</div>

* `search` : à utiliser lorsque vous partez d’une requête et devez découvrir des résultats.
* `scrape` : à utiliser lorsque vous avez déjà une URL et voulez le contenu de la page.
* `interact` : à utiliser lorsque la page nécessite des clics, des formulaires ou des actions du navigateur après le scraping.

<div id="search">
  ## Recherche
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `search` pour trouver des URL pertinentes à partir d’une requête et, si besoin, scraper chaque résultat. Vous pouvez restreindre les requêtes à un seul site avec l’opérateur standard `site:` (par exemple, `site:docs.firecrawl.dev crawl webhooks`) avant de scraper une URL spécifique.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  limit: 5
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "firecrawl pricing",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  enterprise: ["anon"],
  scrape_options: [
    formats: [
      "markdown",
      [type: "json", prompt: "Extract pricing tiers and limits."]
    ],
    only_main_content: true,
    proxy: :enhanced
  ],
  timeout: 60000
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Paramètres
</div>

* `query` : Chaîne de requête de recherche. Toujours requise.
* `sources` : Sources dans lesquelles effectuer la recherche. Valeurs prises en charge : `"web"`, `"news"`, `"images"` ou des objets au format `[%{type: "web" | "news" | "images"}]`. À utiliser pour contrôler quels tableaux de résultats sont renvoyés.
* `categories` : Filtres de résultats. Valeurs prises en charge : `"github"`, `"research"`, `"pdf"` ou des objets au format `[%{type: "github" | "research" | "pdf"}]`. À utiliser pour restreindre les résultats par catégorie.
* `limit` : Nombre maximal de résultats à renvoyer. À utiliser pour limiter la taille de la réponse.
* `tbs` : Chaîne de filtre de recherche temporel. À utiliser pour limiter la recherche à une plage de temps (par ex. `qdr:d`).
* `location` : Chaîne de localisation pour des résultats localisés. À utiliser pour les requêtes géociblées.
* `country` : Code pays ISO pour des résultats de recherche géociblés. À utiliser avec `location`.
* `timeout` : Délai d’expiration de la requête en millisecondes. À utiliser pour limiter le temps de recherche.
* `ignore_invalid_urls` : Exclut les URL invalides des résultats. À utiliser lors du chaînage vers d’autres points de terminaison.
* `enterprise` : Options de recherche Enterprise. Valeurs prises en charge : `"anon"` (ZDR anonymisé) ou `"zdr"` (ZDR de bout en bout).
* `scrape_options` : Options de scrape appliquées à chaque résultat (mêmes clés que `scrape`). À utiliser lorsque vous souhaitez récupérer le contenu avec les résultats.

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `scrape` si vous connaissez déjà l’URL et souhaitez récupérer le contenu dans des formats spécifiques.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`Firecrawl.scrape_and_extract_from_url(params, opts)`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com",
  formats: ["markdown"]
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/docs",
  formats: [
    "markdown",
    [type: "json", prompt: "Extract headings and slugs."],
    [type: "changeTracking", modes: ["git-diff"], tag: "docs"],
    [type: "screenshot", full_page: true, viewport: [width: 1280, height: 720]]
  ],
  actions: [
    [type: "click", selector: "button#accept"],
    [type: "scroll", direction: "down"]
  ],
  parsers: [[type: "pdf", mode: "auto", max_pages: 5]],
  location: [country: "US", languages: ["en-US"]],
  proxy: :enhanced,
  profile: [name: "docs-session", save_changes: true],
  max_age: 3_600_000,
  min_age: 0,
  store_in_cache: true,
  zero_data_retention: false
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Paramètres
</div>

* `url` : URL à scraper. Toujours requis.
* `formats` : Formats de sortie à inclure. Chaînes de format prises en charge : `markdown`, `summary`, `html`, `rawHtml`, `links`, `images`, `screenshot`, `json`, `changeTracking`, `branding`, `audio`. Objets de format : `json` `[type: "json", prompt?, schema?]`, `changeTracking` `[type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?]`, `screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`.
* `headers` : En-têtes HTTP supplémentaires. À utiliser pour l’authentification, les cookies ou le user-agent.
* `include_tags` : Inclure uniquement le contenu de ces balises HTML. À utiliser pour cibler l’extraction.
* `exclude_tags` : Exclure le contenu de ces balises HTML. À utiliser pour supprimer le bruit.
* `only_main_content` : Extraire uniquement le contenu principal. À utiliser pour exclure la navigation et le pied de page.
* `timeout` : Délai d’expiration du scrape en millisecondes. À utiliser pour limiter la durée d’exécution.
* `wait_for` : Délai avant le scraping en millisecondes. À utiliser pour les pages rendues en JS.
* `mobile` : Émuler un appareil mobile. À utiliser pour les mises en page mobiles.
* `parsers` : Liste de parseurs ou de configurations. Configuration PDF prise en charge : `[type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: number]` ou entrées sous forme de chaîne (par exemple, `"pdf"`).
* `actions` : actions du Browser à exécuter avant le scraping. actions prises en charge : `wait` `[type: "wait", milliseconds? | selector?]`, `screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`, `click` `[type: "click", selector, all?]`, `write` `[type: "write", text]`, `press` `[type: "press", key]`, `scroll` `[type: "scroll", direction: "up" | "down", selector?]`, `scrape` `[type: "scrape"]`, `executeJavascript` `[type: "executeJavascript", script]`, `pdf` `[type: "pdf", format?, landscape?, scale?]` avec les formats `A0`-`A6`, `Letter`, `Legal`, `Tabloid`, `Ledger`.
* `location` : Configuration de localisation `[country, languages]`. À utiliser pour les pages propres à une zone géographique.
* `skip_tls_verification` : Ignorer la vérification du certificat TLS. À utiliser pour les certificats auto-signés.
* `remove_base64_images` : Supprimer les images base64 de la sortie markdown. À utiliser pour réduire la taille du payload.
* `block_ads` : Activer le blocage des publicités et des cookies. À utiliser pour réduire le bruit.
* `proxy` : Mode proxy `:basic`, `:enhanced`, `:auto`.
* `max_age` : Accepter le contenu en cache plus récent que cet âge (ms). À utiliser pour privilégier les résultats du cache.
* `min_age` : Retourner uniquement le contenu en cache ayant au moins cet âge (ms). À utiliser pour des lectures depuis le cache uniquement.
* `store_in_cache` : Stocker le résultat dans le cache/l’index. À utiliser lorsque vous souhaitez le réutiliser.
* `profile` : Profil Browser persistant `[name, save_changes]`. À utiliser pour conserver l’état de la session.
* `zero_data_retention` : Activer la zero data retention pour le scrape. À utiliser lorsque ZDR est activé pour votre équipe.

<div id="interact">
  ## Interact
</div>

<div id="why-use-it">
  ### Pourquoi l’utiliser
</div>

Utilisez `interact` lorsque la page nécessite des clics, le remplissage de formulaires ou d’autres actions dans le Browser après une extraction.

<div id="preferred-sdk-method">
  ### Méthode SDK recommandée
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

<div id="simple-example">
  ### Exemple simple
</div>

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.url())"
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### Exemple complexe
</div>

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.title())",
  language: :node,
  timeout: 60,
  origin: "agent-docs"
)

IO.inspect(response.body)
```

<div id="parameters">
  ### Paramètres
</div>

* `job_id` : ID de tâche de scraping (issu de `metadata.scrapeId`). Toujours obligatoire.
* `code` : Code à exécuter dans le sandbox du Browser. Obligatoire.
* `language` : Langage d’exécution `:python`, `:node` ou `:bash` (par défaut `:node`). À choisir en fonction de votre code.
* `timeout` : Délai d’exécution maximal en secondes. À utiliser pour limiter les exécutions trop longues.
* `origin` : Balise d’origine facultative. À utiliser pour l’attribution des requêtes.

<div id="notes">
  ## Remarques
</div>

* Les paramètres sont transmis sous forme de listes de mots-clés en snake&#95;case, validées par `NimbleOptions`.
* Les fonctions sont générées automatiquement à partir de la spécification OpenAPI ; leurs noms suivent les identifiants d’opération OpenAPI.
* Chaque fonction possède une variante bang qui déclenche une exception en cas d’erreur (par exemple, `search_and_scrape!`).
* Le SDK Elixir expose uniquement `interact` sous forme de code (sans paramètre `prompt`).
* Toutes les fonctions acceptent les options `api_key` et `base_url` pour des redéfinitions par requête.

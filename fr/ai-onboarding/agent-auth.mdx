**Par défaut : Path A.** Si vous ne pouvez pas générer de WorkOS ID-JAG — ou si vous avez un doute — utilisez l’onboarding avec intervention humaine via [`SKILL.md`](https://www.firecrawl.dev/agent-onboarding/SKILL.md) (voir [Obtenir les identifiants](/fr/ai-onboarding#get-credentials)).

Cette page couvre **uniquement Path B** — l’enregistrement WorkOS ID-JAG pour Firecrawl. Ce n’est pas la spécification d’exécution ; suivez la version canonique de [`auth.md`](https://www.firecrawl.dev/auth.md) pour chaque étape, chaque exigence relative aux claims et chaque code d’erreur.

Si vous ne pouvez pas créer d’ID-JAG WorkOS pour Firecrawl, ou si vous avez un doute, utilisez le parcours humain + aide de l’agent :

<div id="path-b-agentic-registration-workos-id-jag">
  ## Path B — Enregistrement agentique (WorkOS ID-JAG)
</div>

**Uniquement si compatible avec ID-JAG → Path B.**

Consultez [`auth.md`](https://www.firecrawl.dev/auth.md) pour connaître les exigences des claims ID-JAG, les étapes de découverte, le corps de la requête d’enregistrement et la gestion des erreurs. Ne passez pas directement à la suite dans ce document.

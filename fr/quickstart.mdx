<div id="setup-your-development">
  ## Configurez votre environnement de développement
</div>

Découvrez comment mettre à jour votre documentation en local et la déployer publiquement.

<div id="edit-and-preview">
  ### Modifier et prévisualiser
</div>

<AccordionGroup>
  <Accordion icon="github" title="Cloner votre documentation en local">
    Au cours de l’onboarding, nous avons créé un dépôt sur votre GitHub avec
    le contenu de votre documentation. Vous pouvez trouver ce dépôt sur notre
    [tableau de bord](https://dashboard.mintlify.com). Pour cloner le dépôt
    en local, suivez ces
    [instructions](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)
    dans votre terminal.
  </Accordion>

  <Accordion icon="rectangle-terminal" title="Prévisualiser les modifications">
    La prévisualisation vous permet de vérifier que vos modifications s’affichent comme prévu. Nous avons créé une
    interface en ligne de commande pour afficher ces changements en local. 1. Installez le
    [Mintlify CLI](https://www.npmjs.com/package/mintlify) pour prévisualiser les
    modifications de la documentation en local avec cette commande : `npm i -g mintlify`
    2. Exécutez la commande suivante à la racine de votre documentation (là où
    se trouve `mint.json`) : `mintlify dev`
  </Accordion>
</AccordionGroup>

<div id="deploy-your-changes">
  ### Déployez vos modifications
</div>

<AccordionGroup>
  <Accordion icon="message-bot" title="Installez notre application GitHub">
    Notre application GitHub déploie automatiquement vos modifications sur votre site de documentation, vous évitant ainsi de gérer vous-même les déploiements. Vous trouverez le lien d’installation sur votre [tableau de bord](https://dashboard.mintlify.com). Une fois le bot correctement installé, une coche devrait apparaître à côté du hash du commit du dépôt.
  </Accordion>

  <Accordion icon="rocket" title="Poussez vos modifications">
    [Validez et poussez vos modifications vers
    Git](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push)
    pour qu’elles soient prises en compte sur votre site de documentation. Si après un push vous ne voyez pas que l’application GitHub a déployé vos modifications, vous pouvez également mettre à jour manuellement votre documentation via notre [tableau de bord](https://dashboard.mintlify.com).
  </Accordion>
</AccordionGroup>

</div>

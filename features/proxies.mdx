Firecrawl routes all requests through proxies to ensure reliability and access. You can control the proxy type and geographic location to handle sites with varying levels of complexity.

- Three proxy types: `basic`, `enhanced`, and `auto`
- Location-based routing with support for 25+ countries
- Automatic fallback to US proxies when no location is specified

## Proxy types

Firecrawl supports three proxy types, specified with the `proxy` parameter:

| Type | Description | Speed | Cost |
|------|-------------|-------|------|
| `basic` | Standard proxies for scraping most sites. Works for the majority of requests. | Fast | Standard scrape credits |
| `enhanced` | Enhanced proxies for complex sites that require higher reliability and privacy. [Learn more about Enhanced Mode](/features/enhanced-mode). | Slower | 5 credits per page |
| `auto` | Starts with `basic` and automatically retries with `enhanced` if the first attempt fails. Only charges enhanced pricing if the retry is needed. | Varies | Standard or 5 credits per page |

<Info>For detailed information on using enhanced proxies, including credit costs and retry strategies, see the [Enhanced Mode documentation](/features/enhanced-mode).</Info>

## Setting a proxy location

You can request a specific proxy location by setting the `location.country` parameter to an [ISO 3166-1 alpha-2](https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2) country code. For example, to use a Brazilian proxy, set `location.country` to `BR`.

For full details, see the [API reference for `location.country`](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location).

If you do not specify a location, Firecrawl automatically uses US proxies.

<Info>If you request a country where a proxy is not available, Firecrawl will use the closest available region (EU or US) and set the browser location to your requested country.</Info>

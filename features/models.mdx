Firecrawl Agent offers two models optimized for different extraction tasks. Pick a model based on task complexity and cost requirements.

- `spark-1-mini`: efficient and cost-effective, handles most extraction tasks (default)
- `spark-1-pro`: higher accuracy for complex research and multi-domain extraction

| Feature | Spark 1 Mini | Spark 1 Pro |
|---------|--------------|-------------|
| **Cost** | 60% cheaper | Standard |
| **Accuracy** | Standard | Higher |
| **Speed** | Fast | Fast |
| **Reasoning** | Standard | Advanced |
| **Multi-domain** | Good | Excellent |
| **Best for** | Most tasks | Complex tasks |

<Tip>
Start with Spark 1 Mini (default). It handles most extraction tasks well at 60% lower cost. Switch to Pro only for complex multi-domain research or when accuracy is critical.
</Tip>

## Spark 1 Mini (default)

`spark-1-mini` is the default model, optimized for cost-efficient extraction on straightforward tasks.

**Use Mini when you are:**
- Extracting simple data points (contact info, pricing, metadata)
- Working with well-structured websites
- Running high-volume extraction jobs
- Prioritizing cost efficiency

**Example use cases:** product prices from e-commerce sites, contact information from company pages, basic metadata from articles, simple data point lookups.

## Spark 1 Pro

`spark-1-pro` is the flagship model, designed for maximum accuracy on complex extraction tasks.

**Use Pro when you are:**
- Performing complex competitive analysis across multiple domains
- Extracting data that requires deep reasoning
- Dealing with ambiguous or hard-to-find data
- Running tasks where accuracy is critical

**Example use cases:** multi-domain competitive analysis, complex research tasks requiring reasoning, nuanced information extraction from multiple sources, critical business intelligence gathering.

## Specifying a model

Pass the `model` parameter to select which model to use:

<CodeGroup>

```python Python
from firecrawl import Firecrawl

firecrawl = Firecrawl(api_key="fc-YOUR_API_KEY")

# Using Spark 1 Mini (default, can be omitted)
result = firecrawl.agent(
    prompt="Find the pricing of Firecrawl",
    model="spark-1-mini"
)

# Using Spark 1 Pro for complex tasks
result = firecrawl.agent(
    prompt="Compare all enterprise features and pricing across Firecrawl, Apify, and ScrapingBee",
    model="spark-1-pro"
)

print(result.data)
```

```js Node

const resultPro = await firecrawl.agent({
```

## Pricing

Both models use dynamic, credit-based pricing that scales with task complexity.

| Model | Credit usage |
|-------|-------------|
| `spark-1-mini` | ~60% fewer credits than Pro for equivalent tasks |
| `spark-1-pro` | Standard credit consumption for maximum accuracy |

<Info>
Credit usage varies based on prompt complexity, data processed, and output structure, regardless of model selected.
</Info>

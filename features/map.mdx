The `/map` endpoint takes a single URL and returns all discoverable URLs on the website. URLs are primarily discovered from the website's sitemap, supplemented with search engine results and previously crawled pages to improve coverage.

- Prompt users to choose which links to scrape
- Quickly discover all pages on a website
- Find pages related to a specific topic using the `search` parameter
- Target specific sections of a site before scraping

## Search

Use the `search` parameter to filter and rank URLs by relevance to a query. Results are ordered from most relevant to least relevant.

<CodeGroup>

```python Python
from firecrawl import Firecrawl

firecrawl = Firecrawl(api_key="fc-YOUR-API-KEY")

result = firecrawl.map(
    url="https://firecrawl.dev",
    search="docs"
)

print(result)
```

```js Node
import Firecrawl from '@mendable/firecrawl-js';

const firecrawl = new Firecrawl({ apiKey: "fc-YOUR-API-KEY" });

const result = await firecrawl.map('https://firecrawl.dev', {
  search: 'docs'
});

console.log(result);
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v2/map \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -d '{
    "url": "https://firecrawl.dev",
    "search": "docs"
  }'
```

</CodeGroup>

## Location and language

Specify a country and preferred languages to get location-relevant results. When you set the `location` object, Firecrawl uses an appropriate proxy and emulates the corresponding language and timezone settings. The default location is `US`.

<CodeGroup>
<MapLocationPython />
<MapLocationNode />
<MapLocationCURL />
</CodeGroup>

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `country` | `string` | `"US"` | ISO 3166-1 alpha-2 country code (e.g., `US`, `AU`, `DE`, `JP`) |
| `languages` | `string[]` | (location default) | Preferred languages and locales in order of priority |

For more details about supported locations, refer to the [Proxies documentation](/features/proxies).

## Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `url` | `string` | (required) | The base URL to map |
| `search` | `string` | (none) | Search query to filter and rank URLs by relevance |
| `limit` | `integer` | `5000` | Maximum number of URLs to return (max `100000`) |
| `sitemap` | `string` | `"include"` | Sitemap handling: `"include"` (sitemap + other methods), `"skip"` (ignore sitemap), `"only"` (only sitemap URLs) |
| `includeSubdomains` | `boolean` | `true` | Include subdomains of the website |
| `ignoreQueryParameters` | `boolean` | `true` | Exclude URLs that contain query parameters |
| `ignoreCache` | `boolean` | `false` | Bypass the sitemap cache (cached for up to 7 days) to retrieve fresh URLs |
| `timeout` | `integer` | (none) | Timeout in milliseconds |
| `location` | `object` | (none) | Location and language settings (see [Location and language](#location-and-language)) |

## Considerations

The `/map` endpoint prioritizes speed, so it may not capture every URL on a website. It primarily relies on the website's sitemap, supplemented by cached crawl data and search engine results. For a more thorough and up-to-date list of URLs, consider using the [/crawl](/features/crawl) endpoint instead.

Firecrawl caches previously scraped pages and returns a recent copy when one is available. Set the `maxAge` parameter to control how fresh a cached result must be, and get results up to 500% faster when you don't need real-time content.

- Works with `/scrape`, `/crawl`, and `/batch/scrape`
- Returns cached results in milliseconds instead of seconds
- Defaults to a 2-day freshness window (`172800000` ms)

## How it works

Every scrape stores the result in a cache keyed by URL. On subsequent requests, Firecrawl checks whether a cached copy exists that is newer than your `maxAge` value. If it does, the cached copy is returned instantly. If not, Firecrawl scrapes fresh, updates the cache, and returns the new result.

| Scenario | What happens |
|----------|-------------|
| Cached copy is newer than `maxAge` | Cached result returned instantly |
| Cached copy is older than `maxAge` | Fresh scrape performed, cache updated |
| No cached copy exists | Fresh scrape performed, result cached |
| `maxAge` set to `0` | Cache bypassed, always scrapes fresh |

## Basic usage

Add `maxAge` to your scrape request. The value is in milliseconds (for example, `3600000` = 1 hour).

<CodeGroup>

```python Python
from firecrawl import Firecrawl

firecrawl = Firecrawl(api_key="fc-YOUR-API-KEY")

result = firecrawl.scrape(
    "https://example.com/pricing",
    formats=["markdown"],
    max_age=3600000  # 1 hour in milliseconds
)

print(result.markdown)
```

```js Node
import Firecrawl from '@mendable/firecrawl-js';

const firecrawl = new Firecrawl({ apiKey: "fc-YOUR-API-KEY" });

const result = await firecrawl.scrape('https://example.com/pricing', {
  formats: ['markdown'],
  maxAge: 3600000 // 1 hour in milliseconds
});

console.log(result.markdown);
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v2/scrape \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR-API-KEY" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": ["markdown"],
    "maxAge": 3600000
  }'
```

</CodeGroup>

## When to use caching

**Good candidates:**
- Documentation, articles, and product pages
- Bulk processing jobs where content doesn't change between runs
- Development and testing workflows
- Building knowledge bases from relatively static content

**Skip caching for:**
- Real-time data (stock prices, live scores, breaking news)
- Frequently updated content where freshness is critical
- Time-sensitive applications

## Common maxAge values

| Duration | Value (ms) | Use case |
|----------|-----------|----------|
| 5 minutes | `300000` | Semi-dynamic content |
| 1 hour | `3600000` | Content that updates hourly |
| 1 day | `86400000` | Daily-updated content |
| 2 days | `172800000` | Default value |
| 1 week | `604800000` | Relatively static content |

## Force a fresh scrape

Set `maxAge: 0` to bypass the cache entirely and always scrape fresh. This sends every request through the full scraping pipeline, so requests will take longer and are more likely to fail. Use a non-zero `maxAge` if you don't need real-time content on every request.

<CodeGroup>

```python Python
result = firecrawl.scrape(
    "https://example.com/pricing",
    formats=["markdown"],
    max_age=0  # Always scrape fresh
)
```

```js Node
const result = await firecrawl.scrape('https://example.com/pricing', {
  formats: ['markdown'],
  maxAge: 0 // Always scrape fresh
});
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v2/scrape \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR-API-KEY" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": ["markdown"],
    "maxAge": 0
  }'
```

</CodeGroup>

## Skip storing in cache

Set `storeInCache: false` if you don't want to store the result for a request. This scrapes the page without updating the cache, so future requests will still use the previously cached version.

<CodeGroup>

```python Python
result = firecrawl.scrape(
    "https://example.com/pricing",
    formats=["markdown"],
    store_in_cache=False
)
```

```js Node
const result = await firecrawl.scrape('https://example.com/pricing', {
  formats: ['markdown'],
  storeInCache: false
});
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v2/scrape \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR-API-KEY" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": ["markdown"],
    "storeInCache": false
  }'
```

## Crawl with caching

The same caching benefits apply when crawling multiple pages. Pass `maxAge` inside `scrapeOptions` to return cached results for pages that have been scraped recently.

<CodeGroup>

```python Python
from firecrawl import Firecrawl
from firecrawl.v2.types import ScrapeOptions

firecrawl = Firecrawl(api_key="fc-YOUR-API-KEY")

result = firecrawl.crawl(
    "https://example.com",
    limit=100,
    scrape_options=ScrapeOptions(
        formats=["markdown"],
        max_age=3600000  # Use cached data if less than 1 hour old
    )
)

for page in result.data:
    print(f"URL: {page.metadata.source_url}")
    print(f"Content: {page.markdown[:200]}...")
```

```js Node
import Firecrawl from '@mendable/firecrawl-js';

const firecrawl = new Firecrawl({ apiKey: "fc-YOUR-API-KEY" });

const result = await firecrawl.crawl('https://example.com', {
  limit: 100,
  scrapeOptions: {
    formats: ['markdown'],
    maxAge: 3600000 // Use cached data if less than 1 hour old
  }
});

result.data.forEach(page => {
  console.log(`URL: ${page.metadata.sourceURL}`);
  console.log(`Content: ${page.markdown.substring(0, 200)}...`);
});
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v2/crawl \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer fc-YOUR-API-KEY" \
  -d '{
    "url": "https://example.com",
    "limit": 100,
    "scrapeOptions": {
      "formats": ["markdown"],
      "maxAge": 3600000
    }
  }'
```

</CodeGroup>

Each page in the crawl benefits from cached results individually. Pages with a recent cache entry return instantly, while pages without one are scraped fresh.

## Configuration reference

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `maxAge` | `number` | `172800000` (2 days) | Maximum age of cached content in milliseconds. Set to `0` to always scrape fresh. |
| `storeInCache` | `boolean` | `true` | Whether to store the scrape result in the cache for future requests. |

## Important details

- **Credits**: Cached results still cost 1 credit per page. Caching improves speed and latency, not credit usage.
- **Freshness guarantee**: You will never receive data older than your specified `maxAge`. If no cached copy meets the freshness requirement, Firecrawl scrapes fresh automatically.
- **Cache scope**: The cache is shared across your team. If a teammate recently scraped the same URL, your request can benefit from their cached result.

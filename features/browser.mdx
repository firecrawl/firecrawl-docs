Launch scalable cloud browser sessions and execute Python, JavaScript, and bash remotely.

Firecrawl Browser Sandbox gives your agents a secure, fully managed browser environment. No local setup, no Chromium installs, no driver compatibility issues. Each session runs in an isolated, disposable sandbox that scales from one browser to hundreds without managing infrastructure.

Browser works alongside Firecrawl's [scrape](/features/scrape) and [search](/features/search) endpoints. Use scrape for fast, single-page extraction. Use Browser when your workflow needs navigation, form fills, multi-step flows, or interactive content that scraping can't reach.

Available via API, CLI, SDK, and [Vercel AI SDK](/developer-guides/llm-sdks-and-frameworks/vercel-ai-sdk).

## When to Use Browser

| Use Case | Right Tool |
|----------|-----------|
| Extract content from a known URL | [Scrape](/features/scrape) |
| Search the web and get results | [Search](/features/search) |
| Navigate pagination, fill forms, click through flows | **Browser** |
| Multi-step workflows with interaction | **Browser** |
| Parallel browsing across many sites | **Browser** (each session is isolated) |

## Use Cases

- **Competitive intelligence** - Browse competitor sites, navigate search forms and filters, extract pricing and features into structured data
- **Knowledge base ingestion** - Navigate help centers, docs, and support portals that require clicks, pagination, or authentication
- **Market research** - Launch parallel browser sessions to build datasets from job boards, real estate listings, or legal databases

To execute JavaScript instead, pass `language: "js"`:

```bash cURL
curl -X POST "https://api.firecrawl.dev/v2/browser/execute" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "YOUR_SESSION_ID",
    "code": "await page.goto(\"https://example.com\"); const title = await page.title(); console.log(title);",
    "language": "js"
  }'
```

## agent-browser

[agent-browser](https://github.com/vercel-labs/agent-browser) is a headless browser CLI with 40+ commands -it's pre-installed in the sandbox. Use it with the `--bash` flag and the CLI auto-injects `--cdp` so it connects to your active session.

<CodeGroup>

```bash Navigate & Snapshot
firecrawl browser execute --bash "agent-browser open https://example.com"
firecrawl browser execute --bash "agent-browser snapshot"
firecrawl browser execute --bash "agent-browser screenshot"
```

```bash Interact
firecrawl browser execute --bash "agent-browser click @e5"
firecrawl browser execute --bash "agent-browser fill @e3 'search query'"
firecrawl browser execute --bash "agent-browser scrape"
```

</CodeGroup>

### TTL Configuration

Sessions have two TTL controls:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `ttlTotal` | 300s (5 min) | Maximum session lifetime (30-3600s) |
| `ttlWithoutActivity` | - | Auto-close after inactivity (10-3600s) |

## Connecting via CDP

Every session exposes a CDP WebSocket URL. The execute API and `--bash` flag cover most use cases, but if you need full local control you can connect directly.

<CodeGroup>

<PlaywrightJS />
<PlaywrightPython />

```javascript Puppeteer
import puppeteer from "puppeteer-core";

const browser = await puppeteer.connect({
  browserWSEndpoint: "CDP_URL_FROM_SESSION",
});

const [page] = await browser.pages();
await page.goto("https://example.com");
console.log(await page.title());

await browser.close();
```

```bash agent-browser
agent-browser open https://example.com --cdp "CDP_URL_FROM_SESSION"
agent-browser snapshot --cdp "CDP_URL_FROM_SESSION"
agent-browser screenshot --cdp "CDP_URL_FROM_SESSION"
```

</CodeGroup>

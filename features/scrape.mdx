Firecrawl converts any web page into clean markdown, structured data, screenshots, or HTML. It handles the hard parts for you: proxies, caching, rate limits, JavaScript rendering, dynamic content, and PDFs.

For the full parameter list, see the [Scrape Endpoint API Reference](https://docs.firecrawl.dev/api-reference/endpoint/scrape).

## Output formats

You can request one or more output formats per scrape. The response keys match the formats you choose.

| Format | Key | Description |
|--------|-----|-------------|
| Markdown | `markdown` | Clean markdown representation of the page |
| Summary | `summary` | AI-generated summary of the page content |
| HTML | `html` | Cleaned version of the page's HTML |
| Raw HTML | `rawHtml` | Unmodified HTML as received from the page |
| Screenshot | `screenshot` | Page screenshot (supports `fullPage`, `quality`, `viewport` options). URLs expire after 24 hours |
| Links | `links` | All links found on the page |
| JSON | `json` | Structured data extraction (see [Extract structured data](#extract-structured-data)) |
| Images | `images` | All image URLs from the page |
| Branding | `branding` | Brand identity and design system (see [Extract brand identity](#extract-brand-identity)) |

### Branding profile structure

The `branding` object contains the following properties:

| Property | Description |
|----------|-------------|
| `colorScheme` | Detected color scheme (`"light"` or `"dark"`) |
| `logo` | URL of the primary logo |
| `colors` | Brand colors: `primary`, `secondary`, `accent`, `background`, `textPrimary`, `textSecondary`, `link`, `success`, `warning`, `error` |
| `fonts` | Array of font families used on the page |
| `typography` | `fontFamilies` (primary, heading, code), `fontSizes` (headings and body), `fontWeights` (light, regular, medium, bold), `lineHeights` (different text types) |
| `spacing` | `baseUnit` (base spacing in pixels), `borderRadius`, `padding`, `margins` |
| `components` | UI component styles: `buttonPrimary`, `buttonSecondary`, `input` |
| `icons` | Icon style information |
| `images` | Brand images (logo, favicon, og:image) |
| `animations` | Animation and transition settings |
| `layout` | Layout configuration (grid, header/footer heights) |
| `personality` | Brand personality traits (tone, energy, target audience) |

## Location and language

Specify a country and preferred languages to get location-relevant content. When you provide location settings, Firecrawl uses an appropriate proxy and emulates the corresponding language and timezone.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `country` | `string` | `"US"` | ISO 3166-1 alpha-2 country code (e.g., `"US"`, `"AU"`, `"DE"`, `"JP"`) |
| `languages` | `string[]` | Auto | Preferred languages and locales in order of priority. Defaults to the language of the specified country |

## Caching and maxAge

Firecrawl serves results from cache by default when a recent copy is available, which can speed up scrapes by up to 5x.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `maxAge` | `number` | `172800000` | Cache freshness window in milliseconds (default is 2 days). Set to `0` to always fetch fresh content |
| `storeInCache` | `boolean` | `true` | Set to `false` to prevent Firecrawl from caching results for this request |

<Note>
Setting `maxAge` to `0` bypasses the cache entirely. Every request goes through the full scraping pipeline, which takes longer and is more likely to fail. Use a non-zero `maxAge` if freshness on every request is not critical.
</Note>

Requests that include `changeTracking` bypass the cache, so `maxAge` is ignored.

### Force fresh content

<CodeGroup>

```python Python
from firecrawl import Firecrawl
firecrawl = Firecrawl(api_key='fc-YOUR_API_KEY')

doc = firecrawl.scrape(url='https://example.com', max_age=0, formats=['markdown'])
print(doc)
```

```js Node
import Firecrawl from '@mendable/firecrawl-js';

const firecrawl = new Firecrawl({ apiKey: "fc-YOUR-API-KEY" });

const doc = await firecrawl.scrape('https://example.com', { maxAge: 0, formats: ['markdown'] });
console.log(doc);
```

```bash cURL
curl -s -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com",
    "maxAge": 0,
    "formats": ["markdown"]
  }'
```
</CodeGroup>

## Batch scraping multiple URLs

Batch scraping lets you scrape multiple URLs in a single job. You submit a list of URLs and receive a job ID to check status, similar to the `/crawl` endpoint.

The SDK provides two methods: synchronous (waits for results) and asynchronous (returns a job ID for polling).

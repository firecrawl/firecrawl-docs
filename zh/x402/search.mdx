<Info>
  **按次付费的搜索端点**

  通过此端点，你可以按次使用加密货币支付，在无需订阅的情况下使用 Firecrawl 的强大搜索能力。非常适合一次性搜索，或在订阅前进行测试。
</Info>

<Warning>
  **实验性功能**

  这是一个实验性端点。功能、定价和可用性可能会在不经通知的情况下变更。我们建议在生产环境中使用标准的订阅式搜索端点。
</Warning>

<div id="authentication">
  ## 身份验证
</div>

此端点需要使用特殊的支付请求头，而非标准的 API 密钥：

```
X-Payment: {{paymentHeader}}
```

<div id="how-it-works">
  ## 工作原理
</div>

On-Demand Search 端点将网页搜索（SERP）与 Firecrawl 的抓取能力相结合，为任意查询返回完整页面内容，和常规的 `/search` 端点相同，但采用按次计费。

在请求中包含 `scrapeOptions`，并设置 `formats: ["markdown"]`，即可为每个搜索结果返回完整的 Markdown 内容；否则将默认仅返回 SERP 结果（url、title、description）。

<div id="request-parameters">
  ### 请求参数
</div>

标准 `/search` 端点中的所有参数均受支持，但有以下例外：

<Note>
  **重要限制**：对于钱包端点，`limit` 参数最多为 **10 个结果**（标准端点则为 100）。
</Note>

有关可用参数、支持的查询运算符及其说明的完整列表，请参阅[标准 /search 端点文档](/zh/api-reference/endpoint/search)。

<div id="example-request">
  ## 示例请求
</div>

```bash
curl -X POST https://api.firecrawl.dev/v1/x402/search \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <token>' \
  -H 'X-Payment: {{paymentHeader}}' \
  -d '{
    "query": "2025/26 赛季英超赛程安排",
    "limit": 10,  # 最多返回 10 个结果
    "scrapeOptions": {
      "formats": ["markdown"],
      "onlyMainContent": true
    }
  }'
```

<div id="response">
  ## 响应
</div>

响应格式与标准的 `/search` 端点完全相同：

```json
{
  "success": true,
  "data": [
    {
      "title": "页面标题",
      "description": "来自搜索结果的页面说明",
      "url": "https://example.com",
      "markdown": "# 页面内容\n\n如果请求了抓取，这里将返回 Markdown 内容...",
      "metadata": {
        "title": "页面标题",
        "description": "Meta 描述"
        "sourceURL": "https://example.com",
        "statusCode": 200
      }
    }
  ]
}
```

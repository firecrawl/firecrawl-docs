**默认使用路径 A。** 如果你无法签发 WorkOS ID-JAG，或不确定是否可以，请使用带人工参与的 [`SKILL.md`](https://www.firecrawl.dev/agent-onboarding/SKILL.md) 引导流程 (请参见[获取凭证](/zh/ai-onboarding#get-credentials)) 。

本页**仅介绍路径 B**——适用于 Firecrawl 的 WorkOS ID-JAG 注册。它不是执行规范；具体每一步、声明要求和错误代码，请遵循权威的 [`auth.md`](https://www.firecrawl.dev/auth.md)。

<div id="path-a-human-agent-assist-default">
  ## 路径 A — 人工 + 代理协助 (默认)
</div>

**没有 ID-JAG 或不确定 → 路径 A (默认) 。**

如果你无法为 Firecrawl 创建 WorkOS ID-JAG，或者不确定，请使用人工 + 代理协助路径：

<Card title="路径 A — 人工 + 代理协助（默认）" icon="user" href="https://www.firecrawl.dev/agent-onboarding/SKILL.md">
  浏览器登录、CLI 安装、技能、MCP 连接配置，以及 Dashboard API 密钥。
</Card>

<div id="path-b-agentic-registration-workos-id-jag">
  ## 路径 B — 代理注册 (WorkOS ID-JAG)
</div>

**仅在具备 ID-JAG 能力时 → 路径 B。**

请查阅 [`auth.md`](https://www.firecrawl.dev/auth.md)，了解 ID-JAG 声明要求、发现步骤、注册请求体和错误处理。不要在该文档中跳过前面的内容。

<Card title="权威规范 — auth.md" icon="file-lines" href="https://www.firecrawl.dev/auth.md">
  面向具备身份感知能力的代理的机器注册说明。请按顺序完成步骤 1–4。
</Card>

如需了解支持的方法、限制和错误，请参见权威 [`auth.md`](https://www.firecrawl.dev/auth.md) 规范。

<div id="discovery">
  ## 发现
</div>

“发现”步骤位于 [`auth.md`](https://www.firecrawl.dev/auth.md) 的第 1 步。如果你已经有 [`auth.md`](https://www.firecrawl.dev/auth.md) 的直接链接，直接获取即可——无需查看此页面。

<div id="what-happens-next">
  ## 接下来该怎么做
</div>

按照 [`auth.md`](https://www.firecrawl.dev/auth.md) 中的步骤 1–4 进行操作：发现 → 在 `POST /agent/auth` 注册 → 使用返回的 API 密钥作为访问 `https://api.firecrawl.dev` 的 Bearer Token。

<div id="welcome-to-firecrawl">
  ## 欢迎使用 Firecrawl
</div>

[Firecrawl](https://firecrawl.dev?ref=github) 是一项 API 服务，可接收 URL、进行爬取，并将内容转换为干净的 Markdown。我们会爬取所有可访问的子页面，并为每个页面提供干净的 Markdown。无需提供 sitemap。

<div id="how-to-use-it">
  ## 如何使用？
</div>

我们在托管版本中提供易用的 API。你可以在[此处](https://firecrawl.dev/playground)找到 Playground 和文档。你也可以选择自托管后端。

查看以下资源以开始：

* **API**： [文档](https://docs.firecrawl.dev/api-reference/introduction)
* **SDK**： [Python](https://docs.firecrawl.dev/sdks/python)、[Node](https://docs.firecrawl.dev/sdks/node)
* **LLM 框架**： [LangChain（Python）](https://python.langchain.com/docs/integrations/document_loaders/firecrawl/)、[LangChain（JS）](https://js.langchain.com/docs/integrations/document_loaders/web_loaders/firecrawl)、[LlamaIndex](https://docs.llamaindex.ai/en/latest/examples/data_connectors/WebPageDemo/#using-firecrawl-reader)、[Crew.ai](https://docs.crewai.com/)、[Composio](https://composio.dev/tools/firecrawl/all)、[PraisonAI](https://docs.praison.ai/firecrawl/)、[Superinterface](https://superinterface.ai/docs/assistants/functions/firecrawl)、[Vectorize](https://docs.vectorize.io/integrations/source-connectors/firecrawl)
* **低代码框架**： [Dify](https://dify.ai/blog/dify-ai-blog-integrated-with-firecrawl)、[Langflow](https://docs.langflow.org/)、[Flowise AI](https://docs.flowiseai.com/integrations/langchain/document-loaders/firecrawl)、[Cargo](https://docs.getcargo.io/integration/firecrawl)、[Pipedream](https://pipedream.com/apps/firecrawl/)
* **社区 SDK**： [Go](https://docs.firecrawl.dev/sdks/go)、[Rust](https://docs.firecrawl.dev/sdks/rust)（v1）
* **其他**： [Zapier](https://zapier.com/apps/firecrawl/integrations)、[Pabbly Connect](https://www.pabbly.com/connect/integrations/firecrawl/)
* **自托管：** 请参考[此处](/zh/contributing/self-host)的指南。

想要新的 SDK 或集成？欢迎在 GitHub 提交[issue](https://github.com/firecrawl/firecrawl/issues)告诉我们。

<div id="features">
  ### 功能
</div>

* [**Scrape**](#scraping): 抓取一个 URL，并以适用于 LLM 的格式获取其内容（Markdown、摘要、通过 [JSON 模式](#json-mode) 提供的结构化数据、截图、HTML）
* [**Crawl**](#crawling): 抓取网页上的所有 URL，并以适用于 LLM 的格式返回内容
* [**Map**](/zh/features/map): 输入一个网站，极速获取该网站的所有 URL
* [**Search**](/zh/features/search): 搜索全网，并获取结果的完整内容
* [**Extract**](/zh/features/extract): 使用 AI 从单页、多页或整站提取结构化数据。

<div id="powerful-capabilities">
  ### 强大功能
</div>

* **适配 LLM 的 formats**：markdown、summary、structured data、screenshot、HTML、links、metadata、images
* **解决棘手问题**：proxies、anti-bot 机制、动态内容（JS 渲染）、输出解析、编排
* **极速**：数秒内返回结果——为高速度与高吞吐场景而生。
* **高度可定制**：排除标签、通过自定义 headers 爬取登录后页面、设置最大爬取深度等……
* **媒体解析**：PDF、DOCX、图像。
* **可靠性优先**：旨在获取所需数据——再难也能拿到。
* **actions**：在提取数据前执行 click、scroll、input、wait 等操作

你可以在我们的[文档](https://docs.firecrawl.dev/api-reference/v2-introduction)中查看 Firecrawl 的全部功能及使用方式

<div id="open-source-vs-cloud">
  ## 开源 vs 云端
</div>

Firecrawl 是一个开源项目，采用 [AGPL-3.0 许可证](https://github.com/mendableai/firecrawl/blob/main/LICENSE)。

为提供最佳产品体验，我们在开源版本的同时提供托管版 Firecrawl。云端方案使我们能够持续创新，并为所有用户提供高质量、可持续的服务。

Firecrawl Cloud 可在 [firecrawl.dev](https://firecrawl.dev) 使用，并提供一系列开源版本中不具备的功能：

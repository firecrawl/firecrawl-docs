<div id="base-url">
  ## 基础 URL
</div>

所有请求都使用以下基础 URL：

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## 身份验证
</div>

进行身份验证时，请在请求中包含 `Authorization` 请求头。该请求头的值应为 `Bearer fc_123456789`，其中 `fc_123456789` 表示你的 API 密钥（API Key）。

```bash
Authorization: Bearer fc_123456789
```

​

<div id="response-codes">
  ## 响应状态码
</div>

Firecrawl 使用常规的 HTTP 状态码来表示请求结果。

通常，2xx HTTP 状态码表示成功，4xx 表示与用户相关的错误，5xx 表示服务器或基础设施问题。

Status | Description
--- | ---
200 | 请求成功。
400 | 请检查参数是否正确。
401 | 未提供 API key。
402 | 需要付款。
404 | 找不到请求的资源。
429 | 已超出速率限制。
5xx | 表示 Firecrawl 服务器错误。

有关所有可能的 API 错误的详细说明，请参阅 Error Codes 部分。

<div id="rate-limit">
  ## 速率限制
</div>

Firecrawl API 采用速率限制机制，以确保服务的稳定性与可靠性。该限制适用于所有端点，并根据在特定时间窗口内发起的请求数量进行计算。

当你超出速率限制时，将收到 429 响应码。

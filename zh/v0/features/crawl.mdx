Firecrawl 会深度爬取网站，在规避各类反爬/拦截机制的同时确保全面的数据提取。其工作方式如下：

1. **URL 分析：**
   从指定的 URL 开始，优先根据站点地图识别链接并爬取网站；若未找到站点地图，则沿页面内的链接继续爬取。

2. **递归遍历：**
   递归跟进每个链接，覆盖所有子页面。

3. **内容抓取：**
   从每个访问的页面收集内容，并处理 JavaScript 渲染、速率限制等复杂情况。

4. **结果汇总：**
   将采集的数据转换为干净的 markdown 或结构化输出，非常适合用于 LLM 处理或其他任务。

这种方法可确保从任意起始 URL 进行全面的爬取与数据收集。

<div id="crawl-endpoint">
  ### /crawl 端点
</div>

用于抓取某个 URL 及其所有可访问的子页面。此操作会提交一个爬取任务，并返回用于检查爬取状态的任务 ID。

<div id="usage">
  ### 用法
</div>

<CodeGroup>

```python Python
from firecrawl import FirecrawlApp

app = FirecrawlApp(api_key="YOUR_API_KEY")

crawl_result = app.crawl_url('mendable.ai', {'crawlerOptions': {'excludes': ['blog/*']}})

# 获取 Markdown
for result in crawl_result:
    print(result['markdown'])
```

```js JavaScript
import FirecrawlApp from '@mendable/firecrawl-js';

// 使用你的 API 密钥初始化 FirecrawlApp
const app = new FirecrawlApp({ apiKey: 'YOUR_API_KEY' });

// 爬取网站
const crawlResult = await app.crawlUrl('mendable.ai', { crawlerOptions: { excludes: ['blog/*'] } });

// 输出 Markdown
console.log(crawlResult.map(result => result.markdown));
```

```go Go
import (
  "fmt"
  "log"

  "github.com/mendableai/firecrawl-go"
)

func main() {
  app, err := firecrawl.NewFirecrawlApp("YOUR_API_KEY")
  if err != nil {
    log.Fatalf("Failed to initialize FirecrawlApp: %v", err)
  }

  crawlResult, err := app.CrawlURL("mendable.ai", nil)
  if err != nil {
    log.Fatalf("Failed to crawl URL: %v", err)
  }

  fmt.Println(crawlResult)
}
```

```rust Rust
use firecrawl::FirecrawlApp;

#[tokio::main]
async fn main() {
  // 使用 API 密钥初始化 FirecrawlApp
  let api_key = "YOUR_API_KEY";
  let api_url = "https://api.firecrawl.dev";
  let app = FirecrawlApp::new(api_key, api_url).expect("Failed to initialize FirecrawlApp");

  // 爬取该 URL
  let crawl_params = json!({
    "crawlerOptions": {
        "excludes": ["blog/*"]
    }
  });

  let crawl_result = app
      .crawl_url("https://example.com", Some(crawl_params), true, 2, None)
      .await;

  // 打印爬取结果
  match crawl_result {
      Ok(data) => println!("Crawl Result:\n{}", data),
      Err(e) => eprintln!("Crawl failed: {}", e),
  }
}
```

```bash cURL
curl -X POST https://api.firecrawl.dev/v0/crawl \
    -H 'Content-Type: application/json' \
    -H 'Authorization: Bearer YOUR_API_KEY' \
    -d '{
      "url": "https://mendable.ai"
    }'
```

<div id="job-id-response">
  ### 作业 ID 响应
</div>

如果你不使用 SDK，或更倾向于通过 webhook 或其他轮询方式，你可以将 `wait_until_done` 设置为 `false`。
这会返回一个 jobId。

对于 cURL，/crawl 始终会返回一个 jobId，你可以用它来检查抓取任务的状态。

```json
{ "jobId": "1234-5678-9101" }
```

<div id="check-crawl-job">
  ### 检查爬取任务
</div>

用于查看爬取任务的状态并获取其结果。

<CodeGroup>

```python Python
status = app.check_crawl_status(job_id)
```

```js JavaScript
const status = await app.checkCrawlStatus(jobId);
```

```go Go
status, err := app.CheckCrawlStatus(jobId)
if err != nil {
  log.Fatalf("Failed to check crawl status: %v", err)
}
```

```rust Rust
let status = match app.check_crawl_status(jobId).await {
    Ok(status) => status,
    Err(e) => panic!("Failed to check crawl status: {:?}", e),
};

println!("Crawl Status: {:?}", status);
```

```bash cURL
curl -X GET https://api.firecrawl.dev/v0/crawl/status/1234-5678-9101 \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer YOUR_API_KEY'
```

<div id="firecrawl-javascripttypescript-agent-quickstart">
  # Firecrawl JavaScript/TypeScript 代理快速入门
</div>

本文件是面向外部代理的权威快速入门指南，由 SDK 源代码和 v2 OpenAPI 规范生成。

<div id="install">
  ## 安装
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="authenticate">
  ## 身份验证
</div>

```ts
import { Firecrawl } from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // 可选，适用于自托管部署
});
```

<div id="when-to-use-what">
  ## 何时使用哪一个
</div>

* `search`：当你从查询词开始，需要发现相关页面时使用。
* `scrape`：当你已经有 URL，需要获取页面内容时使用。
* `interact`：当页面需要点击、填写表单或在抓取后执行浏览器 actions 时使用。

<div id="search">
  ## 搜索
</div>

### 为什么使用它

使用 `search` 可根据查询词找到相关 URL，并且还可以选择抓取每条结果。抓取特定 URL 之前，你可以使用标准的 `site:` 运算符将查询限定在单个站点内 (例如 `site:docs.firecrawl.dev crawl webhooks`) 。

### 推荐的 SDK 方法

`client.search(query, options)`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const results = await client.search(
  "site:docs.firecrawl.dev crawl webhooks",
  { limit: 5 }
);

console.log(results.web?.[0]);
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const results = await client.search("firecrawl pricing", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  scrapeOptions: {
    formats: [
      "markdown",
      { type: "json", prompt: "Extract pricing tiers and limits." }
    ],
    onlyMainContent: true,
    actions: [
      { type: "wait", milliseconds: 1000 },
      { type: "scroll", direction: "down" }
    ],
    proxy: "stealth"
  },
  integration: "agent-quickstart",
  origin: "agent-docs"
});
```

<div id="parameters">
  ### 参数
</div>

* `query`：搜索查询字符串。始终必填。
* `sources`：要搜索的来源。支持的值：`"web"`、`"news"`、`"images"`，或 `{ type: "web" | "news" | "images" }`。用于控制返回哪些结果数组。
* `categories`：结果筛选条件。支持的值：`"github"`、`"research"`、`"pdf"`，或 `{ type: "github" | "research" | "pdf" }`。用于按类别限制结果。
* `limit`：返回结果的最大数量。用于限制响应大小。
* `tbs`：基于时间的搜索筛选字符串。用于按时间范围限制结果 (例如 `qdr:d`) 。
* `location`：用于本地化结果的位置字符串。用于发起地理位置相关查询。
* `ignoreInvalidURLs`：从结果中排除无效 URL。适用于将结果传递给其他端点时。
* `timeout`：请求超时时间 (毫秒) 。用于限制搜索时长。
* `scrapeOptions`：应用于每个结果的抓取选项 (与 `scrape` 使用相同字段) 。适用于希望结果中包含内容的场景。
* `integration`：集成标识符。用于跟踪/分析。
* `origin`：可选的来源标签。用于请求归因。

<div id="scrape">
  ## 抓取
</div>

### 为什么使用它

当你已经知道 URL，并且希望以特定 formats 获取内容时，请使用 `scrape`。

### 推荐的 SDK 方法

`client.scrape(url, options)`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const doc = await client.scrape("https://example.com", {
  formats: ["markdown"]
});

console.log(doc.markdown);
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const doc = await client.scrape("https://example.com/docs", {
  formats: [
    "markdown",
    { type: "json", prompt: "Extract headings and their slugs." },
    { type: "changeTracking", modes: ["git-diff"], tag: "docs" },
    { type: "screenshot", fullPage: true, viewport: { width: 1280, height: 720 } },
    {
      type: "attributes",
      selectors: [{ selector: "a", attribute: "href" }]
    },
    { type: "query", prompt: "What is the pricing model?" }
  ],
  onlyMainContent: true,
  waitFor: 1500,
  actions: [
    { type: "click", selector: "button#accept" },
    { type: "scroll", direction: "down" }
  ],
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  location: { country: "US", languages: ["en-US"] },
  proxy: "stealth",
  profile: { name: "docs-session", saveChanges: true },
  maxAge: 3600000
});
```

<div id="parameters">
  ### 参数
</div>

* `url`: 要抓取的 URL。始终为必填项。
* `formats`: 要包含的输出格式。支持的格式字符串：`markdown`、`html`、`rawHtml`、`links`、`images`、`screenshot`、`summary`、`changeTracking`、`json`、`attributes`、`branding`、`audio`。格式对象：`json` `{ type: "json", prompt?, schema? }` (需要 `prompt` 或 `schema`) 、`changeTracking` `{ type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag? }`、`screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`、`attributes` `{ type: "attributes", selectors: [{ selector, attribute }] }`、`query` `{ type: "query", prompt }`，以及适用于任意字符串格式的 `{ type: "<format>" }`。
* `headers`: 附加的 HTTP 请求头。用于身份验证、Cookie 或 User-Agent。
* `includeTags`: 仅包含这些 HTML 标签中的内容。用于缩小提取范围。
* `excludeTags`: 排除这些 HTML 标签中的内容。用于去除噪声。
* `onlyMainContent`: 仅提取主要内容。用于去掉导航栏和页脚。
* `timeout`: 抓取超时时间 (毫秒) 。用于限制运行时长。
* `waitFor`: 抓取前的延迟时间 (毫秒) 。用于 JS 渲染页面。
* `mobile`: 模拟移动设备。用于移动端布局。
* `parsers`: 解析器列表或配置。支持的 PDF 配置：`{ type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number }`，或字符串条目 (例如 `"pdf"`) 。
* `actions`: 抓取前要运行的浏览器 actions。支持的 actions：`wait` `{ type: "wait", milliseconds? | selector? }`、`screenshot` `{ type: "screenshot", fullPage?, quality?, viewport? }`、`click` `{ type: "click", selector }`、`write` `{ type: "write", text }`、`press` `{ type: "press", key }`、`scroll` `{ type: "scroll", direction: "up" | "down", selector? }`、`scrape` `{ type: "scrape" }`、`executeJavascript` `{ type: "executeJavascript", script }`、`pdf` `{ type: "pdf", format?, landscape?, scale? }`，支持的格式有 `A0`-`A6`、`Letter`、`Legal`、`Tabloid`、`Ledger`。
* `location`: 地理位置配置 `{ country, languages }`。用于特定地区的页面。
* `skipTlsVerification`: 跳过 TLS 证书验证。用于自签名证书。
* `removeBase64Images`: 从 markdown 输出中移除 base64 图片。用于减小负载大小。
* `fastMode`: 启用快速模式。适用于速度优先的场景。
* `useMock`: 使用命名的 mock fixture (内部/测试) 。
* `blockAds`: 启用广告和 Cookie 封禁。用于减少噪声。
* `proxy`: 代理模式 `"basic"`、`"stealth"`、`"enhanced"`、`"auto"`，或自定义代理字符串。用于有反爬机制的网站。
* `maxAge`: 接受缓存中未超过此时长 (毫秒) 的内容。用于优先命中缓存。
* `minAge`: 仅返回缓存中至少已存在此时长 (毫秒) 的内容。用于仅从缓存读取。
* `storeInCache`: 将结果存储到缓存/索引中。适用于希望复用结果时。
* `profile`: 持久化浏览器配置 `{ name, saveChanges }`。用于保留会话状态。
* `integration`: 集成标识符。用于跟踪/分析。
* `origin`: 可选的来源标签。用于请求归因。

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么要使用它
</div>

如果页面在抓取后还需要点击、填写表单或执行其他浏览器操作，请使用 `interact`。

<div id="preferred-sdk-method">
  ### 推荐使用的 SDK 方法
</div>

`client.interact(jobId, args)`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const run = await client.interact("scrape-job-id", {
  code: "console.log(await page.url())"
});

console.log(run.stdout);
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const run = await client.interact("scrape-job-id", {
  prompt: "Open the pricing page and list the plan names.",
  language: "node",
  timeout: 60,
  origin: "agent-docs"
});

console.log(run.output);
```

<div id="parameters">
  ### 参数
</div>

* `jobId`：抓取任务 ID (来自 `document.metadata.scrapeId`) 。始终为必填项。
* `code`：在浏览器沙箱中执行的代码。用于脚本化交互。
* `prompt`：提供给浏览器代理的自然语言指令。适合在你希望由代理驱动操作时使用。
* `language`：运行时语言，可选 `"python"`、`"node"` 或 `"bash"` (默认值为 `"node"`) 。用于匹配你的代码。
* `timeout`：执行超时时间 (以秒为单位) 。用于限制长时间运行。
* `origin`：可选的来源标签。用于请求归因。

<div id="notes">
  ## 注意事项
</div>

* 在 JS/TS 中，参数使用 camelCase。
* 已弃用的别名：`scrapeExecute`、`stopInteractiveBrowser`、`deleteScrapeBrowser` (请使用 `interact` 和 `stopInteraction`) 。
* `Firecrawl` 是统一的 v2 客户端；v1 仍可通过 `.v1` 用于兼容旧版。

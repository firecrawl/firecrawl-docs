<div id="firecrawl-java-agent-quickstart">
  # Firecrawl Java 代理快速入门
</div>

本文件是面向外部代理的权威快速入门文档，根据 SDK 源代码和 v2 OpenAPI 规范生成。

<div id="install">
  ## 安装
</div>

```gradle
dependencies {
  implementation("com.firecrawl:firecrawl-java:1.2.0")
}
```

<div id="authenticate">
  ## 认证
</div>

```java
import com.firecrawl.client.FirecrawlClient;

FirecrawlClient client = FirecrawlClient.builder()
    .apiKey(System.getenv("FIRECRAWL_API_KEY"))
    .build();
```

<div id="when-to-use-what">
  ## 何时使用哪种
</div>

* `search`：当你从查询词开始，需要先发现相关结果时使用。
* `scrape`：当你已经有 URL，并且想获取页面内容时使用。
* `interact`：当页面需要点击、填写表单，或在抓取后执行浏览器 actions 时使用。

<div id="search">
  ## 搜索
</div>

### 为什么使用它

使用 `search` 可根据查询词找到相关 URL，并可选择抓取每个结果。你还可以先用标准 `site:` 操作符将查询范围限制在单个站点内 (例如 `site:docs.firecrawl.dev crawl webhooks`) ，然后再抓取特定 URL。

### 推荐的 SDK 方法

`client.search(query, options)`

<div id="simple-example">
  ### 简单示例
</div>

```java
import com.firecrawl.models.SearchData;

SearchData data = client.search("site:docs.firecrawl.dev crawl webhooks");
System.out.println(data.getWeb());
```

<div id="complex-example">
  ### 复杂示例
</div>

```java
import com.firecrawl.models.SearchData;
import com.firecrawl.models.SearchOptions;
import com.firecrawl.models.ScrapeOptions;

SearchData data = client.search(
    "firecrawl pricing",
    SearchOptions.builder()
        .sources(java.util.List.of("web", "news"))
        .categories(java.util.List.of("research"))
        .limit(10)
        .tbs("qdr:w")
        .location("San Francisco,California,United States")
        .ignoreInvalidURLs(true)
        .scrapeOptions(
            ScrapeOptions.builder()
                .formats(java.util.List.of(
                    "markdown",
                    java.util.Map.of("type", "json", "prompt", "Extract pricing tiers and limits.")
                ))
                .onlyMainContent(true)
                .proxy("stealth")
                .build()
        )
        .integration("agent-quickstart")
        .build()
);
```

<div id="parameters">
  ### 参数
</div>

* `query`：搜索查询字符串。始终必填。
* `sources`：要搜索的数据来源。支持的值：`"web"`、`"news"`、`"images"`，或 `{type: "web" | "news" | "images"}` 映射。用于控制返回哪些结果数组。
* `categories`：结果筛选条件。支持的值：`"github"`、`"research"`、`"pdf"`，或 `{type: "github" | "research" | "pdf"}` 映射。用于按类别限制结果。
* `limit`：返回结果的最大数量。用于限制响应大小。
* `tbs`：基于时间的搜索筛选字符串。用于按时间范围限制结果 (例如 `qdr:d`) 。
* `location`：用于返回本地化结果的位置字符串。用于地理位置相关的查询。
* `ignoreInvalidURLs`：从结果中排除无效 URL。用于将结果传递给其他 endpoints 时。
* `timeout`：请求超时时间 (毫秒) 。用于限制搜索耗时。
* `scrapeOptions`：应用于每个结果的抓取选项 (与 `scrape` 的字段相同) 。用于在返回结果时同时获取内容。
* `integration`：集成标识符。用于跟踪/分析。

<div id="scrape">
  ## 抓取
</div>

<div id="why-use-it">
  ### 为什么要使用它
</div>

当你已经知道 URL，并且希望以特定 formats 获取内容时，请使用 `scrape`。

### 推荐的 SDK 方法

`client.scrape(url, options)`

<div id="simple-example">
  ### 简单示例
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com",
    ScrapeOptions.builder()
        .formats(java.util.List.of("markdown"))
        .build()
);

System.out.println(doc.getMarkdown());
```

<div id="complex-example">
  ### 复杂示例
</div>

```java
import com.firecrawl.models.Document;
import com.firecrawl.models.LocationConfig;
import com.firecrawl.models.ScrapeOptions;

Document doc = client.scrape(
    "https://example.com/docs",
    ScrapeOptions.builder()
        .formats(java.util.List.of(
            "markdown",
            java.util.Map.of("type", "json", "prompt", "Extract headings and slugs."),
            java.util.Map.of("type", "changeTracking", "modes", java.util.List.of("git-diff"))
        ))
        .onlyMainContent(true)
        .waitFor(1500)
        .actions(java.util.List.of(
            java.util.Map.of("type", "click", "selector", "button#accept"),
            java.util.Map.of("type", "scroll", "direction", "down")
        ))
        .parsers(java.util.List.of(java.util.Map.of("type", "pdf", "mode", "auto", "maxPages", 5)))
        .location(LocationConfig.builder()
            .country("US")
            .languages(java.util.List.of("en-US"))
            .build())
        .proxy("stealth")
        .maxAge(3_600_000L)
        .build()
);
```

<div id="parameters">
  ### 参数
</div>

* `url`：要抓取的 URL。始终为必填项。
* `formats`：要包含的输出格式。支持的格式字符串：`markdown`、`summary`、`html`、`rawHtml`、`links`、`images`、`screenshot`、`json`、`changeTracking`、`branding`、`audio`。格式对象：`json` `{type: "json", prompt?, schema?}`、`changeTracking` `{type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?}`、`screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`。
* `headers`：附加 HTTP 标头。可用于 auth、cookies 或 user-agent。
* `includeTags`：仅包含这些 HTML 标签中的内容。用于缩小提取范围。
* `excludeTags`：排除这些 HTML 标签中的内容。用于去除噪声。
* `onlyMainContent`：仅提取正文内容。用于去掉导航栏/页脚。
* `timeout`：抓取超时时间 (毫秒) 。用于限制运行时长。
* `waitFor`：抓取前的等待时间 (毫秒) 。用于 JS 渲染的页面。
* `mobile`：模拟移动设备。用于移动端布局。
* `parsers`：解析器列表或配置。支持的 PDF 配置：`{type: "pdf", mode?: "fast" | "auto" | "ocr", maxPages?: number}`，或字符串条目 (例如 `"pdf"`) 。
* `actions`：抓取前要执行的浏览器 actions。支持的 actions：`wait` `{type: "wait", milliseconds? | selector?}`、`screenshot` `{type: "screenshot", fullPage?, quality?, viewport?}`、`click` `{type: "click", selector, all?}`、`write` `{type: "write", text}`、`press` `{type: "press", key}`、`scroll` `{type: "scroll", direction: "up" | "down", selector?}`、`scrape` `{type: "scrape"}`、`executeJavascript` `{type: "executeJavascript", script}`、`pdf` `{type: "pdf", format?, landscape?, scale?}`，其中格式包括 `A0`-`A6`、`Letter`、`Legal`、`Tabloid`、`Ledger`。
* `location`：location 配置 (`country`、`languages`) 。用于特定地理区域的页面。
* `skipTlsVerification`：跳过 TLS 证书验证。用于自签名证书。
* `removeBase64Images`：从 markdown 输出中移除 base64 图片。用于减小 payload 大小。
* `blockAds`：启用广告和 cookie 封禁。用于减少噪声。
* `proxy`：代理模式 `"basic"`、`"stealth"`、`"enhanced"`、`"auto"`，或自定义代理 URL。
* `maxAge`：接受小于此时长 (毫秒) 的缓存内容。用于优先命中缓存。
* `storeInCache`：将结果存入缓存/索引。用于需要复用时。
* `integration`：集成标识符。用于跟踪/分析。

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么要使用它
</div>

当页面在抓取后还需要点击、填写表单或执行其他浏览器 actions 时，请使用 `interact`。

<div id="preferred-sdk-method">
  ### 推荐使用的 SDK 方法
</div>

`client.interact(jobId, code, language, timeout, origin)`

<div id="simple-example">
  ### 简单示例
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());"
);

System.out.println(exec.getStdout());
```

<div id="complex-example">
  ### 复杂示例
</div>

```java
import com.firecrawl.models.BrowserExecuteResponse;

BrowserExecuteResponse exec = client.interact(
    "scrape-job-id",
    "console.log(await page.title());",
    "node",
    60,
    "agent-docs"
);
```

<div id="parameters">
  ### 参数
</div>

* `jobId`: 抓取任务 ID (来自 `document.metadata.scrapeId`) 。始终必填。
* `code`: 在浏览器沙箱中执行的代码。用于脚本化交互。
* `language`: 运行时语言，可选 `"python"`、`"node"` 或 `"bash"` (默认值为 `"node"`) 。用于与你的代码保持一致。
* `timeout`: 执行超时时间 (秒) 。用于限制运行时间过长的任务。
* `origin`: 可选的来源标签。用于请求归因。

<div id="notes">
  ## 注意事项
</div>

* 在 Java 中，参数采用 camelCase。
* `ScrapeOptions` 和 `SearchOptions` 使用构建器模式。
* 已弃用的别名：`scrapeExecute` 和 `deleteScrapeBrowser` (请改用 `interact` 和 `stopInteractiveBrowser`) 。
* Java SDK 仅支持基于代码的 `interact` (不支持 `prompt` 参数) 。

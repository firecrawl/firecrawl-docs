<div id="firecrawl-elixir-agent-quickstart">
  # Firecrawl Elixir 代理快速入门
</div>

本文件是面向外部代理的标准快速入门文档，由 SDK 源代码和 v2 OpenAPI 规范自动生成。

<div id="install">
  ## 安装
</div>

在 `mix.exs` 中添加依赖：

```elixir
def deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

然后运行：

```bash
mix deps.get
```

<div id="authenticate">
  ## 认证
</div>

```elixir
# config/runtime.exs 或 config/config.exs
config :firecrawl, api_key: "fc-your-api-key"
```

或者在每次调用时传入：

```elixir
Firecrawl.scrape_and_extract_from_url([url: "https://example.com"], api_key: "fc-your-api-key")
```

<div id="when-to-use-what">
  ## 何时使用哪一种
</div>

* `search`：当你从查询词开始，需要先发现结果时使用。
* `scrape`：当你已经有 URL，并想获取页面内容时使用。
* `interact`：当页面需要点击、填写表单或在抓取后执行浏览器 actions 时使用。

<div id="search">
  ## 搜索
</div>

### 为何使用它

使用 `search` 可根据查询发现相关 URL，并可选择抓取每条结果。抓取特定 URL 前，你还可以使用标准的 `site:` 运算符将查询限定在单个站点内 (例如，`site:docs.firecrawl.dev crawl webhooks`) 。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### 基础示例
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  limit: 5
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, response} = Firecrawl.search_and_scrape(
  query: "firecrawl pricing",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:w",
  location: "San Francisco,California,United States",
  country: "US",
  ignore_invalid_urls: true,
  enterprise: ["anon"],
  scrape_options: [
    formats: [
      "markdown",
      [type: "json", prompt: "Extract pricing tiers and limits."]
    ],
    only_main_content: true,
    proxy: :enhanced
  ],
  timeout: 60000
)

IO.inspect(response.body)
```

<div id="parameters">
  ### 参数
</div>

* `query`：搜索查询字符串。始终必填。
* `sources`：要搜索的来源。支持的值：`"web"`、`"news"`、`"images"`，或 `[%{type: "web" | "news" | "images"}]` 形式的对象。用于控制返回哪些结果数组。
* `categories`：结果筛选条件。支持的值：`"github"`、`"research"`、`"pdf"`，或 `[%{type: "github" | "research" | "pdf"}]` 形式的对象。用于按类别限制结果。
* `limit`：返回结果的最大数量。用于限制响应大小。
* `tbs`：基于时间的搜索筛选字符串。用于按时间范围限制结果 (例如 `qdr:d`) 。
* `location`：用于本地化结果的位置字符串。用于地理位置相关查询。
* `country`：用于地理位置定向搜索结果的 ISO 国家/地区代码。与 `location` 搭配使用。
* `timeout`：请求超时时间 (毫秒) 。用于限制搜索时长。
* `ignore_invalid_urls`：从结果中排除无效 URL。在将结果传递给其他端点时使用。
* `enterprise`：企业级搜索选项。支持的值：`"anon"` (匿名化 ZDR) 或 `"zdr"` (端到端 ZDR) 。
* `scrape_options`：应用于每个结果的抓取选项 (与 `scrape` 使用相同的键) 。适用于希望结果中包含内容的场景。

<div id="scrape">
  ## 抓取
</div>

<div id="why-use-it">
  ### 为什么要使用它
</div>

当您已知 URL，并希望以特定格式获取内容时，请使用 `scrape`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`Firecrawl.scrape_and_extract_from_url(params, opts)`

### 简单示例

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com",
  formats: ["markdown"]
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, response} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/docs",
  formats: [
    "markdown",
    [type: "json", prompt: "Extract headings and slugs."],
    [type: "changeTracking", modes: ["git-diff"], tag: "docs"],
    [type: "screenshot", full_page: true, viewport: [width: 1280, height: 720]]
  ],
  actions: [
    [type: "click", selector: "button#accept"],
    [type: "scroll", direction: "down"]
  ],
  parsers: [[type: "pdf", mode: "auto", max_pages: 5]],
  location: [country: "US", languages: ["en-US"]],
  proxy: :enhanced,
  profile: [name: "docs-session", save_changes: true],
  max_age: 3_600_000,
  min_age: 0,
  store_in_cache: true,
  zero_data_retention: false
)

IO.inspect(response.body)
```

<div id="parameters">
  ### 参数
</div>

* `url`: 要抓取的 URL。始终为必填项。
* `formats`: 要包含的输出格式。支持的格式字符串：`markdown`、`summary`、`html`、`rawHtml`、`links`、`images`、`screenshot`、`json`、`changeTracking`、`branding`、`audio`。格式对象：`json` `[type: "json", prompt?, schema?]`、`changeTracking` `[type: "changeTracking", modes: ["git-diff" | "json"], schema?, prompt?, tag?]`、`screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`。
* `headers`: 额外的 HTTP 标头。可用于身份验证、cookie 或 user-agent。
* `include_tags`: 仅包含这些 HTML 标签中的内容。可用于缩小提取范围。
* `exclude_tags`: 排除这些 HTML 标签中的内容。可用于去除噪声。
* `only_main_content`: 仅提取主要内容。可用于去除导航栏/页脚。
* `timeout`: 抓取超时时间 (毫秒) 。可用于限制运行时间。
* `wait_for`: 抓取前的等待时间 (毫秒) 。可用于 JS 渲染的页面。
* `mobile`: 模拟移动设备。可用于移动端布局。
* `parsers`: 解析器列表或配置。支持的 PDF 配置：`[type: "pdf", mode?: "fast" | "auto" | "ocr", max_pages?: number]`，或字符串条目 (例如 `"pdf"`) 。
* `actions`: 抓取前要执行的浏览器 actions。支持的 actions：`wait` `[type: "wait", milliseconds? | selector?]`、`screenshot` `[type: "screenshot", full_page?, quality?, viewport?]`、`click` `[type: "click", selector, all?]`、`write` `[type: "write", text]`、`press` `[type: "press", key]`、`scroll` `[type: "scroll", direction: "up" | "down", selector?]`、`scrape` `[type: "scrape"]`、`executeJavascript` `[type: "executeJavascript", script]`、`pdf` `[type: "pdf", format?, landscape?, scale?]`，其中支持的格式包括 `A0`-`A6`、`Letter`、`Legal`、`Tabloid`、`Ledger`。
* `location`: 位置配置 `[country, languages]`。可用于特定地理位置的页面。
* `skip_tls_verification`: 跳过 TLS 证书验证。可用于自签名证书。
* `remove_base64_images`: 从 markdown 输出中移除 base64 图片。可用于减小 payload 大小。
* `block_ads`: 启用广告和 cookie 封禁。可用于减少噪声。
* `proxy`: 代理模式 `:basic`、`:enhanced`、`:auto`。
* `max_age`: 接受缓存中未超过此时长的内容 (毫秒) 。可用于优先命中缓存。
* `min_age`: 仅返回缓存中至少达到此时长的内容 (毫秒) 。可用于仅从缓存读取。
* `store_in_cache`: 将结果存入缓存/索引。适用于希望复用结果的场景。
* `profile`: 持久化浏览器配置文件 `[name, save_changes]`。可用于保留会话状态。
* `zero_data_retention`: 为本次抓取启用零数据留存。适用于你的团队已启用 ZDR 的情况。

<div id="interact">
  ## 交互
</div>

### 为什么要用它

当页面在抓取后还需要点击、填写表单或执行其他浏览器 actions 时，请使用 `interact`。

### 首选 SDK 方法

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

### 简单示例

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.url())"
)

IO.inspect(response.body)
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, response} = Firecrawl.interact_with_scrape_browser_session(
  "scrape-job-id",
  code: "console.log(await page.title())",
  language: :node,
  timeout: 60,
  origin: "agent-docs"
)

IO.inspect(response.body)
```

<div id="parameters">
  ### 参数
</div>

* `job_id`：抓取任务 ID (来自 `metadata.scrapeId`) 。始终必填。
* `code`：在浏览器沙箱中执行的代码。必填。
* `language`：运行时语言 `:python`、`:node` 或 `:bash` (默认值为 `:node`) 。请根据你的代码选择。
* `timeout`：执行超时时间，单位为秒。用于限制运行时间过长的任务。
* `origin`：可选的来源标签。用于标记请求来源。

<div id="notes">
  ## 注意事项
</div>

* 参数以 snake&#95;case 关键字列表形式传入，并由 `NimbleOptions` 验证。
* 函数根据 OpenAPI 规范自动生成；名称遵循 OpenAPI 的 operation ID。
* 每个函数都有一个 bang 变体，出错时会抛出异常 (例如 `search_and_scrape!`) 。
* Elixir SDK 仅提供基于代码的 `interact` (不包含 `prompt` 参数) 。
* 所有函数都接受 `api_key` 和 `base_url` 选项，用于对单次请求进行覆盖。

<div id="firecrawl-actions-overview">
  ## Firecrawl actions 概览
</div>

| 模块 | 用例 | 最佳适配 |
|--------|----------|----------|
| **抓取网站（Scrape a Website）** | 单页数据提取 | 快速获取数据 |
| **爬取网站（Crawl a Website）** | 多页面内容收集 | 全站抓取 |
| **抽取网站（Extract a Website）** | AI 驱动的结构化提取 | 复杂数据场景 |
| **搜索网站（Search a Website）** | SERP + 全量内容 | 研究自动化 |
| **网站映射（Map a Website）** | URL 发现 | SEO 分析 |

<div id="best-practices">
  ## 最佳实践
</div>

<CardGroup cols={2}>
  <Card title="性能" icon="gauge-high">
    - 单页使用 **Scrape**（最快）
    - 大型站点使用设置了限制的 **Crawl**
    - 合理安排任务，避免触发速率限制
    - 添加错误处理模块
  </Card>

  <Card title="成本优化" icon="piggy-bank">
    - 合理规划调度（每小时/每日/每周）
    - 使用筛选条件避免不必要的运行
    - 设置抓取限制以控制 API 使用量
    - 先在 Firecrawl Playground 中测试
  </Card>
</CardGroup>

<div id="make-vs-zapier-vs-n8n">
  ## Make vs Zapier vs n8n
</div>

| 功能 | Make | Zapier | n8n |
|---------|------|--------|-----|
| **设置** | 可视化构建器，云端 | 零代码，云端 | 自托管或云端 |
| **定价** | 按操作量计费 | 按任务计费 | 固定月费 |
| **集成** | 3,000+ 应用 | 8,000+ 应用 | 400+ 集成 |
| **复杂度** | 进阶工作流 | 简单工作流 | 复杂工作流 |
| **最佳适用** | 可视化自动化，中等复杂度 | 快速自动化 | 开发者可控 |
| **学习曲线** | 中等 | 简单 | 中等到高级 |

<Tip>
**专业提示：** Make 在可视化工作流设计和复杂自动化方面表现突出。非常适合需要比 Zapier 更强控制力、但又偏好可视化构建而非 n8n“代码优先”方式的团队。
</Tip>

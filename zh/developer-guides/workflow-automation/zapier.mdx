<div id="firecrawl-actions">
  ## Firecrawl actions
</div>

| 操作 | 使用场景 |
|--------|----------|
| **Scrape URL** | 快速获取单页数据 |
| **Crawl Website** | 跨多页面的全站抓取 |
| **Extract Structured Data** | 基于 AI 的自定义模式结构化提取 |
| **Search Web** | 结合搜索与抓取的研究自动化 |
| **Map Website** | SEO 分析与站点结构映射 |

<div id="quick-reference">
  ## 快速参考
</div>

<CardGroup cols={2}>
  <Card title="快速开始" icon="play">
    1. 前往 [firecrawl.dev](https://firecrawl.dev) 注册
    2. 获取你的 API 密钥
    3. 在 Zapier 中创建一个 Zap
    4. 使用你的 API 密钥连接 Firecrawl
    5. 选择工作流并启用
  </Card>

  <Card title="最佳实践" icon="lightbulb">
    - 单页场景使用 `/Scrape URL`（更快）
    - 合理设置计划（每小时/每日/每周）
    - 先在 Firecrawl Playground 中测试
    - 为抓取失败添加错误处理
    - 使用筛选条件避免不必要的运行
  </Card>
</CardGroup>

<div id="zapier-vs-n8n">
  ## Zapier 与 n8n 对比
</div>

| 功能 | Zapier | n8n |
|---------|--------|-----|
| **设置** | 零代码，云端部署 | 自托管或云端 |
| **定价** | 按任务计费 | 固定月费 |

| **最佳适用** | 快速自动化、非技术用户 | 自定义逻辑、开发者 |

<div id="browser-actions">
  ## 浏览器 actions
</div>

你可以在抓取之前使用 `actions` 参数执行浏览器交互操作。这使你可以：

* 等待元素加载
* 点击按钮或链接
* 填写表单字段
* 按下按键
* 滚动页面
* 截取页面截图
* 执行自定义 JavaScript
* 生成 PDF

支持的 actions 包括 `wait`、`click`、`write`、`press`、`scroll`、`screenshot`、`scrape`、`executeJavascript` 和 `pdf`。有关包含示例的详细文档，请参阅 [高级抓取指南](https://docs.firecrawl.dev/advanced-scraping-guide#actions-actions)。

<div id="persistent-profiles">
  ## 持久化配置文件
</div>

传递 `profile` 参数，以在抓取和 [交互](/zh/features/interact) 会话之间保留浏览器状态 (cookies、localStorage、sessions) 。使用相同 profile 名称的抓取任务会共享浏览器状态，这对于保持登录状态或保留偏好设置很有用。

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/dashboard",
    "profile": {
      "name": "my-session",
      "saveChanges": true
    }
  }'
```

| 参数                    | 类型      | 必填 | 默认值    | 描述                                                           |
| --------------------- | ------- | -- | ------ | ------------------------------------------------------------ |
| `profile.name`        | string  | 是  | —      | 配置文件名称 (1–128 个字符) 。使用相同名称的抓取会共享浏览器状态。                       |
| `profile.saveChanges` | boolean | 否  | `true` | 当为 `true` 时，交互会话结束后，浏览器状态会保存回该配置文件。设为 `false` 可加载现有数据而不写入更改。 |

<Note>
  同一时间只能有一个会话向某个配置文件保存数据。如果另一个会话已在保存，你将收到 `409` 错误。你仍然可以使用 `saveChanges: false` 打开同一个配置文件，或稍后重试。
</Note>

更多详情，请参见 [Interact — 持久化配置文件](/zh/features/interact#persistent-profiles)。

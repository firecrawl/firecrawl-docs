<div id="base-url">
  ## 基础 URL
</div>

所有请求都使用以下基础 URL：

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## 身份验证
</div>

进行身份验证时，需要包含 Authorization 标头。其值应为 `Bearer fc-123456789`，其中 `fc-123456789` 是你的 API Key。

```bash
Authorization：Bearer fc-123456789
```

​

<div id="response-codes">
  ## 响应状态码
</div>

Firecrawl 使用常见的 HTTP 状态码来表示请求结果。

通常，2xx 表示成功，4xx 表示由客户端引起的错误，5xx 表示基础设施问题。

Status | Description
--- | ---
200 | 请求成功。
400 | 请检查参数是否正确。
401 | 未提供 API 密钥。
402 | 需要付款。
404 | 找不到请求的资源。
429 | 已超出速率限制。
5xx | 表示 Firecrawl 服务器错误。

有关所有可能的 API 错误的详细说明，请参见“错误代码”部分。

<div id="rate-limit">
  ## 速率限制
</div>

Firecrawl API 设有速率限制，以确保服务的稳定性与可靠性。该限制适用于所有端点，依据在特定时间窗口内发起的请求次数进行计算。

当你超出速率限制时，将会收到 429 响应码。

<div id="introduction">
  ### 引言
</div>

要有效地定制我们的产品与服务，不仅要知道客户是谁，更要了解他们在做什么。在推进自助式增长模式时，往往会有大量客户涌入，而我们对他们所知甚少。过去，主动了解这些用户通常十分耗时，需要通过手动收集与分析数据才能获得可执行的洞见。

但借助 LLM 的能力，尤其是其先进的数据抽取能力，我们已将这一流程自动化。通过对客户数据进行 LLM 抽取与分析，我们大幅减轻了工作量，使我们能够比以往更高效地理解并服务我们的客户群。

即便你的技术背景有限，也可以搭建一套自动化流程，用于获取围绕产品方向和线索获取所需的客户关键信息。下面介绍如何使用 [Make](https://make.com/) 和 [Firecrawl](https://www.firecrawl.dev/) 自行完成这一操作。

***

<div id="overview-of-the-tools">
  ### 工具概览
</div>

**Firecrawl**

Firecrawl 是一个用于抓取、搜索和提取的数据平台。它让你从网页获取数据，并将其转换为适合 LLM 的 markdown 或结构化数据。

当我们需要了解客户信息时，可以使用 Firecrawl 的 LLM 提取功能，精确指定要从其网站获取的字段。

**Make.com（原 Integromat）**

Make 是一款自动化平台，允许用户在不具备深厚技术背景的情况下，通过创建自定义工作流连接各类应用和服务。它提供可视化界面，用户可通过拖拽元素来设计自动化流程。

我们可以用 Make 将包含用户数据的电子表格连接到 Firecrawl，仅需少量 JSON 即可完成提取。

## 设置自动化流程

要让我们的自动化跑起来，只需在 Make 中按照三个步骤进行。这里，我们将在该场景中选择三个应用：

Google Sheets - 获取区间值
HTTP - 发送使用 API 密钥的身份验证请求
Google Sheets - 更新一行

我们还要添加“忽略流程控制”的工具，以防出现错误，从而让自动化流程继续运行。

![](https://i.imgur.com/MdCWv30.png)

此自动化将使我们能够从电子表格提取一组链接，发送到 Firecrawl 进行数据提取，然后将所需信息回写到电子表格中。

让我们先配置第一个应用。我们的目标是导出所有 URL，便于将它们发送到 Firecrawl 进行抽取。以下是用于获取这些 URL 的配置：

![](https://i.imgur.com/WHa91kY.png)

**重要* - 我们要确保从第二行开始提取数据。如果包含表头，最终会发生错误。

太好了！既然已经完成配置，接下来准备设置我们的 HTTP 请求。为此，请访问 https://firecrawl.dev 注册并获取 API 密钥（可免费开始）。注册后，前往 https://firecrawl.dev/account 查看你的 API 密钥。

我们将使用 Firecrawl 的 /scrape 端点。该端点可从单个 URL 获取信息，将其转换为干净的 markdown，并据此提取所需数据。我会参照其文档中的 API 参考，在我们的“Make an HTTP request”步骤中填写所有必要参数。

现在在 Make 中，我根据 Firecrawl 的文档来配置这次 API 调用。我们将使用 POST 作为 HTTP 方法，并设置两个请求头。

```
请求头 1：
名称：Authorization
值：Bearer your-API-key

请求头 2：
名称：Content-Type
值：application/json
```

![](https://i.imgur.com/LJ8g142.png)
我们还要设置请求体和内容类型。这里我们将这样做：

```
主体类型：Raw
内容类型：JSON（application/json）
```

我们还会点击“是”以解析我们的响应。这样会自动将响应解析为 JSON。

我们要达成的目标主要体现在请求内容。以下是本用例将使用的请求内容：

```
{
  "url": "1. url(B)",

"pageOptions": {
    "onlyMainContent": true
  },
  "extractorOptions": {
    "mode": "llm-extraction",
    "extractionPrompt": "提取公司简介（用一句话说明公司做什么）、公司行业（如软件、服务、AI 等——应以几个关键词作为标签），以及其服务对象（他们的客户是谁）。如果没有明确信息可回答该问题，写 'no info'。",
    "extractionSchema": {
      "type": "object",
      "properties": {
        "company_description": {
          "type": "string"
        },
        "company_industry": {
          "type": "string"
        },
        "who_they_serve": {
          "type": "string"
        }
      },
      "required": [
        "company_description",
        "company_industry",
        "who_they_serve"
      ]
    }
  }
}
```

![](https://i.imgur.com/DrMc1g2.png)

**注意* 截图中的绿色字段是在 Make 界面中可选的动态项。该块可能不是 `url (B)`，而会成为你数据中的第一个 URL。

![](https://i.imgur.com/D4HCBNe.png)

太棒了！我们已配置好 HTTP 请求。现在来测试一下，确保一切按预期运行。在 Make 中点击“Run once”，就应该会返回数据。

![](https://i.imgur.com/QuQZs0U.png)

运行后，先检查第一个操作。在输出中，我们应该看到“status code: 200”，这表示我们的 API 请求已成功。在输出里点击 data，确认我们拿到了所需的数据。

![](https://i.imgur.com/pm614VA.png)

我们的输出看起来很成功！在 llm&#95;extraction 中，我们看到了从该网站提取的、我们需要的三项数据属性。

**注意* 如果你在第一次操作时收到 `500` 错误，而后续请求返回 `200`，这可能是因为操作尝试在你的数据第一行（表头）上执行。这样会导致稍后将数据导回表格时出现问题！如前所述，请务必从第二行开始。

现在我们已经确认 HTTP 请求运行正常，接下来只需把 Firecrawl 输出的 JSON 写回到我们的电子表格中。

***

现在我们需要把提取的数据写回电子表格。为此，我们将使用 HTTP 请求的输出 JSON，并把相应文本填入对应的表格字段。

我们先连接同一个 Google 表格，并指定 Row Number 条件。在这里我们直接使用 Make 的界面选择“row number”。

![](https://i.imgur.com/BYpPabk.png)

接下来只需指定由 LLM 提取的数据要写入哪些列。我们可以直接在 Make 的界面中完成设置。

![](https://i.imgur.com/219tft2.png)

就这样，现在可以测试我们的自动化了！

***

让我们在 Make 的界面上点击 `run once`，确认一切运行顺畅。自动化会按链接逐条迭代，并实时填充电子表格。

![](https://i.imgur.com/vU1CJlt.png)

成功！借助 Make 和 Firecrawl，我们无需逐一手动访问客户网站，就能提取出关于客户的特定信息。

从这些数据来看，我们对客户的理解正在加深。不过，我们不必局限于这些特定特征。如果需要，我们可以自定义 JSON 和提取提示（Extraction Prompt），以获取关于这些公司的更多信息。

<div id="going-a-step-further">
  ### 使用场景
</div>

LLM 抽取让我们能快速从网络获取与业务相关的关键信息，并用这些自动化完成多种任务。

**产品：**
尤其对自助式产品公司，我们可借助自家产品洞察行业趋势。有哪些排名前 2–3 的行业在使用我们的技术？他们具体用来做什么？这有助于我们聚焦优先客户，做出更好的产品决策。

**业务拓展：**
了解现有用户画像后，我们可寻找相似公司，让他们同样受益于我们的产品。通过类似的自动化流程，我们还能从潜在客户中提取正向信号，判断其与产品的契合度。

我们也可以用这些数据生成更个性化、更具针对性的外联邮件。

**市场调研:**
市场研究机构在进行二次研究上投入大量时间，尤其在细分领域。我们可以通过自动化从多源数据中抽取并组织信息，简化采集流程。该自动化可提升效率并随数据规模扩展，成为快速变化行业中进行战略决策的有力工具。

### 更进一步

这只是一个简单示例，展示了如何使用 LLM 配合静态表格从网站提取相关数据。你可以将其与注册流程动态连接，做得更智能。此外，还可以把它与其他工具打通，进一步提升效率。比如，用提取的内容生成更具个性化的获客文案。

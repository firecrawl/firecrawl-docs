<div id="scrape-endpoint">
  ## /scrape 端点
</div>

全新升级的 `/scrape` 端点经过重新设计，进一步提升了可靠性并简化了使用。新的 `/scrape` 请求体结构如下：

```json
{
  "url": "<字符串>",
  "formats": ["markdown", "html", "rawHtml", "links", "screenshot", "json"],
  "includeTags": ["<字符串>"],
  "excludeTags": ["<字符串>"],
  "headers": { "<键>": "<值>" },
  "waitFor": 123,
  "timeout": 123
}
```

<div id="formats">
  ### formats
</div>

现在你可以选择希望输出的 formats。你可以指定多个输出 formats。支持的 formats 有：

* Markdown (markdown)
* HTML (html)
* 原始 HTML（rawHtml，未经修改）
* 截图（screenshot 或 screenshot@fullPage）
* 链接（links）
* JSON（json）
  默认情况下，输出仅包含 markdown 格式。

<div id="details-on-the-new-request-body">
  ### 关于新请求体的详细说明
</div>

下表概述了 V1 中 `/scrape` 端点请求体参数的变更。

| Parameter | Change | Description |
| --------- | ------ | ----------- |
| `onlyIncludeTags` | 已移动并重命名 | 移至根级，并重命名为 `includeTags`。 |
| `removeTags` | 已移动并重命名 | 移至根级，并重命名为 `excludeTags`。 |
| `onlyMainContent`| 已移动 | 移至根级。默认值为 `true`。 |
| `waitFor`| 已移动 | 移至根级。 |
| `headers`| 已移动 | 移至根级。 |
| `parsePDF`| 已移动 | 移至根级。 |
| `extractorOptions`| 无变化 ||
| `timeout`| 无变化 ||
| `pageOptions` | 已移除 | 不再需要 `pageOptions` 参数。抓取选项已移至根级。 |
| `replaceAllPathsWithAbsolutePaths` | 已移除 | 不再需要 `replaceAllPathsWithAbsolutePaths`。现在每个路径默认都是绝对路径。 |
| `includeHtml`| 已移除 | 请改为在 `formats` 中添加 `"html"`。 |
| `includeRawHtml`| 已移除 | 请改为在 `formats` 中添加 `"rawHtml"`。 |
| `screenshot`| 已移除 | 请改为在 `formats` 中添加 `"screenshot"`。 |
| `fullPageScreenshot`| 已移除 | 请改为在 `formats` 中添加 `"screenshot@fullPage"`。 |
| `extractorOptions` | 已移除 | 请改用 `"json"` 格式，并配合 `jsonOptions` 对象。 |

新的 `json` 格式见 [llm-extract](/zh/features/extract) 部分。

<div id="crawl-endpoint">
  ## /crawl 端点
</div>

我们也对 `v1` 的 `/crawl` 端点进行了更新。请查看下方改进后的请求体：

```json
{
  "url": "<string>",
  "excludePaths": ["<string>"],
  "includePaths": ["<string>"],
  "maxDepth": 2,
  "ignoreSitemap": true,
  "limit": 10,
  "allowBackwardLinks": true,
  "allowExternalLinks": true,
  "scrapeOptions": {
    // 与 /scrape 端点中的选项相同
    "formats": ["markdown", "html", "rawHtml", "screenshot", "links"],
    "headers": { "<key>": "<value>" },
    "includeTags": ["<string>"],
    "excludeTags": ["<string>"],
    "onlyMainContent": true,
    "waitFor": 123
  }
}
```

### 新请求体的详细说明

下表概述了 V1 中 `/crawl` 端点请求体参数的变更。

| 参数 | 变更 | 说明 |
| --------- | ------ | ----------- |
| `pageOptions` | 重命名 | 重命名为 `scrapeOptions`。 |
| `includes` | 移动并重命名 | 移至根级，重命名为 `includePaths`。 |
| `excludes` | 移动并重命名 | 移至根级，重命名为 `excludePaths`。 |
| `allowBackwardCrawling` | 移动并重命名 | 移至根级，重命名为 `allowBackwardLinks`。 |
| `allowExternalLinks` | 已移动 | 移至根级。 |
| `maxDepth` | 已移动 | 移至根级。 |
| `ignoreSitemap` | 已移动 | 移至根级。 |
| `limit` | 已移动 | 移至根级。 |
| `crawlerOptions` | 已移除 | 不再需要 `crawlerOptions` 参数。爬取选项已移至根级。 |
| `timeout` | 已移除 | 请改用 `scrapeOptions` 中的 `timeout`。 |

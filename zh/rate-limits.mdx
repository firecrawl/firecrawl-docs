<div id="concurrent-browser-limits">
  ## 并发浏览器限制
</div>

并发浏览器指的是 Firecrawl 可为你同时处理的网页数量。
你的套餐决定可并行运行的作业数——如果超出该上限，
多余的作业将进入队列等待，直到资源可用。

<div id="standard-plans-most-api-endpoints">
  ### 标准方案（适用于大多数 API 端点）
</div>

| 方案       | 并发浏览器数 |
|------------|-----------------------------|
| Free       | 2                           |
| Hobby      | 5                           |
| Standard   | 50                          |
| Growth     | 100                         |

<div id="extract-plans-extract-api">
  ### 提取套餐（/extract API）
</div>

| 套餐       | 并发浏览器 |
|------------|-----------------------------|
| Free       | 2                           |
| Starter    | 50                          |
| Explorer   | 100                         |
| Pro        | 200                         |

<div id="standard-api">
  ## 标准 API
</div>

以下速率限制适用于标准 API 请求，主要用于防止滥用：

| 方案       | /scrape（次/分钟） | /map（次/分钟） | /crawl（次/分钟） | /search（次/分钟） |
|------------|--------------------|-----------------|-------------------|--------------------|
| Free       | 10                 | 10              | 1                 | 5                  |
| Hobby      | 100                | 100             | 15                | 50                 |
| Standard   | 500                | 500             | 50                | 250                |
| Growth     | 5000               | 5000            | 250               | 2500               |

|        | /crawl/status（次/分钟） |
|------------|------------------------|
| Default    | 1500                   |

上述速率限制用于确保所有用户对 API 的公平使用与可用性。若需更高限额，请通过 [help@firecrawl.com](mailto:help@firecrawl.com) 与我们联系，讨论自定义方案。

<div id="batch-endpoints">
  ### 批处理端点
</div>

批处理端点遵循 /crawl 端点的速率限制。

<div id="extract-api">
  ## Extract API
</div>

| 方案      | /extract（请求/分钟） |
|-----------|------------------------|
| Free      | 10                     |
| Starter     | 100                    |
| Explorer  | 500                    |
| Pro    | 1000                   |
| Enterprise | 自定义                  |

|        | /extract/status（请求/分钟） |
|--------|--------------------------------|
| Free | 500                       |

<div id="fire-1-agent">
  ## FIRE-1（代理）
</div>

涉及 FIRE-1（代理）的请求有单独的速率限制，且每个端点的限额独立计算：

| Endpoint   | Rate Limit (requests/min) |
|------------|---------------------------|
| `/scrape`  | 10                        |
| `/extract` | 10                        |

[Firecrawl MCP 服务器](/zh/mcp-server) (`https://mcp.firecrawl.dev/v2/mcp`) 是默认端点，并提供完整的 Firecrawl 工具集。

<div id="endpoint">
  ## 端点
</div>

```text
https://mcp.firecrawl.dev/v2/mcp-search
```

每个请求都需要身份验证：可直接通过该端点使用 OAuth 进行连接，或在 `Authorization` 请求头中传入 Firecrawl API 密钥。

```json
{
  "mcpServers": {
    "firecrawl-search": {
      "url": "https://mcp.firecrawl.dev/v2/mcp-search",
      "headers": {
        "Authorization": "Bearer fc-YOUR_API_KEY"
      }
    }
  }
}
```

<div id="available-tools">
  ## 可用工具
</div>

搜索专用端点仅提供以下六个只读工具：

| 工具                                  | 用途                     |
| ----------------------------------- | ---------------------- |
| `firecrawl_search`                  | 进行网页搜索并返回排序后的结果        |
| `firecrawl_research_search_papers`  | 搜索研究论文                 |
| `firecrawl_research_inspect_paper`  | 获取论文元数据                |
| `firecrawl_research_related_papers` | 通过引用链接查找相关论文           |
| `firecrawl_research_read_paper`     | 读取论文中的全文段落             |
| `firecrawl_research_search_github`  | 搜索公开仓库的历史记录和 README 文件 |

<div id="differences-from-the-full-mcp-endpoint">
  ## 与完整 MCP 端点 的区别
</div>

* 工具集是固定的，并且为只读。
* Scrape、map、crawl、extract、agent、interact、parse、monitor 和 feedback 工具不可用。
* `firecrawl_search` 不接受 `scrapeOptions`，也无法请求抓取页面内容。

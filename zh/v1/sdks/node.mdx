<div id="usage">
  ## 使用
</div>

1. 从 [firecrawl.dev](https://firecrawl.dev) 获取 API 密钥
2. 将该密钥设置为名为 `FIRECRAWL_API_KEY` 的环境变量，或作为参数传递给 `FirecrawlApp` 类。

以下是一个带有错误处理的 SDK 使用示例：

<ScrapeAndCrawlExampleNode />

<div id="scraping-a-url">
  ### 抓取单个 URL
</div>

若需在包含错误处理的情况下抓取单个 URL，请使用 `scrapeUrl` 方法。该方法接收 URL 作为参数，并返回字典形式的抓取结果。

<ScrapeNodeShort />

<div id="crawling-a-website">
  ### 爬取网站
</div>

要在具备错误处理的情况下爬取网站，请使用 `crawlUrl` 方法。它接受起始 URL 和可选参数。通过 `params` 参数，你可以为爬取任务指定更多选项，例如最大爬取页数、允许的域名，以及输出 formats。
 
<CrawlNodeShort />

<div id="asynchronous-crawling">
  ### 异步抓取
</div>

要异步抓取网站，请使用 `crawlUrlAsync` 方法。它会返回抓取的 `ID`，可用于检查抓取任务的状态。该方法接受起始 URL 和可选参数。通过 `params` 参数，你可以为抓取任务指定其他选项，例如最大抓取页数、允许的域名以及输出格式。

<CrawlAsyncNodeShort />

<div id="checking-crawl-status">
  ### 检查抓取状态
</div>

若需在包含错误处理的情况下检查抓取任务状态，请使用 `checkCrawlStatus` 方法。该方法接收 `ID` 作为参数，并返回该抓取任务的当前状态。

<CheckCrawlStatusNodeShort />

<div id="cancelling-a-crawl">
  ### 取消爬取
</div>

要取消异步爬取作业，请使用 `cancelCrawl` 方法。该方法接收异步爬取的作业 ID 作为参数，并返回取消结果。

<CancelCrawlNodeShort />

<div id="mapping-a-website">
  ### 网站映射
</div>

若需在包含错误处理的情况下对网站进行映射，请使用 `mapUrl` 方法。该方法接收起始 URL 作为参数，并以字典形式返回映射结果。

<div id="crawling-a-website-with-websockets">
  ### 使用 WebSockets 爬取网站
</div>

要通过 WebSockets 爬取网站，请使用 `crawlUrlAndWatch` 方法。它接受起始 URL 和可选参数。`params` 参数可用于为爬取任务指定其他选项，例如最大爬取页面数、允许的域名以及输出 formats。

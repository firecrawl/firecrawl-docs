<div id="usage">
  ## 使用
</div>

1. 在 [firecrawl.dev](https://firecrawl.dev) 获取 API 密钥
2. 将 `API key` 作为参数传入 `FirecrawlApp` 结构体。
3. 设置 `API URL`，并/或将其作为参数传入 `FirecrawlApp` 结构体。默认值为 `https://api.firecrawl.dev`。
4. 设置 `version`，并/或将其作为参数传入 `FirecrawlApp` 结构体。默认值为 `v1`。

<ScrapeAndCrawlGoExample />

<div id="scraping-a-url">
  ### 抓取单个 URL
</div>

要在包含错误处理的情况下抓取单个 URL，请使用 `ScrapeURL` 方法。它接收 URL 作为参数，并返回以字典形式表示的抓取数据。

<ScrapeGoShort />

<div id="crawling-a-website">
  ### 爬取网站
</div>

要爬取网站，请使用 `CrawlUrl` 方法。该方法接受起始 URL 和可选参数。通过 `params` 参数，你可以为爬取任务指定更多选项，例如最大爬取页数、允许的域名以及输出格式。

<CrawlGoShort />

<div id="checking-crawl-status">
  ### 检查爬取状态
</div>

要查看某个爬取任务的状态，请使用 `CheckCrawlStatus` 方法。传入任务 ID 作为参数，即可返回该爬取任务的当前状态。

<CheckCrawlStatusGoShort />

<div id="map-a-website">
  ### 映射网站
</div>

使用 `MapUrl` 从网站生成 URL 列表。可通过 `params` 参数自定义映射过程，例如排除子域或使用站点地图（sitemap）。

<MapGoShort />

<div id="error-handling">
  ## 错误处理
</div>

SDK 会处理 Firecrawl API 返回的错误并抛出相应异常。若请求过程中发生错误，将抛出包含详细错误信息的异常。

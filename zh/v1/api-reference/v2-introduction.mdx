<div id="base-url">
  ## 基础 URL
</div>

所有请求都使用以下基础 URL：

```bash
https://api.firecrawl.dev 
```

<div id="authentication">
  ## 身份验证
</div>

进行身份验证时，必须在请求头中包含 Authorization。其值应为 `Bearer fc-123456789`，其中 `fc-123456789` 是你的 API Key。

```bash
Authorization: Bearer fc-123456789
```

​

<div id="response-codes">
  ## 响应状态码
</div>

Firecrawl 使用常规 HTTP 状态码来表示请求结果。

一般来说，2xx 表示成功，4xx 表示用户侧错误，5xx 表示基础设施问题。

Status | Description
--- | ---
200 | 请求成功。
400 | 请检查参数是否正确。
401 | 未提供 API 密钥。
402 | 需要付款。
404 | 未找到请求的资源。
429 | 超出速率限制。
5xx | 表示 Firecrawl 的服务器错误。

有关所有可能的 API 错误的详细说明，请参见“错误代码”部分。

<div id="rate-limit">
  ## 速率限制
</div>

Firecrawl API 实施速率限制，以确保服务的稳定性与可靠性。该限制适用于所有端点，依据指定时间窗口内的请求次数计算。

当你超过速率限制时，会返回 429 响应码。

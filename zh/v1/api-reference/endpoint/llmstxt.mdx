---
title: "生成 LLMs.txt"
openapi: "v1-openapi POST /llmstxt"
---

LLMs.txt 生成端点会为任意网站创建 LLMs.txt 和 LLMs-full.txt 文件。这些文件以结构化、面向 LLM 的格式呈现网站内容，便于语言模型理解和处理信息。

在找状态端点？请查看 [LLMs.txt 状态](/zh/api-reference/endpoint/llmstxt-get) 端点。

<div id="response-structure">
  ### 响应结构
</div>

响应包含：

* **success**：指示请求是否成功的布尔值
* **id**：生成任务的唯一标识符

<div id="output-format">
  ### 输出格式
</div>

1. **LLMs.txt**

   * 对网站的精炼、结构化摘要
   * 包含关键链接与说明
   * 采用 Markdown 格式，便于解析
   * 示例：

     ```markdown
     # http://example.com llms.txt

     - [页面标题](https://example.com/page)：简要说明
     - [另一个页面](https://example.com/another)：其他说明
     ```

2. **LLMs-full.txt**（当 showFullText 为 true 时）

   * 包含已处理页面的完整内容
   * 保持层级结构
   * 提供更详尽的信息
   * 示例：

     ```markdown
     # http://example.com llms-full.txt

     ## 页面标题

     页面完整内容……

     ## 另一个页面

     更详细的内容……
     ```

<div id="billing">
  ### 计费
</div>

计费以 API 调用和处理的 URL 数量为基础：

* 基础费用：每次 API 调用消耗 1 个积分
* 额外费用：每处理 1 个 URL 消耗 1 个积分
* 可通过 `maxUrls` 参数控制 URL 成本

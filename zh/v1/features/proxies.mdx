Firecrawl 提供不同的代理类型，帮助你在应对不同复杂程度的网站时抓取网站。可以使用 `proxy` 参数指定代理类型。

> 默认情况下，即使你未指定代理类型或位置，Firecrawl 也会通过代理路由所有请求，以提升可靠性和访问成功率。

<div id="location-based-proxy-selection">
  ## 基于位置的代理选择
</div>

Firecrawl 会根据你指定或检测到的位置自动选择最合适的代理，以优化抓取性能与可靠性。但目前并非所有位置都受支持。当前可用位置如下：

| Country Code | Country Name         | Enhanced Mode Support |
|--------------|----------------------|----------------------|
| AE           | United Arab Emirates | No                   |
| AU           | Australia            | No                   |
| BR           | Brazil               | Yes                  |
| CN           | China                | No                   |
| DE           | Germany              | Yes                  |
| FR           | France               | Yes                  |
| GB           | United Kingdom       | No                   |
| JP           | Japan                | No                   |
| QA           | Qatar                | No                   |
| TR           | Turkey               | No                   |
| US           | United States        | Yes                  |
| VN           | Vietnam              | No                   |

<Warning>受支持的代理位置列表最后更新于 2025 年 5 月 15 日。可用性可能会随时间变化。</Warning>

如果你需要不在上述列表中的位置代理，请[联系我们](mailto:help@firecrawl.com)并说明你的需求。

如果你未指定代理或位置，Firecrawl 将基于目标站点和你的请求自动选择最优方案。

<div id="how-to-specify-proxy-location">
  ## 如何指定代理位置
</div>

你可以在请求中通过设置 `location.country` 参数来指定代理位置。例如，如需使用巴西代理，请将 `location.country` 设为 `BR`。

详见 [`location.country` 的 API 参考](https://docs.firecrawl.dev/api-reference/endpoint/scrape#body-location-country)。

<div id="proxy-types">
  ## 代理类型
</div>

Firecrawl 支持三种代理类型：

* **basic**：适用于抓取大多数网站的代理。速度快，通常可用。
* **enhanced**：用于在保持隐私的前提下抓取复杂网站的增强型代理。速度较慢，但在特定网站上更可靠。[了解 Enhanced 模式 →](/zh/features/enhanced-mode)
* **auto**：若 basic 代理失败，Firecrawl 会自动切换为增强型代理重试。若增强型重试成功，本次抓取计费 5 个积分；若首次使用 basic 即成功，则仅按常规费用计费。

***

> **注意：** 有关增强型代理的使用详情（包括积分费用与重试策略），请参阅 [Enhanced 模式文档](/zh/features/enhanced-mode)。

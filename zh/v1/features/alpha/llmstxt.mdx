<div id="introducing-llmstxt-generator-endpoint-alpha">
  ## 推出 LLMs.txt 生成器端点（Alpha）📃
</div>

`/llmstxt` 端点可将任意网站转为干净的、[适用于 LLM 的文本文件](https://www.firecrawl.dev/blog/How-to-Create-an-llms-txt-File-for-Any-Website)。只需提供一个 URL，Firecrawl 会对站点进行抓取并生成 `llms.txt` 和 `llms-full.txt` 文件，可用于任意 LLM 的训练或分析。

<div id="how-it-works">
  ## 工作原理
</div>

LLMs.txt 生成器：

1. 爬取提供的网站 URL 及其关联页面
2. 提取干净、有效的文本内容
3. 生成两种 formats：
   * `llms.txt`：精炼摘要与关键信息
   * `llms-full.txt`：完整文本内容，信息更详尽

**关键参数：**

* **url**：用于生成 LLMs.txt 的网站地址
* **maxUrls**（可选）：爬取的最大页面数（1-100，默认 10）
* **showFullText**（可选）：除 llms.txt 外同时生成 llms-full.txt（默认 false）

详见 [API 参考](/zh/api-reference/endpoint/llmstxt)。

<div id="known-limitations-alpha">
  ## 已知限制（Alpha）
</div>

1. **访问限制**\
   仅支持处理公开可访问的页面。不支持登录保护或付费墙后的内容。

2. **站点规模**\
   在 Alpha 阶段，我们仅允许处理最多 5000 个 URL。

3. **Alpha 状态**\
   作为 Alpha 功能，输出 formats 和处理流程可能会根据反馈进行调整。

<div id="billing-and-usage">
  ## 计费与用量
</div>

计费基于处理的 URL 数量：

* 基础费用：每处理 1 个 URL 消耗 1 个积分
* 可通过 `maxUrls` 参数控制 URL 的处理成本

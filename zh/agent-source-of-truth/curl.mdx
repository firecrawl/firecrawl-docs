<div id="firecrawl-curl-source-of-truth">
  # Firecrawl curl 权威来源
</div>

此文件是面向代理的官方公开权威依据。它根据 Firecrawl v2 OpenAPI 规范和 SDK 接口自动生成。

<div id="authenticate">
  ## 身份验证
</div>

所有请求都在 `Authorization` 请求头中使用 Bearer 令牌。

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="when-to-use-what">
  ## 何时使用哪种方式
</div>

* `search`：从查询词开始，发现相关 URL，也支持 `site:` 限定。
* `scrape`：从已知 URL 获取并提取内容。
* `interact`：当页面需要点击、填写表单或执行抓取后步骤时，运行浏览器 actions 或代码。

<div id="search">
  ## Search
</div>

### 使用场景

当你需要通过查询发现内容，并希望 Firecrawl 返回 URL (以及可选地抓取每个结果) 时，请使用 search。你可以使用标准的 `site:` 运算符将结果限定在某个域名范围内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`POST https://api.firecrawl.dev/v2/search`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -sS "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev webhook retries"
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -sS "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev crawl webhooks",
    "sources": ["web", "news"],
    "categories": ["research"],
    "limit": 10,
    "tbs": "qdr:m",
    "location": "San Francisco,California,United States",
    "country": "US",
    "ignoreInvalidURLs": true,
    "timeout": 60000,
    "scrapeOptions": {
      "formats": [
        "markdown",
        "links",
        { "type": "json", "prompt": "Extract title and key endpoints." }
      ],
      "onlyMainContent": true,
      "includeTags": ["main", "article"],
      "excludeTags": ["nav", "footer"],
      "waitFor": 1000,
      "actions": [
        { "type": "click", "selector": "button#accept" },
        { "type": "wait", "milliseconds": 750 },
        { "type": "scrape" }
      ]
    }
  }'
```

<div id="parameters">
  ### 参数
</div>

| 参数                  | 类型 / 选项                                             | 描述                                              |
| ------------------- | --------------------------------------------------- | ----------------------------------------------- |
| `query`             | `string`                                            | 搜索查询。使用 `site:domain` 将结果限定在某个站点。               |
| `sources`           | `array of source names or typed source objects`     | 指定搜索来源。                                         |
| `categories`        | `array of category names or typed category objects` | 用于过滤结果的类别。                                      |
| `limit`             | `integer`                                           | 返回结果的最大数量。                                      |
| `tbs`               | `string`                                            | 基于时间的搜索过滤器 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。 |
| `location`          | `string`                                            | 搜索结果的地理位置字符串。                                   |
| `country`           | `string`                                            | 用于地理定向搜索的 ISO 国家代码。                             |
| `timeout`           | `integer`                                           | 超时时间，单位为毫秒。                                     |
| `ignoreInvalidURLs` | `boolean`                                           | 排除其他 Firecrawl 端点无法抓取的无效 URL。                   |
| `enterprise`        | `array of enterprise modes such as anon or zdr`     | 为团队启用后可用的企业 ZDR 选项。                             |
| `scrapeOptions`     | `ScrapeOptions`                                     | 应用于每个结果的抓取选项 (字段与 `scrape` 相同) 。                |

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### 为什么使用
</div>

如果你已经有一个 URL，并希望 Firecrawl 返回结构化内容及可选的 formats，请使用 scrape。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`POST https://api.firecrawl.dev/v2/scrape`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -sS "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.firecrawl.dev",
    "formats": ["markdown"]
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -sS "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": [
      "markdown",
      "links",
      { "type": "json", "prompt": "Extract plan names and prices." },
      { "type": "screenshot", "fullPage": true, "quality": 80, "viewport": { "width": 1280, "height": 720 } },
      { "type": "changeTracking", "modes": ["git-diff"], "tag": "pricing" }
    ],
    "headers": { "User-Agent": "FirecrawlDocsBot/1.0" },
    "includeTags": ["main", "article"],
    "excludeTags": ["nav", "footer"],
    "onlyMainContent": true,
    "waitFor": 1000,
    "mobile": false,
    "parsers": [{ "type": "pdf", "mode": "auto", "maxPages": 5 }],
    "actions": [
      { "type": "click", "selector": "#accept" },
      { "type": "write", "text": "firecrawl" },
      { "type": "press", "key": "Enter" },
      { "type": "wait", "milliseconds": 1500 },
      { "type": "scrape" }
    ],
    "location": { "country": "US", "languages": ["en-US"] },
    "skipTlsVerification": true,
    "removeBase64Images": true,
    "blockAds": true,
    "proxy": "auto",
    "maxAge": 86400000,
    "storeInCache": true,
    "profile": { "name": "docs-session", "saveChanges": true }
  }'
```

<div id="parameters">
  ### 参数
</div>

| 参数                    | 类型 / 选项                    | 描述                                                                                                                  |
| --------------------- | -------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| `url`                 | `string`                   | 要抓取的 URL。                                                                                                           |
| `formats`             | `Array`                    | 要包含的输出 formats。可以是字符串或对象，例如 `{type: "json", prompt, schema}` 或 `{type: "screenshot", fullPage, quality, viewport}`。 |
| `onlyMainContent`     | `boolean`                  | 仅保留主要内容。                                                                                                            |
| `includeTags`         | `Array<string>`            | 要包含的标签。                                                                                                             |
| `excludeTags`         | `Array<string>`            | 要排除的标签。                                                                                                             |
| `maxAge`              | `integer`                  | 优先使用不超过此时长的缓存数据。                                                                                                    |
| `minAge`              | `integer`                  | 仅在缓存数据达到最小时长时返回。                                                                                                    |
| `headers`             | `object`                   | 自定义请求头。                                                                                                             |
| `waitFor`             | `integer`                  | 抓取前的等待时间 (毫秒) 。                                                                                                     |
| `mobile`              | `boolean`                  | 模拟移动设备。                                                                                                             |
| `skipTlsVerification` | `boolean`                  | 跳过 TLS 证书验证。                                                                                                        |
| `timeout`             | `integer`                  | 超时时间 (毫秒) 。                                                                                                         |
| `parsers`             | `Array`                    | 解析器配置 (支持 PDF 选项) 。                                                                                                 |
| `actions`             | `Array`                    | 抓取前要执行的浏览器 actions。                                                                                                 |
| `location`            | `object`                   | 请求的地理位置设置。                                                                                                          |
| `location.country`    | `string`                   | ISO 国家/地区代码。                                                                                                        |
| `location.languages`  | `Array<string>`            | 首选语言。                                                                                                               |
| `removeBase64Images`  | `boolean`                  | 从 markdown 输出中移除 base64 图片。                                                                                         |
| `blockAds`            | `boolean`                  | 启用广告和 cookie 弹窗拦截。                                                                                                  |
| `proxy`               | `basic, enhanced, or auto` | 代理模式。                                                                                                               |
| `storeInCache`        | `boolean`                  | 将结果存入缓存。                                                                                                            |
| `profile`             | `object`                   | 持久化浏览器配置，包含 `name` 和 `saveChanges`。                                                                                 |
| `zeroDataRetention`   | `boolean`                  | 在你的账户支持的情况下启用零数据保留。                                                                                                 |

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么使用
</div>

当页面在抓取开始后需要执行浏览器 actions 或运行代码时 (例如点击、填写表单或执行脚本步骤) ，请使用交互。

### 首选 SDK 方法

`POST https://api.firecrawl.dev/v2/scrape/{jobId}/interact`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -sS "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "console.log(document.title)",
    "language": "node"
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -sS "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Click the pricing tab and summarize the plans.",
    "language": "node",
    "timeout": 60
  }'
```

<div id="parameters">
  ### 参数
</div>

| 参数         | 类型 / 选项                 | 描述                                    |
| ---------- | ----------------------- | ------------------------------------- |
| `jobId`    | `string`                | 来自 `data.metadata.scrapeId` 的抓取任务 ID。 |
| `code`     | `string`                | 在浏览器会话中执行的代码。如果未提供 `prompt`，则为必填。     |
| `prompt`   | `string`                | 提供给浏览器代理的自然语言指令。如果未提供 `code`，则为必填。    |
| `language` | `python, node, or bash` | 执行语言。省略时默认为 `node`。                   |
| `timeout`  | `integer`               | 执行超时时间，单位为秒。                          |
| `origin`   | `string`                | 仅限内部使用。保留给 Firecrawl 内部追踪。            |

<div id="notes">
  ## 备注
</div>

* OpenAPI 规范目前将交互接口中的 `code` 标记为必填，但 SDK 也接受 `prompt` 作为替代。实际使用时，填写 `code` 或 `prompt` 均可。

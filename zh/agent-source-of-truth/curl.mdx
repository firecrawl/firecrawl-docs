面向代理的 Firecrawl 规范 curl 权威来源。基于 SDK 源代码和 v2 OpenAPI 规范生成。

<div id="authenticate">
  ## 身份验证
</div>

```bash
export FIRECRAWL_API_KEY="fc-your-api-key"
```

<div id="when-to-use-what">
  ## 何时使用哪一种
</div>

* `search`：当你从查询开始，需要先发现结果时使用。
* `scrape`：当你已经有 URL，并想获取页面内容时使用。
* `interact`：当页面需要点击、填写表单，或在抓取后执行浏览器 actions 时使用。

<div id="search">
  ## Search
</div>

### 使用场景

使用 search 从查询中发现相关页面，然后选择要抓取或交互的 URL。你可以使用标准的 `site:` 运算符将结果限定在某个站点范围内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`POST /search`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev webhook retries"
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/search" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "site:docs.firecrawl.dev crawl webhooks",
    "sources": ["web", "news"],
    "categories": ["research"],
    "limit": 10,
    "tbs": "qdr:m",
    "location": "San Francisco,California,United States",
    "country": "US",
    "ignoreInvalidURLs": true,
    "timeout": 60000,
    "enterprise": ["anon"],
    "scrapeOptions": {
      "formats": [
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract title and key endpoints."}
      ],
      "onlyMainContent": true,
      "includeTags": ["main", "article"],
      "excludeTags": ["nav", "footer"],
      "waitFor": 1000
    }
  }'
```

<div id="parameters">
  ### 参数
</div>

* `query`
  * 类型：string
  * 使用时机：需要搜索查询时。
  * 说明：使用 `site:example.com` 可将结果限制在某个域名内。

* `sources`
  * 类型：来源名称数组或带类型的来源对象数组
  * 使用时机：想控制搜索哪些来源时。
  * 已确认的值：
    * `"web"`：网页索引结果
    * `"news"`：新闻结果
    * `"images"`：图片结果
    * `{ "type": "web" | "news" | "images" }`：带类型的来源对象形式

* `categories`
  * 类型：类别名称数组或带类型的类别对象数组
  * 使用时机：想按类别筛选结果时。
  * 已确认的值：
    * `"github"`：GitHub 相关结果
    * `"research"`：研究和学术结果
    * `"pdf"`：PDF 相关结果
    * `{ "type": "github" | "research" | "pdf" }`：带类型的类别对象形式

* `limit`
  * 类型：number
  * 使用时机：想限制结果数量时。

* `tbs`
  * 类型：string
  * 使用时机：需要按时间筛选时 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。

* `location`
  * 类型：string
  * 使用时机：想获取本地化结果时。

* `country`
  * 类型：string
  * 使用时机：想按 ISO 3166-1 alpha-2 国家/地区代码进行定向时 (例如 `"US"`) 。

* `ignoreInvalidURLs`
  * 类型：boolean
  * 使用时机：想忽略无法被其他 endpoints 抓取的 URL 时。

* `timeout`
  * 类型：number
  * 使用时机：需要以毫秒为单位设置请求超时时间时。

* `enterprise`
  * 类型：字符串数组
  * 使用时机：需要企业级搜索控制时。
  * 已确认的值：
    * `"zdr"`：端到端零数据保留
    * `"anon"`：匿名化零数据保留

* `scrapeOptions`
  * 类型：object
  * 使用时机：想抓取每个搜索结果时 (字段请参见 Scrape 参数) 。

<div id="scrape">
  ## Scrape
</div>

<div id="why-use-it">
  ### 为什么使用
</div>

如果你已经有一个 URL，并希望以一种或多种 formats 获取结构化内容，请使用 scrape。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`POST /scrape`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://docs.firecrawl.dev",
    "formats": ["markdown"]
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/pricing",
    "formats": [
      "markdown",
      "links",
      {"type": "json", "prompt": "Extract plan names and prices."},
      {"type": "screenshot", "fullPage": true, "quality": 80, "viewport": {"width": 1280, "height": 720}},
      {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    "headers": {"User-Agent": "FirecrawlDocsBot/1.0"},
    "onlyMainContent": true,
    "waitFor": 1000,
    "parsers": [{"type": "pdf", "mode": "auto", "maxPages": 5}],
    "actions": [
      {"type": "click", "selector": "#accept"},
      {"type": "wait", "milliseconds": 750},
      {"type": "scrape"}
    ],
    "location": {"country": "US", "languages": ["en-US"]},
    "removeBase64Images": true,
    "blockAds": true,
    "proxy": "auto",
    "maxAge": 86400000,
    "minAge": 1,
    "storeInCache": true,
    "profile": {"name": "docs-session", "saveChanges": true},
    "zeroDataRetention": false
  }'
```

<div id="parameters">
  ### 参数
</div>

* `url`
  * 类型：string
  * 使用场景：想抓取某个特定页面时。

* `formats`
  * 类型：格式字符串数组或格式对象数组
  * 使用场景：需要多种输出格式时。
  * 已确认的格式字符串：
    * `"markdown"`：markdown 内容
    * `"html"`：清理后的 HTML
    * `"rawHtml"`：原始 HTML
    * `"links"`：页面链接
    * `"images"`：图片 URL
    * `"screenshot"`：截图输出
    * `"summary"`：摘要输出
    * `"changeTracking"`：变更追踪输出
    * `"json"`：JSON 提取
    * `"branding"`：品牌资料输出
    * `"audio"`：音频提取
  * 格式对象字段：
    * `type`：上述格式字符串之一
    * `prompt`, `schema`：`type: "json"` 时的 JSON 提取选项
    * `modes`, `schema`, `prompt`, `tag`：`type: "changeTracking"` 时的变更追踪选项
    * `fullPage`, `quality`, `viewport`：`type: "screenshot"` 时的截图选项

* `headers`
  * 类型：object
  * 使用场景：需要自定义请求头时。

* `includeTags`
  * 类型：字符串数组
  * 使用场景：只想包含特定 HTML 标签时。

* `excludeTags`
  * 类型：字符串数组
  * 使用场景：想排除特定 HTML 标签时。

* `onlyMainContent`
  * 类型：boolean
  * 使用场景：想去除导航栏、页脚和其他模板化内容时。

* `timeout`
  * 类型：number
  * 使用场景：需要以毫秒为单位设置超时时间时。

* `waitFor`
  * 类型：number
  * 使用场景：需要等待页面渲染完成时 (毫秒) 。

* `mobile`
  * 类型：boolean
  * 使用场景：想使用移动端视口时。

* `parsers`
  * 类型：解析器字符串数组或对象数组
  * 使用场景：需要控制文件解析时。
  * 已确认的值：
    * `"pdf"`
    * `{ "type": "pdf", "mode": "fast" | "auto" | "ocr", "maxPages": number }`

* `actions`
  * 类型：action 对象数组
  * 使用场景：需要轻量级抓取前操作时。
  * 已确认的 action 类型：
    * `wait`：必须提供 `milliseconds` 或 `selector`
    * `screenshot`：`fullPage`、`quality`、`viewport` 为可选项
    * `click`：必须提供 `selector`，`all` 为可选项
    * `write`：必须提供 `text` (先点击使输入框获得焦点) 
    * `press`：必须提供 `key`
    * `scroll`：必须提供 `direction` (`up` 或 `down`) ，`selector` 为可选项
    * `scrape`：无额外字段
    * `executeJavascript`：必须提供 `script`
    * `pdf`：`format` (A0、A1、A2、A3、A4、A5、A6、Letter、Legal、Tabloid、Ledger) 、`landscape`、`scale` 为可选项

* `location`
  * 类型：包含 `country` 和 `languages` 的 object
  * 使用场景：需要按地理位置或语言进行抓取时。

* `skipTlsVerification`
  * 类型：boolean
  * 使用场景：需要跳过 TLS 验证时。

* `removeBase64Images`
  * 类型：boolean
  * 使用场景：想从 markdown 输出中移除 base64 图片时。

* `blockAds`
  * 类型：boolean
  * 使用场景：想启用广告和 Cookie 弹窗封禁时。

* `proxy`
  * 类型：string
  * 使用场景：需要控制代理时。
  * 已确认的值：`"basic"`、`"enhanced"`、`"auto"`

* `maxAge`
  * 类型：number
  * 使用场景：想使用不超过最大时长的缓存数据时 (毫秒) 。

* `minAge`
  * 类型：number
  * 使用场景：仅在缓存数据至少达到此时长时才使用时 (毫秒) 。

* `storeInCache`
  * 类型：boolean
  * 使用场景：希望 Firecrawl 缓存结果时。

* `profile`
  * 类型：包含 `name` 和可选 `saveChanges` 的 object
  * 使用场景：想使用可持久化的浏览器配置文件，并在多次抓取和交互之间共享时。

* `zeroDataRetention`
  * 类型：boolean
  * 使用场景：想为此次抓取启用零数据保留时。

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么使用
</div>

当页面在抓取开始后需要执行浏览器 actions 或运行代码时，请使用交互。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`POST /scrape/{jobId}/interact`

<div id="simple-example">
  ### 简单示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "console.log(await page.title());",
    "language": "node",
    "timeout": 60
  }'
```

<div id="complex-example">
  ### 复杂示例
</div>

```bash
curl -X POST "https://api.firecrawl.dev/v2/scrape/<jobId>/interact" \
  -H "Authorization: Bearer $FIRECRAWL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Click the pricing tab and summarize the plans.",
    "language": "node",
    "timeout": 60
  }'
```

<div id="parameters">
  ### 参数
</div>

* `jobId`
  * 类型：string
  * 适用场景：你已有抓取任务 ID。

* `code`
  * 类型：string
  * 适用场景：你想在浏览器会话中运行代码。

* `prompt`
  * 类型：string
  * 适用场景：你希望浏览器代理按自然语言指令执行。

* `language`
  * 类型：string
  * 适用场景：你需要指定特定的运行时环境。
  * 可用值：`"python"`、`"node"`、`"bash"`

* `timeout`
  * 类型：number
  * 适用场景：你需要以秒为单位设置执行超时时间。

<div id="notes">
  ## 注意事项
</div>

* 多步骤的浏览器工作流请使用 `interact`；`scrape` 的 `actions` 更适合抓取前的小型准备步骤。

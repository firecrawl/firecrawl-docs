<div id="firecrawl-nodejs-source-of-truth">
  # Firecrawl Node.js 权威来源
</div>

此文件是面向代理的官方权威公开来源。它根据 Firecrawl JS SDK 源代码和 v2 OpenAPI 规范生成。

<div id="install">
  ## 安装
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="authenticate">
  ## 身份验证
</div>

```ts
import Firecrawl from "@mendable/firecrawl-js";

const client = new Firecrawl({
  apiKey: process.env.FIRECRAWL_API_KEY,
  // apiUrl: "https://api.firecrawl.dev" // 可选，用于覆盖自托管实例地址
});
```

<div id="when-to-use-what">
  ## 何时使用哪种方式
</div>

* `search`：从查询出发，发现相关 URL，包括使用 `site:` 约束。
* `scrape`：从已知 URL 获取并提取内容。
* `interact`：当页面需要点击、填写表单或执行抓取后步骤时，运行浏览器 actions 或代码。

<div id="search">
  ## Search
</div>

### 使用场景

当你需要通过查询发现内容，并希望 Firecrawl 返回 URL (以及可选地抓取每个结果) 时，请使用 search。你可以使用标准的 `site:` 运算符将结果限定在某个域名范围内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`client.search(query, options?)`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev webhook retries");
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const results = await client.search("site:docs.firecrawl.dev crawl webhooks", {
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignoreInvalidURLs: true,
  timeout: 60000,
  scrapeOptions: {
    formats: [
      "markdown",
      "links",
      { type: "json", prompt: "Extract title, summary, and key endpoints." }
    ],
    onlyMainContent: true,
    includeTags: ["main", "article"],
    excludeTags: ["nav", "footer"],
    waitFor: 1000,
    actions: [
      { type: "click", selector: "button#accept" },
      { type: "wait", milliseconds: 750 },
      { type: "scrape" }
    ]
  }
});
```

<div id="parameters">
  ### 参数
</div>

| 参数                          | 类型 / 选项             | 描述                                              |
| --------------------------- | ------------------- | ----------------------------------------------- |
| `query`                     | `string`            | 搜索查询。使用 `site:domain` 将结果限定在特定站点。               |
| `options.sources`           | `来源名称数组或带类型的来源对象数组` | 要搜索的来源。                                         |
| `options.categories`        | `类别名称数组或带类型的类别对象数组` | 用于筛选结果的类别。                                      |
| `options.limit`             | `number`            | 返回结果的最大数量。                                      |
| `options.tbs`               | `string`            | 基于时间的搜索过滤器 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。 |
| `options.location`          | `string`            | 搜索结果的位置字符串。                                     |
| `options.ignoreInvalidURLs` | `boolean`           | 排除其他 Firecrawl 端点无法抓取的无效 URL。                   |
| `options.timeout`           | `number`            | 超时时间，单位为毫秒。                                     |
| `options.scrapeOptions`     | `ScrapeOptions`     | 应用于每个结果的抓取选项 (字段与 `scrape` 相同) 。                |
| `options.integration`       | `string`            | 仅供内部使用。保留用于 Firecrawl 内部追踪。                     |
| `options.origin`            | `string`            | 仅供内部使用。保留用于 Firecrawl 内部追踪。                     |

<div id="scrape">
  ## 抓取
</div>

### 为什么使用

当你已经有一个 URL，并且希望 Firecrawl 返回结构化内容和可选的 formats 时，请使用 `scrape`。

### 首选 SDK 方法

`client.scrape(url, options?)`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const doc = await client.scrape("https://docs.firecrawl.dev", {
  formats: ["markdown"]
});
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const doc = await client.scrape("https://example.com/pricing", {
  formats: [
    "markdown",
    "links",
    { type: "json", prompt: "Extract plan names and prices." },
    { type: "screenshot", fullPage: true, quality: 80, viewport: { width: 1280, height: 720 } },
    { type: "changeTracking", modes: ["git-diff"], tag: "pricing" }
  ],
  headers: {
    "User-Agent": "FirecrawlDocsBot/1.0"
  },
  includeTags: ["main", "article"],
  excludeTags: ["nav", "footer"],
  onlyMainContent: true,
  waitFor: 1000,
  mobile: false,
  parsers: [{ type: "pdf", mode: "auto", maxPages: 5 }],
  actions: [
    { type: "click", selector: "#accept" },
    { type: "write", text: "firecrawl" },
    { type: "press", key: "Enter" },
    { type: "wait", milliseconds: 1500 },
    { type: "scrape" }
  ],
  location: { country: "US", languages: ["en-US"] },
  skipTlsVerification: true,
  removeBase64Images: true,
  fastMode: true,
  blockAds: true,
  proxy: "auto",
  maxAge: 86400000,
  storeInCache: true,
  profile: { name: "docs-session", saveChanges: true }
});
```

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么使用它
</div>

当页面在开始抓取后需要执行浏览器 actions 或运行代码 (如点击、填写表单或脚本步骤) 时，请使用 interact。

### 首选 SDK 方法

`client.interact(jobId, args)`

<div id="simple-example">
  ### 简单示例
</div>

```ts
const doc = await client.scrape("https://example.com", { formats: ["markdown"] });
const jobId = doc.metadata?.scrapeId;
if (!jobId) throw new Error("scrape 响应中缺少 scrapeId");

const result = await client.interact(jobId, {
  prompt: "点击定价选项卡并总结相关方案。",
  language: "node"
});
```

<div id="complex-example">
  ### 复杂示例
</div>

```ts
const result = await client.interact("<scrapeJobId>", {
  code: "console.log(await page.title());",
  language: "node",
  timeout: 60
});
```

<div id="parameters">
  ### 参数
</div>

| 参数              | 类型 / 选项              | 描述                                      |
| --------------- | -------------------- | --------------------------------------- |
| `jobId`         | `string`             | `document.metadata.scrapeId` 中的抓取任务 ID。 |
| `args.code`     | `string`             | 在浏览器会话中执行的代码。若未提供 `prompt`，则此项必填。       |
| `args.prompt`   | `string`             | 提供给浏览器代理的自然语言指令。若未提供 `code`，则此项必填。      |
| `args.language` | `python、node 或 bash` | 执行语言。省略时默认为 `node`。                     |
| `args.timeout`  | `number`             | 执行超时时间，单位为秒。                            |
| `args.origin`   | `string`             | 仅供内部使用。保留给 Firecrawl 的内部追踪。             |

<div id="notes">
  ## 说明
</div>

* 已弃用的别名：`scrapeExecute`、`stopInteractiveBrowser` 和 `deleteScrapeBrowser` 对应 `interact` 和 `stopInteraction`。
* 顶层 `Firecrawl` 客户端会直接提供 v2 方法；旧版 v1 可通过 `client.v1` 使用。

<div id="search">
  ## 何时使用哪种方式
</div>

* `search`：从查询出发，发现相关 URL，包括 `site:` 限制。
* `scrape`：从已知 URL 获取并提取内容。
* `interact`：当页面需要点击、填写表单或执行抓取后步骤时，运行浏览器 actions 或代码。



## Search

### 使用场景

当你需要通过查询发现内容，并希望 Firecrawl 返回 URL（以及可选地抓取每个结果）时，请使用 search。你可以使用标准的 `site:` 运算符将结果限定在某个域名范围内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="simple-example">
  ### 推荐的 SDK 方法
</div>

`client.search(query, **options)`

<div id="complex-example">
  ### 简单示例
</div>

```py
results = client.search("site:docs.firecrawl.dev webhook retries")
```

<div id="parameters">
  ### 复杂示例
</div>

```py
from firecrawl.v2.types import ScrapeOptions

results = client.search(
    "site:docs.firecrawl.dev crawl webhooks",
    sources=["web", "news"],
    categories=["research"],
    limit=10,
    tbs="qdr:m",
    location="San Francisco,California,United States",
    ignore_invalid_urls=True,
    timeout=300000,
    scrape_options=ScrapeOptions(
        formats=[
            "markdown",
            "links",
            {"type": "json", "prompt": "Extract title and key endpoints."}
        ],
        only_main_content=True,
        include_tags=["main", "article"],
        exclude_tags=["nav", "footer"],
        wait_for=1000,
        actions=[
            {"type": "click", "selector": "button#accept"},
            {"type": "wait", "milliseconds": 750},
            {"type": "scrape"}
        ]
    )
)
```

### 参数

| 参数                  | 类型 / 选项     | 描述                                                                             |
| --------------------- | --------------- | -------------------------------------------------------------------------------- | -------- | ----------------- | ---------- | ----------- | ----------------------------- |
| `query`               | `str`           | 搜索查询。使用 `site:domain` 将结果限定在特定站点。                              |
| `sources`             | `List["web"     | "news"                                                                           | "images" | {"type": "web"    | "news"     | "images"}]` | 要搜索的来源。                |
| `categories`          | `List["github"  | "research"                                                                       | "pdf"    | {"type": "github" | "research" | "pdf"}]`    | 用于筛选结果的类别。          |
| `limit`               | `int`           | 返回结果的最大数量。SDK 模型中默认值为 `5`。                                     |
| `tbs`                 | `str`           | 基于时间的搜索过滤器（例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`）。                  |
| `location`            | `str`           | 搜索结果的地理位置字符串。                                                       |
| `ignore_invalid_urls` | `bool`          | 排除其他 Firecrawl 端点无法抓取的无效 URL。                                      |
| `timeout`             | `int`           | 超时时间（毫秒）。SDK 模型中默认值为 `300000`。                                  |
| `scrape_options`      | `ScrapeOptions` | 应用于每个结果的抓取选项（字段与 `scrape` 相同，另加 `min_age`）。               |
| `integration`         | `str`           | 仅供内部使用。保留用于 Firecrawl 内部追踪。                                      |


## 抓取

<div id="preferred-sdk-method">
  ### 为什么使用
</div>

当你已经有一个 URL，并希望 Firecrawl 返回结构化内容和可选 formats 时，请使用 scrape。

### 首选 SDK 方法

`client.scrape(url, **options)`

<div id="complex-example">
  ### 简单示例
</div>

```py
doc = client.scrape("https://docs.firecrawl.dev", formats=["markdown"])
```

<div id="parameters">
  ### 复杂示例
</div>

```py
doc = client.scrape(
    "https://example.com/pricing",
    formats=[
        "markdown",
        "links",
        {"type": "json", "prompt": "Extract plan names and prices."},
        {"type": "screenshot", "full_page": True, "quality": 80, "viewport": {"width": 1280, "height": 720}},
        {"type": "changeTracking", "modes": ["git-diff"], "tag": "pricing"}
    ],
    headers={"User-Agent": "FirecrawlDocsBot/1.0"},
    include_tags=["main", "article"],
    exclude_tags=["nav", "footer"],
    only_main_content=True,
    wait_for=1000,
    mobile=False,
    parsers=[{"type": "pdf", "mode": "auto", "max_pages": 5}],
    actions=[
        {"type": "click", "selector": "#accept"},
        {"type": "write", "text": "firecrawl"},
        {"type": "press", "key": "Enter"},
        {"type": "wait", "milliseconds": 1500},
        {"type": "scrape"}
    ],
    location={"country": "US", "languages": ["en-US"]},
    skip_tls_verification=True,
    remove_base64_images=True,
    fast_mode=True,
    block_ads=True,
    proxy="auto",
    max_age=86400000,
    store_in_cache=True,
    profile={"name": "docs-session", "save_changes": True}
)
```

## 交互

<div id="preferred-sdk-method">
  ### 为什么使用
</div>

当页面在抓取开始后需要执行浏览器 actions 或运行代码时，请使用 interact (例如点击、填写表单或执行脚本化步骤) 。

### 首选 SDK 方法

`client.interact(job_id, code=None, prompt=None, language="node", timeout=None)`

<div id="complex-example">
  ### 简单示例
</div>

```py
doc = client.scrape("https://example.com", formats=["markdown"])
job_id = doc.metadata.scrape_id if doc.metadata else None
if not job_id:
    raise ValueError("scrape 响应中缺少 scrape_id")

result = client.interact(
    job_id,
    prompt="Click the pricing tab and summarize the plans.",
    language="node"
)
```

<div id="parameters">
  ### 复杂示例
</div>

```py
result = client.interact(
    "<scrape_job_id>",
    code="print(await page.title())",
    language="python",
    timeout=60
)
```

### 参数

| 参数         | 类型 / 选项             | 描述                                         |                   |                   |
| ---------- | ------------------- | ------------------------------------------ | ----------------- | ----------------- |
| `job_id`   | `str`               | 来自 `document.metadata.scrape_id` 的抓取任务 ID。 |                   |                   |
| `code`     | `str`               | 在浏览器会话中执行的代码。若未提供 `prompt`，则此项必填。          |                   |                   |
| `prompt`   | `str`               | 提供给浏览器代理的自然语言指令。若未提供 `code`，则此项必填。         |                   |                   |
| `language` | `&quot;python&quot; | &quot;node&quot;                           | &quot;bash&quot;` | 执行语言。默认值为 `node`。 |
| `timeout`  | `int`               | 执行超时时间 (秒，范围 1–300) 。                      |                   |                   |
| `origin`   | `str`               | 仅供内部使用。保留用于 Firecrawl 内部追踪。                |                   |                   |



<div id="source-of-truth">
  ## 说明
</div>

* 已弃用的别名：`scrape_execute`、`stop_interactive_browser` 和 `delete_scrape_browser` 会映射为 `interact` 和 `stop_interaction`。
* `ScrapeOptions` 包含 `min_age`，但 `scrape` 这个便捷方法未提供该参数。如需使用，请通过 `search(..., scrape_options=...)` 传入 `ScrapeOptions`。
* 顶层 `Firecrawl` 客户端直接提供 v2 方法；旧版 v1 可通过 `client.v1` 使用。

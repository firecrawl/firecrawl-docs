<div id="firecrawl-java-source-of-truth">
  # Firecrawl Java 权威来源
</div>

此文件是供代理使用的公开权威来源。它根据 Firecrawl Java SDK 源代码和 v2 OpenAPI 规范生成。

<div id="install">
  ## 安装
</div>

Maven：

```xml
<dependency>
  <groupId>com.firecrawl</groupId>
  <artifactId>firecrawl-java</artifactId>
  <version>1.2.0</version>
</dependency>
```

Gradle：

```gradle
implementation("com.firecrawl:firecrawl-java:1.2.0")
```

<div id="authenticate">
  ## 认证
</div>

```java
import com.firecrawl.client.FirecrawlClient;

FirecrawlClient client = FirecrawlClient.builder()
    .apiKey(System.getenv("FIRECRAWL_API_KEY"))
    .build();
```

<div id="when-to-use-what">
  ## 何时使用哪种方式
</div>

* `search`：从查询出发查找相关 URL，包括 `site:` 限定。
* `scrape`：从已知 URL 获取并提取内容。
* `interact`：当页面需要点击、填写表单或执行抓取后的步骤时，运行浏览器 actions 或代码。

<div id="search">
  ## 搜索
</div>

### 为什么使用它

当你需要根据查询词发现结果，并希望 Firecrawl 返回 URL (也可选择抓取每个结果) 时，请使用 search。你可以使用标准的 `site:` 运算符，将结果限制在某个域名下，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`client.search(query, options)`

<div id="simple-example">
  ### 简单示例
</div>

```java
SearchData results = client.search("site:docs.firecrawl.dev webhook retries");
```

<div id="complex-example">
  ### 复杂示例
</div>

```java
import com.firecrawl.models.SearchOptions;
import com.firecrawl.models.ScrapeOptions;
import com.firecrawl.models.JsonFormat;

SearchOptions options = SearchOptions.builder()
    .sources(List.of("web", "news"))
    .categories(List.of("research"))
    .limit(10)
    .tbs("qdr:m")
    .location("San Francisco,California,United States")
    .ignoreInvalidURLs(true)
    .timeout(60000)
    .scrapeOptions(
        ScrapeOptions.builder()
            .formats(List.of(
                "markdown",
                "links",
                JsonFormat.builder().prompt("Extract title and key endpoints.").build()
            ))
            .onlyMainContent(true)
            .waitFor(1000)
            .build()
    )
    .build();

SearchData results = client.search("site:docs.firecrawl.dev crawl webhooks", options);
```

<div id="parameters">
  ### 参数
</div>

| 参数                          | 类型 / 选项                                                             | 描述                                               |
| --------------------------- | ------------------------------------------------------------------- | ------------------------------------------------ |
| `query`                     | `String`                                                            | 搜索查询。使用 `site:domain` 可将结果限定在某个站点内。              |
| `options.sources`           | `List<Object>` (`"web"`、`"news"`、`"images"` 或 `{type: "web"}` 映射对象) | 要搜索的来源。                                          |
| `options.categories`        | `List<Object>` (`"github"`、`"research"`、`"pdf"`)                    | 用于筛选结果的类别。                                       |
| `options.limit`             | `Integer`                                                           | 返回结果的最大数量。                                       |
| `options.tbs`               | `String`                                                            | 基于时间的搜索筛选条件 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。 |
| `options.location`          | `String`                                                            | 搜索结果的地区字符串。                                      |
| `options.ignoreInvalidURLs` | `Boolean`                                                           | 排除其他 Firecrawl 端点无法抓取的无效 URL。                    |
| `options.timeout`           | `Integer`                                                           | 超时时间 (毫秒) 。                                      |
| `options.scrapeOptions`     | `ScrapeOptions`                                                     | 应用于每个结果的抓取选项 (字段与 `scrape` 相同) 。                 |
| `options.integration`       | `String`                                                            | 仅供内部使用。保留给 Firecrawl 埋点。                         |

<div id="scrape">
  ## 抓取
</div>

### 为什么要用它

如果你已经有了 URL，并希望 Firecrawl 返回结构化内容和可选的 formats，请使用 `scrape`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`client.scrape(url, options)`

<div id="simple-example">
  ### 简单示例
</div>

```java
Document doc = client.scrape(
    "https://docs.firecrawl.dev",
    ScrapeOptions.builder()
        .formats(List.of("markdown"))
        .build()
);
```

<div id="complex-example">
  ### 复杂示例
</div>

```java
import com.firecrawl.models.ScrapeOptions;

ScrapeOptions options = ScrapeOptions.builder()
    .formats(List.of(
        "markdown",
        "links",
        Map.of("type", "json", "prompt", "Extract plan names and prices."),
        Map.of("type", "screenshot", "fullPage", true, "quality", 80, "viewport", Map.of("width", 1280, "height", 720)),
        Map.of("type", "changeTracking", "modes", List.of("git-diff"), "tag", "pricing")
    ))
    .headers(Map.of("User-Agent", "FirecrawlDocsBot/1.0"))
    .includeTags(List.of("main", "article"))
    .excludeTags(List.of("nav", "footer"))
    .onlyMainContent(true)
    .waitFor(1000)
    .mobile(false)
    .parsers(List.of(Map.of("type", "pdf", "mode", "auto", "maxPages", 5)))
    .actions(List.of(
        Map.of("type", "click", "selector", "#accept"),
        Map.of("type", "write", "text", "firecrawl"),
        Map.of("type", "press", "key", "Enter"),
        Map.of("type", "wait", "milliseconds", 1500),
        Map.of("type", "scrape")
    ))
    .location(LocationConfig.builder().country("US").languages(List.of("en-US")).build())
    .skipTlsVerification(true)
    .removeBase64Images(true)
    .blockAds(true)
    .proxy("auto")
    .maxAge(86400000L)
    .storeInCache(true)
    .build();

Document doc = client.scrape("https://example.com/pricing", options);
```

<div id="parameters">
  ### 参数
</div>

| 参数                            | 类型 / 选项                                                            | 描述                                                                                                                  |
| ----------------------------- | ------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------- |
| `url`                         | `String`                                                           | 要抓取的 URL。                                                                                                           |
| `options.formats`             | `List<Object>`                                                     | 要包含的输出formats。可以是字符串，也可以是格式对象，例如 `{type: "json", prompt, schema}` 或 `{type: "screenshot", fullPage, quality, viewport}`。 |
| `options.headers`             | `Map<String, String>`                                              | 自定义请求头。                                                                                                             |
| `options.includeTags`         | `List<String>`                                                     | 要包含的标签。                                                                                                             |
| `options.excludeTags`         | `List<String>`                                                     | 要排除的标签。                                                                                                             |
| `options.onlyMainContent`     | `Boolean`                                                          | 仅保留主要内容。                                                                                                            |
| `options.timeout`             | `Integer`                                                          | 超时时间 (毫秒) 。                                                                                                         |
| `options.waitFor`             | `Integer`                                                          | 抓取前的等待时间 (毫秒) 。                                                                                                     |
| `options.mobile`              | `Boolean`                                                          | 模拟移动设备。                                                                                                             |
| `options.parsers`             | `List<Object>`                                                     | 解析器配置 (支持 PDF 选项) 。                                                                                                 |
| `options.actions`             | `List<Map<String, Object>>`                                        | 抓取前要执行的浏览器 actions。                                                                                                 |
| `options.location`            | `LocationConfig`                                                   | 请求的位置设置。                                                                                                            |
| `options.skipTlsVerification` | `Boolean`                                                          | 跳过 TLS 证书验证。                                                                                                        |
| `options.removeBase64Images`  | `Boolean`                                                          | 从 markdown 输出中移除 base64 图片。                                                                                         |
| `options.blockAds`            | `Boolean`                                                          | 启用广告和 Cookie 弹窗拦截。                                                                                                  |
| `options.proxy`               | `String` (`"basic"`, `"stealth"`, `"enhanced"`, `"auto"`，或自定义 URL) | 代理模式。                                                                                                               |
| `options.maxAge`              | `Long`                                                             | 优先使用不超过此时长的缓存数据。                                                                                                    |
| `options.storeInCache`        | `Boolean`                                                          | 将结果存入缓存。                                                                                                            |
| `options.integration`         | `String`                                                           | 仅供内部使用。保留给 Firecrawl 埋点使用。                                                                                          |

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么要使用它
</div>

当页面在开始抓取后还需要执行浏览器 actions 或运行代码时 (例如点击、填写表单或按脚本步骤操作) ，请使用 interact。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`client.interact(jobId, code, language, timeout, origin)`

<div id="simple-example">
  ### 简单示例
</div>

```java
Document doc = client.scrape("https://example.com");
String jobId = doc.getMetadata() != null ? String.valueOf(doc.getMetadata().get("scrapeId")) : null;
if (jobId == null || jobId.isBlank()) {
    throw new IllegalStateException("抓取响应中缺少 scrapeId");
}

BrowserExecuteResponse result = client.interact(jobId, "console.log(document.title)");
```

<div id="complex-example">
  ### 复杂示例
</div>

```java
BrowserExecuteResponse result = client.interact(
    "<scrapeJobId>",
    "console.log(await page.title());",
    "node",
    60,
    null
);
```

<div id="parameters">
  ### 参数
</div>

| 参数         | 类型 / 选项                                   | 描述                                      |
| ---------- | ----------------------------------------- | --------------------------------------- |
| `jobId`    | `String`                                  | `document.metadata.scrapeId` 中的抓取任务 ID。 |
| `code`     | `String`                                  | 在浏览器会话中执行的代码。                           |
| `language` | `String` (`"python"`, `"node"`, `"bash"`) | 执行语言。省略时默认为 `node`。                     |
| `timeout`  | `Integer`                                 | 执行超时时间，单位为秒 (1-300) 。                   |
| `origin`   | `String`                                  | 仅供内部使用。保留给 Firecrawl 埋点。                |

<div id="notes">
  ## 注意事项
</div>

* Java SDK 的交互 API 仅支持 `code`。其他 SDK 还额外提供了 `prompt` 以便使用。
* 已弃用的别名：`scrapeExecute` 映射为 `interact`，`deleteScrapeBrowser` 映射为 `stopInteractiveBrowser`。

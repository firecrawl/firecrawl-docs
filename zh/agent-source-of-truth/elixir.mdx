<div id="firecrawl-elixir-source-of-truth">
  # Firecrawl Elixir 权威信息源
</div>

此文件是面向代理的公开权威信息源。它由 Firecrawl Elixir SDK (OpenAPI 格式) 和 v2 OpenAPI 规范生成。

<div id="install">
  ## 安装
</div>

```elixir
# mix.exs
defp deps do
  [
    {:firecrawl, "~> 1.0"}
  ]
end
```

<div id="authenticate">
  ## 认证
</div>

在 `config/runtime.exs` 中配置你的 API 密钥：

```elixir
config :firecrawl, api_key: System.fetch_env!("FIRECRAWL_API_KEY")
```

或者在每次请求中传入 `api_key`：

```elixir
{:ok, resp} = Firecrawl.search_and_scrape([query: "site:docs.firecrawl.dev webhook retries"], api_key: "fc-...")
```

<div id="when-to-use-what">
  ## 何时该用哪一种
</div>

* `search`：从查询出发发现相关 URL，包括 `site:` 限制。
* `scrape`：从已知 URL 获取并提取内容。
* `interact`：当页面需要点击、填写表单或执行抓取后的操作时，运行浏览器 actions 或代码。

<div id="search">
  ## 搜索
</div>

<div id="why-use-it">
  ### 为什么使用它
</div>

当你需要根据查询词发现相关内容，并希望 Firecrawl 返回 URL (也可以选择抓取每个结果) 时，请使用 search。你可以使用标准的 `site:` 运算符，将结果限定在某个域名内，例如 `site:docs.firecrawl.dev crawl webhooks`。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`Firecrawl.search_and_scrape(params, opts)`

<div id="simple-example">
  ### 简单示例
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(query: "site:docs.firecrawl.dev webhook retries")
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, resp} = Firecrawl.search_and_scrape(
  query: "site:docs.firecrawl.dev crawl webhooks",
  sources: ["web", "news"],
  categories: ["research"],
  limit: 10,
  tbs: "qdr:m",
  location: "San Francisco,California,United States",
  ignore_invalid_urls: true,
  timeout: 60000,
  scrape_options: [
    formats: [
      "markdown",
      "links",
      %{type: "json", prompt: "Extract title and key endpoints."}
    ],
    only_main_content: true,
    include_tags: ["main", "article"],
    exclude_tags: ["nav", "footer"],
    wait_for: 1000,
    actions: [
      %{type: "click", selector: "button#accept"},
      %{type: "wait", milliseconds: 750},
      %{type: "scrape"}
    ]
  ]
)
```

<div id="parameters">
  ### 参数
</div>

| 参数                    | 类型 / 选项                                                 | 描述                                               |
| --------------------- | ------------------------------------------------------- | ------------------------------------------------ |
| `query`               | `String`                                                | 搜索查询。使用 `site:domain` 可将结果限制在特定站点内。              |
| `sources`             | `List` (`"web"`、`"news"`、`"images"` 或包含 `type` 的映射)     | 要搜索的来源。                                          |
| `categories`          | `List` (`"github"`、`"research"`、`"pdf"` 或包含 `type` 的映射) | 用于筛选结果的类别。                                       |
| `limit`               | `integer`                                               | 返回结果的最大数量。                                       |
| `tbs`                 | `String`                                                | 基于时间的搜索筛选条件 (例如 `qdr:d`、`qdr:w`、`sbd:1,qdr:m`) 。 |
| `location`            | `String`                                                | 搜索结果的地区字符串。                                      |
| `country`             | `String`                                                | 用于地理定向搜索结果的 ISO 国家代码。                            |
| `enterprise`          | `List[String]` (`"anon"`、`"zdr"`)                       | 为你的团队启用时可用的企业级 ZDR 选项。                           |
| `ignore_invalid_urls` | `boolean`                                               | 排除其他 Firecrawl 端点无法抓取的无效 URL。                    |
| `timeout`             | `integer`                                               | 超时时间 (毫秒) 。                                      |
| `scrape_options`      | `keyword`                                               | 应用于每条结果的抓取选项 (字段与 `scrape` 相同) 。                 |

<div id="scrape">
  ## 抓取
</div>

<div id="why-use-it">
  ### 为什么使用它
</div>

如果你已经有 URL，并希望 Firecrawl 返回结构化内容和可选的 formats，请使用 `scrape`。

### 首选 SDK 方法

`Firecrawl.scrape_and_extract_from_url(params, opts)`

<div id="simple-example">
  ### 简单示例
</div>

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://docs.firecrawl.dev",
  formats: ["markdown"]
)
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, resp} = Firecrawl.scrape_and_extract_from_url(
  url: "https://example.com/pricing",
  formats: [
    "markdown",
    "links",
    %{type: "json", prompt: "Extract plan names and prices."},
    %{type: "screenshot", fullPage: true, quality: 80, viewport: %{width: 1280, height: 720}},
    %{type: "changeTracking", modes: ["git-diff"], tag: "pricing"}
  ],
  headers: %{"User-Agent" => "FirecrawlDocsBot/1.0"},
  include_tags: ["main", "article"],
  exclude_tags: ["nav", "footer"],
  only_main_content: true,
  wait_for: 1000,
  mobile: false,
  parsers: [%{type: "pdf", mode: "auto", maxPages: 5}],
  actions: [
    %{type: "click", selector: "#accept"},
    %{type: "write", text: "firecrawl"},
    %{type: "press", key: "Enter"},
    %{type: "wait", milliseconds: 1500},
    %{type: "scrape"}
  ],
  location: [country: "US", languages: ["en-US"]],
  skip_tls_verification: true,
  remove_base64_images: true,
  block_ads: true,
  proxy: "auto",
  max_age: 86400000,
  store_in_cache: true,
  profile: [name: "docs-session", saveChanges: true]
)
```

<div id="interact">
  ## 交互
</div>

<div id="why-use-it">
  ### 为什么使用它
</div>

当页面在开始抓取后需要进行浏览器 actions 或执行代码时 (如点击、填写表单或脚本步骤) ，请使用 interact。

<div id="preferred-sdk-method">
  ### 推荐的 SDK 方法
</div>

`Firecrawl.interact_with_scrape_browser_session(job_id, params, opts)`

<div id="simple-example">
  ### 简单示例
</div>

```elixir
{:ok, scrape_resp} = Firecrawl.scrape_and_extract_from_url(url: "https://example.com")
job_id = get_in(scrape_resp.body, ["data", "metadata", "scrapeId"])

{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  job_id,
  code: "console.log(document.title)",
  language: :node
)
```

<div id="complex-example">
  ### 复杂示例
</div>

```elixir
{:ok, resp} = Firecrawl.interact_with_scrape_browser_session(
  "<scrape_job_id>",
  code: "console.log(await page.title());",
  language: :node,
  timeout: 60
)
```

<div id="parameters">
  ### 参数
</div>

| 参数         | 类型 / 选项   | 描述                                    |        |                       |
| ---------- | --------- | ------------------------------------- | ------ | --------------------- |
| `job_id`   | `String`  | 来自 `data.metadata.scrapeId` 的抓取任务 ID。 |        |                       |
| `code`     | `String`  | 在浏览器会话中执行的代码。必填。                      |        |                       |
| `language` | `:python  | :node                                 | :bash` | 执行语言。省略时默认值为 `:node`。 |
| `timeout`  | `integer` | 执行超时时间 (秒) 。                          |        |                       |
| `origin`   | `String`  | 仅供内部使用。保留给 Firecrawl 的埋点功能使用。         |        |                       |

<div id="notes">
  ## 说明
</div>

* Elixir 客户端函数遵循 OpenAPI 规范，并接受 snake&#95;case 形式的关键字参数；这些参数会被转换为 camelCase 的 JSON 字段。

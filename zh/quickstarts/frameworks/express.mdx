<div id="overview">
  ## 概述
</div>

在 Express 服务器路由中使用 Firecrawl Node SDK。

<div id="prerequisites">
  ## 先决条件
</div>

* Firecrawl API 密钥
* Express 应用
* Node.js 18+

<div id="install">
  ## 安装
</div>

```bash
npm install @mendable/firecrawl-js
```

<div id="configure-environment">
  ## 配置环境
</div>

将 API 密钥存储在 `.env` 文件或进程环境变量中。

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## 添加服务器路由
</div>

server.js

```js
import express from 'express';
import Firecrawl from '@mendable/firecrawl-js';

const app = express();
app.use(express.json());

const client = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

app.post('/api/firecrawl', async (req, res) => {
  const { url } = req.body;
  const doc = await client.scrape(url, { formats: ['markdown'] });
  res.json({ markdown: doc.markdown });
});

app.listen(3000, () => console.log('Server listening on :3000'));
```

<div id="call-it-from-your-app">
  ## 在应用中调用
</div>

```js
const response = await fetch('http://localhost:3000/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.markdown);
```

<div id="notes">
  ## 注意事项
</div>

* 请将 Firecrawl 调用放在服务器端；绝不要在客户端打包产物中暴露 API 密钥。

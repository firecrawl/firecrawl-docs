<div id="overview">
  ## 概述
</div>

使用 Nuxt 的服务器端 API 路由，确保你的 Firecrawl API 密钥不会暴露。

<div id="prerequisites">
  ## 前提条件
</div>

* Firecrawl API 密钥
* Nuxt 3 应用
* Node.js 18+

<div id="install">
  ## 安装
</div>

无需额外安装任何包；Nuxt 的服务端路由可直接调用 fetch。

<div id="configure-environment">
  ## 配置环境
</div>

将 API 密钥添加到 `.env` 中，并通过运行时配置公开该密钥。

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  runtimeConfig: {
    firecrawlApiKey: process.env.FIRECRAWL_API_KEY,
  },
});
```

<div id="add-a-server-route">
  ## 添加服务端路由
</div>

server/api/firecrawl.post.ts

```ts
export default defineEventHandler(async (event) => {
  const { url } = await readBody(event);
  const config = useRuntimeConfig();

  return await $fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${config.firecrawlApiKey}`,
      'Content-Type': 'application/json',
    },
    body: { url, formats: ['markdown'] },
  });
});
```

<div id="call-it-from-your-app">
  ## 从你的应用中调用
</div>

```ts
const response = await fetch('/api/firecrawl', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

<div id="notes">
  ## 注意事项
</div>

* 请将 Firecrawl API 密钥保存在 `runtimeConfig` 中，避免暴露给客户端。

<div id="overview">
  ## 概述
</div>

使用 Deno Deploy 的 Fetch API 调用 Firecrawl。

<div id="prerequisites">
  ## 前置条件
</div>

* Firecrawl API 密钥
* Deno Deploy 项目

<div id="install">
  ## 安装
</div>

无需额外安装软件包；Deno 已内置 fetch。

<div id="configure-environment">
  ## 配置环境
</div>

在 Deno Deploy 项目的环境变量中设置 API 密钥。

```bash
FIRECRAWL_API_KEY=fc-your-api-key
```

<div id="add-a-server-route">
  ## 添加服务器路由
</div>

main.ts

```ts
Deno.serve(async (req) => {
  const { url } = await req.json();
  const response = await fetch('https://api.firecrawl.dev/v2/scrape', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${Deno.env.get('FIRECRAWL_API_KEY')}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ url, formats: ['markdown'] }),
  });

  return new Response(await response.text(), {
    headers: { 'Content-Type': 'application/json' },
  });
});
```

<div id="call-it-from-your-app">
  ## 在应用中调用
</div>

```js
const response = await fetch('https://your-app.deno.dev', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'https://example.com' }),
});

const data = await response.json();
console.log(data.data?.markdown ?? data);
```

<div id="notes">
  ## 注意事项
</div>

* Deno Deploy 仅允许向外发起 fetch 请求；请将所有 Firecrawl 调用保留在服务端。

</div>

在 Launch Week III 的最后一天，我们推出了全新与升级的集成，让你更轻松将 Firecrawl 连接到现有的工具和平台。

从自动化平台到 AI 管道，Firecrawl 现已与 20 多项服务集成，加速你从网页数据到工作流执行的全流程。

<div id="key-integration-updates">
  ### 关键集成更新
</div>

- **Discord Bot：** 直接在你的服务器中触发抓取并接收结构化结果。[了解更多](https://github.com/mendableai/firecrawl-discord-bot)

- **n8n 集成：** 将 Firecrawl 接入自定义自动化流程。[了解更多](https://n8n.io/)
- **Langflow：** 将 Firecrawl 代理无缝嵌入 Langflow 流水线。[了解更多](https://docs.langflow.org/)
- **LlamaIndex：** 搭配 LlamaIndex 使用 Firecrawl，以智能方式丰富与检索数据。[了解更多](https://www.llamaindex.ai/)
- **Dify：** 将 Firecrawl 集成到 Dify，自动化 AI 工作流。[了解更多](https://dify.ai/)

更多集成即将推出——如果你有缺少的集成，欢迎告诉我们。

<div id="day-6-firecrawl-mcp">
  ## 第 6 天 – Firecrawl MCP
</div>

![Firecrawl MCP](/images/launch-week/lw3-d6-1.webp)

<div id="overview">
  ### 概览
</div>

今天我们发布了对 Firecrawl MCP 服务器的一次重大升级，这是我们在 LLM 连接的抓取工作流中对模型上下文协议（MCP）的实现。

此版本为 MCP 带来了 FIRE-1（代理）支持，借助 /scrape 端点和 /extract 端点，帮助你跨越登录、按钮等交互障碍，获取背后隐藏的数据。

我们还为本地使用引入了 SSE（服务器发送事件）支持，让设置与实时集成比以往更加轻松。

这些更新让将网页数据以流式方式接入 LLM 流水线更为简单，由智能代理承担繁重工作。

<div id="key-features">
  ### 关键特性
</div>

* 支持 FIRE-1（代理）：通过模型上下文协议（MCP）的 /scrape 和 /extract 端点使用 Web 操作代理。
* 交互感知：可自动应对复杂的网页障碍。
* 支持 SSE（服务器发送事件）：实时流式输出——可在本地或通过我们托管的 MCP 服务器。
* 即插即用：几乎零配置即可集成到您的技术栈。

<div id="day-4-llmstxtnew">
  ## 第 4 天 – LLMstxt.new
</div>

![LLMs.txt 主图](/images/launch-week/lw3-d4-1.webp)

<div id="overview">
  ### 概览
</div>

今天我们发布 [http://llmstxt.new](http://llmstxt.new)——以最快速度将任意网站转换为适用于 LLM 的干净、整合文本文件。

只需在任意 URL 前加上 `llmstxt.new/`，即可获得一份纯 `.txt` 文件，针对 AI 训练与推理进行了优化。没有模板、没有噪声——只有有价值的内容。

基于 Firecrawl 构建，这款工具让为 LLM 工作流准备真实的网页内容变得轻而易举。

![LLMs.txt 生成器界面](/images/launch-week/lw3-d4-2.webp)

### 关键功能  

- **即用即取**：只需在任意 URL 前添加 `llmstxt.new/`。  
- **两种输出**：`llms.txt` 提供精简摘要，`llms-full.txt` 提供完整内容。  
- **对 API 友好**：可通过 `http://llmstxt.new/{YOUR_URL}` 调用，或使用 Firecrawl API 密钥获取完整输出。  
- **LLM 就绪格式**：面向训练与推理场景设计。  

<div id="day-3-extract-v2">
  ## 第 3 天 – /extract v2
</div>

![/extract v2 主图](/images/launch-week/lw3-d3-l.webp)

<div id="overview">
  ### 概览
</div>

今天我们正式发布 **/extract v2**，这是我们抽取系统的一次重大升级——由 FIRE-1（代理）驱动。

借助对**分页、多步流程和动态交互**的完整支持，extract v2 远超我们在一月推出的版本。现在还可以在**无需提供 URL**的情况下抽取数据，借助内置搜索层定位所需内容。

我们从底层彻底重构——更强的模型、更优的架构，并在内部基准测试中实现了显著的性能提升。

![Extract v2 in action](/images/launch-week/lw3-d3-2.webp)

### 关键功能  

- **集成 FIRE-1（代理）**：自动处理页面操作、导航与交互。  
- **多页面提取**：跨分页或分步流程提取数据。  
- **无需提供 URL**：使用内置搜索即可在无直接链接时提取内容。  
- **更高准确性**：全新模型与架构带来更好、更快的结果。  

<div id="day-2-fire-1-agent">
  ## 第 2 天 - FIRE-1（代理）
</div>

![FIRE-1（代理）可视化](/images/launch-week/lw3d2-1.webp)

<div id="overview">
  ### 概览
</div>

来认识 **FIRE-1**，这是 Firecrawl 的首个 AI 代理，旨在将网页抓取能力提升到新高度。凭借智能导航与交互能力，FIRE-1 远超传统抓取方式。

从处理分页到与按钮、链接等动态站点元素交互，FIRE-1 能实现强大且具上下文感知的抓取与抽取工作流。
![FIRE-1 Agent Visualization](/images/launch-week/lw3-d2-4.webp)

### 关键功能

- **智能导航**：自动在分页或多步骤内容中进行导航。
- **动态交互**：点击按钮、跟随链接，并与 JavaScript 渲染的组件进行交互。

<div id="day-1-change-tracking">
  ## 第 1 天 - 变更跟踪
</div>

![变更跟踪主图](/images/launch-week/lw3d1.webp)

<div id="overview">
  ### 概览
</div>

变更跟踪是一项强大的功能，可用于长期监控并检测网页内容的变化。该功能在 JavaScript 和 Python SDK 中均可用。
![Change Tracking](/images/launch-week/lw3d12.webp)

### 关键功能

- **检测变更**：识别网页自上次抓取以来是否发生变化。
- **查看具体变更**：获取两次抓取之间具体变化的详细信息。
- **结构化数据**：接收关于变更的结构化数据。
- **可见性控制**：管理您网站上变更内容的可见性。

使用 `changeTracking` 格式，您可以高效监控网站变更，并获取关于上次抓取时间戳、两版页面比对结果，以及当前页面/URL 可见性的完整信息。
